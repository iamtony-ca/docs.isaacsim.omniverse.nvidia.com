Physics raycast sensor#

물리 레이캐스트 센서는 물리 레이캐스트를 사용하여 센서 프림과 주변 지오메트리 사이의 거리를 측정합니다. Isaac Sim의 고정 패턴 센서와 달리, 물리 레이캐스트 센서는 레이당 명시적 원점 오프셋, 방향 벡터, 선택적 시간 오프셋을 받아들여 고체 상태 센서, 회전 센서, 빔 커튼 등 다양한 구성에 적합합니다.

매 물리 스텝마다, 센서는 프림의 월드 공간 위치(레이당 원점 오프셋 추가)에서 지정된 방향으로 레이를 발사합니다. rayTimeOffsets가 제공된 경우, 현재 물리 스텝의 시간 창 내에 시간 오프셋이 속하는 레이 하위 집합만 발사되어 여러 스텝에 걸쳐 스위핑 패턴을 생성합니다.

Isaac Sim 규칙의 전체 목록은 Isaac Sim Conventions 문서를 참조하십시오.

물리 레이캐스트 센서 속성

  1. enabled 파라미터는 센서가 실행 중인지 여부를 결정합니다.

  2. numRays(부호 없는 정수) 파라미터는 공인 레이 수를 지정합니다. rayOrigins와 rayDirections는 각각 정확히 이 개수의 요소를 가져야 합니다. Raycast.create() 또는 Raycast 작성 생성자를 사용할 때 자동으로 설정됩니다.

  3. minRange 파라미터는 스테이지 길이 단위로 최소 감지 범위를 지정합니다. 레이는 origin + direction * minRange에서 시작합니다.

  4. maxRange 파라미터는 스테이지 길이 단위로 최대 감지 범위를 지정합니다.

  5. rayOrigins 파라미터는 센서의 로컬 좌표 프레임에서 레이당 원점 이동을 지정합니다.

  6. rayDirections 파라미터는 센서의 로컬 좌표 프레임에서 레이당 발사 방향 벡터를 지정합니다. 벡터는 사용 전에 정규화됩니다.

  7. rayTimeOffsets 파라미터는 레이당 시간 오프셋(초)을 지정합니다. 제공된 경우, 센서는 현재 물리 스텝 내에 오프셋이 속하는 레이만 발사하여 스위핑 패턴을 활성화합니다. 스윕 주기는 max(rayTimeOffsets)입니다.

  8. outputFrameOfReference 파라미터는 히트 위치와 법선의 좌표 프레임을 선택합니다. SENSOR는 센서의 로컬 좌표 프레임에서 결과를 반환하고; WORLD는 월드 좌표로 결과를 반환합니다.

  9. reportHitPrimPaths 파라미터는 각 히트 표면의 USD 프림 경로 확인을 활성화합니다.

전체 USD 속성 정의는 Raycast Sensor schema reference를 참조하십시오.

Note

모든 센서 속성은 시뮬레이션이 시작될 때 한 번 읽힙니다. 시뮬레이션이 재생 중인 동안 속성 값을 변경해도 효과가 없습니다; 변경 사항을 적용하려면 시뮬레이션을 중지하고 재시작하십시오.

GUI#

Creating a physics raycast sensor#

GUI에서 물리 레이캐스트 센서를 생성하려면:

  1. 상단 메뉴 바에서 Create > Physics > Physics Scene을 클릭하여 Physics Scene을 생성하십시오. 오른쪽 Stage 패널에 PhysicsScene Prim이 생성되었는지 확인하십시오.

  2. Stage 패널에서 선택적으로 부모 프림을 선택하십시오.

  3. 상단 메뉴 바에서 Create > Sensors > Physics Raycast Sensor를 클릭하고 사전 설정 구성 중 하나를 선택하십시오:

    • Solid State Physics Raycast Sensor: 수평 및 수직 시야각을 설정할 수 있는 직사각형 격자 레이.

    • Rotating Physics Raycast Sensor: 1Hz 스위핑 패턴을 생성하는 시간 오프셋과 함께 360도에 분산된 레이.

    • Beam Curtain Physics Raycast Sensor: 근접 감지를 위해 수직으로 펼쳐진 평행 레이.

  4. 센서의 위치와 방향을 변경하려면, 센서 프림을 선택하고 Property 탭에서 Transform 속성을 수정하십시오.

  5. 센서 속성을 변경하려면 Raw USD Properties 섹션을 확장하여 범위, 레이 지오메트리, 출력 프레임 설정을 수정하십시오.

Physics raycast sensor example#

물리 레이캐스트 센서 예제를 실행하려면:

  1. Windows > Examples > Robotics Examples에서 Robotics Examples 탭을 활성화하십시오.

  2. Robotics Examples > Sensors > Physics Raycast Sensor > Load Scene을 클릭하십시오.

  3. 세 개의 물리 레이캐스트 센서가 생성되었는지 확인하십시오: 고체 상태 센서(녹색 레이), 회전 센서(파란색 레이), 빔 커튼 센서(빨간색 레이).

  4. Play 버튼을 눌러 시뮬레이션을 시작하십시오.

  5. 뷰포트의 디버그 레이 시각화와 예제 창의 히트 수 / 최소 깊이 판독값을 확인하십시오.

OmniGraph workflow#

다음은 OmniGraph를 사용하여 물리 레이캐스트 센서 데이터를 읽고 시각화하는 튜토리얼입니다.

Scene setup#

  1. Create > Physics > Physics Scene으로 Physics Scene을 생성하십시오.

  2. 충돌 지오메트리를 추가하십시오(예: Create > Mesh > Cube로 큐브를 추가하고 Add > Physics > Colliders Preset 적용).

  3. Create > Physics > GroundPlane으로 지면을 추가하십시오.

  4. Create > Sensors > Physics Raycast Sensor > Solid State Physics Raycast Sensor로 물리 레이캐스트 센서를 생성하십시오.

OmniGraph setup#

이 센서에서 판독값을 수집하기 위한 OmniGraph를 설정하려면:

  1. Window > Graph Editors > Action Graph로 이동하고 새 탭에서 New Action Graph를 선택하여 새 액션 그래프를 생성하십시오.

  2. 그래프에 다음 노드를 추가하십시오:

    • On Playback Tick (omni.graph.action.OnPlaybackTick): 매 시뮬레이션 타임스텝마다 그래프를 실행합니다.

    • Isaac Read Physics Raycast Sensor (isaacsim.sensors.physics.IsaacReadRaycastSensor): 물리 레이캐스트 센서를 읽습니다. Property 탭에서 Physics Raycast Sensor Prim을 센서 프림의 경로(예: /World/Sensors/Solid_State_Physics_Raycast_Sensor)로 설정하십시오.

    • Debug Draw RayCast (isaacsim.util.debug_draw.DebugDrawRayCast): 뷰포트에서 레이를 시각화합니다.

  3. Debug Draw RayCast 노드를 구성하십시오:

    • inputs:doTransform을 False로 설정하십시오. 읽기 노드는 이미 월드 공간 빔 원점과 끝점을 제공합니다; 추가 변환을 적용하면 잘못된 시각화가 생성됩니다.

  4. 필수 연결 5개를 모두 갖추어 노드를 연결하십시오:

    • On Playback Tick outputs:tick → Isaac Read Physics Raycast Sensor inputs:execIn

    • Isaac Read Physics Raycast Sensor outputs:execOut → Debug Draw RayCast inputs:exec

    • Isaac Read Physics Raycast Sensor outputs:beamOrigins → Debug Draw RayCast inputs:beamOrigins

    • Isaac Read Physics Raycast Sensor outputs:beamEndPoints → Debug Draw RayCast inputs:beamEndPoints

    • Isaac Read Physics Raycast Sensor outputs:numRays → Debug Draw RayCast inputs:numRays

    Important

    numRays 연결이 필요합니다. 없으면 Debug Draw 노드는 기본적으로 0개의 레이로 설정되어 아무것도 렌더링하지 않습니다. 마찬가지로, 읽기 노드의 빔 원점과 끝점이 이미 월드 좌표로 되어 있으므로 doTransform은 False로 설정해야 합니다.

  5. Play 버튼을 누르십시오. 올바르게 설정되었다면 뷰포트에서 센서에서 히트 점까지 레이 라인이 나타납니다.

Programmatic OmniGraph setup#

og.Controller를 사용하여 동일한 그래프를 프로그래밍 방식으로 생성할 수도 있습니다:

# Programmatic OmniGraph setup: Physics Raycast Sensor + Debug Draw visualization
import omni.graph.core as og

sensor_prim_path = "/World/Sensors/Solid_State_Physics_Raycast_Sensor"

action_graph, _, _, _ = og.Controller.edit(
    {"graph_path": "/World/ActionGraph", "evaluator_name": "execution"},
    {
        og.Controller.Keys.CREATE_NODES: [
            ("OnPlaybackTick", "omni.graph.action.OnPlaybackTick"),
            ("ReadRaycast", "isaacsim.sensors.physics.IsaacReadRaycastSensor"),
            ("DebugDraw", "isaacsim.util.debug_draw.DebugDrawRayCast"),
        ],
        og.Controller.Keys.SET_VALUES: [
            ("ReadRaycast.inputs:raycastSensorPrim", sensor_prim_path),
            ("DebugDraw.inputs:doTransform", False),
        ],
        og.Controller.Keys.CONNECT: [
            ("OnPlaybackTick.outputs:tick", "ReadRaycast.inputs:execIn"),
            ("ReadRaycast.outputs:execOut", "DebugDraw.inputs:exec"),
            ("ReadRaycast.outputs:beamOrigins", "DebugDraw.inputs:beamOrigins"),
            ("ReadRaycast.outputs:beamEndPoints", "DebugDraw.inputs:beamEndPoints"),
            ("ReadRaycast.outputs:numRays", "DebugDraw.inputs:numRays"),
        ],
    },
)

Note

시각화가 실패할 수 있는 단순한 설정과의 주요 차이점:

  • doTransform은 False여야 합니다: 읽기 노드는 월드 공간 좌표를 출력합니다. Debug Draw 노드의 doTransform 입력은 기본적으로 추가 행렬 변환을 적용하여 레이를 잘못된 위치로 이동시킵니다.

  • numRays를 연결해야 합니다: 없으면 드로우 노드는 렌더링할 레이 수를 알 수 없어 기본값인 0이 됩니다.

  • 실행 체인이 완전해야 합니다: execIn → execOut → exec는 읽기 노드가 출력을 채운 후 드로우 노드가 실행되도록 보장합니다.

Standalone Python#

Creating a physics raycast sensor#

PhysicsScene, 충돌 지오메트리, 센서 부모 Xform을 추가하는 다음 스니펫을 사용하여 씬을 준비하십시오.

import omni.usd
from pxr import Gf, Sdf, UsdGeom, UsdPhysics

stage = omni.usd.get_context().get_stage()

UsdGeom.SetStageUpAxis(stage, UsdGeom.Tokens.z)
UsdGeom.SetStageMetersPerUnit(stage, 1.0)

UsdPhysics.Scene.Define(stage, Sdf.Path("/World/PhysicsScene"))

ground = UsdGeom.Cube.Define(stage, "/World/GroundPlane")
ground.GetSizeAttr().Set(1.0)
ground.AddTranslateOp().Set(Gf.Vec3d(0, 0, -0.05))
ground.AddScaleOp().Set(Gf.Vec3f(50, 50, 0.1))
UsdPhysics.CollisionAPI.Apply(ground.GetPrim())

wall = UsdGeom.Cube.Define(stage, "/World/Obstacles/Wall")
wall.GetSizeAttr().Set(1.0)
wall.AddTranslateOp().Set(Gf.Vec3d(5, 0, 1.5))
wall.AddScaleOp().Set(Gf.Vec3f(0.2, 8, 3))
UsdPhysics.CollisionAPI.Apply(wall.GetPrim())

UsdGeom.Xform.Define(stage, "/World/Sensors")

Using the Python API#

Raycast.create()(작성 클래스)로 물리 레이캐스트 센서를 생성하고 반환된 작성 객체를 RaycastSensor로 래핑하여 런타임 데이터에 접근합니다. ray_origins와 ray_directions 배열을 같은 길이로 제공해야 합니다. 경로에는 부모 프림 경로가 포함되어야 합니다.

import math

from isaacsim.sensors.experimental.physics import Raycast, RaycastSensor

# Generate a simple grid of ray directions for a solid state physics raycast sensor.
h_count, v_count = 10, 5
h_fov, v_fov = 60.0, 20.0
origins = []
directions = []
for vi in range(v_count):
    v_angle = math.radians(-v_fov / 2 + v_fov * vi / max(v_count - 1, 1))
    for hi in range(h_count):
        h_angle = math.radians(-h_fov / 2 + h_fov * hi / max(h_count - 1, 1))
        dx = math.cos(v_angle) * math.cos(h_angle)
        dy = math.cos(v_angle) * math.sin(h_angle)
        dz = math.sin(v_angle)
        origins.append([0.0, 0.0, 0.0])
        directions.append([dx, dy, dz])

sensor = RaycastSensor(
    Raycast.create(
        "/World/Sensors/Physics_Raycast_Sensor",
        min_range=0.4,
        max_range=100.0,
        ray_origins=origins,
        ray_directions=directions,
        output_frame="WORLD",
        translations=[[0.0, 0.0, 1.5]],
    )
)

Using time offsets#

스위핑 패턴의 센서를 생성하려면 ray_time_offsets를 제공하십시오. 레이는 현재 물리 스텝의 시간 창 내에 시간 오프셋이 속하는 경우에만 발사됩니다. 스윕 주기는 max(ray_time_offsets)와 같습니다.

import math

from isaacsim.sensors.experimental.physics import Raycast, RaycastSensor

# Generate rays for a rotating physics raycast sensor with time offsets.
# Each azimuthal column is assigned a time offset within the sweep period.
# Only rays whose offsets fall in the current physics step are fired.
v_count = 8
azimuth_steps = 36
v_fov = 30.0
rotation_rate = 1.0
period = 1.0 / rotation_rate

origins = []
directions = []
time_offsets = []
for ai in range(azimuth_steps):
    h_angle = math.radians(360.0 * ai / azimuth_steps)
    t_offset = period * ai / azimuth_steps
    for vi in range(v_count):
        v_angle = math.radians(-v_fov / 2 + v_fov * vi / max(v_count - 1, 1))
        dx = math.cos(v_angle) * math.cos(h_angle)
        dy = math.cos(v_angle) * math.sin(h_angle)
        dz = math.sin(v_angle)
        origins.append([0.0, 0.0, 0.0])
        directions.append([dx, dy, dz])
        time_offsets.append(t_offset)

sensor = RaycastSensor(
    Raycast.create(
        "/World/Sensors/Rotating_Physics_Raycast_Sensor",
        min_range=0.4,
        max_range=100.0,
        ray_origins=origins,
        ray_directions=directions,
        ray_time_offsets=time_offsets,
        output_frame="WORLD",
        translations=[[0.0, 0.0, 1.5]],
    )
)

Using the RaycastSensor runtime#

RaycastSensor 클래스는 기존 Raycast 작성 객체 또는 기존 IsaacRaycastSensor 프림을 래핑하여 런타임 데이터에 접근합니다. 새 프림의 설정 및 생성은 Raycast.create()로 하십시오.

from isaacsim.sensors.experimental.physics import Raycast, RaycastSensor

sensor = RaycastSensor(
    Raycast.create(
        "/World/Sensors/My_Sensor",
        ray_origins=[[0, 0, 0], [0, 0, 0]],
        ray_directions=[[1, 0, 0], [0, 1, 0]],
        min_range=0.4,
        max_range=100.0,
        output_frame="WORLD",
    )
)

# After simulation starts:
frame = sensor.get_data()
print(f"Depths: {frame['depths']}")
print(f"Hit positions: {frame['hit_positions']}")

Reading sensor output#

물리 레이캐스트 센서는 Play 시 동적으로 생성됩니다. RaycastSensor.get_sensor_reading()으로 원시 센서 데이터를 읽거나, RaycastSensor.get_data()로 구조화된 딕셔너리를 얻을 수 있습니다. 판독값에는 깊이, 히트 위치, 히트 법선, 선택적으로 히트 프림 경로가 포함됩니다.

다음 스니펫은 위의 스니펫 중 하나를 사용하여 센서 프림을 생성했다고 가정합니다.

from isaacsim.sensors.experimental.physics import Raycast, RaycastSensor

sensor = RaycastSensor(
    Raycast.create(
        "/World/Sensors/Physics_Raycast_Sensor",
        ray_origins=[[0.0, 0.0, 0.0]],
        ray_directions=[[1.0, 0.0, 0.0]],
    )
)
reading = sensor.get_sensor_reading()

if reading.is_valid:
    print(f"Ray count: {reading.ray_count}")
    print(f"Depths: {reading.depths}")
    print(f"Hit positions: {reading.hit_positions}")
    print(f"Hit normals: {reading.hit_normals}")

get_sensor_reading() 함수는 다음 속성을 가진 RaycastSensorReading 객체를 반환합니다:

  • is_valid: 판독값에 유효한 데이터가 있는지 여부.

  • ray_count: 판독값의 레이 수.

  • time: 이 판독값의 시뮬레이션 시간(초).

  • depths: 스테이지 길이 단위로 레이당 히트 거리. 미스된 레이는 maxRange를 반환합니다.

  • hit_positions: outputFrameOfReference에 지정된 프레임에서 Nx3 배열로 레이당 히트 위치.

  • hit_normals: 히트 점에서 Nx3 배열로 레이당 표면 법선.

  • hit_prim_paths: 히트 표면의 레이당 USD 프림 경로(reportHitPrimPaths가 활성화된 경우에만 채워짐).

  • ray_origins_world: Nx3 배열로 레이당 월드 공간 원점.

  • ray_end_points_world: Nx3 배열로 레이당 월드 공간 끝점(디버그 시각화에 유용).

get_data() 함수는 depths, hit_positions, hit_normals, hit_prim_paths, time, physics_step를 포함하는 구조화된 딕셔너리를 반환합니다. ray_origins_world와 ray_end_points_world는 원시 get_sensor_reading() 결과에서만 사용할 수 있습니다.
