Deploying Policies in Isaac Sim#

이 튜토리얼의 목표는 Isaac Lab에서 훈련된 정책을 배포하는 과정을 예제를 통해 설명하고 로봇 정의 파일을 탐색하는 것입니다.

Isaac Sim에서 정책을 배포하려는 여러 사용 사례가 있습니다. 예를 들어 더 복잡한 로코모션을 달성하도록 로봇을 활성화하거나, 시뮬레이션 환경에서 내비게이션 및 위치 추적과 같은 다른 스택과 정책을 테스트하고 통합하거나, ROS 2와 같은 기존 브릿지를 사용하여 인터페이스하는 것 등이 있습니다.

Learning Objectives#

이 튜토리얼에서는 정책 기반 로봇 예제를 살펴봅니다:

  1. H1 및 Spot 평지 지형 정책 컨트롤러 데모

  2. Isaac Lab에서 정책 훈련 및 내보내기

  3. Isaac Lab에서 환경 파라미터 파일 읽기

  4. 로봇 정의 클래스

  5. 위치에서 토크로 변환

  6. 디버깅 팁

  7. Sim to Real 배포

Demos#

먼저 Windows > Examples > Robotics Examples를 활성화하면 Robotics Examples 탭이 열립니다.

Unitree H1 Humanoid Example#

  1. Unitree H1 휴머노이드 예제는 빈 스테이지를 생성하여 접근할 수 있습니다.

  2. Robotics Examples > POLICY > Humanoid를 사용하여 예제를 엽니다.

  3. LOAD를 눌러 씬을 엽니다.

이 예제는 Isaac Lab에서 훈련된 H1 Flat Terrain Policy를 사용하여 휴머노이드의 로코모션을 제어합니다.

컨트롤:

  • 앞으로: UP ARROW / NUM 8

  • 왼쪽 회전: LEFT ARROW / NUM 4

  • 오른쪽 회전: RIGHT ARROW / NUM 6

Boston Dynamics Spot Quadruped Example#

  1. Boston Dynamics Spot 사족 보행 예제는 빈 스테이지를 생성하여 접근할 수 있습니다.

  2. Robotics Examples > POLICY > Quadruped를 사용하여 예제를 엽니다.

  3. LOAD를 눌러 씬을 엽니다.

이 예제는 Isaac Lab에서 훈련된 Spot Flat Terrain Policy를 사용하여 사족 보행 로봇의 로코모션을 제어합니다.

컨트롤:

  • 앞으로: UP ARROW / NUM 8

  • 뒤로: BACK ARROW / NUM 2

  • 왼쪽 이동: LEFT ARROW / NUM 4

  • 오른쪽 이동: RIGHT ARROW / NUM 6

  • 왼쪽 회전: N / NUM 7

  • 오른쪽 회전: M / NUM 9

참고

Standalone 예제 워크플로우와 예제에 사용된 정책 파일은 isaac sim policy example extension document를 참조하세요.

Training and Exporting Policies in Isaac Lab#

Training#

Isaac Lab에서 정책을 훈련하는 것이 정책 배포의 첫 번째 단계입니다. 기존 또는 커스텀 정책 훈련에 대한 Isaac Lab 튜토리얼을 참조하세요.

위 예제에서 사용된 정책은 Unitree H1 휴머노이드를 위한 Isaac-Velocity-Flat-H1-v0와 Boston Dynamics Spot 로봇을 위한 Isaac-Velocity-Flat-Spot-v0입니다.

참고

예를 들어 Isaac Lab 2.0에서는 다음 명령을 사용하여 H1 평지 지형 정책을 훈련합니다:

./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/train.py --task Isaac-Velocity-Flat-H1-v0 --headless

Exporting#

RSL_rl을 사용하여 훈련된 정책은 Isaac Lab 워크스페이스 내의 scripts/reinforcement_learning/rsl_rl/play.py를 사용하여 내보낼 수 있습니다. 내보낸 파일은 exported 폴더에 생성됩니다.

다른 프레임워크로 훈련된 정책이나 반복 스냅샷을 사용하여 추론하는 것도 가능하지만, 신경망 구조와 같은 추가 데이터가 필요할 수 있습니다. 더 자세한 내용은 원하는 프레임워크의 문서를 따르세요.

참고

예를 들어 Isaac Lab 2.0에서는 다음 명령을 사용하여 H1 평지 지형 정책을 내보냅니다:

./isaaclab.sh -p scripts/reinforcement_learning/rsl_rl/play.py --task Isaac-Velocity-Flat-H1-v0 --num_envs 32

참고

예제에 사용된 훈련된 정책 파일은 여기에서 다운로드할 수 있습니다.

Understanding the Environment Parameter File#

agent.yaml과 env.yaml은 훈련된 정책과 함께 생성되어 정책 설정을 설명하며 logs/rsl_rl/<task_name>/<time>/params/ 폴더에 있습니다.

  • agent.yaml은 신경망 파라미터를 설명합니다.

  • env.yaml은 환경 및 로봇 설정을 설명합니다.

아래 스니펫은 Isaac-Velocity-Flat-H1-v0에서 가져온 것입니다.

Simulation Setup#

sim:
physics_prim_path: /physicsScene
dt: 0.005
render_interval: 4
gravity: !!python/tuple
- 0.0
- 0.0
- -9.81
enable_scene_query_support: false
use_fabric: true
disable_contact_processing: true
use_gpu_pipeline: true
device: cuda:0

첫 번째 스니펫은 시뮬레이션 환경을 설명합니다. 시뮬레이션 물리는 0.005s (200hz)로 실행되어야 하며, 중력은 9.81m/s^2로 아래를 향합니다.

Robot Setup#

scene:robot:init_state 섹션은 로봇의 초기 위치, 방향, 속도 및 기본 관절 위치와 속도를 설명합니다.

init_state:
  pos: !!python/tuple
  - 0.0
  - 0.0
  - 1.05
  rot: &id003 !!python/tuple
  - 1.0
  - 0.0
  - 0.0
  - 0.0
  lin_vel: &id001 !!python/tuple
  - 0.0
  - 0.0
  - 0.0
  ang_vel: *id001
  joint_pos:
    .*_hip_yaw: 0.0
    .*_hip_roll: 0.0
    .*_hip_pitch: -0.28
    .*_knee: 0.79
    .*_ankle: -0.52
    torso: 0.0
    .*_shoulder_pitch: 0.28
    .*_shoulder_roll: 0.0
    .*_shoulder_yaw: 0.0
    .*_elbow: 0.52
  joint_vel:
    .*: 0.0

scene:robot:init_state:actuators 섹션은 노력 및 속도 제한, 강성 및 감쇠와 같은 로봇 관절 속성을 설명합니다.

actuators:
  legs:
    class_type: omni.isaac.lab.actuators.actuator_pd:ImplicitActuator
    joint_names_expr:
    - .*_hip_yaw
    - .*_hip_roll
    - .*_hip_pitch
    - .*_knee
    - torso
    effort_limit: 300
    velocity_limit: 100.0
    stiffness:
      .*_hip_yaw: 150.0
      .*_hip_roll: 150.0
      .*_hip_pitch: 200.0
      .*_knee: 200.0
      torso: 200.0
    damping:
      .*_hip_yaw: 5.0
      .*_hip_roll: 5.0
      .*_hip_pitch: 5.0
      .*_knee: 5.0
      torso: 5.0
    armature: null
    friction: null

Observations Parameters#

관측 파라미터는 정책에 필요한 관측값과 관측값에 적용해야 하는 스케일 또는 클리핑 팩터를 설명합니다.

observations:
    policy:
        concatenate_terms: true
        enable_corruption: true
        base_lin_vel:
        func: omni.isaac.lab.envs.mdp.observations:base_lin_vel
        params: {}
        noise:
            func: omni.isaac.lab.utils.noise.noise_model:uniform_noise
            operation: add
            n_min: -0.1
            n_max: 0.1
        clip: null
        scale: null

Actions Parameters#

액션 파라미터는 정책이 출력하는 액션과 액션에 적용해야 하는 스케일링 팩터 및 오프셋을 설명합니다.

actions:
    joint_pos:
        class_type: omni.isaac.lab.envs.mdp.actions.joint_actions:JointPositionAction
        asset_name: robot
        debug_vis: false
        joint_names:
        - .*
        scale: 0.5
        offset: 0.0
        use_default_offset: true

Commands Parameters#

마지막으로, 커맨드 섹션은 정책의 커맨드 유형과 허용 가능한 커맨드 범위를 설명합니다.

commands:
    base_velocity:
        class_type: omni.isaac.lab.envs.mdp.commands.velocity_command:UniformVelocityCommand
        resampling_time_range: !!python/tuple
        - 10.0
        - 10.0
        debug_vis: true
        asset_name: robot
        heading_command: true
        heading_control_stiffness: 0.5
        rel_standing_envs: 0.02
        rel_heading_envs: 1.0
        ranges:
            lin_vel_x: !!python/tuple
            - 0.0
            - 1.0
            lin_vel_y: *id006
            ang_vel_z: !!python/tuple
            - -1.0
            - 1.0
            heading: !!python/tuple
            - -3.141592653589793
            - 3.141592653589793

Policy Controller Class#

로봇 정의 클래스는 로봇 prim을 정의하고, 로봇 정책을 임포트하고, 로봇 설정을 설정하고, 관측 텐서를 구성하고, 최종적으로 정책 제어 액션을 로봇에 적용합니다.

Constructor#

Constructor는 로봇 USD를 스폰하고 로봇 제어를 위한 단일 아티큘레이션 오브젝트를 생성합니다.

Load Policy#

이 클래스는 정책 파일과 정책 컨트롤러가 Isaac Sim 환경을 설정하는 데 사용할 해당 환경 파일을 로드합니다.

Initialize#

initialize 함수는 시뮬레이션이 시작된 후 한 번 호출해야 합니다. 이 함수의 목적은 로봇 설정을 정책에 맞추는 것으로, 로봇 노력 모드, 제어 모드, 관절 게인, 관절 최대 노력, 관절 최대 속도, 아티큘레이션 루트를 설정합니다.

_set_articulation_prop#

이 함수는 아티큘레이션 루트 속성을 파싱하고 이 속성들을 로봇에 설정합니다.

_compute_action#

이 함수는 관측에서 액션을 계산합니다.

_compute_observation#

이 함수는 상속된 클래스에 의해 오버로드되어야 하며 매 물리 스텝마다 advance()에 의해 호출됩니다. 이 함수의 목적은 정책이 예상하는 형식으로 관측 텐서를 생성하는 것입니다. 예를 들어, 아래 코드 스니펫은 H1 평지 지형 정책을 위한 관측 텐서를 생성합니다.

obs = torch.zeros(69, device=torch.device(str(self.robot._device)))
# Base lin vel
obs[:3] = lin_vel_b.squeeze()
# Base ang vel
obs[3:6] = ang_vel_b.squeeze()
# Gravity
obs[6:9] = gravity_b.squeeze()
# Command
obs[9:12] = command
# Joint states
current_joint_pos = wp.to_torch(self.robot.get_dof_positions())
current_joint_vel = wp.to_torch(self.robot.get_dof_velocities())
obs[12:31] = current_joint_pos - self.default_pos
obs[31:50] = current_joint_vel - self.default_vel
# Previous Action
obs[50:69] = self._previous_action

참고

env.yaml에 지정된 관측 스케일을 관측 항에 곱하는 것을 잊지 마세요.

Forward#

이 함수는 상속된 클래스에 의해 오버로드되어야 하며 매 물리 스텝마다 호출되어 로봇에 대한 제어 액션을 생성합니다. 예를 들어, 아래 코드 스니펫은 H1 평지 지형 정책을 위한 컨트롤을 생성합니다.

if self._policy_counter % self._decimation == 0:
    obs = self._compute_observation(command)
    self.action = self._compute_action(obs)
    self._previous_action = self.action.clone()

self.robot.set_dof_position_targets(positions=wp.from_torch(self.default_pos + (self.action * self._action_scale)))
self._policy_counter += 1

참고

  • 정책은 매 스텝마다 호출할 필요가 없으며, env.yaml의 decimation 파라미터를 참조하세요.

  • env.yaml에 지정된 액션 스케일을 액션 출력에 곱하는 것을 잊지 마세요.

경고

위치 기반 컨트롤의 경우 관절을 원하는 위치로 순간 이동시키므로 set_joint_position()을 사용하지 마세요.

Position to Torque Controls#

일부 로봇은 출력으로 토크 제어가 필요할 수 있습니다. 정책이 위치를 출력으로 생성하는 경우 위치를 토크로 변환해야 합니다. 이를 수행하는 방법은 여러 가지가 있으며, 여기서는 액추에이터 네트워크를 사용하여 위치를 토크로 변환합니다.

액추에이터 네트워크 클래스는 source/extensions/isaacsim.robot.policy.examples/isaacsim/robot/policy/examples/utils/actuator_network.py에 정의되어 있습니다. Anymal 로봇의 액추에이터 네트워크 정책은 Content Browser의 SAMPLES > POLICY > ANYMAL_POLICIES에 저장되어 있습니다.

Import Policy#

LSTMSeaNetwork 구현을 위해 Anymal Flat Terrain Policy 클래스의 다음 스니펫을 사용하여 정책 파일을 헬퍼 액추에이터 네트워크에 로드합니다:

def initialize(self, physics_sim_view=None) -> None:
    """
    Initialize the articulation interface and set up drive mode.

    Args:
        physics_sim_view: The physics simulation view
    """
    super().initialize(physics_sim_view=physics_sim_view, control_mode="effort")

    # Actuator network
    assets_root_path = get_assets_root_path()
    file_content = omni.client.read_file(
        assets_root_path + "/Isaac/IsaacLab/ActuatorNets/ANYbotics/anydrive_3_lstm_jit.pt"
    )[2]
    file = io.BytesIO(memoryview(file_content).tobytes())
    self._actuator_network = LstmSeaNetwork()
    self._actuator_network.setup(file, self.default_pos)
    self._actuator_network.reset()

Run the Actuator Network#

advance 함수에서 로코모션 정책의 위치 출력을 액추에이터 네트워크에 삽입하고 다음 스니펫을 사용하여 토크를 로봇에 적용합니다:

current_joint_pos = self.get_joint_positions()
current_joint_vel = self.get_joint_velocities()

joint_torques, _ = self._actuator_network.compute_torques(
    current_joint_pos, current_joint_vel, self._action_scale * self.action
)

self.set_joint_efforts(joint_torques)

Debugging Tips#

로봇이 즉시 작동하지 않는 경우 다음 팁을 사용하여 디버깅을 시작할 수 있습니다:

Verify Your Policy#

Isaac Lab에서 플레이하여 정책이 올바르게 작동하는지 확인하는 것부터 시작할 수 있습니다.

워크플로우에 맞는 올바른 play.py를 사용하고 올바른 태스크를 선택하는 것을 잊지 마세요.

Verify the Robot Joint Properties#

Robot Joint Order#

정책이 Isaac Lab에서 작동한다면 로봇의 관절 순서, 관절 속성, 기본 관절 위치를 확인해야 합니다.

관절 순서를 확인하려면 에셋 USD를 열고 로봇 prim으로 아티큘레이션을 생성하고 시뮬레이션을 시작하고 아티큘레이션을 초기화한 후 dof_names 함수를 호출하세요.


# Open your USD and PLAY the simulation before running this snippet
# Change the path to the robot you want to inspect
prim = Articulation(paths="/World/Robot")
print(str(prim.dof_names))

Isaac Sim 에셋과 Isaac Lab에서 훈련에 사용한 에셋 모두의 dof_names를 출력하고 이름과 순서가 정확히 일치하는지 확인하세요.

아래 ANYmal 로봇은 잘못된 순서의 제어 커맨드로 인해 넘어지고 있습니다.

Default Joint Position#

관절 위치를 확인한 후 기본 관절 위치가 올바르게 삽입되었는지 확인하세요. 관절 위치가 올바르지 않으면 로봇 관절이 올바른 위치로 이동하지 않습니다.

예를 들어, 아래 비디오에서 발목 관절이 잘못 설정되어 H1 휴머노이드가 발끝으로 걷는 "문워크"를 하고 있습니다.

Robot Joint Properties#

관절이 너무 많이 또는 너무 적게 움직이는 것이 관찰되면 관절 속성이 올바르게 설정되지 않은 것일 수 있습니다.


# Open your USD and PLAY the simulation before running this snippet
# Change the path to the robot you want to inspect
prim = Articulation(paths="/World/Robot")
print("DOF names:", prim.dof_names)
print("DOF types:", prim.dof_types)
print("DOF limits:", prim.get_dof_limits())
print("DOF gains (stiffness, damping):", prim.get_dof_gains())
print("DOF max efforts:", prim.get_dof_max_efforts())
print("DOF max velocities:", prim.get_dof_max_velocities())
print("DOF drive types:", prim.get_dof_drive_types())
print("DOF friction:", prim.get_dof_friction_properties())
print("DOF armatures:", prim.get_dof_armatures())

그런 다음 Isaac Lab에서 생성된 env YAML 파일과 관절 속성을 비교할 수 있습니다. DOF의 속성에 대한 아티큘레이션 API 문서를 확인하세요.

예를 들어, 아래 비디오에서 Spot 로봇의 강성과 감쇠가 너무 높게 설정되어 저작동 움직임이 발생합니다.

예를 들어, 아래 비디오에서 H1 로봇의 팔 강성과 감쇠가 너무 낮게 설정되어 과도한 움직임이 발생합니다.

Verify the Simulation Environment#

로봇이 정확히 일치하고 추론 예제가 여전히 작동하지 않는다면 시뮬레이션 파라미터를 확인해야 합니다.

Physics Scene#

물리 씬은 Time Steps Per Second (Hz)로 시간 스텝을 설명하므로 env.yaml의 dt 파라미터의 역수를 취하여 올바르게 설정하세요. 또한 물리 씬 속성을 env.yaml 파일의 physx 섹션과 맞추세요.

예를 들어, 아래 비디오에서 시간 스텝이 컨트롤러가 예상하는 500Hz 대신 60Hz로 설정되었습니다.

Verify the Observation and Action Tensor#

마지막으로 관측 및 액션 텐서를 확인하고, 텐서 구조가 올바른지, 텐서에 전달되는 데이터가 올바른지, 입력 및 출력에 올바른 스케일 팩터가 적용되었는지 확인하세요.

또한 정책의 액션 출력이 아티큘레이션의 예상 입력 유형과 일치하고 로봇을 올바르게 구동하기 위한 올바른 순서인지 확인하세요.

Sim To Real Deployment#

축하합니다. 이제 로봇과 정책이 Isaac Sim에서 올바르게 작동하고 나머지 스택과 함께 테스트했습니다. 이제 실제 로봇에 배포할 시간입니다.

Spot 로봇에 강화 학습 정책을 배포하는 방법에 대한 이 글을 읽어보세요.
