OmniGraph via Python Scripting Tutorial#

OmniGraph는 시각적 스크립팅 도구이지만 Python 스크립팅 인터페이스도 제공합니다. 이 튜토리얼은 Python을 사용하여 Action Graph를 스크립팅하는 몇 가지 예제를 제공합니다.

Learning Objectives#

이 튜토리얼에서는

  • 순수 Python API를 사용하여 OmniGraph를 스크립팅하는 예제를 안내합니다

  • OmniGraph의 기본 개념과 자주 사용되는 매개변수를 소개하고 스크립팅된 예제로 보여줍니다

Getting Started#

Prerequisites

  • 이 튜토리얼을 시작하기 전에 GUI 튜토리얼 시리즈, 특히 Isaac Sim OmniGraph Tutorial과 Omniverse Script Editor를 검토하세요.

  • Core API 튜토리얼 시리즈, 특히 Hello World를 검토하여 Python을 통한 익스텐션 워크플로우와 Python 독립 실행형 워크플로우에 익숙해지세요.

Code Snippets#

Creating a Graph#

먼저 모든 시뮬레이션 프레임에서 콘솔에 "Hello World"를 출력하는 간단한 Action Graph를 구성해 봅니다.

  1. 'Window > Script Editor'를 열고 다음 코드를 붙여넣습니다:

    import omni.graph.core as og

keys = og.Controller.Keys
graph_handle, list_of_nodes, _, _ = og.Controller.edit(
    {"graph_path": "/action_graph", "evaluator_name": "execution"},
    {
        keys.CREATE_NODES: [("tick", "omni.graph.action.OnTick"), ("print", "omni.graph.ui_nodes.PrintText")],
        keys.SET_VALUES: [
            ("print.inputs:text", "Hello World"),
            (
                "print.inputs:logLevel",
                "Warning",
            ),  # setting the log level to warning so we can see the printout in terminal
        ],
        keys.CONNECT: [("tick.outputs:tick", "print.inputs:execIn")],
    },
)

  2. 'Run'을 눌러 스크립트를 실행합니다. Stage 트리에 새 프림 /action_graph가 생성되는 것을 확인할 수 있습니다.

  3. Stage에서 프림을 확장하면 "tick"과 "print" 노드가 그래프 아래에 나열됩니다. 이 노드들은 Stage의 다른 프림처럼 접근할 수 있습니다.

  4. "play"를 눌러 시뮬레이션을 시작합니다. 매 프레임마다 콘솔에 "Hello World"가 출력되는 것을 확인할 수 있습니다.

  5. Window > Graph Editors > Action Graph로 이동하여 그래프 에디터를 엽니다.

  6. 오른쪽 Stage 트리에서 새로 생성된 그래프를 선택한 상태에서, 그래프 에디터 창의 'Edit Action Graph' 아이콘을 클릭하여 그래프를 엽니다. 선으로 연결된 두 노드가 표시됩니다.

Editing a Graph#

그래프가 생성되면 그래프의 항목을 조작하는 특정 API가 있습니다.

Getting and Setting Attribute Values

Script Editor에서 다른 탭을 열고 아래 스니펫을 붙여넣고 실행합니다.

import omni.graph.core as og

# get existing value from an attribute
existing_text = og.Controller.attribute("/action_graph/print.inputs:text").get()
print("Existing Text: ", existing_text)

# set new value
og.Controller.attribute("/action_graph/print.inputs:text").set("New Texts to print")

이것은 "Print Text" 노드의 값을 "Hello World"에서 "New Texts to print"로 변경합니다. 그러나 이 변경은 그래프의 첫 번째 틱이 실행될 때까지 적용되지 않습니다. 따라서 스크립트 에디터에서 'Run'을 누르면 그래프가 아직 틱되지 않았으므로 현재 값을 가져와 스크립트 에디터의 콘솔에 "Existing Text: Hello World"라는 단일 문자열을 출력합니다(터미널을 사용하는 경우 터미널에도, "Info"를 포함하면 Omniverse 메인 콘솔에도 출력됩니다).

이제 'Play'를 눌러 시뮬레이션을 시작합니다. 틱당 하나의 문자열 속도로 업데이트된 텍스트 "New Texts to print"가 터미널 또는 Omniverse 메인 콘솔에 출력됩니다(스크립트 에디터의 콘솔에는 아님).

Adding Nodes and Connections

스크립트 에디터에서 세 번째 탭을 열어 기존 그래프에 노드를 추가하고 더 많은 연결을 만듭니다.

import omni.graph.core as og

og.Controller.create_node("/action_graph/new_node_name", "omni.graph.nodes.ConstantString")
og.Controller.attribute("/action_graph/new_node_name.inputs:value").set("This is a new node")
og.Controller.connect("/action_graph/new_node_name.inputs:value", "/action_graph/print.inputs:text")

"new_node_name"이라는 새 노드가 생성되어 "Print Text" 노드에 연결됩니다. 그래프 에디터(Window > Graph Editors > Action Graph)를 열어 두면 두 개가 아닌 세 개의 노드가 서로 연결된 것을 볼 수 있습니다.

Graph Execution#

기본적으로 그래프는 매 프레임마다 평가됩니다. 그래프를 호출할 때만 평가되도록 이 동작을 변경할 수 있습니다.

각 그래프를 명시적으로 호출할 때만 실행되도록 트리거할 수도 있습니다. 이를 위해 그래프를 "On Demand"로 실행하도록 설정할 수 있는 "pipeline_stage"라는 특수 매개변수가 있습니다. 대부분의 경우 그래프 생성 시 이 변수를 설정합니다:

  1. Stage 트리에서 이전 그래프를 선택하고 'Delete' 키를 눌러 삭제합니다.

  2. 스크립트 에디터에서 새 탭을 열고 다음 코드를 붙여넣습니다

    import omni.graph.core as og

keys = og.Controller.Keys
demand_graph_handle, _, _, _ = og.Controller.edit(
    {
        "graph_path": "/ondemand_graph",
        "evaluator_name": "execution",
        "pipeline_stage": og.GraphPipelineStage.GRAPH_PIPELINE_STAGE_ONDEMAND,
    },
    {
        keys.CREATE_NODES: [("tick", "omni.graph.action.OnTick"), ("print", "omni.graph.ui_nodes.PrintText")],
        keys.SET_VALUES: [("print.inputs:text", "On Demand Graph"), ("print.inputs:logLevel", "Warning")],
        keys.CONNECT: [("tick.outputs:tick", "print.inputs:execIn")],
    },
)

  3. 스크립트 에디터에서 'Run'을 누릅니다. 새 그래프 /ondemand_graph가 생성됩니다.

  4. "play"를 눌러 시뮬레이션을 시작합니다. 명시적으로 평가를 호출하지 않았으므로 이 그래프에서는 아무것도 출력되지 않습니다.

  5. 그래프를 수동으로 트리거하려면 다른 탭을 열고 demand_graph_handle.evaluate()를 붙여넣습니다

  6. 시뮬레이션이 여전히 실행 중인지 확인합니다. 스크립트 에디터에서 'Run'을 클릭합니다. 콘솔에 "On Demand Graph"가 한 번 출력되는 것을 확인할 수 있습니다.

대안으로, 기존 그래프에 대해 demand_graph_handle.change_pipeline_stage(og.GraphPipelineStage.GRAPH_PIPELINE_STAGE_ONDEMAND)를 사용하여 설정할 수도 있습니다

물리 콜백 및/또는 렌더링 콜백에 그래프를 연결하는 더 심층적인 예제는 standalone_examples/api/isaacsim.core.experimental.api/omnigraph_triggers.py에서 찾을 수 있습니다
