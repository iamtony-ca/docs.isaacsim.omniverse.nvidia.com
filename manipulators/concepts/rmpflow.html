RMPflow#

Deprecated

새로운 개발을 위해서는 Lula보다 개선된 인터페이스와 추가 기능을 제공하는 최신 Robot Motion (Experimental) API 사용을 고려하세요.

Riemannian Motion Policy (RMP)는 대부분의 Isaac Sim 매니퓰레이터 제어의 기반이 되는 모션 생성 도구 세트입니다. 지능적인 충돌 회피로 로봇을 위한 부드러운 궤적을 생성합니다.

리만 모션 폴리시(Riemannian Motion Policy), 또는 RMP는 때로는 관성 행렬이라고 불리는 행렬 \(M(q, \dot{q})\)를 동반하는 가속도 폴리시로, 고전 역학에서 용어를 빌려왔지만 리만 계량(Riemannian metric) 개념과도 밀접하게 관련됩니다.

리만 기하학의 기법을 활용하여, RMPflow는 여러 (경쟁할 수 있는) 목표와 제약 조건을 나타내는 RMP들을 단일 전역 가속도 폴리시로 결합하는 프레임워크입니다. 이 프레임워크 내에서 로컬 RMP는 임의의 수의 중간 작업 공간(엔드 이펙터의 작업 공간 포함, 작업 공간 제어를 일반화)에서 정의될 수 있습니다. 세부 사항은 *RMPflow: A computational graph for automatic motion policy generation*를 참조하세요.

넓게 정의하면, 모션 폴리시는 로봇의 현재 상태(예: 일반화 좌표계에서의 위치 및 속도)를 받아 그 상태의 원하는 변화를 나타내는 양을 반환하는 수학적 함수입니다. 이러한 폴리시는 하나 이상의 목표나 제약 조건, 환경 상태를 나타내는 변수에 암묵적으로 의존할 수 있습니다. 가속도 폴리시는 출력이 원하는 가속도 \(\ddot q = \pi(q, \dot{q})\)인 모션 폴리시로, 2차 미분 방정식을 발생시킵니다.

위치 또는 속도 제어로 로봇을 제어하는 목적으로, 일반적으로 출력이 위치 또는 속도인 모션 폴리시가 사용됩니다. 이러한 폴리시는 오일러 적분과 같은 수치 적분 방식을 사용하여 가속도 폴리시로부터 생성될 수 있습니다.

RMPflow Debugging Features 섹션은 MotionPolicy 인터페이스의 일부가 아닌 RMPflow에 속하는 함수들을 검토합니다. 이미 지원되는 로봇을 제어하기 위해 RMPflow와 상호 작용할 수 있습니다. RMPflow의 내부 메커니즘에 관심이 있거나 지원되지 않는 로봇에 대해 RMPflow를 설정하려는 경우, RMPflow 문서를 계속 읽으세요.

여기에서 기본 사항을 검토한 후, 새 로봇을 위한 RMPflow 설정에 대한 실용적인 조언을 위해 RMPflow Tuning Guide도 참조하세요.

RMPflow Debugging Features#

RmpFlow 인스턴스와 직접 상호 작용함으로써, 다른 MotionPolicy 구현에서 사용할 수 없는 기능에 접근할 수 있습니다. 개발자들이 NVIDIA Isaac Sim에서 시뮬레이션 로봇 Articulation에서 Motion Policy Algorithm을 분리하고자 하는 것은 일반적입니다. 예를 들어, 시뮬레이션 로봇이 느리게 움직일 때, MotionPolicy 또는 PD 게인이 부적절하게 튜닝되었는지 판단하는 것이 중요하지만, PD 게인과 MotionPolicy 모두 로봇 조인트를 구동하는 역할을 할 때 이것은 어려울 수 있습니다 (Outputs: Robot Joint Targets 참조).

RMPflow는 스테이지의 일부로 알고리즘의 내부 상태를 명확하게 보여주는 시각화 함수를 제공합니다. RMPflow는 내부적으로 월드의 장애물을 피하기 위해 충돌 구를 사용합니다. 이 구들은 RmpFlow.visualize_collision_spheres()를 호출하여 시간에 따라 시각화할 수 있습니다. RmpFlow.stop_visualizing_collision_spheres()가 호출되면 시각화가 중지됩니다. 공칭 엔드 이펙터 위치는 마찬가지로 RmpFlow.visualize_end_effector_position()과 RmpFlow.stop_visualizing_end_effector()으로 시각화할 수 있습니다.

시각화 함수 자체만으로는 RMPflow의 로봇 내부 표현이 합리적인지 확인하는 데 사용될 수 있지만, 시뮬레이션 로봇을 RmpFlow 내부 로봇 표현에서 분리하는 데는 도움이 되지 않습니다.

각 프레임에서 RmpFlow.compute_joint_targets(active_joint_positions,...)가 호출될 때, 시각화가 active_joint_positions를 사용하도록 업데이트됩니다. 이 동작은 RmpFlow.set_ignore_state_updates(True)를 사용하여 끌 수 있습니다. RmpFlow가 "상태 업데이트를 무시"할 때, active_joint_positions 인수를 무시하기 시작하고, 대신 로봇의 물리 시뮬레이션과 완전히 독립적이라고 가정하여 로봇의 예상 상태를 내부적으로 추적하기 시작합니다. RmpFlow가 시뮬레이터에서 상태 업데이트를 무시하도록 설정되고 시각화 함수가 사용되면, 바람직하지 않은 로봇 동작이 RmpFlow에서 비롯된 것인지 로봇 Articulation과 그 PD 게인에서 비롯된 것인지 쉽게 판단할 수 있습니다.

RMPflow Configuration#

새로운 로봇에 RMPflow를 설정하려면 세 개의 파일이 필요합니다:

  • URDF(범용 로봇 설명 파일) - 로봇 운동학 지정에 사용됩니다.

    조인트 및 링크 이름과 함께. 각 조인트의 위치 제한도 필요합니다. URDF의 다른 속성들은 무시되며 생략할 수 있습니다. 여기에는 질량, 관성 모멘트, 시각적, 충돌 메시가 포함됩니다.

  • 보조 로봇 설명 파일 (YAML 형식). 로봇의 구성 공간(c-space)을 정의하는

    작동 조인트 목록을 열거하는 것 외에도, 이 파일에는 기본 c-space 구성과 충돌 회피에 사용되는 충돌 구 세트를 지정하는 섹션이 포함됩니다. 이 파일은 비작동 조인트의 고정 위치를 지정하는 데도 사용할 수 있습니다.

  • YAML 형식의 RMPflow 설정 파일 - 활성화된 모든 RMP에 대한 파라미터를 포함합니다.

일반적인 수학적 프레임워크로서, RMPflow는 개별 RMP가 취해야 하는 형태를 규정하지 않습니다. Lula(및 이를 확장한 NVIDIA Isaac Sim)의 특정 RMPflow 구현은 다양한 조작 작업에서 부드럽고 반응적인 동작을 생성하는 것으로 경험적으로 밝혀진 미리 지정된 RMP 세트를 노출합니다.

C-Space Target RMP (c-space_target_rmp)#

목적: 로봇의 기본 c-space 구성을 지정하며, 중복성 해결에 사용됩니다.

정의: 이 RMP의 가속도는 위치 게인과 감쇠 게인을 가진 PD 컨트롤러와 유사한 방정식으로 주어지지만, C-space 거리가 임계값을 초과할 때 위치 항의 크기가 제한됩니다. 이 제한은 구성이 목표에서 멀리 있을 때 과도한 힘을 방지합니다. \(q\)를 전체 구성 벡터로 정의하면:

\[\ddot q = k_p r(q_0 - q) - k_d \dot q\,,\]

여기서 "강건 제한 함수" \(r(p)\)는 다음과 같이 주어집니다:

\[\begin{split}r(p) = \left \{ \begin{array}{cl} p, & ||p|| < \theta \\ \theta\, p / ||p|| & \textrm{otherwise.} \end{array} \right.\end{split}\]

관성 행렬은 항등 행렬에 비례합니다:

\[M = \mu I\]

c-space_target_rmp 섹션의 RMPflow 설정 파일에는 추가 inertia 파라미터 \(m\)이 포함됩니다. 이 파라미터가 0이 아닌 경우, 관성 행렬이 \(M = mI\)인 제로 c-space 가속도 \(\ddot q = 0\)에 해당하는 개념적으로 별개의 RMP 도입으로 이어집니다.

Target RMP (target_rmp)#

목적: 엔드 이펙터를 지정된 위치 목표로 이동시킵니다.

정의: c-space 목표 RMP와 유사하게, 이 RMP의 가속도는 PD 컨트롤러와 유사하지만, 위치 보정 벡터의 크기를 제한하는 전략이 약간 다릅니다.

\[\ddot x = k_p (x_0 - x) / (||x_0-x|| + \epsilon) - k_d \dot x\]

관성 행렬은 목표 방향 벡터 \(n\)인 순위 결핍 메트릭 \(S = n n^T`\)와 항등 행렬 \(I\) 사이를 혼합합니다.

직관적으로, \(S\)는 목표 방향에만 관심을 갖습니다(다른 RMP들, 예를 들어 장애물 회피 RMP가 직교 방향을 제어하도록 함).

\(I\)는 모든 방향에 관심을 갖습니다.

\(S\)의 기여는 목표에서 멀리 있을수록 더 크며, 장애물이 시스템을 더 효과적으로 밀어낼 수 있게 하고, \(I\)는 목표 근처에서 지배적이 되어 더 빠른 수렴을 유도합니다.

혼합은 방사형 기저 함수, 구체적으로 가우시안으로 제어되며, 목표에서 멀리 있을 때 최소 상수 값에서 목표 근처에서 1로 전환됩니다.

목표 근처에서는 추가적인 비선형 "근접 부스트" 승수가 켜집니다. 이 인수는 가우시안 형태를 취합니다:

\[M = \left[\beta(x) b + (1-\beta(x))\right] \left[\alpha(x) M_\textrm{near} + (1-\alpha(x)) M_\textrm{far} \right]\]

여기서

\[\begin{split}\begin{array}{l} \alpha(x) = (1-\alpha_\textrm{min})\exp \left(\frac{-||x_0-x||^2}{2 \sigma_a^2}\right) + \alpha_\textrm{min} \\ \beta(x) = \exp \left(-\frac{||x_0 - x||^2}{2 \sigma_b^2}\right) \\ M_\textrm{near} = \mu_\textrm{near} I \\ M_\textrm{far} = \mu_\textrm{far} S = \frac{\mu_\textrm{far}}{||x_0-x||^2} (x_0-x)(x_0-x)^T\,. \end{array}\end{split}\]

Axis Target RMP (axis_target_rmp)#

목적: 엔드 이펙터 프레임의 x-, y-, 또는 z-축을 목표 방향으로 이동시킵니다. 이 RMP는 일반 방향 목표(세 축 각각에 대한 축 목표 RMP가 추가됨)와 단일 축의 정렬만 원하는 "부분 포즈" 목표에 사용됩니다.

Note

부분 포즈 목표는 Motion Generation 익스텐션에서 지원되지 않습니다.

정의:

(위치) 목표 RMP와 유사하게, 축 목표 RMP는 "근접 부스팅"을 지원하지만, 동시에 목표 RMP가 활성화된 경우에만 지원됩니다. 이 경우, 부스팅 강도를 제어하는 것은 위치 목표까지의 거리 (\(||x_0-x||\))입니다.

현재 및 원하는 축 방향은 각각 \(n\)과 \(n_0\)으로 표시되는 단위 벡터로 나타냅니다. 가속도는 다음과 같이 주어집니다:

\[\ddot n = k_p (n_0 - n) - k_d \dot n\,\]

위치 목표(즉, 목표 RMP)가 활성화된 경우, 메트릭은 다음 형태를 갖습니다:

\[M_\textrm{boosted} = \left[\beta(x) b + (1-\beta(x))\right] \mu I\,\]

여기서:

\[\beta(x) = \exp \left(-\frac{||x_0 - x||^2}{2 \sigma_b^2}\right)\,\]

위치 목표가 활성화되지 않은 경우, 다음과 같이 단순화됩니다:

\[M = \mu I\,.\]

Joint Limit RMP (joint_limit_rmp)#

목적: 조인트 제한을 회피합니다.

정의: 이것은 단일 c-space 좌표(조인트)와 로봇에 대한 URDF에 지정된 해당 상한 또는 하한 조인트 제한에 의존하는 1차원 RMP입니다. 로봇이 \(N\)개의 조인트를 가진 경우, 총 \(2N\)개의 조인트 제한 RMP가 도입됩니다. URDF에 지정된 조인트 제한은 RMPflow 설정 파일의 joint_limit_buffers 배열에 양수 패딩 값을 입력하여 더 보수적으로 만들 수 있습니다. 주어진 조인트에 대해, 동일한 패딩 값이 상한 및 하한 제한 모두에 사용됩니다.

이 RMP의 작업 공간은 이동되고 스케일된 c-space 좌표로 구성되며, 상한 또는 하한 조인트 제한까지의 스케일된 거리를 측정합니다. 일반성을 잃지 않고 하한 조인트 제한 RMP를 고려합니다. \(q\)가 주어진 조인트의 c-space 좌표이고, \(q_\textrm{upper}\)와 \(q_\textrm{lower}\)가 해당 조인트의 상한 및 하한 제한인 경우, 다음과 같이 정의합니다:

\[x = \frac{q - q_\textrm{lower}}{q_\textrm{upper} - q_\textrm{lower}}\,\]

해당 좌표의 가속도는 다음과 같이 주어집니다:

\[\ddot x = \frac{k_p}{x^2/\ell_p^2 + \epsilon_p} - k_d \dot x\,\]

메트릭(관성 행렬)은 다음과 같이 주어지는 스칼라입니다:

\[m = \left(1 - \frac{1}{1+\exp(-\dot x/v_m)}\right) \frac{\mu}{x/\ell_m + \epsilon_m}\,\]

Joint Velocity Limit RMP (joint_velocity_cap_rmp)#

목적: 최대 조인트 속도를 제한합니다.

정의: 이 RMP는 주어진 조인트의 속도 크기가 지정된 제한에 접근할 때 감쇠를 적용합니다.

이것은 다음과 같이 주어지는 가속도를 가진 1차원 RMP입니다:

\[\ddot q = -k_d\,\textrm{sgn}(\dot q) \left(|\dot q| - (v_\textrm{max} - v_r)\right)\,\]

메트릭(관성 행렬)은 다음과 같이 주어지는 스칼라입니다:

\[\begin{split}m = \left \{ \begin{array}{cl} 0, & |\dot q| < (v_\textrm{max} - v_r) \\ \frac{\mu}{1 - \left(|\dot q| - (v_\textrm{max} - v_r)\right)^2 / v_r^2} & \textrm{otherwise.} \end{array} \right\end{split}\]

메트릭은 사용자가 지정한 감쇠 영역 밖에서는 0이므로, 이 RMP를 비활성화합니다. 또한, \(\dot q\)가 \(v_\textrm{max}\)에 접근할 때 메트릭의 발산을 방지하기 위해 클리핑이 적용됩니다.

파라미터:

\(q\)가 라디안으로 표현되는 회전 조인트를 가정합니다. 조인트가 프리즘형인 경우, max_velocity와 velocity_damping_region의 단위는 m/s가 됩니다.

Collision Avoidance RMP (collision_rmp)#

목적: 환경의 장애물과의 충돌을 회피합니다.

정의: 이것은 작업 공간이 로봇의 특정 충돌 구(로봇 설명 YAML 파일에 지정됨)에서 환경의 장애물까지의 거리를 측정하는 단일 좌표로 구성된 1차원 RMP입니다. 해당 좌표를 \(x\)로 표시하면, 가속도는 다음과 같이 주어집니다:

\[\ddot x = k_p \exp(-x / \ell_p) - k_d \left[1 - \frac{1}{1 + \exp(-\dot x/v_d)} \right] \frac{\dot x}{x/\ell_d + \epsilon_d}\,\]

메트릭(관성 행렬)은 다음과 같이 주어지는 스칼라입니다:

\[m = \left[1 - \frac{1}{1 + \exp(-\dot x/v_d)} \right] g(x) \frac{\mu}{x / \ell_m + \epsilon_m}\,\]

여기서 \(g(x)\)는 \(x\)가 0에서 \(r\)로 변화함에 따라 1에서 0으로 부드럽게 변하는 구분적 다항식입니다

\[\begin{split}g(x) = \left \{ \begin{array}{cl} x^2/r^2 -2s/r + 1, & x\le r \\ 0, & x\gt r \end{array} \right.\end{split}\]

Damping RMP (damping_rmp)#

목적: 제어 프레임(예: 엔드 이펙터)의 목표에 대한 속도를 기반으로 추가적인 비선형 감쇠를 기여합니다.

정의: 이것은 제어 프레임의 원점에서 목표까지의 거리를 측정하는 단일 좌표 \(x\)로 작업 공간이 구성된 1차원 RMP입니다. 가속도는 다음과 같이 주어집니다:

\[\ddot x = -k_d |\dot x|\dot x\]

그리고 메트릭은 다음과 같습니다:

\[M = \mu |\dot x| I\,\]

damping_rmp 섹션의 RMPflow 설정 파일에는 추가 inertia 파라미터 \(m\)이 포함됩니다. 이 파라미터가 0이 아닌 경우, 관성 행렬이 \(M = mI\)인 제로 가속도 \(\ddot x = 0\)에 해당하는 개념적으로 별개의 RMP 도입으로 이어집니다.
