Known Issues#

General#

  1. 일부 Windows 시스템에서는 다음과 같은 오류가 발생할 수 있습니다:

    OSError: [WinError 126] The specified module could not be found. Error loading "C:\path\to\omni.isaac.ml_archive\pip_prebundle\torch\lib\fbgemm.dll" or one of its dependencies.

    이 문제는 빌드 도구가 없어서 발생합니다. 해결하려면 Visual Studio 2022를 설치한 후 Visual Studio 인터페이스를 통해 MSVC v143 - VS 2022 c++ x64/86 build tools를 설치하세요.

  2. Replicator의 Scatter3D OmniGraph 노드는 world를 사용하는 스테이지에서 호출될 때 물리 시뮬레이션을 중단시킵니다.

  3. NVIDIA Isaac Sim을 헤드리스(headless)로 실행하여 원격 클라이언트에 연결한 상태에서 종료 시 다음 오류가 발생할 수 있으며, 무시해도 됩니다:

    [ext: omni.physx] shutdown
Fatal Python error: Segmentation fault

Thread 0x00007f46f8faa740 (most recent call first):
File "..._build/target-deps/kit_sdk_release/_build/linux-x86_64/release/extsPhysics/omni.physx/omni/physx/scripts/extension.py", line 30 in on_shutdown
File "..._build/target-deps/kit_sdk_release/_build/linux-x86_64/release/plugins/bindings-python/omni/ext/impl/_internal.py", line 225 in shutdown_all
File "..._build/target-deps/kit_sdk_release/_build/linux-x86_64/release/plugins/bindings-python/omni/ext/impl/_internal.py", line 261 in shutdown_all_extensions
File "..._build/target-deps/kit_sdk_release/_build/linux-x86_64/release", line 3 in <module>

  4. 윈도우 컨테이너에서 실행 시 다음 오류들은 무시할 수 있으며, 잠시 기다리면 앱이 계속 실행됩니다:

    ERROR: Could not find a version that satisfies the requirement psutil (from versions: none)
ERROR: No matching distribution found for psutil
WARNING: Retrying (Retry(total=4, connect=None, read=None, redirect=None, status=None)) after connection broken by 'NewConnectionError('<pip._vendor.urllib3.connection.VerifiedHTTPSConnection object at 0x7fcdcc284dd8>: Failed to establish a new connection: [Errno -3] Temporary failure in name resolution',)': /simple/psutil/``

  5. Ctrl-C로 독립형(standalone) Python 스크립트를 종료할 때 두 번 눌러야 할 수 있습니다.

  6. URDF에서 둘 이상의 에셋이 동일한 머티리얼 이름을 포함하면 하나의 머티리얼만 생성됩니다. 파라미터가 달라도 마찬가지입니다. 예를 들어 두 메시에 "material"이라는 이름의 머티리얼이 있고 하나는 파란색, 다른 하나는 빨간색이면, 두 메시 모두 빨간색이나 파란색 중 하나로 표시됩니다. 텍스처 머티리얼에도 동일하게 적용됩니다.

  7. MJCF 임포터는 파일 선택 대화상자에 내장 북마크를 표시하지 않습니다. 북마크는 콘텐츠 창에서 계속 사용할 수 있으며 파일 선택 대화상자에 복사하여 붙여넣을 수 있습니다.

  8. Windows에서 검은 화면이 표시되면 시작 시 --vulkan 커맨드 라인 인수를 사용하세요.

  9. 핀홀이 아닌 어안 렌즈 카메라에는 Debug Visualization이 표시되지 않습니다. 해당 기능이 구현되지 않았기 때문입니다.

  10. 사용 가능한 VRAM을 초과하는 뷰포트 해상도를 지정하면 애플리케이션에서 ERROR_OUT_OF_DEVICE_MEMORY 오류가 발생합니다. 이후 해상도를 낮은 값으로 줄이면 크래시가 발생할 수 있습니다.

  11. OmniGraph와 함께 isaacsim.core.api의 World 또는 SimulationContext를 사용할 때, World 또는 SimulationContext가 초기화되기 전에 그래프를 생성해야 합니다.

  12. replicator의 rep.new_layer() 기능(에셋을 배치하고 무작위화할 새 레이어를 생성)을 사용하면 해당 에셋을 사용하는 시뮬레이션 시나리오에서 문제가 발생할 수 있습니다. 이 경우 rep.new_layer() 사용을 생략할 수 있습니다.

  13. Python을 통해 실행할 때 물리 오브젝트를 선택한 후 화면의 다른 오브젝트를 선택하면 하나 이상의 omni.kit.manupulator 오류가 발생할 수 있습니다. 이 오류는 코드 실행에 영향을 미치지 않으므로 무시해도 됩니다.

  14. OmniGraph 노드에서 기본값으로 에셋을 드래그 앤 드롭한 경우 Play를 누르기 전에 씬과 함께 저장하고 다시 로드하여 모든 값이 올바르게 설정되었는지 확인하세요.

  15. UI에서 ./isaac-sim.streaming.sh 종료 시 오류가 발생합니다.

    [Error] [carb.livestream-rtc.plugin] nvstPushStreamData timeout for eye 0, stream 0000000000000000.

  16. ROS 동기화를 사용하는 Cortex 샘플은 실행 FPS가 25 FPS 미만으로 떨어지면 비정상적으로 동작하거나 작업을 실행하지 못할 수 있습니다.

  17. 합성 데이터 생성을 위한 무작위화 머티리얼이 제때 로드되지 않으면 rt_subframes를 최소 2 이상으로 설정해야 합니다.

  18. 평행 메커니즘을 가진 일부 그리퍼(Robotiq 2F-85, 2F-C2 등)는 나머지 그리퍼와 함께 움직이지 않는 링크가 있습니다.

  19. Multiple Robot ROS2 Navigation 튜토리얼은 CPU 사용률이 높습니다. 로봇이 충돌하거나 위치 파악 문제가 발생한다면, Nav2 스택이 센서 데이터와 제대로 동기화하지 못해 컨트롤러 명령을 놓치는 것이 원인일 가능성이 높습니다.

  20. Isaac Sim 실행 중 많은 경고와 메시지가 표시될 수 있습니다. 다음 커맨드 라인 인수를 사용하여 로그 출력을 줄일 수 있습니다:

    --/log/level=error --/log/fileLogLevel=error --/log/outputStreamLevel=error

  21. Windows에서 Replicator를 사용하여 내장 백엔드로 S3 버킷에 쓸 때, AWS 설정 파일 대신 환경 변수에 자격 증명을 설정해야 할 수 있습니다. 이는 Windows의 boto3에서 경로 파싱 오류가 발생할 수 있기 때문입니다.

  22. XR 익스텐션은 Windows에서 제대로 작동하지 않습니다.

  23. Windows에서 독립형 예제를 실행할 때, 일부 시나리오에서 애플리케이션 종료 시 스레드가 제대로 정리되지 않을 수 있습니다. 애플리케이션은 정상적으로 종료되므로 일반적으로 무시해도 됩니다. 우회 방법으로는 standalone_app.close()를 호출하기 전에 standalone_app.update()를 여러 번 추가하세요.

    Windows fatal exception: access violation

Thread 0x00000634 (most recent call first):

  24. ROS 2 QoS Profile OmniGraph 노드는 다른 필드를 업데이트하기 전에 createProfile 입력을 수동으로 "Custom"으로 변경하지 않으면 커스텀 프로파일을 저장할 수 없습니다.

  25. 일부 멀티 GPU 시스템에서 렌더 프로덕트를 생성하면 메인 뷰포트가 검게 변할 수 있지만, 렌더 프로덕트는 계속 올바르게 동작합니다.

  26. USD to URDF Exporter:

    • 콜라이더 메시가 비주얼에 잘못 포함될 수 있습니다. URDF 파일에서 수동으로 제거할 수 있습니다.

    • Body와 Joint는 URDF 파일에서 알파벳 순서로 작성됩니다. URDF 파일에서 수동으로 순서를 변경할 수 있습니다.

    • 로봇 구조에 따라 서로 다른 프레임이 병합되면서 일부 바디 이름이 덮어씌워질 수 있습니다. 출력을 검토하여 정확한지 확인하세요.

    • URDF 익스포터는 제한이 없는 경우 조인트 노력(effort) 및 속도 제한을 inf로 추가합니다. URDF 파서가 Float에서 inf 값을 지원하지 않으면 URDF가 올바르게 임포트되지 않을 수 있습니다.

  17. omni.replicator.character 또는 omni.replicator.agent 의존성 관련 문제가 발생하면, 해당 익스텐션이 isaacsim.replicator.agent로 이름이 변경되었습니다. 코드를 이에 맞게 수정하세요.

  18. RTX 센서 저작 클래스(isaacsim.sensors.experimental.rtx.Lidar, isaacsim.sensors.experimental.rtx.Radar, 또는 isaacsim.sensors.experimental.rtx.Acoustic)에서 aux_output_level을 설정하면 로그에 다음 경고가 나타날 수 있습니다. 이는 무해합니다. usdrt Fabric 캐시는 VtArray<std::string> 속성을 미러링하지 않지만, 속성은 USD prim에 올바르게 설정되어 Replicator 파이프라인에서 읽힙니다.

    [Warning] [usdrt.population.plugin] [UsdNoticeHandler] Unhandled attribute type VtArray<std::string> (prim attribute: _replicator:rendervar:GenericModelOutput:channels)

  19. Ubuntu에서 omni.sensors.nv.lidar.lidar_core.plugin의 CUDA 드라이버 오류는 시스템 수준의 CUDA 설치와 omni.sensors 런타임 컴파일 라이브러리 간의 불일치로 인해 발생할 수 있습니다.

    [Error] [omni.sensors.nv.lidar.lidar_core.plugin] CUDA Driver CALL FAILED at line 522: the provided PTX was compiled with an unsupported toolchain.
[Error] [omni.sensors.nv.lidar.lidar_core.plugin] CUDA Driver CALL FAILED at line 548: named symbol not found

    한 가지 해결 방법은 다음과 같이 LD_LIBRARY_PATH 환경 변수를 설정하는 것입니다. </path/to/isaac_sim_installation>을 로컬 Isaac Sim 설치 경로로 교체하세요.

    export LD_LIBRARY_PATH=</path/to/isaac_sim_installation>/extscache/omni.sensors.nv.common-2.5.0-coreapi+lx64.r.cp310/bin:$LD_LIBRARY_PATH

  20. 마임(mimic) 조인트가 있는 로봇에서 Physics Inspector의 "failed to find internal joint" 오류는 마임 조인트 기능에 영향을 주지 않으므로 무시해도 됩니다.

    [Error] [omni.physx.plugin] Usd Physics: failed to find internal joint object for PhysxMimicJointAPI at /Franka/panda_hand/panda_finger_joint2. Please ensure that the prim is a supported joint type and is part of an articulation.

  21. Windows에서 코드 (error = 206)와 함께 발생하는 omni.kit.telemetry 익스텐션 시작 오류는 파일 경로가 길이 제한을 초과하여 발생합니다. omni.telemetry.transmitter.exe의 파일 경로가 260자를 초과하지 않는지 확인하세요.

  22. 앱이 시작된 후 Windows fatal exception: int divide by zero 오류 메시지가 발생하면, MSI Afterburner와 같은 GPU 오버클럭 소프트웨어가 원인일 수 있습니다. 해당 소프트웨어를 비활성화하여 문제를 해결해 보세요.

  23. tkinter와 관련된 Python 오류는 사용자가 Isaac Sim에 내장된 Python 배포판과 함께 tkinter를 사용하려 하기 때문입니다. 이는 지원되지 않습니다.

    File "/path/to/isaac_sim/installation/kit/python/lib/python3.11/tkinter/__init__.py", line 38, in <module>
    import _tkinter # If this fails your Python may not be configured for Tk
    ^^^^^^^^^^^^^^^
ModuleNotFoundError: No module named '_tkinter'

  24. depth sensor AOV 사용 시 오류가 발생합니다. AOV 번호(예: 아래의 38)는 선택한 AOV에 따라 달라질 수 있습니다.

    [Error] [rtx.postprocessing.plugin] DepthSensor: Texture sizes do not match: inColorTexDesc 1920x1080x1:11@0 inDepthTexDesc 1500x843x1:33@0
[Error] [rtx.postprocessing.plugin] DepthSensor: Failed to allocate view resources for view 1 device 0
[Error] [carb.scenerenderer-rtx.plugin] Failed to export AOV 38 to render product. The renderer did not generate the AOV texture

Hang#

  1. Windows에서 익스텐션 매니저를 사용할 때 의존성 창을 클릭하면 뷰포트가 검게 변하고 멈출 수 있습니다.

  2. Firefox에서 WebRTC 클라이언트는 Play 버튼을 클릭하면 몇 초 후에 멈출 수 있습니다. Google Chrome 또는 Chromium 브라우저를 사용하는 것을 권장합니다.

  3. Nucleus 로그인을 위한 브라우저 팝업이 로그인을 완료하기 전에 닫히면 Isaac Sim이 멈출 수 있습니다. Isaac Sim을 강제 재시작해야 합니다.

Crash#

  1. OmniGraph에서 컴파운드 노드를 사용하면 크래시가 발생할 수 있습니다. OmniGraph에서 컴파운드 노드 사용을 권장하지 않습니다.

  2. Python 가비지 컬렉터가 관련 오브젝트를 정리하기 전에 physics.tensors 익스텐션을 종료하면 크래시가 발생할 수 있습니다. 이를 방지하려면 익스텐션을 언로드하기 전에 Python에서 관련 tensor API 오브젝트를 수동으로 None으로 설정하세요.

  3. 멀티 GPU(MGPU) 시스템에서 RTX Lidar를 사용하면 로그에 CUDA 오류 700 메시지와 함께 치명적인 앱 크래시가 발생할 수 있습니다. 우회 방법으로는 멀티 GPU 렌더링을 비활성화하여 Isaac Sim을 실행하세요:

    ./isaac-sim.sh --/renderer/multiGpu/enabled=false

    기타 멀티 GPU 설정은 Multi-GPU를 참조하세요.
