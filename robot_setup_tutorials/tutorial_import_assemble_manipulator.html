Tutorial 6: Setup a Manipulator#

Learning Objectives#

이것은 4개의 튜토리얼 시리즈 중 첫 번째 매니퓰레이터 튜토리얼입니다. 이 튜토리얼에서는 URDF 파일에서 Universal Robots의 UR10e 로봇과 Robotiq의 2F-140 그리퍼를 NVIDIA Isaac Sim에 가져오고 하나의 아티큘레이션 아래 연결하는 방법을 보여줍니다.

30분 튜토리얼

Isaac Sim은 항상 Python 3.12를 사용하므로 이 튜토리얼에서 사용되는 UR 설명 패키지와 모든 ROS 패키지는 Python 3.12 환경에서 사용 가능해야 합니다. 패키지를 가져오는 방법은 플랫폼에 따라 다릅니다:

  • Ubuntu 24.04 + ROS 2 Jazzy — 사전 빌드된 ros-jazzy-ur-description apt 패키지를 설치하세요. 시스템 Python(3.12)이 Isaac Sim과 이미 일치합니다.

  • Ubuntu 22.04 + ROS 2 Humble 또는 Jazzy — 시스템 Python이 3.10이므로 포함된 build_ros.sh 스크립트를 사용하여 Python 3.12에 대해 워크스페이스를 클론하고 재빌드해야 합니다.

  • Windows + Pixi 기반 ROS 2 Jazzy — Pixi 환경에 UR 설명 패키지를 추가하세요(pixi add ros-jazzy-ur-description). Pixi 관리 ROS 2 Jazzy는 이미 Python 3.12에서 실행됩니다. Pixi 설정은 ROS 2 Installation (Other Platforms)를 참조하세요. WSL2는 ROS 기반 URDF 가져오기 워크플로우에 지원되지 않습니다 — 대신 콘텐츠 브라우저의 사전 빌드된 USD 파일을 사용하세요.

Attention

Windows에서 ROS 2 Humble(Pixi)은 이 튜토리얼에서 지원되는 구성이 아닙니다. Windows에서는 Pixi를 사용하는 ROS 2 Jazzy만 지원됩니다. Windows에서 ROS 2 Jazzy로 전환하거나 Linux 구성으로 이동하여 이 튜토리얼을 그대로 따라하세요.

이 페이지 상단의 Build Environment 배너에서 구성을 확인하거나 선택하여 설정 단계를 확인하세요. 선택 사항은 이 페이지 나머지 부분의 플랫폼별 명령을 결정합니다.

Prerequisites#

  • NVIDIA Isaac Sim을 처음 사용하는 경우, 이 튜토리얼을 시작하기 전에 Wheeled Robot Set Up Tutorials 튜토리얼을 완료하세요.

  • 이 튜토리얼을 시작하기 전에 ROS 2 설치 ROS 2 Installation (Default)를 검토하세요.

  • URDF 임포터 URDF Importer Extension 튜토리얼을 검토하세요.

  • ROS가 소싱된 터미널에서 선택한 구성에 맞는 xacro를 설치하세요(페이지 상단의 Build Environment 배너 참조):

    sudo apt install ros-$ROS_DISTRO-xacro

    pixi add ros-$ROS_DISTRO-xacro

    Attention

    Windows에서 ROS 2 Humble(Pixi)은 지원되는 구성이 아닙니다. Windows에서 ROS 2 Jazzy로 전환하거나 Linux 구성으로 이동하여 이 튜토리얼을 그대로 따라하세요.

  • 콘텐츠 브라우저의 Isaac Sim/Samples/Rigging/Manipulator/import_manipulator/ 경로에서 import_manipulator 폴더를 찾습니다.

Build and Install the UR Description Package#

이 페이지 상단의 Build Environment 선택기에서 선택한 구성에 대한 단계를 따르세요.

사전 빌드된 UR 설명 패키지를 설치하고 ROS 2 Jazzy를 소싱합니다:

sudo apt install ros-jazzy-ur-description
source /opt/ros/jazzy/setup.bash

그런 다음 같은 터미널에서 Isaac Sim을 실행합니다:

./isaac-sim.sh

Ubuntu 22.04에서는 시스템 Python(3.10)이 Isaac Sim에서 사용하는 Python 3.12와 일치하지 않으며 UR 설명 패키지를 Python 3.12에서 기본적으로 사용할 수 없습니다. 포함된 build_ros.sh 스크립트로 패키지를 클론하고 재빌드하세요.

Note

ROS 워크스페이스에 사용자 정의 ROS 2 패키지를 설정하는 방법에 대한 자세한 내용은 Isaac Sim ROS Workspaces를 참조하세요.

  1. Isaac Sim ROS Workspace로 이동한 후 배포별 워크스페이스의 src 폴더로 이동합니다:

    cd <path to Isaac Sim ROS Workspace>
cd jazzy_ws/src

    cd <path to Isaac Sim ROS Workspace>
cd humble_ws/src

  2. ROS 배포와 일치하는 Universal Robots ROS 2 Description 저장소의 브랜치를 클론합니다:

    git clone --branch jazzy https://github.com/UniversalRobots/Universal_Robots_ROS2_Description.git

    git clone --branch humble https://github.com/UniversalRobots/Universal_Robots_ROS2_Description.git

  3. Isaac Sim ROS Workspace 루트로 돌아가서 Python 3.12에 대해 빌드합니다:

    cd ../..
./build_ros.sh

  4. Python 3.12 ROS 환경을 소싱하고 Isaac Sim을 실행합니다.

    source build_ws/jazzy/jazzy_ws/install/local_setup.bash
source build_ws/jazzy/isaac_sim_ros_ws/install/local_setup.bash
./isaac-sim.sh

    source build_ws/humble/humble_ws/install/local_setup.bash
source build_ws/humble/isaac_sim_ros_ws/install/local_setup.bash
./isaac-sim.sh

Attention

Ubuntu 24.04에서 ROS 2 Humble은 ROS 2 Installation (Default)에서 공식 지원되는 구성이 아닙니다. Ubuntu 24.04에서 ROS 2 Jazzy로 전환하거나 Ubuntu 22.04에서 ROS 2 Humble로 이동하여 이 튜토리얼을 그대로 따라하세요.

Windows에서 이 튜토리얼의 URDF 가져오기 워크플로우는 Pixi 기반 ROS 2 Jazzy 설치에서만 지원됩니다. Windows ROS 2 설정과 Pixi 환경에 대한 UR 설명 패키지 설치 또는 빌드는 ROS 2 Installation (Other Platforms)를 따르세요. WSL2를 사용하는 경우 ROS 기반 가져오기 단계를 건너뛰고 콘텐츠 브라우저의 Isaac Sim/Samples/Rigging/Manipulator/import_manipulator/에서 사전 빌드된 USD 파일을 사용하세요.

Attention

Windows에서 ROS 2 Humble(Pixi)은 ROS 2 Installation (Default)에서 지원되는 구성이 아닙니다. Windows에서 ROS 2 Jazzy로 전환하거나 Linux 구성으로 이동하여 이 튜토리얼을 그대로 따라하세요. ROS 없이 Windows에서 UR10e를 사용해야 하는 경우 콘텐츠 브라우저의 Isaac Sim/Samples/Rigging/Manipulator/import_manipulator/에서 사전 빌드된 USD 파일을 사용하세요.

Import the UR10e Robot#

Enable the ROS 2 Robot Description URDF Importer Extension#

  1. Window > Extensions로 이동합니다.

  2. 검색창에 URDF를 입력하고 ROS 2 Robot Description URDF Importer Extension을 찾습니다.

  3. 찾을 수 없는 경우 검색창에서 @feature 필터를 제거하세요.

  4. 여전히 찾을 수 없는 경우 Isaac Sim이 ROS가 소싱된 동일한 터미널에서 실행되었는지 확인하세요.

  5. ENABLE 토글 버튼을 클릭하여 익스텐션을 활성화합니다.

  6. ENABLE 바로 오른쪽의 AUTOLOAD 체크박스를 선택합니다.

Launch the URDF Publisher Topic#

  1. 네이티브 ROS 2 환경으로 새 터미널을 열고, 구성에 맞는 ROS 2를 소싱한 후 UR10e 설명을 실행합니다.

    Important

    위에서 Isaac Sim을 실행하는 데 사용한 Python 3.12 build_ws 셸을 재사용하지 마세요. build_ws 경로는 매칭되는 ROS 2 브리지를 Isaac Sim으로 소싱하기 위한 것입니다. ros2 launch 명령에는 OS 네이티브 ROS 2 설치를 사용하세요(또는 OS에서 기본적으로 사용할 수 없는 배포판의 경우 Docker 컨테이너 사용).

    source /opt/ros/jazzy/setup.bash
ros2 launch ur_description view_ur.launch.py ur_type:=ur10e

    네이티브 ROS 2 Humble 설치를 소싱합니다. ur_description이 아직 없는 경우 apt에서 설치합니다:

    sudo apt install ros-humble-ur-description
source /opt/ros/humble/setup.bash
ros2 launch ur_description view_ur.launch.py ur_type:=ur10e

    또는 colcon build로 ur_description을 humble_ws에 네이티브로(Python 3.10) 빌드한 후 apt 패키지 대신 humble_ws/install/local_setup.bash를 소싱합니다.

Import the UR10e Robot into Isaac Sim#

  1. Isaac Sim으로 이동합니다.

  2. File > Import from the ROS 2 URDF Node로 이동합니다.

    • ROS2 Node 필드에 robot_state_publisher를 입력하고 Find Node를 클릭합니다.

    • USD Output 필드에서 원하는 출력 경로를 선택합니다 (예: ~/Desktop/).

    • Robot Type 필드에서 Manipulator를 선택합니다.

    • Base Type 필드에서 Fixed를 선택합니다.

Set Gains Using the Gain Tuner#

임포터는 UR 로봇의 게인을 자동으로 설정하지 않습니다. Gain Tuner를 사용하여 UR 로봇의 게인을 설정할 수 있습니다. 이 튜토리얼에서는 gain tuner를 사용하여 UR 로봇의 자연 주파수와 감쇠비를 설정합니다. 이는 다음과 같이 정의됩니다:

\[ \begin{align}\begin{aligned}\omega_n = \sqrt{\frac{K_p}{m}}\\\zeta = \frac{K_d}{2 m \omega_n}\end{aligned}\end{align} \]

\(\omega_n\)은 자연 주파수이고, \(\zeta\)는 감쇠비이며, \(m\)은 조인트 양쪽의 로봇 질량을 기반으로 계산된 조인트 관성입니다. 감쇠비에서 \(\zeta = 1.0\)은 임계 감쇠 시스템, \(\zeta < 1.0\)은 부족 감쇠, \(\zeta > 1.0\)은 과도 감쇠입니다.

Gain Tuner Extension을 사용하여 UR 로봇의 게인을 설정하고 검증합니다.

  1. Tools > Robotics > Asset Editors > Gain Tuner로 이동합니다.

  2. Gain Tuner 창의 Robot Selection 드롭다운에서 Stage의 ur 아티큘레이션을 선택합니다.

  3. Tune Gains 패널에서 로봇과 그리퍼 손가락 조인트의 게인을 조정할 수 있습니다. Test Gains Settings 패널로 테스트하세요. 먼저 자연 주파수를 300으로, 감쇠비를 1.0으로 설정하세요.

Hint

전체 로봇의 게인을 조정하기 어려운 경우 먼저 소규모 조인트 그룹에 대한 게인을 결정하는 것을 권장합니다. 다음은 게인 조정을 위한 몇 가지 팁입니다:

  • 자연 주파수가 높을수록 로봇이 목표 위치에 더 빠르게 반응합니다. 감쇠비가 낮을수록 로봇이 목표 위치에 더 빠르게 도달합니다.

  • 결과 플롯에서 로봇이 목표 위치에 미달하는 경우 Nat. Freq.를 약간 높이세요.

  • 결과 플롯에서 로봇이 목표 위치를 초과하는 경우 Nat. Freq.를 약간 낮추고 Damping Ratio를 높이세요.

  • 중력을 비활성화하면 게인을 더 명확하게 볼 수 있습니다.

  • 함께 움직일 것으로 예상되는 조인트만 게인 테스트하세요. 게인 테스트 순서는 Sequence 드롭다운으로 선택할 수 있습니다.

  • 실제로 그렇게 빠르게 움직일 것으로 예상되지 않는 조인트의 최대 속도를 줄이세요. Gains Test의 기본값은 USD에 기록된 최대 속도입니다.

Note

Gain Tuner에 대한 자세한 내용은 Gain Tuner Extension을, 로봇의 게인 조정 방법에 대한 자세한 내용은 Tutorial 11: Tuning Joint Drive Gains를 참조하세요.

참고로 결과 USD 파일은 콘텐츠 브라우저의 Isaac Sim/Samples/Rigging/Manipulator/import_manipulator/ur10e/ur_gains_tuner/ur.usda 경로에서 확인할 수 있습니다.

Connect the UR10e Robot with the Robotiq 2F-140 Gripper#

실제 로봇이 다양한 작업에 맞게 도구를 교체할 수 있는 것처럼, 시뮬레이션 로봇도 동일한 기능으로 이점을 얻습니다. 이 섹션에서는 UR10e 로봇과 Robotiq 2F-140 그리퍼를 연결하는 두 가지 방법을 설명합니다.

Robot Assembler를 사용하여 UR10e 로봇과 2F-140 그리퍼를 연결합니다.

  1. 마지막 활동에서 생성된 UR10e USD 파일(ur.usda)을 엽니다.

  2. 이전에 만든 robotiq_2f_140.usd 파일을 Stage로 드래그 앤 드롭합니다.

  3. Tools > Robotics > Asset Editor > Robot Assembler로 이동하여 Robot Assembler를 엽니다.

    • Base Robot에서 Select Base Robot을 /ur로, Attach Point를 wrist_3_link으로 설정합니다.

    • Attach Robot에서 Select Attach Robot을 /ur/robotiq_2f_140으로, Attach Point를 robotiq_arg2f_base_link으로 설정합니다.

    • Assembly Namespace를 Gripper로 설정합니다.

  4. Begin Assembling Process를 클릭하여 프로세스를 시작합니다.

  5. 부착 지점 방향을 조정하여 엔드이펙터가 그리퍼에 올바르게 부착되도록 합니다. Z +90을 클릭하여 z축을 중심으로 그리퍼를 90도 회전합니다.

  6. Assemble and Simulate를 클릭하여 프로세스를 테스트합니다.

  7. End Simulation And Finish를 클릭하여 프로세스를 완료합니다.

  8. File > Save로 이동하거나 Ctrl+S를 눌러 에셋을 저장합니다.

Run the Simulation#

  1. Stage 패널에서 ur 프림을 선택합니다.

  2. 우측 하단의 Property Editor에서 Variants 섹션을 찾습니다.

  3. Gripper 옆에서 None을 선택하면 그리퍼가 로봇에서 제거됩니다.

  4. Gripper 옆에서 robotiq_2f_140을 선택하면 그리퍼가 로봇에 추가됩니다.

  5. File > Save로 이동하거나 Ctrl+S를 눌러 에셋을 저장합니다.

Note

그리퍼가 있는 완성된 로봇팔 에셋은 콘텐츠 브라우저의 Isaac Sim/Samples/Rigging/Manipulator/import_manipulator/ur10e/ur_gripper/ur.usda 경로에서 확인할 수 있습니다.
