Workflows#

Isaac Sim은 더 큰 솔루션의 컴포넌트이며 단독으로 사용할 수도 있습니다. 따라서 동일한 것을 달성하기 위한 여러 가지 사용 방법이 있습니다. 이러한 다양한 방법을 워크플로우라고 합니다. Isaac Sim에서 개발할 때 세 가지 주요 워크플로우가 있습니다:

  • GUI

  • Extensions

  • Standalone Python

모든 것과 서로 어떻게 연결되어 있는지 기본적으로 이해하려면 Quick Start Tutorials을 진행하는 것을 권장합니다.

Workflows#

주요 기능과 권장 사용법 요약:

GUI

  • Key features: 가상 세계를 구성하고 시뮬레이션하기 위한 시각적이고 직관적인 특화 도구.

  • Recommended usage: 세계 구축, 로봇 조립, 센서 부착, OmniGraph를 사용한 비주얼 프로그래밍, ROS 브리지 초기화.

Extension

  • Key features: 스테이지와의 상호 작용을 허용하는 비동기 실행, 변경 사항을 즉시 반영하는 hot reloading, 실시간 시뮬레이션을 위한 적응형 물리 스텝.

  • Recommended usage: Python 스니펫 테스트, 대화형 GUI 구축, 커스텀 애플리케이션 모듈, 실시간 민감 애플리케이션.

Standalone Python

  • Key features: 물리 및 렌더링 스텝의 타이밍 제어, 헤드리스 모드 실행 가능.

  • Recommended usage: 강화 학습을 위한 대규모 훈련, 체계적인 세계 생성 및 수정.

Combining Workflows#

GUI에서 수행할 수 있는 대부분의 작업은 Python을 사용하여 수행할 수 있습니다. GUI에서의 작업과 Python에서의 작업을 전환할 수 있습니다. GUI 내에서 만든 모든 것은 USD 파일의 일부로 저장할 수 있습니다.

예를 들어, GUI를 사용하여 세계를 생성하고 로봇에 필요한 작업을 포함할 수 있습니다. 그런 다음 전체 USD 파일을 독립 실행형 Python 스크립트로 가져와 필요에 따라 체계적으로 속성을 수정합니다.

Extensions and the GUI#

Extensions는 Omniverse Kit 기반 애플리케이션의 핵심 빌딩 블록입니다. 개별적으로 빌드된 애플리케이션 모듈로 다양한 Omniverse 애플리케이션에서 사용할 수 있습니다. Isaac Sim의 대부분의 도구는 확장 기능으로 구축됩니다. 프로젝트 필요에 따라 확장 기능 세트를 활성화 및 비활성화할 수 있습니다.

GUI workflow는 Isaac Sim 시작 시 기본적으로 로드되는 확장 기능 모음을 사용합니다. 이것들은 가상 세계 구축, 로봇 조립, 물리 검사, 렌더링, 재료 속성, 성능 프로파일링 시 자주 사용되는 일반 도구이며, 비주얼 프로그래밍, USD 스테이지 및 에셋 관리, 로봇 공학 애플리케이션을 위한 도구를 포함합니다.

Next steps: Templates를 사용하여 자체 확장 기능을 구축하고, 모두 확장 기능 기반인 Examples Browser의 대화형 예제를 탐색하는 방법을 배웁니다.

Python Standalone and in an Extension#

Extension 및 Standalone Python 워크플로우는 모든 함수에 동일한 API를 사용합니다. 그러나 로봇 관절 상태를 지속적으로 출력하거나 명령하는 경우 차이가 있습니다.

Python in an Extension – Script Editor를 사용하면 Python을 사용하여 비동기적으로 Stage와 상호 작용할 수 있습니다. 이는 Python API가 USD 스테이지와 상호 작용하고 있음을 의미합니다.

확장 기능의 Python은 렌더링 및 물리 스테핑을 차단하지 않고 실행됩니다. 물리 및 렌더링 스텝과 상호 작용하거나 차단될 가능성이 있는 작업을 수행하려면 해당 함수가 작동하도록 관련 콜백 및 비동기 함수를 명시적으로 삽입해야 합니다. 확장 기능 애플리케이션에서 렌더링은 뷰포트가 열리는 순간 스테핑되고 물리는 Play 버튼을 누를 때 스테핑됩니다.

Standalone Python – Isaac Sim의 독립 실행형 Python 버전을 사용하려면 Python 스크립트를 사용하여 실행합니다. 스크립트 내에서 GUI 인터페이스를 열거나 헤드리스 모드로 실행할지 제어할 수 있습니다.

독립 실행형 Python에서는 렌더링과 물리를 수동으로 스텝할 수 있으므로, 일련의 명령이 완료된 후에만 스테핑이 발생함을 보장할 수 있습니다. 이러한 함수들은 독립 실행형 워크플로우를 모든 무작위화 작업이 다음 스텝 전에 완료되어야 하는 행동 훈련, ROS에서 메시지 발행 속도를 제어해야 하는 경우, 성능 향상을 위한 헤드리스 실행과 같은 사용 사례에 이상적으로 만듭니다.

Next steps: Hello World로 첫 번째 독립 실행형 애플리케이션을 실행하는 방법과 Python 개발을 위한 Jupyter Notebook 또는 Visual Studio Code (VS Code)와 같은 개발 도구 사용 방법을 배웁니다.

Hot Reloading for Extensions#

Python 기반 확장 기능은 "hot reload" 기능도 갖고 있습니다. 이는 Isaac Sim이 실행 중에 기반 코드를 변경하고 파일을 저장한 후 Isaac Sim을 종료하거나 재시작하지 않고도 애플리케이션에 변경 사항이 반영되는 것을 볼 수 있음을 의미합니다. 이는 애플리케이션에서 빠르게 반복할 수 있는 강력한 기능입니다.
