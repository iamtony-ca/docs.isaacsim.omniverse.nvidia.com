Teleoperation Synthetic Data Generation#

Isaac Sim의 텔레오퍼레이션은 VR 헤드셋과 컨트롤러로 로봇을 제어하고, 결과 동작을 데모 데이터로 캡처하여 로봇 학습을 위한 합성 데이터셋을 생성하는 재생을 지원합니다.

이 튜토리얼은 isaacsim.replicator.teleop 및 isaacsim.replicator.teleop.ui 확장을 다룹니다. 런타임은 VR 컨트롤러에서 로봇 팔, 그리퍼, 부유 엔드 이펙터, 이동 베이스를 구동합니다. UI는 해당 런타임을 6개의 접을 수 있는 패널이 있는 Teleop 창으로 노출합니다. isaacsim.replicator.episode_recorder 확장은 녹화, 재생, 오프라인 합성 데이터 생성을 처리합니다.

Learning objectives#

이 튜토리얼을 완료하면 다음을 수행할 수 있습니다:

  • Isaac Teleop 런타임을 통해 Isaac Sim을 CloudXR 호환 VR 헤드셋에 연결합니다.

  • Teleop 창에서 Floating, IK, Grasp, Locomotion 컨트롤러를 구성합니다.

  • VR 컨트롤러 또는 디버그 모드의 화면 마커와 슬라이더로 로봇을 조작합니다.

  • YAML 프로파일로 완전한 텔레오퍼레이션 설정을 저장하고 다시 로드합니다.

  • Episode Recorder로 텔레오퍼레이션 에피소드를 HDF5로 녹화합니다.

  • 녹화된 에피소드를 Replicator 라이터를 통해 재생하여 합성 데이터셋을 생성합니다.

Getting started#

Prerequisites#

  • Isaac Sim을 빌드하고 실행할 수 있어야 합니다.

  • PyPI에서 Isaac Teleop을 설치합니다:

    python -m pip install "isaacteleop[cloudxr,retargeters]~=1.0.0"

  • 호스트 머신과 동일한 네트워크에 CloudXR 호환 VR 헤드셋이 필요합니다. 이 튜토리얼의 컨트롤러 버튼 매핑은 Meta Quest 3을 대상으로 하며, 다른 헤드셋은 동일한 OpenXR 액션을 통해 다른 버튼 의미를 제공할 수 있습니다.

  • 로봇이 있는 스테이지. 워크플로우를 학습하는 동안 기본 제공 시나리오 스테이지 중 하나(예: teleop_scenario_floating_xarm_dex3.usd)를 사용합니다.

Note

디버그 모드는 VR 입력을 드래그 가능한 USD 마커와 화면 슬라이더로 대체합니다. 헤드셋, CloudXR 또는 Isaac Teleop 패키지가 필요하지 않습니다. 디버그 모드만 사용할 계획이라면 아래 CloudXR 단계를 건너뛰세요. VR 없이 조작하기(디버그 모드)를 참조하세요.

Start CloudXR and connect the headset#

별도의 터미널에서 Isaac Teleop CloudXR 런타임을 시작하고 전체 텔레오퍼레이션 세션 동안 실행 상태로 유지합니다:

python -m isaacteleop.cloudxr

헤드셋 브라우저에서 Isaac Teleop Web Client를 열고 표시된 연결 단계에 따라 헤드셋을 페어링합니다. CloudXR이 실행 중이고 헤드셋이 연결되면 웹 클라이언트로 돌아가지 않고 Isaac Sim에서 나머지 워크플로우를 완료합니다 — 앱 실행, Teleop 창 열기, Connect 클릭.

Running modes#

텔레오퍼레이션 런타임은 두 가지 Isaac Sim 실행 구성에서 작동합니다. 두 구성 모두 Teleop UI를 로드하고 이 튜토리얼에서 설명하는 모든 컨트롤러를 지원합니다.

  • 2D 모니터(컨트롤러 추적만) — 데스크톱 뷰포트는 평면 화면에서 장면을 렌더링합니다. 헤드셋과 컨트롤러는 CloudXR을 통해 런타임에 포즈 추적을 제공하지만 장면을 스테레오 렌더링하지는 않습니다. 이것이 기본 모드이며 특별한 앱이 필요하지 않습니다:

    ./isaac-sim.sh

  • VR 헤드셋(스테레오 렌더링) — XR VR 경험 앱(isaacsim.exp.base.xr.vr.kit)을 실행합니다. 헤드셋은 1인칭 3D 뷰를 위한 스테레오 렌더링 뷰포트를 수신하고, 데스크톱 타임라인 버튼에 해당하는 헤드셋 내 Play / Stop UI를 노출합니다. 데스크톱 창은 UI 상호 작용에 사용할 수 있습니다:

    ./isaac-sim.xr.vr.sh

Open the Teleop window#

확장은 자동으로 로드됩니다. Tools > Replicator > Teleop에서 Teleop 창을 엽니다:

CloudXR 런타임이 실행 중이고 헤드셋이 연결되면, Session 패널의 Connect를 클릭하여 텔레오퍼레이션 세션을 시작합니다.

Quick start#

기본 제공 시나리오 스테이지와 일치하는 프로파일을 페어링하고, 연결하고, Play를 누릅니다. 프로파일은 스테이지에 대해 모든 컨트롤러를 해결하므로 수동 설정이 필요하지 않습니다.

  1. 기본 제공 시나리오 스테이지 중 하나(예: teleop_scenario_floating_xarm_dex3.usd)를 엽니다.

  2. Teleop 창의 Profiles 패널에서 일치하는 프로파일(floating_xarm_dex3.yaml)을 선택하고 Load를 클릭합니다. 모든 컨트롤러가 스테이지에 대해 해결되고 Enable 버튼이 활성화됩니다.

  3. Session을 펼치고 Connect를 클릭합니다. 헤드셋 없이는 Session > Debug를 펼치고 대신 Debug Tracking을 체크합니다.

  4. 타임라인에서 Play를 누릅니다.

  5. VR 컨트롤러를 이동하거나(디버그 모드에서는 화면 마커를 드래그하여) 로봇을 조작합니다.

프로파일은 시나리오에 필요한 컨트롤러만 활성화합니다:

  • Floating Controller — 자유 강체 그리퍼 또는 엔드 이펙터를 VR 컨트롤러 포즈로 추적합니다.

  • IK Controller — 역운동학을 통해 관절형 팔을 구동하여 엔드 이펙터가 VR 컨트롤러를 추적하게 합니다.

  • Grasp Controller — VR 트리거를 그리퍼 열기와 닫기에 매핑합니다.

  • Locomotion — 썸스틱으로 로봇 베이스 또는 VR 원점을 이동합니다.

예를 들어, floating_xarm_dex3.yaml은 Floating, Grasp, Locomotion 컨트롤러를 활성화하고, IK 프로파일은 Floating 대신 IK를 활성화합니다. 녹화 및 재생을 포함한 자세한 단계별 버전은 워크플로우 안내를 참조하세요.

Overview#

확장은 두 계층으로 나뉩니다:

  • isaacsim.replicator.teleop — VR 입력, 프레임 마커, 4개의 컨트롤러(Floating, IK, Grasp, Locomotion)를 처리하는 런타임으로 TeleopManager가 관리합니다.

  • isaacsim.replicator.teleop.ui — 6개의 접을 수 있는 패널이 있는 Teleop 창: Profiles, Session, Floating Controller, IK Controller, Grasp Controller, Locomotion.

모든 컨트롤러는 동일한 3단계 수명 주기를 따릅니다:

  1. Apply는 프림 경로를 검증하고 컨트롤러 리소스를 준비합니다.

  2. Enable은 다음 Play를 위해 컨트롤러를 활성화합니다.

  3. Clear는 리소스를 해제하지만 빠른 재구성을 위해 프림 경로를 유지합니다.

컨트롤러는 타임라인이 재생 중일 때만 활성화되며 Stop 시 자동으로 비활성화됩니다. 게인, 회전 오프셋, 속도 슬라이더는 Play 중 실시간으로 편집 가능하며 세션 전반에 걸쳐 유지됩니다. 모든 패널의 완전한 상태는 Profiles 패널에서 YAML 프로파일로 저장할 수 있습니다.

Episode Recorder 창은 녹화 및 재생을 처리합니다. TeleopManager가 활성 상태인 경우, 해당 창에서 열린 세션은 UI에서 선택된 관절, 강체 및 Xform 채널 외에도 텔레오퍼레이션 컨트롤러, 조준 포즈 및 헤드 포즈 채널을 자동으로 캡처합니다. 녹화된 HDF5 파일은 오프라인 합성 데이터 파이프라인에 공급됩니다. 스크립트 워크플로우의 경우 build_teleop_recorder는 텔레오퍼레이션 및 장면 레코더블로 미리 구성된 동등한 레코더를 반환합니다.

UI window overview#

Teleop 창에는 6개의 접을 수 있는 패널이 있으며, 아래에서 위에서 아래 순서로 설명합니다. 별도의 Episode Recorder 창은 녹화 및 재생을 처리합니다. 녹화 및 재생을 참조하세요.

Profiles#

Profiles 패널은 모든 다른 패널의 완전한 상태를 단일 YAML 파일로 저장하고 복원합니다.

  • Dir — 텔레오퍼레이션 프로파일 파일의 작업 디렉토리. 기본값은 확장과 함께 제공되는 기본 제공 프로파일입니다. 폴더 아이콘을 클릭하여 사용자 정의 디렉토리를 찾습니다.

  • Profile dropdown — 작업 디렉토리에서 찾은 모든 .yaml 파일을 나열합니다.

  • Load — 선택된 프로파일을 읽고 모든 패널에 적용합니다. 스테이지에 참조된 프림이 포함된 경우 컨트롤러가 즉시 해결되며, 그렇지 않으면 UI 필드가 채워지고 해결되지 않은 경로가 보고됩니다.

  • Save — 인라인 Name 필드와 Confirm 버튼을 엽니다. 파일 이름(.yaml 없이)을 입력하고 Confirm을 클릭하여 현재 패널 상태를 디스크에 저장합니다. 같은 이름의 파일이 이미 존재하면 Overwrite profile 대화 상자가 확인을 요청합니다; Overwrite를 클릭하여 교체하거나 Cancel을 클릭하여 기존 파일을 유지합니다.

  • Validate — 현재 스테이지에 대해 모든 패널 설정을 확인하고 상태 표시줄에 오류 및 경고 수를 보고합니다. 자세한 문제는 콘솔에 출력됩니다.

  • Delete(휴지통 아이콘) — 선택된 프로파일 파일을 디스크에서 영구적으로 제거합니다.

Session#

Session 패널은 VR 연결, 프레임 마커, XR Anchor(사용자 정의 앵커 프림과 헤드셋 오프셋 및 회전), 디버그 컨트롤을 관리합니다.

Connection#

  • Connect / Disconnect — Isaac Teleop CloudXR 세션에 대한 OpenXR 연결을 설정하거나 해제합니다.

  • Status — 현재 연결 상태를 표시합니다: 빨간색(Disconnected), 초록색(Connected - markers active), 또는 노란색(중간 상태, 예: No data).

Frame Markers#

Frame Markers 하위 섹션은 /Teleop/Markers/TrackingOrigin 아래에 4개의 프레임 축 프림으로 라이브 VR 포즈를 보여줍니다 — 원점, Left, Right, Head. 마커는 Connect 및 Debug Tracking 활성화 시 자동으로 생성됩니다; 여기서 수동으로 생성하거나 제거할 수도 있습니다.

  • Show — 4개의 프레임 축 마커를 생성하고 VR 포즈 스트리밍을 시작합니다.

  • Remove — 마커를 삭제하고 추적을 중지합니다.

  • Scale — 모든 마커의 시각적 축 길이를 조정합니다.

XR Anchor#

XR Anchor 하위 패널은 VR 헤드셋과 컨트롤러가 장면에서 나타나는 위치를 결정하는 모든 컨트롤을 그룹화합니다: 앵커가 따라가는 프림(Custom Anchor), 포즈별 Coordinate Frame 변환, 그리고 그 앵커 위에 적용되는 헤드셋 오프셋, 회전, 고정 높이 컨트롤. 이름은 동일한 개념이 Navigation Settings > Physical World USD Anchor > Custom USD Anchor 아래에 노출되는 Kit의 VR 프로파일 메뉴를 반영합니다.

  • Coordinate Frame — 들어오는 VR 포즈를 변환하는 방법을 선택합니다:

    • Isaac Sim (Z-up) — Isaac Sim 스테이지 규칙에 맞게 포즈가 일치하도록 Y-up에서 Z-up 회전을 적용합니다(기본값).

    • Raw (no conversion) — 포즈를 변경 없이 그대로 전달합니다.

  • Custom Anchor — VR 헤드셋과 컨트롤러가 앵커되는 장면 프림. Set을 클릭하여 경로를 검증하고 프림의 월드 변환을 매 프레임 실시간으로 추적하기 시작합니다. 사용자 정의 경로가 활성화된 후 동일한 행 버튼이 Clear로 변경됩니다. Clear는 활성 앵커를 /Teleop/Markers/ 아래의 기본 제공 원점 마커로 되돌리고 마커를 월드 (0, 0, 0)로 재설정합니다; 입력된 경로는 필드에 유지됩니다. 활성 앵커를 재지정하려면 Clear를 클릭하고, 필요하면 경로를 편집한 다음 Set을 다시 클릭합니다. 행의 휴지통 아이콘을 사용하여 필드 텍스트를 지웁니다. 예약된 /Teleop/Markers/ 네임스페이스 아래의 경로는 Set 시 기본 제공 원점으로 폴백됩니다.

  • Offset — VR 헤드셋 카메라의 위치 오프셋(미터 단위)(X, Y, Z 필드가 있는 한 행). Custom Anchor 없이는 절대 월드 위치이고, Custom Anchor가 있으면 해당 프림에 상대적입니다.

  • Rotation — 헤드셋 카메라 요가 Custom Anchor 프림을 추적하는 방법:

    • Fixed — 프림 회전을 완전히 무시합니다.

    • Follow Prim — 요가 프림을 추적합니다(롤과 피치는 제거됨).

    • Follow (Smoothed) — slerp 감쇠가 있는 Follow Prim과 동일합니다.

  • Smooth — slerp 시간 상수(초 단위), Smoothed 모드에서만 사용됩니다. 값이 낮을수록 더 빠른 추적이 가능하고, 높을수록 더 부드럽습니다.

  • Fixed Height — 헤드셋 카메라 Z를 초기 값에 고정하여 Custom Anchor 프림이 위아래로 이동할 때 수직 흔들림을 방지합니다.

Note

텔레오퍼레이션 확장은 세션 동안 Kit의 XR 프로파일 앵커(VR Profile > Navigation Settings > Physical World USD Anchor 아래에서 설정)를 소유합니다. Connect 시 Kit을 custom anchor 모드로 전환하여 /World/XRAnchor를 가리키고, 위의 Custom Anchor 프림과 오프셋, 회전, 스무딩, 좌표 프레임 컨트롤에서 해당 프림을 매 프레임 구동합니다. 활성 사용자 정의 앵커를 재지정하려면 먼저 지운 다음 새 경로를 설정하세요. Kit의 프로파일 수준 Adjust for User Height 설정(Navigation Settings 아래)은 관련 없습니다 — 장면 진입 시 카메라를 이동하는 반면, 여기의 Fixed Height는 텔레오퍼레이션 세션 동안 Z를 첫 프레임 값에 고정합니다.

Debug#

디버그 모드는 VR 컨트롤러 입력을 드래그 가능한 USD 마커와 화면 슬라이더로 대체하여 VR 하드웨어 없이도 모든 컨트롤러를 사용할 수 있습니다. 단계별 안내는 디버그 모드를 참조하세요.

  • Write Backend — 모든 텔레오퍼레이션 쓰기에 사용되는 전역 XformPrim 백엔드를 재정의합니다. 옵션: USD(일반 속성 쓰기), USD-RT(Fabric 계층), Fabric(가장 빠른 경로, Fabric Scene Delegate 필요).

  • Debug Tracking 체크박스 — 합성 포즈 입력을 활성화합니다. 라이브 VR 연결과 상호 배타적입니다: 먼저 연결을 끊거나, 연결하기 전에 디버그 추적을 비활성화하세요.

  • L Grasp / R Grasp — VR 트리거 압력(0-1)을 시뮬레이션하는 슬라이더. Grasp Controller에 trigger_value로 직접 공급됩니다.

  • Slide X / Slide Y — Locomotion 좌우 및 전후 슬라이드를 위한 왼쪽 썸스틱을 시뮬레이션하는 슬라이더(-1 ~ 1).

  • Turn — Locomotion 요를 위한 오른쪽 썸스틱을 시뮬레이션하는 슬라이더(-1 ~ 1).

  • Up / Down — 수직 이동을 위한 오른쪽 측면 페이스 버튼을 시뮬레이션하는 홀드 버튼.

  • Carry Origin — 왼쪽 기본 페이스 버튼을 시뮬레이션하는 홀드 버튼. 누르고 있으면 입력이 활성화되고, 놓으면 지워집니다. Locomotion 컨트롤러는 상승 에지에서 Carry Tracking Space를 토글합니다.

Floating Controller#

Floating Controller는 속도 기반 PD 제어를 사용하여 자유 강체가 VR 컨트롤러 포즈를 추적하도록 구동합니다. 관절 체인에 속하지 않은 엔드 이펙터나 그리퍼에 사용합니다. 각 측면(Left / Right)에는 자체 접을 수 있는 하위 패널이 있습니다.

대상 프림은 강체여야 합니다. Floating Controller로 관절형 그리퍼를 제어하려면, 관절 루트 조인트를 강체에 연결하고 Floating Controller를 해당 강체로 지정합니다. 그리퍼 관절은 자식으로 함께 이동하고, Grasp Controller는 독립적으로 손가락 조인트를 구동합니다.

  • Prim Path — 구동할 강체 프림. + 버튼을 클릭하여 뷰포트에서 프림을 선택하거나 경로를 붙여넣습니다. Apply를 클릭하여 검증합니다. 경로 필드, + 버튼, 휴지통 버튼은 구성되면 잠깁니다; 재구성하려면 Clear를 클릭합니다.

  • Target Rot(X, Y, Z 콤보가 있는 한 행) — 축별 로컬 회전 오프셋(90도 단위: -180, -90, 0, +90, +180). 다른 그리퍼와 엔드 이펙터는 다른 로컬 프레임 규칙을 가지므로 이 오프셋은 제어되는 물체의 전방 축이 VR 컨트롤러 방향과 일치하도록 정렬합니다. 예를 들어, 로컬 Z가 전방이 아닌 측면을 향하는 그리퍼는 90도 Y 오프셋으로 수정할 수 있습니다. Play 중 조정 가능하며 텔레오퍼레이션 프로파일에 저장됩니다.

  • Pos Kp / Kd — 위치 비례 및 미분 게인. Kp가 높을수록 물체가 목표에 더 빠르게 스냅됩니다; Kd는 진동을 감쇠합니다.

  • Rot Kp / Kd — 방향 비례 및 미분 게인. 위치 게인과 동일한 원리가 회전 추적에 적용됩니다.

  • Enable / Disable — 다음 Play를 위해 컨트롤러를 활성화하거나 비활성화합니다. 상태 전환: Configured → Standby → Active(Play 시).

  • Clear — 프림 경로를 유지하면서 컨트롤러 리소스를 제거합니다.

IK Controller#

IK Controller는 역운동학을 통해 관절형 로봇 팔을 구동하여 엔드 이펙터가 VR 컨트롤러 포즈를 추적하게 합니다. 각 측면(Left / Right)에는 자체 접을 수 있는 하위 패널이 있습니다.

대상 프림은 관절이어야 합니다. IK 솔버는 관절 루트에서 선택된 엔드 이펙터 링크까지의 조인트 체인에서 작동합니다. 일반적인 설정(예: 그리퍼가 부착된 UR3e 팔)에서는 손목 링크를 엔드 이펙터로 선택하여 IK가 팔 조인트만 해결하도록 합니다. 그리퍼 조인트는 Grasp Controller에 의해 별도로 구동됩니다.

Articulation and end effector#

  • Prim Path — 관절 루트 프림. Apply를 클릭하여 검증합니다. 성공하면 EE Link 드롭다운이 운동학 체인의 모든 바디 링크로 채워집니다.

  • EE Link — 체인에서 IK 목표로 사용할 링크를 선택합니다. 그리퍼 조인트를 IK 해결에서 제외하려면 마지막 팔 링크(예: 손목)를 선택합니다. 체인의 마지막 링크가 기본으로 선택됩니다.

  • Clear — 솔버 및 관절 리소스를 제거하지만 빠른 재구성을 위해 프림 경로를 유지합니다.

Solver selection#

  • Solver 드롭다운 — IK 백엔드를 선택합니다. 각 솔버는 타임라인을 중지하지 않고 Play 중 핫 스왑할 수 있습니다:

    Solver

    Description

    Position-based

    단일 스텝 야코비안 차분 IK. 구성 가능한 Method 드롭다운을 지원합니다.

    Velocity-based

    추적 적극성을 제어하는 비례 Gain 슬라이더가 있는 속도 공간 IK. Method 드롭다운도 지원합니다.

    Levenberg-Marquardt

    프레임당 다중 반복 감쇠 최소 제곱. 메서드 또는 게인 컨트롤 없음.

    PINK

    조인트 한계 적용 및 자세 정규화가 있는 Pinocchio 백엔드를 사용하는 작업 기반 QP IK. 아래에 설명된 추가 튜닝을 노출합니다.

  • Method 드롭다운 — Position-based와 Velocity-based 솔버에서만 표시됩니다. 야코비안 역전 전략을 선택합니다:

    • Damped LS — 가장 안정적인 기본값; 특이점을 잘 처리합니다.

    • Pseudoinverse — 조건이 좋을 때 직접 추적; 특이점 근처에서 덜 안정적입니다.

    • Transpose — 가장 저렴한 업데이트; 게인에 민감할 수 있습니다.

    • SVD — 강력한 특이값 필터링; 일반적으로 가장 무거운 계산.

Rotation offset and tuning#

  • EE Rot(X, Y, Z 콤보가 있는 한 행) — 축별 로컬 회전 오프셋(90도 단위: -180, -90, 0, +90, +180). Floating Controller의 Target Rot와 동일한 목적: IK 목표를 정렬하여 로봇의 툴 팁 또는 그리퍼가 VR 컨트롤러와 동일한 방향을 향하도록 합니다. 런타임에 조정 가능하며 프로파일에 저장됩니다.

  • VR Target Filter — 들어오는 VR 목표 포즈에 대한 지수 이동 평균(EMA) 저역 통과 필터. 범위 0.0–0.95. 값이 높을수록 지터가 줄어들지만 지연이 추가됩니다. 기본값 0.0(필터링 없음).

  • Max Joint Step — 시뮬레이션 스텝당 최대 관절 각도 변화에 대한 안전 클램프(라디안). 실제 속도 제한으로 작용하지 않고 갑작스러운 관절 점프를 방지합니다. 기본값 0.0(비활성화).

  • Gain — (Velocity-based 솔버 전용) 엔드 이펙터가 VR 목표를 얼마나 적극적으로 추적하는지를 제어하는 비례 게인. 1–5는 부드럽고 보수적인 추적; 10–20은 빠름; 30 이상은 진동할 수 있음.

PINK-specific tuning#

PINK 솔버가 선택된 경우에만 이 컨트롤이 나타납니다:

  • Task Gain — PINK FrameTask 응답 게인. 값이 높을수록 추적이 더 적극적이고, 낮을수록 부드러워집니다.

  • Posture — 자세 정규화 비용. 값이 높을수록 팔이 현재 자세에 더 가깝게 유지되고, 낮을수록 엔드 이펙터 작업에 더 많은 자유를 줍니다.

  • QP 드롭다운 — 이차 프로그래밍 솔버 백엔드. 백엔드 간 해결 품질과 성능을 비교하는 데 사용합니다.

  • LM Damp — FrameTask Levenberg-Marquardt 감쇠. 값이 높을수록 어려운 구성에서 안정성이 향상되지만 응답이 느려집니다.

Enable and status#

  • Enable / Disable — 다음 Play를 위해 IK 컨트롤러를 활성화하거나 비활성화합니다. Play 중 목표에 도달할 수 있을 때 Active, VR 목표가 팔의 작업 공간을 벗어날 때 Out of reach로 상태가 표시됩니다. 목표가 도달 가능한 자세로 돌아오면 추적이 자동으로 재개됩니다.

Grasp Controller#

Grasp Controller는 VR 트리거의 아날로그 값(0 = 열림, 1 = 완전히 닫힘)을 그리퍼 조인트 드라이브 목표에 매핑합니다. 그리퍼는 매우 다양합니다 — 평행 조 그리퍼는 단일 드라이브 조인트를 가지고, 다중 손가락 핸드는 여러 손가락에 걸쳐 수십 개의 조인트를 가질 수 있습니다 — 따라서 컨트롤러는 선형 0–1 트리거 값에서 각 조인트의 목표 위치로의 매핑을 정의하는 YAML 구성 파일에 의존합니다. 각 측면(Left / Right)에는 독립적인 구성을 가진 자체 접을 수 있는 하위 패널이 있습니다.

  • Prim Path — 그리퍼 관절 프림. Apply를 클릭하여 경로를 검증하고 현재 선택된 구성을 한 번에 로드합니다. 필드는 구성 후 잠깁니다; 재구성하려면 Clear를 클릭합니다.

  • Config 드롭다운 — 확장과 함께 제공되는 기본 제공 그래스프 구성을 선택합니다. 항목을 선택하면 즉시 옆의 경로 필드가 업데이트되고 측면이 Config changed - click Apply로 재설정되므로 Enable이 활성화되기 전에 Apply를 다시 클릭해야 합니다.

  • Config path field(Config 옆의 편집 가능한 텍스트 필드) — 그래스프 구성 YAML에 대한 전체 경로 또는 builtin:// URI. 사용자 정의 그리퍼나 그래스프 스타일에 자신만의 구성 파일을 사용하려면 여기에 사용자 정의 경로를 입력합니다. 이 필드를 편집해도 다시 Apply를 클릭해야 합니다.

  • Enable / Disable — 이 측면의 트리거 추적을 활성화하거나 비활성화합니다.

  • Clear — 빠른 재구성을 위해 경로를 유지하면서 그래스프 리소스를 제거합니다.

Play 중 트리거 압력은 VR 컨트롤러 또는 L Grasp / R Grasp 디버그 슬라이더에서 읽힙니다. 구성에 나열된 각 조인트에 대해 컨트롤러는 현재 트리거 값을 기반으로 열림과 닫힘 목표 값 사이를 선형적으로 보간합니다.

Config file format#

각 구성 파일은 구동할 조인트, 입력 범위, 그에 해당하는 목표 범위(도 단위)를 나열합니다. 자신만의 구성 파일을 작성하여 사용자 정의 그리퍼를 지원하거나 동일한 핸드에서 대체 그래스프 스타일을 정의합니다 — 예를 들어 5손가락 핸드에서 핀치 그래스프 대 전체 손바닥 그래스프.

단순 그리퍼 — xarm_grasp.yaml은 단일 드라이브 조인트를 매핑합니다:

joints:
  - name: "drive_joint"
    input_range: [0.0, 1.0]
    target_range: [0.0, 48.0]

다중 손가락 핸드 — dex3_grasp.yaml은 세 손가락에 걸쳐 7개의 조인트를 매핑하며, 각각 자체 목표 범위를 가집니다:

joints:
  - name: "right_hand_index_0_joint"
    input_range: [0.0, 1.0]
    target_range: [0.0, 90.0]
  - name: "right_hand_index_1_joint"
    input_range: [0.0, 1.0]
    target_range: [0.0, 80.0]
  - name: "right_hand_middle_0_joint"
    input_range: [0.0, 1.0]
    target_range: [0.0, 90.0]
  - name: "right_hand_middle_1_joint"
    input_range: [0.0, 1.0]
    target_range: [0.0, 80.0]
  - name: "right_hand_thumb_0_joint"
    input_range: [0.0, 1.0]
    target_range: [0.0, 0.0]
  - name: "right_hand_thumb_1_joint"
    input_range: [0.0, 1.0]
    target_range: [0.0, -60.0]
  - name: "right_hand_thumb_2_joint"
    input_range: [0.0, 1.0]
    target_range: [0.0, -60.0]

Locomotion#

Locomotion 컨트롤러는 VR 썸스틱 및 페이스 버튼 입력을 사용하여 프림을 운동학적으로 이동합니다. 수평 이동은 프림의 방향을 사용하여 월드 지면 평면에 투영되므로 대상 프림의 로컬 프레임 방향에 관계없이 축이 올바르게 유지됩니다.

두 가지 워크플로우를 지원합니다:

  • Robot base — 프림 경로를 로봇 베이스 링크로 설정합니다. 썸스틱 입력이 로봇을 이동하며 부착된 팔과 그리퍼가 따라옵니다. Carry Tracking Space(왼쪽 기본 버튼)를 토글하여 로봇과 함께 VR 원점을 함께 이동합니다.

  • VR origin — 프림 경로를 기본 제공 추적 공간 원점 마커(/Teleop/Markers/TrackingOrigin)로 설정합니다. Locomotion 프림이 VR 원점이므로 Carry가 암묵적입니다. 물리적 베이스가 없는 부유 그리퍼에 사용합니다.

컨트롤:

  • Prim Path — 이동할 프림. Apply를 클릭하여 검증합니다.

  • Slide Step — 전체 입력 시 앱 업데이트당 슬라이드 거리. 왼쪽 썸스틱 이동(앞, 뒤, 측면)과 오른쪽 페이스 버튼 수직 이동을 구동합니다.

  • Turn Step — 전체 오른쪽 썸스틱 요 입력 시 앱 업데이트당 회전 각도.

  • Enable / Disable — 다음 Play를 위해 로코모션을 활성화하거나 비활성화합니다.

  • Clear — 프림 경로를 유지하면서 구성된 상태를 제거합니다.

Play 중 컨트롤러는 다음 VR 입력을 읽습니다:

  • Left thumbstick — 월드 지면 평면에서 앞뒤(Y) 및 좌우(X) 슬라이드.

  • Right thumbstick — 좌우 요 회전.

  • Right face buttons — A(기본)는 아래로, B(보조)는 월드 Z를 따라 위로 이동합니다(Meta 스타일 컨트롤러 레이아웃).

  • Left primary face button(X on Meta-style controllers) — Carry Tracking Space 모드를 토글합니다. 활성화되면 로코모션도 베이스 피벗 주위의 회전을 포함하여 Tracking Space 프림을 베이스와 함께 이동합니다. Locomotion 프림이 추적 공간 원점인 경우 Carry가 암묵적이며 토글은 추가 효과가 없습니다.

Record and replay (Episode Recorder)#

Episode Recorder 창(isaacsim.replicator.episode_recorder.ui, Tools > Replicator > Episode Recorder에서 열림)은 물리 스텝당 시뮬레이션 상태를 다중 에피소드 HDF5 파일에 기록하고 Kit 타임라인을 통해 재생합니다. 모든 스테이지에서 작동합니다. TeleopManager가 활성 상태인 경우, 텔레오퍼레이션 컨트롤러, 조준 포즈, 헤드 포즈 채널이 install_teleop_session_injector를 통해 창에서 열린 모든 세션에 추가됩니다.

녹화 세션은 많은 에피소드가 포함된 하나의 HDF5 파일입니다. 에피소드는 타임라인 Play에서 자동 시작되고 타임라인 Stop에서 자동 종료됩니다. 창 버튼, VR 녹화 버튼, 스크립트 호출자는 그 위에 수동 시작, 종료 또는 토글 에지를 추가하며, 모두 동일한 기본 세션을 구동합니다.

Targets and output#

  • USD Root — 검색 헬퍼가 스캔하는 프림 경로. /World가 합리적인 기본값입니다.

  • Discover — 루트 아래의 모든 관절(ArticulationRootAPI 사용), 강체(RigidBodyAPI 사용), 일반 Xform 프림을 나열합니다. 일반 Xform은 항상 포함되므로 로코모션으로 구동되는 로봇 베이스 큐브, 수동으로 배치된 트래커, 관절 아래의 시각적 툴 팁이 별도의 옵트인 없이 표시됩니다.

  • Discovered Targets(접을 수 있고 스크롤 가능) — 루트 아래에서 발견된 관절과 프림. 기록할 각 목표의 확인란을 선택합니다; 각 선택은 HDF5 파일 내의 그룹 또는 데이터셋에 매핑됩니다.

  • Output Dir — HDF5 파일이 작성되는 디렉토리. 기본값은 <cwd>/_episode_recorder; 없으면 생성됩니다.

  • Export Scene(Output Dir 필드 옆) — 현재 스테이지의 평면화된 USD를 <output_dir>/stage_snapshot.usd로 stage_snapshot.sidecar.json과 함께 저장합니다. 스냅샷은 장면 수준이므로 장면당 한 번 클릭하면 충분합니다: 이후 Open Session 호출이 파일을 감지하고 해당 기본 이름을 HDF5 stage_snapshot 속성에 자동으로 기록합니다.

  • File Prefix — 파일 이름 접두사. 최종 경로는 {prefix}_{timestamp}.hdf5입니다.

  • Auto-start recording on timeline Play — 체크 시(기본값), Play를 누르면 자동으로 새 에피소드가 시작됩니다. Start / End(또는 VR 버튼)를 눌렀을 때만 기록하려면 체크를 해제합니다; 타임라인은 에피소드 캡처 없이 재생될 수 있습니다.

  • Pose Backend(레코드 측) — 레코더의 틱당 배치 XformPrim.get_world_poses() 읽기에 사용되는 백엔드를 선택합니다. 옵션: usd(기본값; 순수 USD 읽기), usdrt(IFabricHierarchy를 통한 Fabric Scene Delegate), fabric(Fabric Scene Delegate 직접). Fabric 지원 옵션은 Fabric Scene Delegate가 활성화된 경우 안전한 속도 향상이며, 비활성화 시 carb 경고와 함께 usd로 폴백합니다. 텔레오퍼레이션 쓰기 경로를 제어하는 Teleop Session > Debug 패널의 Write Backend와는 다릅니다.

Session and episode control#

  • Open Session / Close Session — 단일 토글 버튼. 열면 레코더가 HDF5 파일을 생성하고 시뮬레이션 이벤트를 구독하며 파일 이름이 아래에 표시됩니다. 세션이 열려 있는 동안 모든 구성 옵션이 잠깁니다.

  • Start / End — 열린 세션 내에서 에피소드를 수동으로 시작하거나 종료하는 단일 토글 버튼. 세션이 열려 있는 경우에만 활성화됩니다. VR 왼쪽 Y 버튼(아래 참조)과 Auto-start recording on timeline Play가 활성화된 경우 타임라인 PLAY / STOP 후크에 의해서도 구동됩니다.

  • Binding badge — Start / End 옆에 렌더링되는 작은 점선 레이블. 현재 이 레코더에 연결된 모든 외부 입력(예: TeleopManager에 의해 연결된 VRRecordingButton)이 있으면 초록색으로 켜집니다. 툴팁은 각 바인딩의 레이블과 디스패치하는 명령(start / end / toggle)을 열거합니다. 외부 바인딩이 활성화되지 않으면 비어 있습니다.

  • Status label — 버튼 아래의 색상 코딩 피드백:

    • Idle(흐림).

    • Session open - N articulation(s), M prim(s)(노란색).

    • Recording episode #K(초록색).

    • Standby - K episode(s) captured(노란색).

    • Session closed (K episode(s))(초록색).

    오류와 경고는 빨간색과 노란색으로 표시됩니다.

VR recording button#

TeleopManager는 생성 시 VRRecordingButton을 통해 Meta Quest 왼쪽 Y 버튼(VRButton.LEFT_SECONDARY)을 toggle 명령에 자동으로 연결하고 그 수명 동안 바인딩을 유지합니다. 한 번 누르면 새 에피소드가 시작되고, 두 번째로 누르면 종료됩니다. 바인딩은 상승 에지로 트리거되므로 버튼을 누르고 있어도 재트리거되지 않습니다. 세션이 열려 있지 않으면 디스패치가 아무 작업도 하지 않습니다.

이 바인딩은 Meta Quest 3에서만 테스트되었습니다; 다른 헤드셋은 동일한 OpenXR 액션을 통해 다른 버튼 의미를 제공할 수 있습니다.

Replay#

Replay 하위 섹션(접을 수 있으며 기본적으로 접힘)은 이전에 기록된 HDF5를 Kit 타임라인을 통해 재생합니다. 재생은 녹화와 상호 배타적입니다: 세션이 열려 있는 동안 Replay 컨트롤이 잠기고, 재생이 연결된 동안 녹화 컨트롤이 잠깁니다.

전송 행은 텍스트 레이블 대신 Kit 타임라인 스타일 글리프 버튼을 사용합니다: 재생/중지, 일시 정지, 뒤로 이동, 앞으로 이동.

  • File — HDF5 세션 파일의 전체 경로. Latest를 사용하여 현재 Output Dir에서 가장 최신 {prefix}_*.hdf5를 입력합니다.

  • Load — HDF5를 열고 Episode 드롭다운에 모든 에피소드 이름과 프레임 수를 채웁니다. 드롭다운 옆의 정보 레이블은 선택된 에피소드에 대해 success=True/False를 표시하므로 포기한 테이크가 한눈에 보입니다. 로드 후 HDF5에서 참조된 프림 경로가 현재 스테이지에서 해결되지 않으면 상태 아래에 빨간색 경고 행이 나타납니다 — 재생을 시작하기 전에 일치하는 장면(또는 내보낸 stage_snapshot.usd)을 여세요.

  • Pose Backend(재생 측) — 재플레이어의 계층당 배치 포즈 쓰기에 사용되는 백엔드를 선택합니다. 옵션은 레코드 측 선택기(usd / usdrt / fabric)와 동일합니다. usd가 권장 기본값입니다 — 조상 순서 계층 분할과 USD 쓰기의 조합이 이동하는 xform 아래에 중첩된 관절의 부모 지연 떨림을 방지합니다. usdrt와 fabric은 평면 장면 벤치마킹을 위해 예약되어 있으며 중첩된 계층에서 한 프레임 부모 지연이 발생할 수 있습니다. Load 시 적용됩니다.

  • Play / Stop 글리프 — EpisodeReplayer.start_replay를 구동합니다. 각 Kit 앱 업데이트는 하나의 기록된 프레임을 적용하고 Kit 타임라인을 기록된 sim_time으로 탐색합니다(재생하지 않음), 따라서 스테이지 작성 USD 애니메이션이 물리를 스텝하지 않고 함께 재생됩니다. 포즈 쓰기는 익명 USD 서브레이어에 저장되므로 루트 스테이지가 변경되지 않습니다. 중지하거나(또는 비루프 모드에서 마지막 프레임에 도달하면) 해당 서브레이어가 팝되어 모든 프림이 재생 전 포즈로 돌아갑니다; HDF5 세션은 로드된 상태로 유지되므로 즉시 새 재생을 시작할 수 있습니다.

  • Pause 글리프 — 현재 프레임에서 재생을 일시 정지합니다; 마지막 적용된 프레임이 스테이지에 유지됩니다. 다시 누르면 중단된 곳에서 재개됩니다. Stop 글리프는 여전히 익명 서브레이어를 팝합니다.

  • Step Backward / Step Forward 글리프 — 이전 또는 다음 기록된 프레임을 적용하고 자동으로 재생을 일시 정지합니다. 녹화를 한 프레임씩 검사하거나 재개하기 전에 특정 순간으로 탐색하는 데 사용합니다.

  • Seek timeline — 체크 시(기본값), 각 적용된 프레임도 해당 프레임의 기록된 sim_time으로 Kit 타임라인을 탐색하여 스테이지 작성 USD 애니메이션이 녹화와 동기화 상태를 유지합니다. 기록된 프림 포즈만 재생하고 타임라인을 그대로 두려면 체크를 해제합니다(스테이지에 작성된 애니메이션이 없는 경우 유용).

  • Progress label — 재생 상태 아래에 현재 적용된 프레임을 Frame X / N으로 표시합니다. 동일한 카운터가 1초 간격과 첫 번째 및 마지막 프레임에서 터미널로 출력됩니다.

재생은 순수 USD이고 타임라인 탐색 전용입니다 — 재플레이어는 타임라인을 재생하지 않으며 물리 엔진을 호출하지 않습니다. 텔레오퍼레이션 컨트롤러(Floating, IK, Grasp, Locomotion)는 재생 중 비활성 상태를 유지하여 중지된 시뮬레이션에 대해 타임라인을 재생하면 발생하는 Simulation view object is invalidated 오류를 방지합니다. 시작/중지 수명 주기는 터미널에 [EpisodeRecorder][UI] Replay: starting (episode ..., N frames, file=...)와 Replay: stopped (reason=user | finished | stage_closed)를 출력하며, 주기적인 Replay: frame X/N 진행 줄도 출력됩니다.

재생이 작동하려면 HDF5에 기록된 모든 프림 경로가 로드된 스테이지에 존재해야 합니다. Replay 패널은 누락된 경로를 오류 없이 경고와 함께 건너뛰는 관대한 재플레이어(ReplayPolicy(strictness="best_effort"))를 사용합니다. 재현 가능한 설정을 보장하려면 녹화 전에 Export Scene을 한 번 클릭하세요; 결과 stage_snapshot.usd는 재생 전에 작성된 스테이지를 재현하기 위해 모든 머신에서 열 수 있습니다.

HDF5 file layout#

각 세션은 에피소드당 하나의 그룹이 있는 하나의 HDF5 파일을 생성합니다. 데이터셋은 에피소드당 사전 할당되며 end_episode에서 실제 길이로 잘립니다.

<file>.hdf5                             # one file per open_session()
├── @schema_version, @created_at, manifest/, ...  # file-level attrs + manifest
├── @stage_snapshot                     # optional, set by Export Scene
└── episodes/
    ├── episode_00000/                  # @episode_index, @started_at, @ended_at,
    │   │                               # @num_frames, @success (optional),
    │   │                               # @user_metadata (optional, JSON)
    │   ├── meta/time/
    │   │   ├── sim_time            (N,)     float64
    │   │   ├── physics_step        (N,)     int64
    │   │   └── wall_time           (N,)     float64
    │   ├── state/<name>/                  # articulation, xform, or rigid body (UI naming)
    │   │   ├── positions           (N, L, 3)  float32   # articulation: per-link world position
    │   │   ├── orientations        (N, L, 4)  float32   # articulation: per-link wxyz
    │   │   ├── position            (N, 3)     float32   # xform / rigid body
    │   │   └── orientation         (N, 4)     float32   # wxyz
    │   └── teleop/                        # present when a live TeleopManager is active
    │       ├── <side>/{trigger, squeeze, thumbstick_x, thumbstick_y}     (N,)    float32
    │       ├── <side>/{primary_click, secondary_click, thumbstick_click} (N,)    uint8
    │       ├── <side>/aim_position          (N, 3)  float32   # record_aim_pose=True
    │       ├── <side>/aim_orientation       (N, 4)  float32   # wxyz
    │       └── head/{position, orientation} (N, 3 | 4)  float32   # record_head_pose=True
    ├── episode_00001/ ...
    └── episode_00002/ ...

관절의 경우 L은 기록된 링크 수(관절 루트와 모든 UsdGeom.Xformable 후손)입니다. 링크 목록은 Open Session에서 동결되어 매니페스트에 저장되므로 재플레이어가 동일한 프림 세트에 바인딩합니다. DOF, 속도, 드라이브 목표 채널이 없습니다: 모든 그리퍼 드라이브 조인트는 자식 링크의 기록된 월드 포즈를 통해 재현되므로 텔레오퍼레이션 로직을 실행하지 않고도 열기/닫기 그리퍼 재생이 작동합니다.

EpisodeReplayer.list_episodes는 에피소드별 재생을 위해 episodes/episode_NNNNN 그룹을 반복합니다.

Recorded data vs. replayed data#

레코더는 프레임당 두 종류의 데이터를 캡처합니다:

  • World state(state/<name>/ 아래, 기록된 관절, Xform 또는 강체당 하나의 HDF5 그룹) — 기록된 모든 프림의 월드 포즈. 관절의 경우 링크별 포즈 배열; 강체와 Xform의 경우 단일 루트 포즈. 이것이 재플레이어가 적용하는 유일한 데이터입니다.

  • Teleop input channels(teleop/<side>/... 아래, 녹화 시 라이브 TeleopManager가 활성 상태일 때만 존재) — 트리거, 압력, 썸스틱, 버튼 클릭, 선택적 OpenXR 조준 포즈 및 헤드 포즈 채널. 오프라인 분석, 정책 학습 및 재시뮬레이션을 위해 기록됩니다; 재플레이어는 이를 완전히 무시합니다.

조준 포즈 및 헤드 포즈 캡처는 carb 설정 /persistent/exts/isaacsim.replicator.teleop/record/record_aim_pose와 .../record_head_pose로 제어됩니다(둘 다 기본값 True). 건너뛰려면 세션을 열기 전에 Script Editor에서 토글합니다(carb.settings.get_settings().set_bool(...)).

재생 시 EpisodeReplayer.apply_frame은 XformPrim.set_world_poses를 통해 모든 프림(및 모든 관절 링크)의 기록된 월드 포즈를 익명 USD 서브레이어에 씁니다. 물리가 스텝되지 않으며, DOF가 쓰이지 않으며, IK가 해결되지 않으며, 트리거 명령이 재디스패치되지 않으며, OpenXR 입력이 소비되지 않습니다. 텔레오퍼레이션 컨트롤러(Floating, IK, Grasp, Locomotion)는 비활성 상태를 유지합니다. 재생은 순전히 USD 포즈 재생입니다.

Programmatic recordables (cameras, attributes)#

Episode Recorder 창은 USD Root 아래의 관절, 강체, 일반 Xform만 자동 검색합니다. 추가 채널을 캡처하려면 — 일반적으로 합성 데이터 파이프라인의 카메라 궤적이나 오프라인 분석을 위한 임의의 USD 속성 — 레코더를 프로그래밍 방식으로 구성하고 세션을 열기 전에 추가 레코더블을 add합니다:

from isaacsim.replicator.episode_recorder import (
    CameraRecordable,
    AttributeRecordable,
)
from isaacsim.replicator.teleop import build_teleop_recorder

recorder = build_teleop_recorder(
    output_dir="/tmp/demos",
    teleop_manager=teleop_manager,
    articulations={"robot": "/World/teleop/robot"},
)
recorder.add(CameraRecordable(
    group="cameras/wrist",
    prim_path="/World/teleop/robot/.../wrist_cam",
))
recorder.add(AttributeRecordable(
    group="env/light_intensity",
    prim_path="/World/Lights/key_light",
    attribute_name="intensity",
))
recorder.open_session()

CameraRecordable은 카메라의 월드 포즈와 USD 내부 파라미터(초점 거리, 수평 및 수직 조리개, 클리핑 범위)를 매 프레임 캡처합니다; 해상도는 세션 매니페스트에 한 번 저장됩니다. 재생 시 동일한 채널이 기록된 카메라 궤적을 익명 서브레이어에 다시 작성하므로 카메라 프림에 부착된 Replicator 렌더 제품이 추가 배선 없이 이를 가져옵니다.

AttributeRecordable은 프레임당 프림의 단일 USD 속성을 캡처합니다. 포즈가 아닌 환경 상태(광원 강도, 재질 파라미터, 사용자 정의 작성 속성 등)에 사용합니다.

서드파티 확장도 register_session_injector로 세션 인젝터를 등록하여 UI 창에서 열린 모든 세션에 채널을 추가할 수 있습니다 — install_teleop_session_injector가 텔레오퍼레이션 컨트롤러, 조준 포즈, 헤드 포즈 채널을 제공하는 데 사용하는 동일한 메커니즘입니다.

Teleop profiles#

텔레오퍼레이션 프로파일은 Teleop 창의 모든 패널의 완전한 상태를 캡처하는 단일 YAML 파일입니다. Teleop 창 상단의 Profiles 패널을 사용하여 프로파일을 저장, 로드, 삭제합니다. 기본 제공 프로파일은 source/extensions/isaacsim.replicator.teleop/data/teleop_profiles/ 아래의 확장과 함께 제공됩니다; Dir 필드를 사용자 정의 폴더로 지정하여 기본 제공 프로파일과 함께 자신의 프로파일을 관리합니다.

로드되면 프로파일은 모든 섹션을 순서대로 적용합니다: 세션 전역 설정 먼저, 그 다음 각 컨트롤러 패널. 참조된 프림이 현재 스테이지에 있으면 컨트롤러가 즉시 해결되어 Enable할 준비가 됩니다. 스테이지가 일치하지 않는 경우(다른 로봇이나 누락된 프림), UI 필드는 여전히 채워지고 해결되지 않은 경로가 상태 표시줄에 보고됩니다.

Bimanual floating grippers (VR origin locomotion)#

floating_xarm_dex3.yaml은 이중 부유 그리퍼 설정을 구성합니다. Floating Controller는 각 그리퍼를 자유 강체로 구동하고, Locomotion은 VR 원점 마커를 목표로 하여 썸스틱 입력이 전체 VR 작업 공간을 재배치합니다.

Session — 컨트롤러가 구성되기 전에 적용되는 전역 설정:

session:
  coordinate_system: isaac_sim       # Z-up coordinate conversion
  tracking_space_enabled: false
  tracking_space_path: ''            # empty = built-in origin marker
  marker_scale: 0.05
  anchor_x: 0.0
  anchor_y: 0.0
  anchor_z: 0.0
  anchor_rotation_mode: fixed
  anchor_smoothing: 1.0
  anchor_fixed_height: true

Floating — PD 게인과 회전 오프셋이 있는 측면별 강체 컨트롤러. 두 측면이 활성화되어 각각 다른 그리퍼 루트 프림을 가리킵니다:

floating:
  left:
    enabled: true
    settings:
      prim_path: /World/teleop_xarm_dex3/.../xarm_gripper_rigid_root
      pos_kp: 20.0
      pos_kd: 0.5
      orient_kp: 20.0
      orient_kd: 0.2
      target_rot_x_deg: 180
      target_rot_y_deg: 0
      target_rot_z_deg: 90
  right:
    enabled: true
    settings:
      prim_path: /World/teleop_xarm_dex3/.../dex3_1_r_rigid_root
      pos_kp: 20.0
      pos_kd: 0.5
      orient_kp: 20.0
      orient_kd: 0.2
      target_rot_x_deg: -90
      target_rot_y_deg: 0
      target_rot_z_deg: 90

IK — 그리퍼가 관절이 아닌 부유 강체이므로 어느 측면도 활성화되지 않습니다. 섹션은 기본값으로 여전히 존재하여 프로파일을 로드하면 이전 IK 구성이 재설정됩니다.

Grasp — 각 측면을 그리퍼 관절 프림과 기본 제공 그래스프 구성에 매핑합니다. builtin:// 경로는 확장과 함께 제공되는 YAML 파일로 해결됩니다:

grasp:
  left:
    enabled: true
    prim_path: /World/teleop_xarm_dex3/.../xarm_gripper
    config_path: builtin://xarm_grasp
  right:
    enabled: true
    prim_path: /World/teleop_xarm_dex3/.../dex3_1_r
    config_path: builtin://dex3_grasp

Locomotion — 기본 제공 추적 공간 원점을 구동하여 썸스틱 입력이 전체 텔레오퍼레이션 작업 공간을 이동합니다(VR 원점 워크플로우):

locomotion:
  enabled: true
  settings:
    prim_path: /Teleop/Markers/TrackingOrigin
    linear_step: 0.003333333333333333

Workflow walkthrough#

이 섹션은 빠른 시작을 전체 워크플로우로 확장합니다: 프로파일에서 구성, VR 또는 디버그 모드로 연결, 각 컨트롤러 조작, 에피소드 녹화 및 재생. 데이터를 캡처하려면 Play를 누르기 전에 Episode Recorder 세션을 엽니다.

Configure with a built-in profile#

  1. Tools > Replicator > Teleop을 엽니다.

  2. 위의 기본 제공 시나리오 스테이지 중 하나를 엽니다.

  3. Profiles 패널에서 Dir 필드가 기본 제공 프로파일 디렉토리로 기본 설정됩니다. 드롭다운에서 스테이지와 일치하는 프로파일을 선택하고 Load를 클릭합니다. Floating Controller, IK Controller, Grasp Controller, Locomotion 패널이 로드된 스테이지에 대해 구성되고 Enable 버튼이 활성화됩니다.

  4. Validate를 클릭하여 상태 표시줄에 0 error(s), 0 warning(s)가 보고되는지 확인합니다. 해결되지 않은 프림 경로는 콘솔에 나열됩니다.

사용자 정의 로봇에 대해 컨트롤러를 수동으로 구성하려면 UI 창 개요에 설명된 대로 각 패널을 작업합니다. 수명 주기는 모든 컨트롤러에 동일합니다: 프림 경로를 입력하고, Apply를 클릭하고, 필요에 따라 조정하고, Enable을 클릭합니다.

Connect to VR#

  1. CloudXR 런타임이 실행 중(python -m isaacteleop.cloudxr)이고 헤드셋 웹 클라이언트가 연결되어 있는지 확인합니다.

  2. Session을 펼치고 Connect를 클릭합니다. 상태가 초록색(Connected - markers active)으로 바뀌고 /Teleop/Markers/TrackingOrigin 아래에 4개의 프레임 마커가 나타납니다.

  3. VR 컨트롤러를 이동합니다. Left와 Right 마커가 실시간으로 추적됩니다.

Connect가 실패하면 상태가 빨간색으로 유지됩니다. 가장 일반적인 원인은 CloudXR 프로세스가 중지되었거나 헤드셋 연결이 끊어진 경우입니다; 두 가지를 모두 재시작하고 Connect를 다시 클릭합니다.

Operate without VR (debug mode)#

헤드셋을 사용할 수 없거나, 튜닝을 반복하거나, 헤드리스로 실행할 때 디버그 모드를 사용합니다. 디버그 모드와 VR 모드는 상호 배타적입니다 — 디버그를 활성화하기 전에 VR 연결을 끊고, Connect를 클릭하기 전에 Debug Tracking을 체크 해제합니다.

  1. Session > Debug를 펼치고 Debug Tracking을 체크합니다. 뷰포트에 프레임 마커가 나타나고, Connect 버튼이 비활성화되고, L Grasp, R Grasp, Slide X, Slide Y, Turn, Up, Down, Carry Origin 컨트롤이 활성화됩니다.

  2. 뷰포트에서 Left, Right, Head 마커를 드래그하여 포즈를 설정합니다; TrackingOrigin 부모를 드래그하여 4개의 마커를 모두 이동합니다. 마커 계층은 실제 VR 추적 공간을 반영합니다.

Record an episode#

Episode Recorder 창은 시뮬레이션 상태와(TeleopManager가 활성 상태인 경우) 텔레오퍼레이션 입력 채널을 캡처합니다. 전체 UI 참조는 녹화 및 재생을 참조하세요.

  1. Tools > Replicator > Episode Recorder를 엽니다. 세션 인젝터가 활성 상태로 유지되도록 Teleop 창을 열어 두세요.

  2. USD Root를 /World로 설정하고, Discover를 클릭하고, 녹화할 목표(로봇, 추적된 Xform)를 선택합니다.

  3. 선택 사항 — Export Scene을 한 번 클릭하여 HDF5 출력 옆에 stage_snapshot.usd를 저장합니다. 다른 머신에서의 재생은 이 스냅샷을 이식 가능한 스테이지로 사용할 수 있습니다.

  4. Open Session을 클릭합니다. 구성 컨트롤이 잠기고 파일 이름이 버튼 아래에 나타납니다.

  5. 타임라인에서 Play를 누릅니다. Auto-start recording on timeline Play가 체크된 경우(기본값), 상태가 Recording episode #1으로 초록색이 됩니다. 로봇을 조작합니다. Stop을 눌러 에피소드를 종료합니다. 추가 에피소드에 대해 반복합니다.

  6. VR에서 수동으로 녹화를 토글하려면 Meta Quest 컨트롤러의 왼쪽 Y 버튼을 누릅니다. 각 상승 에지가 에피소드를 시작하거나 종료합니다.

  7. 완료되면 Close Session을 클릭합니다.

Replay an episode#

  1. Episode Recorder 창의 Replay 하위 섹션을 펼칩니다.

  2. Latest를 클릭하여 Output Dir의 가장 최신 HDF5 파일을 입력하고, Load를 클릭합니다. Episode 드롭다운에 모든 에피소드와 프레임 수 및 success 플래그가 나열됩니다.

  3. 에피소드를 선택하고 전송 행의 재생 글리프를 클릭합니다. Kit 타임라인이 각 프레임의 기록된 sim_time으로 탐색합니다. 모든 프림이 기록된 궤적을 통해 이동합니다; 텔레오퍼레이션 컨트롤러는 비활성 상태를 유지합니다.

  4. 일시 정지 및 뒤로/앞으로 이동 글리프를 사용하여 스크럽합니다. Seek timeline을 체크 해제하여 Kit 타임라인을 그대로 둡니다(스테이지에 작성된 애니메이션이 없는 경우 유용).

  5. 중지 글리프를 클릭하여 스테이지를 재생 전 포즈로 되돌립니다. HDF5가 로드된 상태로 유지되므로 즉시 새 재생을 시작할 수 있습니다.

재생이 작동하려면 HDF5에 기록된 모든 프림 경로가 로드된 스테이지에 존재해야 합니다. 원본 스테이지를 사용할 수 없는 경우 Export Scene으로 작성한 stage_snapshot.usd를 엽니다.

Synthetic data generation from recorded episodes#

녹화 및 재생에서 다룬 UI 재생은 Kit 타임라인으로 구동되는 빠른 시각적 미리보기입니다. 오프라인 합성 데이터 생성을 위해서는 EpisodeReplayer를 프레임별로 구동하고 각 apply_frame 후에 rep.orchestrator.step_async를 호출합니다. 이렇게 하면 녹화 시간과 렌더링 시간이 분리되어 비용이 많이 드는 라이터나 DLSS 모드가 텔레오퍼레이션을 느리게 하거나 시간 드리프트 없이 프레임당 실행될 수 있습니다.

Prerequisites#

  • Episode Recorder 창(또는 EpisodeRecorder 하위 클래스)으로 생성된 HDF5 세션.

  • 재생할 USD 스테이지. HDF5의 모든 프림 경로가 이 스테이지에서 해결되어야 합니다. STAGE_URL을 원본 장면의 에셋 서버 경로 또는 내보낸 스냅샷으로 지정합니다 — Episode Recorder 창에서 Export Scene을 클릭하거나 스크립트에서 export_stage_snapshot을 호출하여 HDF5 옆에 stage_snapshot.usd를 생성합니다.

  • Isaac Sim이 실행 중이어야 합니다. 재생을 위해 VR 또는 CloudXR 연결은 필요하지 않습니다.

What the script does#

스크립트는 STAGE_URL을 열고(get_assets_root_path를 통해 해결), CAMERA_PATHS에 나열된 카메라를 해결하고(해결되지 않으면 기본 카메라로 폴백), 카메라 렌더 제품에 BasicWriter(RGB PNG)를 연결하고, 모든 기록된 프레임을 반복합니다 — 각 step_frame 후에 rep.orchestrator.step(또는 Script Editor 변형의 경우 step_async)을 호출합니다. 출력은 현재 작업 디렉토리 옆의 _out_teleop_replay/basic/ 아래에 저장됩니다.

아래의 변형 중 하나를 실행하기 전에 스크립트 상단의 HDF5_PATH와 STAGE_URL을 기록된 세션과 일치하는 USD 스테이지로 지정합니다.

예제는 터미널에서 다음 명령을 사용하여 독립 실행형 애플리케이션으로 실행할 수 있습니다(Windows에서는 python.sh 대신 python.bat 사용):

./python.sh standalone_examples/api/isaacsim.replicator.teleop/sdg_teleop_replay.py

import os

from isaacsim import SimulationApp

simulation_app = SimulationApp(launch_config={"headless": False})

import carb.settings
import omni.kit.app
import omni.replicator.core as rep
import omni.usd
from isaacsim.core.utils.extensions import enable_extension

# Enable the teleop extension before importing modules it owns and before resolving its data path.
# It also transitively pulls in `isaacsim.replicator.episode_recorder`, which is not part of the base kit.
enable_extension("isaacsim.replicator.teleop")

from isaacsim.replicator.episode_recorder import EpisodeReplayer
from isaacsim.storage.native import get_assets_root_path
from pxr import UsdGeom

# Resolve the bundled golden HDF5 episode shipped with the isaacsim.replicator.teleop extension tests.
_TELEOP_EXT_PATH = (
    omni.kit.app.get_app().get_extension_manager().get_extension_path_by_module("isaacsim.replicator.teleop")
)

# Path to the USD stage to replay against; every prim path in the HDF5 must resolve on this stage.
STAGE_URL = "/Isaac/Samples/Replicator/Teleop/teleop_scenario_floating_xarm_dex3.usd"
HDF5_PATH = os.path.join(
    _TELEOP_EXT_PATH,
    "isaacsim",
    "replicator",
    "teleop",
    "tests",
    "data",
    "_episode_recorder",
    "episode_floating_xarm_dex3.hdf5",
)
CAMERA_PATHS = [
    "/World/teleop_xarm_dex3/gripper_origin_xform/xarm_gripper_root_xform/xarm_gripper/xarm_gripper_base_link/xarm_view_cam",
    "/World/teleop_xarm_dex3/gripper_origin_xform/dex3_1_r_root_xform/dex3_1_r/right_hand_palm_link/dex3_view_cam",
]
EPISODE_INDEX = 0
RESOLUTION = (512, 512)
NUM_CAPTURES = 10  # Number of frames to capture, evenly distributed across the episode


def run_example():
    print("[TeleopReplay] Starting replay example")
    if not HDF5_PATH:
        print("[TeleopReplay] HDF5 path not provided, exiting")
        return
    if not os.path.isfile(HDF5_PATH):
        print(f"[TeleopReplay] HDF5 session file does not exist: '{HDF5_PATH}', exiting")
        return
    print(f"[TeleopReplay] HDF5 session: {HDF5_PATH}")

    # Load the authored USD stage so every prim path in the HDF5 resolves.
    assets_root_path = get_assets_root_path()
    if assets_root_path is None:
        print("[TeleopReplay] Could not find Isaac Sim assets folder, exiting")
        return
    stage_path = assets_root_path + STAGE_URL
    print(f"[TeleopReplay] Opening stage: {stage_path}")
    omni.usd.get_context().open_stage(stage_path)
    print("[TeleopReplay] Stage opened")

    # Drive writers manually via rep.orchestrator.step, not via timeline play.
    rep.orchestrator.set_capture_on_play(False)

    # Set DLSS to Quality mode (2) for best SDG results (Options: 0 (Performance), 1 (Balanced), 2 (Quality), 3 (Auto)
    carb.settings.get_settings().set("rtx/post/dlss/execMode", 2)

    # Resolve CAMERA_PATHS to UsdGeom.Camera prims; fall back to a default (5,5,5)
    # camera looking at the origin when CAMERA_PATHS is empty or none resolve.
    stage = omni.usd.get_context().get_stage()
    valid_camera_paths: list[str] = []
    for path in CAMERA_PATHS:
        prim = stage.GetPrimAtPath(path) if path else None
        if prim is None or not prim.IsValid():
            print(f"[TeleopReplay] Camera path '{path}' not found in stage, skipping")
            continue
        if not prim.IsA(UsdGeom.Camera):
            print(f"[TeleopReplay] Prim at '{path}' is not a UsdGeom.Camera (type={prim.GetTypeName()}), skipping")
            continue
        valid_camera_paths.append(path)

    render_products = []
    if valid_camera_paths:
        print(f"[TeleopReplay] Using {len(valid_camera_paths)} scene camera(s): {valid_camera_paths}")
        for i, cam_path in enumerate(valid_camera_paths):
            render_products.append(rep.create.render_product(cam_path, RESOLUTION, name=f"ReplayRP_{i}"))
    else:
        if CAMERA_PATHS:
            print(
                "[TeleopReplay] No valid scene cameras found in CAMERA_PATHS, falling back to default (5,5,5) camera."
            )
        cam = rep.functional.create.camera(position=(5, 5, 5), look_at=(0, 0, 0), name="ReplayCamera")
        render_products.append(rep.create.render_product(cam, RESOLUTION, name="ReplayRP"))
