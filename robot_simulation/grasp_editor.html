Grasp Editor#

Learning Objectives#

이 튜토리얼에서는 NVIDIA Isaac Sim의 Grasp Editor 익스텐션을 사용하여 특정 그리퍼/객체 쌍에 대한 그립을 수동으로 작성하고 시뮬레이션하는 방법을 설명합니다. 이 그립들은 isaac_grasp YAML 파일에 저장되며, 그리퍼를 위치로 이동하고 원하는 객체를 파지하는 모션 생성 알고리즘과 함께 가져와 사용할 수 있습니다.

Getting Started#

Grasp Editor 익스텐션 사용을 시작하려면 NVIDIA Isaac Sim에서 에셋을 준비해야 합니다.

  • 파지 가능한 Articulation이 있어야 합니다. 부동 그리퍼이거나 팔에 부착된 그리퍼일 수 있습니다.

  • 파지하려는 객체의 USD 버전이 있어야 합니다.

그리퍼와 객체 모두에 대해 위치를 나타내는 데 사용해야 하는 USD 프레임을 식별할 준비가 되어 있어야 합니다. 이것은 종종 객체 메시의 중앙과 그리퍼 베이스에 있는 프레임입니다.

이 튜토리얼에서 사용된 스테이지를 여기에서 다운로드하고 따라할 수 있습니다.

다운로드 후 압축을 풉니다. 실행 중인 NVIDIA Isaac Sim 인스턴스에서 File > Open을 클릭하고 grasp_editor_tutorial.usd 파일을 선택합니다.

What is an Isaac Grasp File?#

Grasp Editor 익스텐션의 출력은 isaac_grasp 파일 형식의 YAML입니다. 단일 isaac_grasp 파일은 특정 그리퍼/객체 쌍에 대한 그립 목록을 저장합니다. 파일은 간단한 형식을 따릅니다:

 1format: isaac_grasp
 2format_version: 1.0
 3
 4object_frame: /World/mug
 5gripper_frame: /World/panda_hand
 6
 7grasps:
 8  grasp_0:
 9      confidence: 1.0
10      position: [-0.04346, 0.06759, 0.19895]
11      orientation: {w: 0.00332, xyz: [0.98453, 0.16837, 0.04837]}
12      cspace_position:
13        panda_finger_joint1: 0.00943
14      pregrasp_cspace_position:
15        panda_finger_joint1: 0.04

isaac_grasp 파일은 Grasp Editor 익스텐션에서 시작될 필요가 없습니다. Grasp Editor 익스텐션은 isaac_grasp 파일을 작성하고 시각화 및 검증을 위해 다른 곳에서 작성된 그립을 가져오는 데 모두 유용합니다.

그립은 그리퍼와 객체의 상대적 위치로 정의됩니다. 이 상대적 위치가 의미를 가지려면 그리퍼와 객체 위치에 대한 대표 프레임을 선택해야 합니다. Grasp Editor는 이 프레임들을 두 가지 다른 방식으로 처리합니다:

  • 내보낼 때 대표 프레임의 USD 경로가 object_frame과 gripper_frame 필드 아래 isaac_grasp 파일에 기록됩니다.

  • 가져올 때 object_frame과 gripper_frame 필드는 무시됩니다. isaac_grasp 파일이 외부에서 작성될 수 있기 때문입니다(USD를 전혀 거치지 않을 수도 있음).

  • 따라서 Grasp Editor를 가져오기에 사용할 때 올바른 USD 프레임을 식별하는 것은 사용자의 책임입니다.

isaac_grasp 파일의 각 그립에는 고유한 이름(예: grasp_0)이 있습니다. 명명된 그립의 필드는 다음과 같습니다:

  • confidence: 그립의 품질을 설명하는 파라미터.

  • position: 객체 프레임에 대한 그리퍼 프레임의 이동.

  • orientation: 객체 프레임에 대한 그리퍼 프레임의 방향.

  • cspace_position: 그리퍼를 제어하는 데 사용되는 모든 조인트에 대한 조인트 위치 사전. 이 조인트 위치는 그리퍼가 객체를 적극적으로 파지하는 상태입니다.

  • pregrasp_cspace_position: 그리퍼를 제어하는 데 사용되는 모든 조인트에 대한 조인트 위치 사전. 이 조인트 위치는 그리퍼의 열린 위치를 나타냅니다.

전체적으로 그립은 pregrasp_cspace_position에 있는 동안 그리퍼를 올바른 상대적 위치와 방향으로 이동한 다음 조인트가 cspace_position에 있을 때까지 그리퍼를 닫아 실제로 적용할 수 있습니다. 세계 기준 프레임에서 객체의 위치와 방향이 \(T_o, R_o\)로 주어지고, position과 orientation 필드가 상대 변환 \(^oT_g, ^o\!\!R_g\)(즉, 객체 기준 프레임에 따른 그리퍼의 이동과 회전)을 지정하는 경우, 세계 기준 프레임에서 그리퍼의 원하는 위치 \(T_g , R_g\)는 다음과 같습니다:

\[\begin{split}T_g = R_o \cdot {^oT_g + T_o} \\ R_g = R_o \cdot {^o\!R_g}\end{split}\]

Using the Grasp Editor#

Selection Frame#

Grasp Editor는 isaac_grasp 파일을 작성하고 가져오는 데 사용할 수 있는 UI 기반 익스텐션입니다. NVIDIA Isaac Sim에서 Grasp Editor는 Tools > Robotics > Grasp Editor의 툴바에서 찾을 수 있습니다. 첫 번째 단계는 스테이지에 Articulation과 객체를 추가하는 것입니다. Articulation은 독립적인 그리퍼이거나 로봇 팔에 부착된 그리퍼일 수 있습니다. 객체는 연관된 메시가 있는 Articulation이 아닌 것이면 됩니다.

Selection Frame을 채우려면:

  1. 관심 있는 Articulation과 객체를 선택합니다. 객체의 프림 경로는 원하는 프림을 우클릭하고 "Copy Prim Path"를 선택하여 복사할 수 있습니다.

  2. isaac_grasp 파일의 내보내기 경로를 선택합니다. 이것은 '.yaml'로 끝나야 합니다.

내보내기 파일에 몇 가지 규칙이 적용됩니다:

  • Grasp Editor는 선택된 내보내기 파일에 그립 시퀀스를 작성하는 데 사용할 수 있지만 기존 파일 수정은 지원하지 않습니다.

  • 이미 존재하는 내보내기 경로가 제공되면 기존 파일이 새 isaac_grasp 파일로 덮어씌워집니다.

이 튜토리얼에서는 Panda 핸드 그리퍼(Franka Emika Panda 로봇에서 분리된)와 머그컵 사이의 그립을 작성합니다. "Ready"를 클릭하면 Grasp Editor가 다음을 수행합니다:

  • 패널의 각 필드를 검증합니다.

  • 선택한 객체 프림(머그컵)에 필요한 모든 변환을 수행하여 파지 가능하게 만듭니다. 구체적으로, 객체에 충돌 기하학이 있고 외부 힘에 의해 이동될 수 있도록 Usd Physics의 Rigid Body와 Collision API를 적용합니다.

Note

Grasp Editor는 객체 에셋에 대한 이러한 변경 사항을 되돌리지 않으므로 이러한 변경 사항이 특별히 원하지 않는 한 USD 스테이지를 저장하지 않는 것이 가장 좋습니다.

Warning

머그컵이 "사라질" 수 있다는 알려진 문제가 있습니다. 이것은 시각적 버그입니다. "STOP"을 누른 다음 "PLAY"를 다시 누르면 다시 나타납니다.

Select Frames of Reference#

이 패널에서 그리퍼와 객체의 공간 위치를 설명하는 데 사용해야 하는 기준 프레임을 선택할 수 있습니다. 이 패널을 이해하고 계속 진행하기 전에 올바른 선택을 하는 것이 중요합니다.

대부분의 모션 생성 알고리즘은 기본적으로 USD 파일을 사용하지 않습니다. 모션 생성 알고리즘은 URDF 파일을 참조하는 것이 일반적입니다. Grasp Editor가 해당 URDF 파일에 정의되지 않은 프레임을 사용하면 작성된 그립은 이러한 모션 생성 알고리즘의 관점에서 의미가 없어집니다.

마찬가지로 객체에 대해 선택된 기준 프레임은 객체가 조작되는 기존 파이프라인에 해당해야 합니다. 예를 들어 카메라가 객체 포즈를 식별하는 데 사용되는 경우 해당 비전 시스템과 관련된 객체의 암묵적 기준 프레임이 있습니다. 이 경우 객체에 대해 선택된 프레임은 이 암묵적 기준 프레임에 해당해야 합니다. USD에 올바른 기준 프레임을 나타내는 프레임이 아직 없으면 선택된 객체 경로 아래 스테이지에 새 프레임을 작성해야 합니다(예: "/World/mug" 아래 중첩).

이 튜토리얼에서는 그리퍼와 객체의 기본 프레임이 사용됩니다. 전체 Franka Panda 로봇이 사용되는 경우 그리퍼의 올바른 기준 프레임은 여전히 panda_hand 프레임입니다. "Finalize"를 클릭하면 이 기준 프레임들이 출력 isaac_grasp 파일에 전역적이 되고 변경할 수 없습니다.

Grasp Editor는 기준 프레임의 USD 경로를 출력 isaac_grasp 파일에 기록하지만, 이 정보는 USD를 사용하지 않는 모션 생성 알고리즘에서는 해석할 수 없습니다.

Joint Settings#

이 메뉴에서 Articulation의 어떤 조인트가 그리퍼의 활성 자유도(DOF)인지 선택해야 합니다. Panda 핸드는 두 손가락 그리퍼이지만 조인트 중 하나는 미믹 조인트입니다. 아래 그림에서 panda_finger_joint1의 값을 변경하면 panda_finger_joint_2가 동시에 움직이는 것을 확인하세요. 이는 Panda 핸드 그리퍼가 단일 DOF로 효과적으로 제어된다는 것을 의미합니다.

그리퍼의 각 활성 DOF는 "Part of Gripper"로 선택해야 합니다. 이렇게 하면 그립이 어떻게 시뮬레이션될지와 출력 isaac_grasp 파일에 기록될 내용을 정의하는 조인트 설정의 새 메뉴가 열립니다.

  • Position When Open: 열린 것으로 간주되는 DOF의 위치. 각 그립은 열린 위치에서 닫힌 위치를 향해 이동하는 방식으로 시뮬레이션됩니다.

  • Position When Closed: 완전히 닫힌 것으로 간주되는 DOF의 위치.

  • Grasp Speed: 시뮬레이션 시 DOF가 열린 위치에서 닫힌 위치를 향해 이동하는 속도.

  • Max Effort Magnitude: 시뮬레이션 시 이 DOF가 객체에 가할 수 있는 최대 힘/토크(N 또는 N*m).

그립을 작성하려면 최소 하나의 DOF가 Gripper의 일부로 표시되어야 합니다. 활성 그리퍼 DOF만 출력 isaac_grasp 파일에 기록됩니다.

Utils#

Utils 메뉴에는 Grasp Editor 사용을 도와주는 두 가지 유용한 유틸리티 기능이 있습니다.

Mask Collision 버튼은 그리퍼와 객체 사이의 충돌을 마스킹합니다. 이것은 그립을 테스트하기 위해 객체를 위치로 이동할 때 도움이 될 수 있습니다. 마스킹된 충돌은 그립이 시뮬레이션될 때 해제됩니다. 그립을 가져올 때 충돌이 자동으로 마스킹됩니다.

시뮬레이션된 그립이 객체와 그리퍼 사이에 완전한 접촉이 없는 것처럼 보이면 "Show Physics Colliders" 버튼을 사용하여 에셋과 관련된 충돌 기하학을 시각화할 수 있습니다. 잘못된 충돌체 기하학을 수정하는 것은 이 익스텐션의 범위 밖이지만 Grasp Editor는 시뮬레이션 없이 그립을 작성할 수 있습니다. 이 상황에서는 충돌을 마스킹하고 시각적으로 위치를 조정할 수 있습니다.

Author a Grasp#

에셋을 올바른 위치로 손으로 이동하는 것은 부정확하므로 시뮬레이션의 도움으로 그립을 작성할 수 있습니다. 아래 그림은 시뮬레이션 작성 워크플로우를 보여줍니다:

  1. 객체를 파지하기에 대략 맞는 위치로 이동합니다.

  2. "Simulate" 버튼을 클릭하여 조인트 설정에 따라 그리퍼를 닫습니다. 그림에서 이것은 머그컵의 입술이 그리퍼 손가락의 정확한 중앙으로 밀려 들어가고 그리퍼 손가락이 객체와의 정확한 접촉 위치에 남게 됩니다.

  3. 시뮬레이션이 완료되면 내보내기 패널이 채워지고 그립을 파일에 기록할 수 있습니다.

이런 방식으로 시뮬레이션이 접촉을 정착시키도록 하면 출력 파일에 기록되는 그립에 높은 신뢰도를 줍니다.

그립 시뮬레이션이 사용 사례를 지원하지 않을 이유가 있을 수 있습니다:

  • 에셋의 물리 충돌체가 정확하지 않습니다.

  • 그리퍼를 열고 닫는 메커니즘이 Grasp Editor에 표현된 것보다 더 복잡합니다.

어느 경우든 Grasp Editor를 최대한 활용하는 가장 좋은 방법은 외부 수단으로 위치를 조정하고 "Skip Sim" 버튼을 클릭하여 시뮬레이션 없이 그립을 내보내는 것입니다. 예를 들어 일부 실제 로봇 그리퍼는 다소 복잡한 메커니즘을 가진 강하게 결합된 자유도를 가집니다. 이러한 그리퍼의 경우 프로그래밍 방식으로 정확한 동작을 복제하고 USD 에셋에 조인트 명령을 전송하는 것이 좋습니다. 이 경우 충돌을 켜고 외부 스크립트나 OmniGraph 노드를 사용하여 그리퍼를 파지 위치로 구동한 다음 Grasp Editor의 내보내기 기능을 사용하여 USD 스테이지의 현재 그립 상태를 isaac_grasp 파일로 내보낼 수 있습니다.

Adding External Forces and Torques#

Grasp Editor의 추가 기능으로 그립 시뮬레이션의 일부로 외부 힘과 토크를 적용할 수 있습니다. 이것은 어떤 그립이 객체에 대해 최상의 힘 클로저를 가지는지 식별하는 데 도움이 될 수 있습니다. 적용되는 힘과 토크의 양은 "Add External Rigid Body Forces" 패널에서 선택할 수 있습니다. 힘과 토크에 대해 단일 스칼라 값을 선택할 수 있습니다. 힘에 대한 0이 아닌 값 \(v\)는 강체의 기본 프레임을 중심으로 각 축을 따라 \(\pm v\) N의 힘을 유발합니다. 마찬가지로 토크의 경우 값 \(v\)는 강체의 기본 프레임을 중심으로 각 축에 대해 \(\pm v\) N*m의 토크를 적용합니다.

아래 그림은 그립을 닫고 3 N의 힘을 적용하는 것을 보여줍니다. 이 테스트는 머그컵이 \([3, 0, 0]\)의 힘을 받아 날아갈 때 실패합니다. 그런 다음 더 작은 힘 값인 0.5 N이 선택되고 머그컵은 힘 아래 움직이지만 그립은 유지됩니다.

Exporting Grasps#

내보내기 프레임은 그립이 완전히 시뮬레이션되거나 시뮬레이션 옵션이 거절된 후에 사용 가능해집니다. "Export"를 클릭하면 스테이지의 현재 상태가 isaac_grasp 파일의 관련 필드를 채우는 데 사용됩니다.

  • confidence 필드는 Export 패널의 "Confidence" 필드 값을 가집니다.

  • 그립의 position과 orientation 필드는 객체의 기준 프레임에서 그리퍼의 상대적 위치를 찾아 결정됩니다. 이것은 Select Frames of Reference에서 정의된 프레임을 사용합니다.

  • cspace_position 필드는 그리퍼의 일부로 표시된 DOF의 현재 위치를 기반으로 결정됩니다.

  • pregrasp_cspace_position 필드는 그리퍼의 일부로 표시된 각 DOF의 Joint Settings의 "Position When Open" 필드에서 가져옵니다.

이 단계에서 여러 그립을 연속으로 작성하고 순차적으로 동일한 isaac_grasp 파일로 내보낼 수 있습니다.

Importing Grasps#

그립을 작성하는 것 외에도 Grasp Editor는 다른 곳에서 작성된 그립을 검증하는 데 사용할 수 있습니다. Import 패널에서 isaac_grasp 파일을 선택하고 Import를 클릭하여 수행할 수 있습니다. 이 튜토리얼은 내보내기에 사용된 것과 동일한 파일을 사용하지만 그럴 필요는 없습니다.

아래 그림에서 여러 그립이 Grasp Editor를 사용하여 작성되고 파일에 기록되었습니다. 이 그립들이 가져와지고 이제 순서대로 빠르게 시각화하고 시뮬레이션할 수 있습니다.

Using Authored Grasps in Isaac Sim#

Grasp Editor는 주로 UI 기반 익스텐션이지만 Python API를 통해 NVIDIA Isaac Sim에서 작성된 그립을 가져오고 사용하는 데 유용한 기능을 제공합니다.

이 섹션에서는 작성된 그립 중 하나를 실행하기 위해 로봇이 어디로 가야 하는지 결정하는 목표로 다음 스테이지를 제시합니다.

다음 함수 스니펫은 Importing Grasps에서 시연된 그립 파일을 가져오고 grasp_1을 복제하기 위해 panda_hand 프레임이 있어야 하는 위치를 결정합니다. 이 함수를 시험하려면 Script Editor에 복사하고 import_file_path="path/to/your/isaac_grasp.yaml" 인수를 함수에 전달하세요.

import isaacsim.core.experimental.utils.xform as xform_utils
from isaacsim.robot_setup.grasp_editor import GraspSpec, import_grasps_from_file


def compute_gripper_pose_for_grasp(
    import_file_path: str,
    mug_reference_frame: str = "/World/mug",
    grasp_name: str = "grasp_1",
) -> tuple:
    """Compute and print the gripper pose target needed to execute a named grasp.

    Args:
        import_file_path: Path to an ``isaac_grasp`` YAML file to import.
        mug_reference_frame: USD prim path of the rigid body whose pose anchors the grasp.
        grasp_name: Name of the grasp inside the file to compute targets for.

    Returns:
        Tuple of ``(gripper_trans_target, gripper_orientation_target)``.
    """
    grasp_spec: GraspSpec = import_grasps_from_file(import_file_path)
    grasp_names = grasp_spec.get_grasp_names()

    mug_trans, mug_quat = xform_utils.get_world_pose(mug_reference_frame, device="cpu")
    mug_trans, mug_quat = mug_trans.numpy(), mug_quat.numpy()

    gripper_trans_target, gripper_orientation_target = grasp_spec.compute_gripper_pose_from_rigid_body_pose(
        grasp_name, mug_trans, mug_quat
    )

    print("Grasp Names:", grasp_names)
    print("Gripper Translation Target:", gripper_trans_target)
    print("Gripper Orientation Target:", gripper_orientation_target)

    return gripper_trans_target, gripper_orientation_target



Grasp Names: ['grasp_0', 'grasp_1', 'grasp_2']
Gripper Translation Target: [ 0.41496072 -0.03612298  0.27738899]
Gripper Orientation Target: [-0.1690746   0.63886658  0.12752551  0.73959483]

코드 스니펫의 결과는 isaac_grasp 파일의 각 그립 이름과 전체 Franka 로봇에서 panda_hand 프레임에 설정해야 하는 이동 및 방향 목표를 보여줍니다. 코드 스니펫은 Grasp Editor에서 선택된 머그컵의 기준 프레임을 사용합니다. 이 그립을 달성하기 위해 모션 생성 알고리즘을 사용하는 것은 이 튜토리얼의 범위 밖입니다.

완전한 기능 세트를 보려면 API Documentation의 GraspSpec 클래스를 확인하세요.
