Tutorial 3: Inspect Asset#

Inspire Hand가 여러 USD 파일로 구성된 방식을 확인했습니다(Tutorial 2). 이제 해당 에셋을 검사하고 검증합니다: 관절, 질량과 관성, 충돌 메시. 이 단계를 건너뛰면 어둠 속에서 튜닝하는 것과 같습니다—잘못된 질량이나 정렬되지 않은 관성은 관절 파라미터가 올바르게 보여도 불안정하거나 비현실적인 동작을 일으킬 수 있으며, 잘못된 콜라이더 유형은 시뮬레이션을 느리게 하거나 혼란스러운 접촉 동작을 유발할 수 있습니다. Isaac Sim의 관절 시각화 도구, Robot Inspector, Physics Debugger, 콜라이더 시각화는 충돌 쌍을 필터링하거나 드라이브를 튜닝하기 전에 에셋에 대한 명확한 그림을 제공합니다.

Learning Objectives#

이 튜토리얼에서 수행할 작업:

  • 활성화하기: 관절 시각화 도구를 활성화하고 관절 유형 해석하기.

  • 활성화하기: 질량 및 관성 시각화 활성화하기.

  • 검증하기: 충돌 메시 및 콜라이더 유형 검증하기.

Prerequisites#

  • Tutorial 2: Asset Structure를 완료합니다.

  • Isaac Sim에서 PhysX 배리언트가 선택된 Inspire Hand 씬이 열려 있어야 합니다.

Module 2.1: Enable Joint Visualizer#

PhysX 백엔드에 맞게 튜닝하므로, PhysX 배리언트로 핸드를 로드합니다. 그런 다음 관절 시각화를 활성화하여 관절 위치와 유형을 한눈에 확인합니다.

Isaac Sim 뷰포트 탐색

  • 카메라 궤도 회전: Alt를 누른 채 마우스 왼쪽 버튼을 누르고 드래그합니다.

  • 제자리 회전(둘러보기): 마우스 오른쪽 버튼을 누른 채 마우스를 이동합니다.

  • 줌: Alt를 누른 채 마우스 오른쪽 버튼(또는 스크롤 휠 사용).

  • 이동: 마우스 가운데 버튼을 누른 채 드래그합니다.

  • 프림에 카메라 포커스: Stage 패널에서 원하는 프림을 선택한 후 F를 누릅니다.

아래 지침을 따르면서 Inspire Hand 모델을 효율적으로 탐색하고 검사하는 데 이 컨트롤을 활용하세요.

  1. Isaac Sim에서 /path/to/Inspire/module_1_start/inspire_hand.usda를 엽니다.

  2. 최상위 레벨 inspire_hand 프림을 선택합니다.

  3. Property 패널에서 Variants로 스크롤하여 PhysX를 선택합니다.

  1. Eye > Show by Type > Physics > Joints로 이동하여 관절 시각화를 활성화합니다.

뷰포트에서 Inspire Hand에 각 관절의 위치와 유형을 식별하는 기즈모가 표시됩니다.

관절 검사 — Stage 패널에서 /Physics 스코프 아래에서 right_index_1_joint를 찾습니다—검지 손가락의 기본 동작을 담당하는 Revolute 관절로, 뷰포트에서 원형 아이콘으로 표시됩니다. 또한 right_index_rubber_1_joint를 찾습니다, 이는 하부 검지 고무 패드를 링크에 부착하는 Fixed 관절로, 시각화에서 사각형 아이콘으로 표시됩니다. right_index_2_joint는 right_index_1_joint의 움직임을 참조하는 미믹 관절입니다(미믹 관절에 대해서는 Tutorial 5에서 자세히 다룹니다). 이러한 관절의 기능과 명명 규칙을 이해하면 Tutorial 5와 6에서 드라이브를 튜닝할 때 유용합니다.

Module 2.2: Robot Inspector (hierarchy and session masking)#

뷰포트에서 관절 기즈모가 보이는 상태에서, Robot Inspector Window는 동일한 관절 구조를 /Physics 아래에서만 프림을 찾을 때보다 스캔하기 쉬운 구조화된 링크 → 관절 트리로 제공합니다—페이로드와 스코프가 레이어 전반에 걸쳐 프림을 분산시킬 때 특히 유용합니다.

  1. Window > Robot Inspector를 엽니다. 창이 기본적으로 Stage 옆에 도킹됩니다.

  2. 로봇 목록에서 Inspire Hand 항목을 선택합니다.

  3. 계층 구조 모드를 Tree(기본값)로 설정합니다: 부모 링크 → 관절 → 자식 링크.

  4. 선택적으로 Flat(모든 링크, 그 다음 모든 관절) 또는 MuJoCo(기반 루트 바디 트리)로 전환하여 레이아웃을 비교합니다; 동일한 기본 관절 구조를 세 가지 다른 방식으로 표시할 수 있습니다.

Deactivate, Bypass, Anchor 열은 전용 세션 서브레이어에 일시적 의견을 적용합니다—이는 USD 파일에 저장되지 않습니다. 디버깅 중 빠른 격리에 유용합니다.

참고

Deactivate, Bypass, Anchor의 아이콘과 동작은 Component Masking 아래에 문서화되어 있습니다.

Robot Inspector가 열려 있을 때, Eye Icon > Show by Type > Physics > Joints가 활성화된 경우 관절 연결선(부모에서 자식으로, 방향 힌트 포함)이 뷰포트에 표시됩니다; 이는 Robot Inspector Window에 설명된 대로 시뮬레이션 재생 중에는 숨겨집니다.

Module 2.3: Verify Mass and Inertia Properties#

질량과 관성은 각 링크가 힘에 어떻게 반응하는지를 정의합니다. 주 관성 축이 링크 지오메트리와 정렬되지 않거나, 질량 값이 너무 작거나 크면 핸드가 비현실적으로 동작할 수 있습니다. Physics Debugger를 통해 바디 축과 Body Mass Axes(주 관성)을 시각화하여 시뮬레이션 실행 전에 문제를 발견할 수 있습니다.

  1. Utilities > Physics Debugger를 엽니다. Physics Debug 패널이 나타납니다.

  1. Simulation Debug Visualization에서:

    • Enabled를 체크합니다.

    • Body Axes를 체크하여 좌표 프레임을 표시합니다.

    • Body Mass Axes를 체크하여 주 관성 축을 표시합니다.

  2. Simulation Control에서 Step을 클릭하여 하나의 시뮬레이션 프레임을 실행하고 시각화를 표시합니다.

Warning

이 단계에서 Play를 누르지 마세요, Isaac Sim이 크래시될 수 있습니다. 대신 Simulation Control을 사용하여 물리 시뮬레이션을 Run하거나 한 프레임씩 Step하세요.

  1. 각 링크에 대해 다음을 확인할 수 있습니다:

    • 질량 중심이 링크 내에 적절히 위치합니다.

    • 주 관성 축이 링크 지오메트리와 정렬됩니다.

    • 관성 값이 합리적으로 보입니다(지나치게 작거나 크지 않음).

Note

정렬되지 않은 주 관성 축은 불안정하거나 비현실적인 동작을 일으킬 수 있습니다. 아래 이미지는 정렬 불량의 예시를 보여줍니다.

Alternative Method: Inspecting Mass and Inertia via the Physics Toolbar#

Physics 도구모음을 사용하여 질량 및 관성 속성을 검사할 수도 있습니다:

  1. Tools > Physics Toolbar로 이동합니다.

  1. 도구모음에서 Rigid Body Selection Mode(큐브 아이콘)와 Mass Distribution Manipulator(저울 아이콘)를 모두 켭니다.

  2. 뷰포트에서 핸드의 임의의 강체 프림을 선택합니다. Mass Properties Info가 표시되어 뷰포트에서 직접 총 질량, 질량 중심, 주 축, 대각 관성에 대한 세부 정보를 제공합니다.

이 방법을 사용하면 Property 패널로 이동하지 않고도 씬의 모든 바디에 대해 질량 분포를 빠르게 검사하고 디버그할 수 있습니다.

Module 2.4: Verify Collision Meshes#

뷰포트에서 보이는 형상이 반드시 물리 엔진이 접촉에 사용하는 것은 아닙니다—그것은 충돌 메시(콜라이더)에 의해 결정됩니다. Tutorial 4에서 충돌 쌍을 필터링하기 전에, 콜라이더를 검사하고 확인합니다: 콜라이더는 강체에 대해 녹색, 정적 바디에 대해 자홍색으로 색상 코딩됩니다.

  1. Eye > Show by Type > Physics > Colliders > All로 이동하여 모든 충돌 형상을 시각화합니다.

콜라이더 유형 설정은 성능과 정확도에 영향을 미칩니다. 하나의 에셋에 유형을 혼합할 수 있습니다. 다관절 손의 경우, 높은 정확도가 필요 없는 부품(예: 손바닥)에는 Convex Hull이 자주 사용되고, 정확한 접촉이 필요한 부품(예: 손끝)에는 Convex Decomposition이 사용됩니다. Convex Decomposition이 가장 좋은 형상을 제공하지만 Convex Hull이나 지오메트리 기반 콜라이더보다 비용이 더 많이 듭니다. 이 튜토리얼에서는 모든 부품에 Convex Hull을 사용합니다.

Summary#

이 튜토리얼에서 다룬 내용:

  • 관절 시각화 도구를 활성화하고 Stage에서 관절 유형(Fixed, Revolute, Mimic)을 식별하기—Tutorial 5와 6에서 튜닝할 동일한 구조입니다.

  • Robot Inspector를 열어 핸드의 운동학적 계층(Flat / Tree / MuJoCo 모드)을 검토하고 세션 마스킹을 이해하기.

  • Physics Debugger를 사용하여 바디 축과 주 관성을 시각화하고 각 링크의 질량 중심 및 관성 정렬이 올바른지 확인하기.

  • 콜라이더 시각화를 켜고 콜라이더 전략(이 시리즈에서는 Convex Hull)을 확인하여, Tutorial 4에서 자기 충돌이 활성화될 때 어떤 형상이 충돌할지 파악하기.

Next Steps#
