Mesh merge tool#

버전 6.0부터 지원 종료: 독립 실행형 isaacsim.util.merge_mesh 확장은 지원이 종료되었으며 향후 릴리스에서 제거될 예정입니다. 대신 URDF/MJCF 임포터의 Merge Mesh 옵션, Scene Optimizer 패널(Window > Utilities > Scene Optimizer), Asset Transformer MergeMeshRule, 또는 Python에서 직접 작업을 호출하여 Scene Optimizer merge 작업을 사용하세요.

About#

메시 병합은 공통 강체 부모를 공유하는 여러 시각적 형상 프림을 단일 메시로 결합합니다. 형상 프림의 수를 줄이면 드로우 콜 오버헤드가 낮아지고 렌더링 및 물리 성능이 향상됩니다.

NVIDIA Isaac Sim은 Scene Optimizer Kit 확장(omni.scene.optimizer.core)을 통해 메시 병합을 수행합니다. Scene Optimizer merge 작업은 결과 메시에 재료를 GeomSubset 항목으로 보존하고, 정점 중복 제거를 지원하며, 메시를 공간적으로 그룹화할 수 있습니다.

NVIDIA Isaac Sim에서 Scene Optimizer 병합 작업을 노출하는 네 가지 통합 경로:

Integration

Where to use it

Best for

URDF / MJCF importer

File > Import > Merge Mesh 체크박스

로봇 에셋의 일회성 임포트 시간 병합

Scene Optimizer panel

Window > Utilities > Scene Optimizer

기존 에셋의 대화형 병합, 임시 사전 설정

Asset Transformer profile

Asset Transformer > MergeMeshRule

반복 가능한 버전 관리 에셋 파이프라인

Python API

isaacsim.asset.importer.utils.merge_mesh_utils

커스텀 스크립트 및 헤드리스 에셋 준비

Using the URDF / MJCF importer#

URDF와 MJCF 임포터 모두 임포터 UI의 Options 섹션 아래에 Merge Mesh 체크박스를 제공합니다. 이 옵션을 활성화하면 변환된 USD 스테이지에서 순차적으로 세 개의 Scene Optimizer 작업이 실행됩니다:

  1. meshCleanup — 중복 정점을 병합하고, 퇴화 면을 제거하고, 비다양체 형상을 수정합니다.

  2. generateNormals 및 generateProjectionUVs — 정점 법선과 투영 UV를 재생성합니다.

  3. merge — 스테이지의 각 강체에 대해 해당 강체 아래에 있는 모든 자식 시각적 메시 프림(default 또는 render 목적을 가진)을 단일 메시로 병합합니다.

임포터는 각 UsdPhysics.RigidBodyAPI 프림을 순회하고 자식 형상을 수집하여 중첩된 강체 경계에서 중지함으로써 병합 그룹을 자동으로 결정합니다. 추가 구성이 필요하지 않습니다.

전체 임포터 워크플로우는 URDF Importer Extension 및 MJCF Importer Extension를 참조하세요.

Using the Scene Optimizer UI#

Scene Optimizer 확장에는 Asset Transformer 규칙 없이 merge 작업을 직접 노출하는 자체 UI 패널이 있습니다. 기존 스테이지에서 일회성 대화형 병합을 수행하거나 merge를 중복 제거 및 재료 통합과 같은 다른 Scene Optimizer 작업과 결합하려면 이 경로를 사용하세요.

Scene Optimizer 패널을 사용하여 메시를 대화형으로 병합하세요.

  1. Scene Optimizer 확장을 활성화합니다. Scene Optimizer는 표준 NVIDIA Isaac Sim 경험에서 기본적으로 활성화되어 있지 않습니다. Extension Manager(Window > Extensions)를 열고 omni.scene.optimizer.bundle을 검색하여 활성화합니다. 번들은 core, ui, analysis, validators 확장을 가져옵니다.

  2. 병합할 에셋을 로드합니다. File > Open을 사용하여 USD를 활성 스테이지에 로드하세요.

  3. Scene Optimizer 패널을 엽니다. Window > Utilities > Scene Optimizer를 선택하세요.

  4. 워크플로우를 선택합니다:

    • 번들 사전 설정을 사용하려면 Load Preset을 클릭하고 병합 관련 사전 설정 중 하나를 선택하세요:

      • Modify Stage - Merge Meshes — 계층 구조를 평탄화하고 재료별로 메시를 /World/Geometry/merged로 병합합니다.

      • Modify Stage - Spatial Merge Meshes — 공간적 경계 볼륨을 기반으로 재료에 관계없이 모든 메시를 병합합니다. 매우 밀집된 장면에 유용합니다.

      • Modify Instances - Merge Meshes — 인스턴스 계층 구조를 보존하면서 각 프로토타입 내부의 메시를 병합합니다. 에셋이 씬그래프 인스턴싱을 사용하는 경우 가장 좋습니다.

    • 선택한 메시 집합을 병합하려면 Add Scene Optimizer Process를 클릭하고 Merge Meshes(merge로 등록된 작업)를 선택하세요. 구성:

      • Mesh Prim Paths: 프림 경로 또는 경로 표현식을 입력하세요(예: /World/Robot/left_wheel//Mesh* — 앞에 있는 //는 left_wheel 아래의 임의 깊이에 있는 후손과 일치합니다). 연필 아이콘을 클릭하여 스테이지에서 경로를 선택하거나 뷰포트에서 프림을 선택한 상태에서 Add를 클릭하세요.

      • Consider Materials: 병합된 메시에 재료별 GeomSubset 파티션을 보존하려면 활성화하세요(레거시 Combine Materials 옵션의 Scene Optimizer 등가).

      • Original Geom Option: 소스 메시를 유지하려면 0, 비활성화하려면 1, 삭제하려면 2.

      • Merge Point: 병합된 메시를 월드 원점에 배치하려면 0, 루트 프림의 변환을 사용하려면 1.

      • Root Path: 병합 결과에 대한 선택적 명시적 경로. 기본값은 Mesh Prim Paths의 첫 번째 항목입니다.

      • Spatial Mode / Spatial Threshold / Spatial Vertex Count: 공간 그룹화가 필요하지 않으면 기본값을 유지하세요.

  5. 사전 정리 단계를 추가합니다(선택 사항이지만 권장). Add Scene Optimizer Process를 두 번 더 클릭하고 Mesh Cleanup(meshCleanup)과 Generate Normals(generateNormals)를 추가하세요. 각 프로세스 카드의 왼쪽 상단에 있는 핸들을 드래그하여 Merge Meshes 전에 실행되도록 하세요. 이는 URDF/MJCF 임포터가 내부적으로 실행하는 파이프라인과 일치합니다.

  6. 실행하고 확인합니다.

    • Execute All을 클릭하세요. 콘솔 로그가 작업이 실행될 때 보고합니다.

    • 스테이지를 검사하세요. 각 병합 그룹은 구성된 Root Path 아래의 단일 메시 프림으로 교체됩니다. Original Geom Option이 1로 설정된 경우, 소스 메시는 비활성화된 프림으로 스테이지에 남아 있습니다.

    • File > Save As로 결과를 저장하세요.

  7. 재사용을 위해 구성을 저장합니다(선택 사항). Save Preset을 클릭하여 현재 프로세스 스택을 JSON 파일에 쓰세요. 동일한 JSON을 나중에 이 UI에서 로드하거나 배치 처리를 위한 Scene Optimizer CLI에 공급할 수 있습니다.

참고

Scene Optimizer UI는 메모리에서 활성 스테이지에 직접 작업을 실행합니다. 병합 후 스테이지를 명시적으로 저장하세요 — 패널은 자체적으로 디스크에 쓰지 않습니다.

Scene Optimizer 패널은 자체 JSON 사전 설정 형식을 읽고 씁니다. 임포터의 파이프라인(정리, 법선 + UV, 병합)을 반영하는 최소 사전 설정은 다음과 같습니다:

[
  {
    "operation": "meshCleanup",
    "paths": [],
    "mergeVertices": true,
    "tolerance": 0.0,
    "mergeBoundaries": true,
    "mergeNeighbors": true,
    "contractDegenerateEdges": true,
    "removeDegenerateFaces": true,
    "removeIsolatedVertices": true,
    "removeDuplicateFaces": true,
    "makeManifold": true
  },
  {
    "operation": "generateNormals",
    "paths": [],
    "binding": 0,
    "replaceExisting": true,
    "weightMode": 0,
    "sharpnessAngle": 60.0,
    "gpuThreshold": 500000
  },
  {
    "operation": "generateProjectionUVs",

Using the asset transformer#

메시 병합을 다른 USD 재구성 규칙(스키마 라우팅, 형상 중복 제거, 재료 라우팅, 인터페이스 생성)과 연결해야 하는 경우, 병합 단계를 Asset Transformer 프로파일로 래핑하세요. Asset Transformer에는 임포터와 동일한 강체 인식 병합을 실행하는 Merge Mesh 프로파일(source/extensions/isaacsim.asset.transformer.rules/data/merge_mesh.json)이 포함됩니다:

{
  "profile_name": "Merge Mesh",
  "version": "1.0",
  "rules": [
    {
      "name": "Merge Meshes",
      "type": "isaacsim.asset.transformer.rules.isaac_sim.merge_mesh.MergeMeshRule",
      "destination": "",
      "params": {},
      "enabled": true
    }
  ],
  "interface_asset_name": null,
  "output_package_root": null,
  "flatten_source": false,
  "base_name": null
}

MergeMeshRule은 스테이지의 모든 강체를 순회하고 각 강체의 자식 시각적 메시에 Scene Optimizer merge 작업을 적용합니다 — 파라미터가 필요하지 않습니다.

전체 정리 + 병합 + 중복 제거 파이프라인을 위해 GeometriesRoutingRule 및 MaterialsRoutingRule과 연결하세요:

{
  "profile_name": "Cleanup + Merge + Dedupe",
  "version": "1.0",
  "rules": [
    {
      "name": "Merge Meshes",
      "type": "isaacsim.asset.transformer.rules.isaac_sim.merge_mesh.MergeMeshRule",
      "destination": "",
      "params": {},
      "enabled": true
    },
    {
      "name": "Route Geometries",
      "type": "isaacsim.asset.transformer.rules.perf.geometries.GeometriesRoutingRule",
      "destination": "payloads",
      "params": {
        "geometries_layer": "geometries.usd",
        "instance_layer": "instances.usda",
        "deduplicate": true
      },
      "enabled": true
    },
    {
      "name": "Route Materials",
      "type": "isaacsim.asset.transformer.rules.perf.materials.MaterialsRoutingRule",
      "destination": "payloads",
      "params": {
        "materials_layer": "materials.usda",
        "download_textures": true
      },
      "enabled": true
    }
  ],
  "interface_asset_name": null,
  "output_package_root": null,
  "flatten_source": false,
  "base_name": null
}

Tools > Robotics > Asset Editors > Asset Transformer > Load Preset으로 프로파일을 로드하고, Output Directory를 설정하고, Execute Actions를 클릭하세요. 전체 규칙 카탈로그는 Asset Transformer Rules Reference를, Asset Transformer GUI 안내는 Asset Transformer Tutorials를 참조하세요.

Using the Python API#

isaacsim.asset.importer.utils.merge_mesh_utils 모듈은 임포터에서 사용하는 Scene Optimizer 작업을 래핑합니다. 헤드리스 에셋 준비, 배치 처리, 또는 임포터나 트랜스포머 흐름에 맞지 않는 커스텀 에셋 "정리"에 사용하세요.

아래 코드 블록은 Isaac Sim의 python.sh로 실행할 수 있는 단일 실행 가능 스크립트 docs/isaacsim/snippets/robot_setup/merge_mesh.py에서 가져온 것입니다. --test를 전달하여 번들된 carter.urdf 테스트 에셋을 가져오고 모든 스니펫을 엔드투엔드로 테스트하세요.

참고

이러한 헬퍼를 호출하기 전에 omni.scene.optimizer.core 확장이 활성화되어 있어야 합니다. 독립 실행형 스크립트는 명시적으로 활성화해야 합니다:

import omni.kit.app

omni.kit.app.get_app().get_extension_manager().set_extension_enabled_immediate("omni.scene.optimizer.core", True)

사용 가능한 헬퍼:

Function

Purpose

clean_mesh_operation(stage)

meshCleanup 실행: 중복 정점 병합, 퇴화 면 제거, 다양체 만들기.

generate_mesh_uv_normals_operation(stage)

정점별 법선과 투영 UV를 재생성하기 위해 generateNormals 및 generateProjectionUVs를 실행합니다.

merge_meshes_operation(stage)

모든 강체를 순회하고 각 강체의 자식 시각적 메시를 병합합니다(병합된 그룹 수를 반환).

merge_mesh(stage, mesh_paths)

명시적 메시 프림 경로 목록을 단일 메시로 병합합니다.

예: 기존 스테이지에 적용된 전체 임포터 등가 파이프라인

from isaacsim.asset.importer.utils import merge_mesh_utils
from pxr import Usd

# stage_path = "/path/to/robot.usd"
stage = Usd.Stage.Open(stage_path)

merge_mesh_utils.clean_mesh_operation(stage)
merge_mesh_utils.generate_mesh_uv_normals_operation(stage)
merged_groups = merge_mesh_utils.merge_meshes_operation(stage)

print(f"Merged {merged_groups} rigid-body mesh group(s)")
stage.Save()

예: 선택한 메시 집합 병합(레거시 "프림 선택 후 Merge 클릭" 워크플로우 대체)

from isaacsim.asset.importer.utils import merge_mesh_utils
from pxr import Usd

# stage_path = "/path/to/asset.usd"
# mesh_paths = [
#     "/World/Jetbot/left_wheel/visual_0",
#     "/World/Jetbot/left_wheel/visual_1",
#     "/World/Jetbot/left_wheel/visual_2",
# ]
stage = Usd.Stage.Open(stage_path)

merge_mesh_utils.merge_mesh(stage, mesh_paths)
stage.Save()

목록의 첫 번째 프림이 병합 루트 및 원점으로 사용되며, 이는 레거시 도구의 "첫 번째 선택 우선" 동작과 일치합니다.

Scene Optimizer operation reference#

더 세밀한 제어를 위해 omni.scene.optimizer.core 작업을 직접 호출하세요. 스테이지에 바인딩된 SceneOptimizerCore 인스턴스와 ExecutionContext를 구성하고, 구성 딕셔너리로 이름별 작업을 디스패치하세요:

from omni.scene.optimizer.core import ExecutionContext, SceneOptimizerCore
from pxr import Usd

# stage_path = "/path/to/asset.usd"
# mesh_paths = ["/World/A", "/World/B", "/World/C"]
stage = Usd.Stage.Open(stage_path)

context = ExecutionContext()
context.set_stage(stage)
context.generateReport = 0
context.captureStats = 0

core = SceneOptimizerCore.getInstance()

core.executeOperation(
    "merge",
    context,
    {
        "meshPrimPaths": list(mesh_paths),
        "considerMaterials": True,
        "materialAlbedoAsVertexColors": False,
        "originalGeomOption": 1,
        "mergePoint": 0,
        "rootPath": mesh_paths[0],
        "considerAllAttributes": True,
        "allowSingleMeshes": False,
        "spatialMode": 0,
        "spatialThreshold": 10.0,
        "spatialMaxSize": 0.0,
        "spatialVertexCount": 10000,
        "spatialDebug": False,
    },
)
stage.Save()

임포터는 재료 라우팅이 에셋 트랜스포머의 MaterialsRoutingRule에 의해 별도로 처리되기 때문에 considerMaterials를 False로 설정합니다. 병합된 메시에 재료별 GeomSubset 파티션을 보존하려면 독립 실행형으로 merge를 실행할 때 True로 설정하세요 — 이것이 레거시 도구의 Combine Materials 옵션의 Scene Optimizer 등가입니다.

Migrating from the legacy mesh merge tool#

Legacy option

Scene Optimizer equivalent

Source Prim (첫 번째 선택 = 원점)

merge_mesh(stage, paths)에 메시 경로를 전달합니다. 첫 번째 경로가 rootPath로 사용됩니다.

Clear Parent Transform

merge 구성에서 mergePoint를 0(월드 원점) 또는 1(루트 프림의 변환)으로 설정합니다.

Deactivate source assets

originalGeomOption으로 제어됩니다: 0은 원본 유지, 1은 비활성화, 2는 삭제.

Combine Materials

merge 구성에서 considerMaterials = True로 설정합니다. 재료 할당이 병합된 메시의 GeomSubset 항목으로 보존됩니다.

원점을 설정하기 위해 빈 Xform을 먼저 선택

rootPath를 사용하여 임의의 프림 경로를 병합 타겟/원점으로 지정하세요.
