Robot Inspector Window#

isaacsim.robot.schema.ui 확장은 스테이지의 로봇 운동학적 구조를 검사하고 시뮬레이션을 위한 구성요소를 선택적으로 마스킹하기 위한 대화형 Robot Inspector 창을 제공합니다.

Opening the Window#

Robot Inspector 창은 Window > Robot Inspector를 통해 접근할 수 있습니다. 기본적으로 Stage 패널 옆에 도킹됩니다.

Hierarchy Display Modes#

창 상단의 모드 선택기는 로봇 계층 구조가 표시되는 방법을 제어합니다. 세 가지 모드를 사용할 수 있습니다:

Mode

Description

Flat

링크와 조인트는 각 로봇 아래에 두 개의 플랫 스코프(Links, Joints)로 나열되며 스키마 관계에 나타나는 순서대로 정렬됩니다. 전체 목록을 빠르게 검토하는 데 유용합니다.

Tree (기본값)

부모 링크 → 조인트 → 자식 링크 체인. 운동학적 순회 순서와 일치하며 아티큘레이션 로봇에 가장 자연스러운 표현입니다.

MuJoCo

베이스 링크에 루트를 둔 트리. 각 링크의 자식이 먼저 나타나고, 링크를 부모에 연결하는 조인트가 마지막 자식 항목으로 나타납니다. MuJoCo MJCF 파일에서 사용하는 body 중심 레이아웃을 반영합니다.

Flat

Tree

MuJoCo

Component Masking#

Robot Inspector는 작성된 USD 레이어를 수정하지 않고 시뮬레이션을 위한 개별 조인트와 링크를 비활성화할 수 있는 구성요소별 마스킹 컨트롤을 제공합니다. 모든 마스킹 의견은 디버깅 도구이며 세션 레이어 스택에 삽입된 전용 익명 서브레이어에 기록되므로 일시적이며 디스크에 저장되지 않습니다.

트리 뷰에는 세 개의 마스킹 열이 나타납니다:

Column

Description

Deactivate ()

시뮬레이션을 위한 조인트 또는 링크를 비활성화합니다. 조인트 비활성화는 jointEnabled = False를 설정합니다. 링크 비활성화는 강체를 비활성화하고, 충돌 메시를 끄고, 비강체 자식 형상을 숨깁니다.

Bypass ()

요소를 비활성화하고 그 주위의 운동학적 체인을 다시 연결합니다. 조인트의 경우 가장 가까운 이전 비마스킹 링크에서 각 이후 비마스킹 링크를 연결하는 고정 조인트가 생성됩니다. 링크의 경우 이전 조인트가 비활성화되고 이후 조인트가 재계산된 로컬 프레임 오프셋을 가진 가장 가까운 비마스킹 조상 링크로 재부모화됩니다.

Anchor ()

body0가 없는 임시 고정 조인트를 생성하여 현재 포즈에서 링크를 월드에 고정합니다. RigidBodyAPI가 적용된 링크에서만 사용할 수 있습니다.

마스킹 작업은 다중 선택을 지원합니다: 여러 프림이 선택된 상태에서 열 아이콘을 클릭하면 선택된 모든 프림에 단일 배치로 작업이 적용됩니다.

참고

마스킹 서브레이어는 새 스테이지가 열리거나 현재 스테이지가 닫힐 때 자동으로 지워집니다. 마스킹 상태는 세션 간에 유지되지 않습니다.

Viewport Joint Visualization#

Robot Inspector 창이 활성화되면, 조인트 위치에서 부모와 자식 링크를 연결하는 오버레이 선이 3D 뷰포트에 그려집니다. 이 시각화는 운동학적 체인의 빠른 시각적 확인을 제공합니다.

  • 연결선에는 부모-자식 관계를 나타내는 방향 화살표가 포함됩니다.

  • 여러 조인트가 동일한 화면 위치에 겹칠 때(밀집된 조인트 클러스터에서 일반적), 오버레이 원이 그려집니다. 원을 클릭하면 해당 위치의 모든 조인트를 나열하는 컨텍스트 메뉴가 열려 개별 조인트를 선택할 수 있습니다.

  • 조인트 시각화는 시뮬레이션 재생 중에 숨겨지고 재생이 중지되면 복원됩니다.

  • 가시성은 Visibility Menu (뷰포트의 Eye Icon) > Show by Type > Physics > Joints 설정으로 제어됩니다.

UI Utility Functions#

isaacsim.robot.schema.ui 확장은 계층 구조 생성 및 구성요소 검사를 위한 추가 유틸리티를 제공합니다. 다음을 통해 접근할 수 있습니다:

from isaacsim.robot.schema.ui.utils import HierarchyMode, generate_robot_hierarchy_stage

Hierarchy Mode#

HierarchyMode는 Robot Inspector와 인메모리 계층 구조 스테이지에서 로봇 계층 구조가 구성되는 방법을 제어하는 열거형입니다:

Value

Description

HierarchyMode.FLAT

Links 스코프 아래의 링크와 Joints 스코프 아래의 조인트, 스키마 관계 순서.

HierarchyMode.LINKED

부모 링크 → 조인트 → 자식 링크 체인 (기본값).

HierarchyMode.MUJOCO

베이스 링크에 루트를 둔 트리. 자식 링크가 각 링크 아래에 먼저 나타나고, 링크를 부모에 연결하는 조인트가 마지막 자식 항목으로 나타납니다.

Hierarchy Stage Generation#

Function

Description

generate_robot_hierarchy_stage(mode)

IsaacRobotAPI가 있는 프림의 현재 스테이지를 스캔하고, 각 로봇의 링크 트리를 구성하고, 인메모리 USD 계층 구조 스테이지를 만듭니다. (hierarchy_stage, path_map, joint_connections)를 반환합니다. mode 파라미터는 HierarchyMode 값을 허용합니다(기본값 LINKED).

반환된 PathMap 객체는 원본 스테이지 경로와 계층 구조 스테이지 경로 간의 양방향 매핑을 제공합니다:

hierarchy_stage, path_map, connections = generate_robot_hierarchy_stage(HierarchyMode.FLAT)
# Map from original stage path to hierarchy path
hier_path = path_map.get_hierarchy_path(original_prim_path)
# Map back from hierarchy path to original stage path
orig_path = path_map.get_original_path(hier_prim_path)

참고

계층 구조 생성 중에 활성 마스킹 서브레이어가 일시적으로 음소거되어 트리가 항상 수정되지 않은 로봇 구조를 반영합니다.

Masking Operations API#

isaacsim.robot.schema.ui 확장은 로봇 구성요소를 마스킹, 우회 및 앵커링하기 위한 프로그래밍 방식 API를 제공합니다. 상태는 MaskingState 싱글톤으로 관리되고 USD 작업은 MaskingOperations로 수행됩니다.

from isaacsim.robot.schema.ui.masking_state import MaskingState
from isaacsim.robot.schema.ui.masking_ops import MaskingOperations

MaskingState#

MaskingState는 어떤 프림이 비활성화(마스킹), 우회 또는 앵커링되었는지 추적하는 싱글톤입니다. 세 개의 독립적인 집합을 유지하고 상태가 변경될 때 구독자에게 알립니다.

Method

Description

MaskingState.get_instance()

싱글톤 MaskingState 인스턴스를 반환합니다.

is_deactivated(original_path)

프림이 마스킹되거나 우회된 경우 True를 반환합니다.

is_bypassed(original_path)

프림이 우회 상태인 경우에만 True를 반환합니다.

is_anchored(original_path)

링크가 월드에 앵커링된 경우 True를 반환합니다.

toggle_deactivated(original_path)

일반 마스크를 전환합니다. 우회된 프림을 마스크 해제하면 먼저 우회 해제됩니다.

toggle_bypassed(original_path)

우회 상태를 전환합니다(마스크 + 체인 재연결).

toggle_anchored(original_path)

링크의 월드 앵커를 전환합니다.

set_deactivated_batch(paths, deactivated)

단일 변경 알림으로 여러 프림의 비활성화 상태를 설정합니다.

set_bypassed_batch(paths, bypassed)

단일 변경 알림으로 여러 프림의 우회 상태를 설정합니다.

set_anchored_batch(paths, anchored)

단일 변경 알림으로 여러 링크의 앵커 상태를 설정합니다.

clear()

모든 마스킹, 우회 및 앵커 상태를 지웁니다.

subscribe_changed(callback)

상태가 변경될 때 호출되는 인수 없는 콜백을 등록합니다.

unsubscribe_changed(callback)

이전에 등록된 변경 콜백을 등록 취소합니다.

MaskingOperations#

MaskingOperations는 전용 익명 서브레이어에서 실제 USD 편집을 수행합니다. 모든 의견은 일시적이며 작성된 레이어에 기록되지 않습니다.

Method

Description

mask_prim(original_path)

시뮬레이션을 위한 조인트(jointEnabled = False) 또는 링크(강체, 충돌, 가시성)를 비활성화합니다.

unmask_prim(original_path)

마스크 의견을 제거하여 기본 레이어 값을 복원합니다.

bypass_prim(original_path)

요소를 마스킹하고 그 주위의 운동학적 체인을 다시 연결합니다. 조인트의 경우 간격을 연결하는 고정 조인트를 생성합니다. 링크의 경우 이전 조인트를 비활성화하고 이후 조인트를 재부모화합니다.

unbypass_prim(original_path)

우회를 제거하여 체인을 원래 상태로 복원합니다.

anchor_link(original_path)

현재 포즈에서 고정 조인트를 생성하여 링크를 월드에 고정합니다.

unanchor_link(original_path)

앵커 고정 조인트를 제거합니다.

clear_all()

마스킹 서브레이어를 완전히 삭제하여 모든 것을 한 번에 되돌립니다.
