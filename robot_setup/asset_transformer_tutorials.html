Asset Transformer Tutorials#

이 튜토리얼은 일반적인 Asset Transformer 워크플로우를 단계별로 안내합니다. 각 튜토리얼은 이전 튜토리얼을 기반으로 하므로 순서대로 따르는 것을 권장합니다.

UI와 개념에 대한 심층적인 설명은 Asset Transformer를 참고하세요.

시작하기 전에 Tools > Robotics > Asset Editors > Asset Transformer에서 Asset Transformer를 엽니다.

Tutorial 1: Transform an Asset Using the Isaac Sim Structure Profile#

이 튜토리얼은 내장된 Isaac Sim Structure 프로파일을 사용하여 로봇 USD 에셋을 권장 Isaac Sim 에셋 구조로 변환하는 방법을 보여줍니다.

Prerequisites#

  • 스테이지에 로드된 로봇 USD 에셋 또는 디스크에 있는 에셋. 이 튜토리얼은 Isaac Sim 에셋의 Isaac/Robots/에 있는 샘플 로봇을 사용합니다. 이 튜토리얼에서는 Isaac/Robots/Fraunhofer/Evobot/evobot.usd 에셋을 사용합니다.

Instructions#

Select the Input Asset#

  1. Input Section에서 현재 열린 스테이지를 변환하려면 Active Stage를 선택하고, 디스크의 로봇 USD 파일을 사용하려면 Pick File을 선택하여 파일을 탐색합니다.

  2. Output Directory를 변환된 에셋 패키지가 저장될 쓰기 가능한 폴더로 설정합니다(예: /tmp/my_robot_transformed/).

  3. 실행 후 출력 파일이 자동으로 열리기를 원하면 Load Restructured File을 체크합니다.

Load the Built-in Profile#

  1. Actions Section에서 Load Preset 버튼을 클릭합니다.

  2. 최근 프리셋 메뉴에서 Isaac Sim Structure를 선택합니다.

  3. 액션 목록에 Isaac Sim Structure 프로파일에 정의된 모든 규칙이 채워집니다.

  1. 액션 목록에서 한두 개의 규칙을 확장하여 매개변수를 확인합니다. 각 규칙에는 Rule Type, Destination, Parameters가 표시됩니다.

Execute the Transformation#

  1. Execute Section에서 Execute Actions를 클릭합니다. 하나 이상의 액션이 활성화되고 출력 디렉토리가 설정된 경우 버튼이 활성화됩니다.

  2. 파이프라인이 완료될 때까지 기다립니다. 콘솔 로그에 각 규칙의 진행 상황이 표시됩니다.

Verify the Output#

  1. Load Restructured File이 체크되어 있으므로 변환된 에셋이 자동으로 열립니다. 로드된 에셋에 지면이 더 이상 포함되지 않는 것을 확인할 수 있습니다. 이는 원본 로봇 에셋의 기본 prim에 없었기 때문이며, 이는 예상되고 원하는 결과입니다.

  2. 출력 폴더 구조를 확인합니다. Isaac Sim 에셋 구조를 따라야 합니다:

output_directory/
├── payloads/
│   ├── base.usd
│   ├── robot.usda
│   ├── geometries.usd
│   ├── instances.usda
│   ├── materials.usda
│   ├── Textures/
│   │   └── ...
│   └── Physics/
│       ├── physics.usda
│       ├── physx.usda
│       └── mujoco.usda
├── <asset_name>.usda          (interface layer)
└── transform_report.json

  1. 로봇 prim 구조를 확인하고, 모든 메시가 이제 참조를 통해 추가되었는지, 그리고 이전의 looks 범위가 이제 비어 있는지 검증합니다. 모든 재질은 이제 materials.usda 레이어에 추가되고 메시를 통해 추가됩니다.

  2. 기본 prim 속성을 확인하고, physx 변형이 선택된 물리 엔진을 위한 변형 세트가 있는지 검증합니다.

  3. transform_report.json을 열어 규칙별 실행 로그를 검토하고 모든 규칙이 성공적으로 완료되었는지 확인합니다.

Tutorial 2: Creating a New Profile#

이 튜토리얼은 처음부터 커스텀 변환 프로파일을 만드는 방법을 보여줍니다. 예제 프로파일은 물리 스키마와 재질을 전용 레이어로 라우팅합니다. 이는 전체 Isaac Sim Structure 처리가 필요하지 않은 에셋에 일반적인 워크플로우입니다.

Instructions#

Clear the Action List#

  1. Clear All Actions를 클릭하여 빈 파이프라인으로 시작합니다.

  2. Profile Settings 프레임을 확장하고 메타데이터를 입력합니다:

    • Profile Name: My Custom Profile

    • Version: 1.0

    • Base Name: base.usd

    • Flatten Source: 소스 에셋에 먼저 평탄화하고 싶은 복잡한 서브레이어 컴포지션이 없는 한 체크를 해제합니다.

Add a Schema Routing Rule#

  1. Add Action을 클릭하여 파이프라인에 새 규칙을 추가합니다.

  2. 새 액션을 확장하고 설정합니다:

    • Rule Type: SchemaRoutingRule을 검색하고 isaacsim.asset.transformer.rules.core.schemas.SchemaRoutingRule을 선택합니다.

    • Name: Route Physics Schemas

    • Destination: payloads/Physics

    • Parameters:

      • stage_name: physics.usda

      • schemas: ["Physics*", "Newton*"] ([+] 버튼을 클릭하여 목록의 각 항목에 새 스키마 줄을 추가합니다)

Add a Materials Routing Rule#

  1. Add Action을 다시 클릭하여 두 번째 규칙을 추가합니다.

  2. 확장하고 설정합니다:

    • Rule Type: MaterialsRoutingRule을 검색하고 isaacsim.asset.transformer.rules.perf.materials.MaterialsRoutingRule을 선택합니다.

    • Name: Route Materials

    • Destination: payloads

    • Parameters:

      • materials_layer: materials.usda

      • textures_folder: Textures

      • deduplicate: true

      • download_textures: true

Add a Geometry Deduplication Rule#

  1. Add Action을 한 번 더 클릭합니다.

  2. 확장하고 설정합니다:

    • Rule Type: GeometriesRoutingRule을 검색하고 isaacsim.asset.transformer.rules.perf.geometries.GeometriesRoutingRule을 선택합니다.

    • Name: Deduplicate Geometries

    • Destination: payloads

    • Parameters:

      • geometries_layer: geometries.usd

      • instance_layer: instances.usda

      • deduplicate: true

      • save_base_as_usda: true

Review the Pipeline#

이제 액션 목록에 세 가지 규칙이 순서대로 포함됩니다:

  1. Route Physics Schemas

  2. Route Materials

  3. Deduplicate Geometries

실행 순서가 올바른지 확인합니다. 필요한 경우 액션을 드래그하여 순서를 변경합니다. 테스트 중에는 체크박스를 사용하여 개별 규칙을 비활성화합니다.

Note

규칙 실행 순서는 중요합니다. 규칙은 순차적으로 실행되며 각 규칙은 이전 규칙이 수정한 작업 스테이지에서 작동합니다. 중복 제거가 일어나기 전에 스키마가 분리되도록 스키마 라우팅 규칙을 재질 및 지오메트리 규칙 앞에 배치합니다.

Tutorial 3: Saving a Profile#

튜토리얼 2에서 커스텀 프로파일을 만든 후 재사용을 위해 저장합니다.

Instructions#

  1. Profile Settings 프레임을 확장하여 프로파일 메타데이터를 확인합니다. Profile Name, Version 및 기타 필드가 올바른지 확인합니다.

  2. Actions Section에서 Save Preset 버튼을 클릭합니다.

  3. 파일 선택기에서 원하는 저장 위치로 이동하고 파일명을 입력합니다(예: my_custom_profile.json).

  4. Save를 클릭합니다.

프로파일이 JSON 파일로 저장되고 나중에 빠르게 불러올 수 있도록 최근 프리셋 목록에 추가됩니다.

저장된 파일을 텍스트 편집기로 열어 검증합니다. 구조는 규칙 프로파일 형식과 일치해야 합니다:

{
   "profile_name": "My Custom Profile",
   "version": "1.0",
   "rules": [
      {
         "name": "Route Physics Schemas",
         "type": "isaacsim.asset.transformer.rules.core.schemas.SchemaRoutingRule",
         "destination": "payloads/Physics",
         "params": {
         "schemas": [
            "Physics*",
            "Newton*"
         ],
         "ignore_schemas": [],
         "stage_name": "physics.usda",
         "prim_names": [
            ".*"
         ],
         "ignore_prim_names": []
         },
         "enabled": true
      },
      {
         "name": "Route Materials",
         "type": "isaacsim.asset.transformer.rules.perf.materials.MaterialsRoutingRule",
         "destination": "payloads",
         "params": {

Tutorial 4: Modifying a Rule Profile#

이 튜토리얼은 기존 프로파일을 불러오고 규칙과 매개변수를 수정하여 다시 저장하는 방법을 보여줍니다. 예제는 튜토리얼 3에서 저장한 프로파일을 수정하여 Interface Connection 규칙을 추가하고 지오메트리 중복 제거 설정을 변경합니다.

Instructions#

Load the Existing Profile#

  1. Load Preset을 클릭하고 이전에 저장한 my_custom_profile.json 파일을 선택합니다. 액션 목록에 튜토리얼 3의 세 가지 규칙이 채워집니다.

Add a New Rule#

  1. Add Action을 클릭하여 네 번째 규칙을 추가합니다.

  2. 확장하고 설정합니다:

    • Rule Type: Interface Connection Rule

    • Name: Connect Interfaces

    • Destination: payloads

    • Parameters:

      • Base Layer: payloads/base.usda

      • Base Connection Type: Reference

      • Generate Folder Variants: true

      • Payloads Folder: payloads

      • Custom Connections: []

      • Default Variant Selections: {}

Modify an Existing Rule#

  1. Deduplicate Geometries 규칙을 확장합니다.

  2. 지오메트리 라우팅을 유지하면서 중복 제거를 비활성화하기 위해 deduplicate 매개변수를 true에서 false로 변경합니다.

Reorder the Rules#

  1. Route Materials 규칙 전에 실행되도록 Deduplicate Geometries 규칙을 두 번째 위치로 드래그합니다.

Disable a Rule#

  1. Route Materials 옆의 활성화 체크박스를 해제하여 파이프라인에서 제거하지 않고 비활성화합니다. 이는 특정 변환을 반복적으로 테스트할 때 유용합니다.

Update Profile Metadata#

  1. Profile Settings를 확장합니다.

  2. 수정 사항을 추적하기 위해 Version을 1.1로 변경합니다.

Save the Modified Profile#

  1. Save Preset을 클릭합니다.

  2. 기존 파일을 덮어쓰거나 새 파일명을 선택합니다.

Tutorial 5: Adding an Asset Transform Pipeline Through Code#

이 튜토리얼은 Asset Transformer API를 사용하여 변환 파이프라인을 프로그래밍 방식으로 실행하는 방법을 보여줍니다. 이는 배치 처리, CI/CD 통합, 또는 커스텀 익스텐션과 워크플로우에 에셋 변환을 포함시킬 때 유용합니다.

아래 코드 블록은 단일 스크립트 docs/isaacsim/snippets/robot_setup/asset_transformer_tutorials.py에서 가져온 것으로, Isaac Sim의 python.sh를 사용하여 독립 실행형 앱으로 실행할 수 있습니다(예: 등록된 규칙 타입 목록 출력).

Load and Run a Saved Profile#

가장 간단한 방법은 기존 프로파일 JSON 파일을 불러와 입력 에셋에 실행하는 것입니다.

import json

from isaacsim.asset.transformer import AssetTransformerManager, RuleProfile

# profile_path = "/path/to/my_custom_profile.json"
# input_stage = "/path/to/my_robot.usd"
# package_root = "/output/my_robot_package"
# Load a saved profile from disk
with open(profile_path, "r") as f:
    profile_data = json.load(f)

profile = RuleProfile.from_dict(profile_data)

# Create the manager and run the transformation
manager = AssetTransformerManager()
report = manager.run(
    input_stage=input_stage,
    profile=profile,
    package_root=package_root,
)

# Inspect results
print(f"Output asset: {report.output_stage_path}")
for result in report.results:
    status = "OK" if result.success else "FAILED"
    print(f"  [{status}] {result.rule.name}")
    if result.error:
        print(f"    Error: {result.error}")

Build a Profile Programmatically#

JSON 파일 없이 코드에서 완전히 프로파일을 정의하려면:

from isaacsim.asset.transformer import (
    AssetTransformerManager,
    RuleProfile,
    RuleSpec,
)

# input_stage = "/path/to/my_robot.usd"
# package_root = "/output/my_robot_package"

profile = RuleProfile(
    profile_name="Code-Defined Profile",
    version="1.0",
    base_name="base.usd",
    flatten_source=False,
    rules=[
        RuleSpec(
            name="Route Physics Schemas",
            type="isaacsim.asset.transformer.rules.core.schemas.SchemaRoutingRule",
            destination="payloads/Physics",
            params={
                "stage_name": "physics.usda",
                "schemas": ["Physics*", "Newton*"],
            },
        ),
        RuleSpec(
            name="Route Materials",
            type="isaacsim.asset.transformer.rules.perf.materials.MaterialsRoutingRule",
            destination="payloads",
            params={
                "materials_layer": "materials.usda",
                "assets_folder": "Textures",
                "download_textures": True,
            },
        ),
        RuleSpec(
            name="Deduplicate Geometries",
            type="isaacsim.asset.transformer.rules.perf.geometries.GeometriesRoutingRule",
            destination="payloads",
            params={
                "geometries_layer": "geometries.usd",
                "instance_layer": "instances.usda",
                "deduplicate": True,
            },
        ),
    ],
)

manager = AssetTransformerManager()
report = manager.run(
    input_stage=input_stage,
    profile=profile,
    package_root=package_root,
)

print(f"Transformation complete: {report.output_stage_path}")

Use the Isaac Sim Structure Profile in Code#

내장된 Isaac Sim Structure 프로파일을 프로그래밍 방식으로 사용하려면 익스텐션의 데이터 디렉토리에서 불러옵니다:

import json
from pathlib import Path

from isaacsim.asset.transformer import AssetTransformerManager, RuleProfile
from isaacsim.core.utils.extensions import get_extension_path_from_name

# input_stage = "/path/to/my_robot.usd"
# package_root = "/output/my_robot_isaacsim_structure"
# Locate the built-in profile shipped with the rules extension (use get_extension_path_from_name so the path is absolute)
ext_path = Path(get_extension_path_from_name("isaacsim.asset.transformer.rules"))
profile_path = ext_path / "data" / "isaacsim_structure.json"

with open(profile_path, "r") as f:
    profile = RuleProfile.from_dict(json.load(f))

manager = AssetTransformerManager()
report = manager.run(
    input_stage=input_stage,
    profile=profile,
    package_root=package_root,
)

Batch-Process Multiple Assets#

표준 Python과 결합하여 여러 에셋을 변환합니다:

import json
from pathlib import Path

from isaacsim.asset.transformer import AssetTransformerManager, RuleProfile

# profile_path = "/path/to/my_custom_profile.json"
# asset_paths = ["/assets/robot_a.usd", "/assets/robot_b.usd", "/assets/robot_c.usd"]
# output_base_dir = "/output"

with open(profile_path, "r") as f:
    profile = RuleProfile.from_dict(json.load(f))

manager = AssetTransformerManager()

for asset_path in asset_paths:
    asset_name = Path(asset_path).stem
    output_dir = f"{output_base_dir.rstrip('/')}/{asset_name}_transformed"

    report = manager.run(
        input_stage=asset_path,
        profile=profile,
        package_root=output_dir,
    )

    all_ok = all(r.success for r in report.results)
    print(f"{asset_name}: {'PASS' if all_ok else 'FAIL'} -> {report.output_stage_path}")

Save and Inspect the Execution Report#

manager.run()이 반환하는 ExecutionReport는 로깅 또는 CI 검증을 위해 JSON으로 직렬화할 수 있습니다:

import json
from pathlib import Path

# report = <ExecutionReport from manager.run() in any example above>
# After running the transformation (report from any example above)
report_path = Path(report.package_root) / "transform_report.json"
with open(report_path, "w") as f:
    json.dump(report.to_dict(), f, indent=2)

print(f"Report saved to: {report_path}")

Discover Available Rule Types#

런타임에 등록된 모든 변환 규칙을 나열하려면:

from isaacsim.asset.transformer import RuleRegistry

registry = RuleRegistry()
for rule_type in registry.list_rule_types():
    print(rule_type)

이는 RuleSpec.type에서 사용할 수 있는 모든 완전한 정규화 규칙 클래스 이름을 출력합니다. 각 규칙에 대한 문서는 Asset Transformer 규칙 참조를 참고하세요.
