Robot Self-Collision Detector#

About#

Robot Self-Collision Detector 확장은 로봇 아티큘레이션의 강체 링크 간의 자체 충돌 쌍을 감지하고 관리합니다. 물리 시뮬레이션 중에 초기 구성에서 겹치는 콜라이더 쌍은 로봇을 불안정하게 만드는 원치 않는 접촉력을 생성합니다. 이 도구는 해당 쌍을 열거하고 UsdPhysics.FilteredPairsAPI를 사용하여 Filtered Pairs로 표시하여 물리 엔진이 해당 접촉을 무시하도록 합니다.

이 확장에 접근하려면 상단 메뉴 바에서 Tools > Robotics > Asset Editors > Robot Self-Collision Detector를 클릭하세요. 이 확장은 기본적으로 활성화됩니다. 비활성화된 경우 Extension Manager에서 isaacsim.robot_setup.collision_detector를 검색하여 다시 활성화할 수 있습니다.

When to Use#

충돌 메시가 휴지 상태에서 겹치는 로봇 에셋을 임포트하거나 작성한 후 이 도구를 사용하세요. 일반적인 시나리오:

  • 조인트 경계에서 인접 링크 콜라이더가 교차하는 새로 임포트된 URDF/MJCF 로봇.

  • 많은 링크가 근접해 있는 밀집된 충돌 형상(예: 정교한 손)이 있는 로봇.

  • 에셋 구조 가이드라인에 따라 기존 에셋에 물리 기능 레이어로 충돌 필터링 추가.

User Interface#

Robot Selection#

  • Robot 드롭다운: Robot Schema가 적용된 스테이지의 모든 프림을 나열합니다. 검사할 로봇을 선택하세요.

  • Check Collisions 버튼: 선택된 로봇에서 충돌 감지를 실행하고 결과 테이블을 채웁니다.

Options#

  • Include environment collisions: 활성화하면 결과에 로봇 링크와 장면의 비로봇 물체(예: 테이블 또는 지면) 간의 충돌 쌍도 포함됩니다.

Collision Pairs Table#

테이블에는 세 개의 열이 있습니다:

Column

Content

Description

Rigid Body A

색상 견본 + 링크 이름

충돌 쌍의 첫 번째 바디. 포컬 아이콘을 클릭하면 Stage 창에서 충돌 프림이 선택됩니다.

Rigid Body B

색상 견본 + 링크 이름

충돌 쌍의 두 번째 바디. 동일한 포컬 아이콘 동작.

Filtered Pair

체크박스

토글하여 FilteredPairsAPI 관계를 추가하면 물리 엔진이 이 접촉을 무시합니다. 해제하면 제거됩니다.

추가 테이블 상호작용:

  • 검색 바: 바디 이름으로 행을 필터링합니다(대소문자 구분 없음).

  • 열 정렬: 헤더 정렬 아이콘을 클릭하여 알파벳 정렬을 전환합니다.

  • 행 선택: 행을 클릭하면 뷰포트에서 두 바디가 서로 다른 색상으로 강조 표시되어 쌍을 시각적으로 쉽게 찾을 수 있습니다. 뷰포트의 다른 곳을 클릭하면 테이블 선택이 해제됩니다.

  • 키보드 탐색: Ctrl+Up / Ctrl+Down을 눌러 행 선택을 위아래로 이동합니다. 여러 행이 선택된 경우 전체 블록이 함께 이동합니다.

  • 일괄 전환: 여러 행을 선택하고 하나의 체크박스를 전환하면 선택된 모든 쌍에 동일한 필터링 상태가 적용됩니다.

  • 충돌 없음: 자체 충돌이 감지되지 않으면 테이블 대신 오버레이 메시지가 표시됩니다.

Viewport Highlighting#

충돌 쌍 행이 선택되면, 두 강체가 64색 팔레트에서 선택된 서로 다른 색상을 사용하여 뷰포트에 강조 표시됩니다. 각 바디는 한눈에 구별할 수 있도록 색상 선택 그룹 윤곽선을 받습니다. 뷰포트 선택 윤곽선 설정이 비활성화된 경우, 패널은 선택 기간 동안 일시적으로 이를 활성화하고 선택이 해제되거나, 패널이 숨겨지거나, 확장이 종료될 때 원래 설정을 복원합니다.

Example Usage: Unitree Dex5 Hand#

이 안내는 Isaac Sim 에셋 라이브러리의 Isaac/Robots/Unitree/Dex5/Dex5-URDF-R.usda 에셋을 사용합니다.

1. Open the Robot Asset#

Dex5 손 에셋을 스테이지에 열거나 참조하세요:

File > Open을 사용하거나 Content Browser에서 Isaac/Robots/Unitree/Dex5/Dex5-URDF-R.usda로 드래그하세요.

2. Open the Self-Collision Detector#

Tools > Robotics > Asset Editors > Robot Self-Collision Detector로 이동하세요.

패널은 Content 창 옆에 편집기 왼쪽 하단에 도킹되어 열립니다.

3. Select the Robot and Detect Collisions#

로봇 드롭다운은 Robot Schema가 적용된 스테이지의 모든 프림으로 자동 채워집니다. 스테이지의 첫 번째 로봇을 자동으로 선택하고 충돌을 확인합니다. 로봇을 수정한 경우 Check Collisions를 클릭하여 결과를 업데이트하세요.

이 도구는 로봇의 현재 구성에서 겹치는 모든 콜라이더 쌍에 대해 물리 엔진을 쿼리하고, 각 콜라이더를 소유 강체 링크에 매핑하고, 테이블을 채웁니다.

4. Inspect a Collision Pair#

행을 클릭하면 뷰포트에서 두 바디가 강조 표시됩니다. 각 바디는 고유한 색상으로 그려지므로 겹침이 예상된 것인지(예: 인접한 손가락 부분) 의도하지 않은 것인지 시각적으로 확인할 수 있습니다.

바디 이름 옆에 있는 포컬 아이콘(십자선)을 클릭하면 뷰포트와 Stage 창에서 해당 바디만 선택됩니다. 테이블 행이 선택되지만(또는 이미 선택된 경우 유지됨), 클릭한 바디만 색상 윤곽선을 받아 쌍의 한쪽을 쉽게 분리할 수 있습니다. 동일한 행의 다른 바디에서 포컬 아이콘을 클릭하면 선택된 행을 변경하지 않고 뷰포트 강조 표시가 전환됩니다.

5. Filter Collision Pairs#

물리 엔진에서 무시해야 할 각 쌍에 대해 Filtered Pair 체크박스를 체크하세요. 이렇게 하면 UsdPhysics.FilteredPairsAPI 관계가 스테이지에 기록됩니다.

여러 쌍을 한 번에 필터링하려면 원하는 행을 선택하고(Ctrl 또는 Shift를 누른 상태에서 클릭), 하나의 체크박스를 전환하세요. 선택된 모든 행이 함께 업데이트됩니다.

참고

충돌 쌍을 해결하는 다른 방법은 충돌 형상을 조정하거나 Robot Poser 확장을 사용하여 로봇 시작 포즈를 수정하는 것입니다. 필터링된 쌍은 자체 충돌 감지기에 표시되므로 어떤 쌍이 무시되고 있는지 쉽게 확인할 수 있습니다. 하지만 쌍이 필터링되지 않고 충돌 엔진에서도 감지되지 않으면 자체 충돌 감지기에 표시되지 않습니다. 로봇의 아티큘레이션 루트에서 자체 충돌이 비활성화된 경우 충돌 엔진에서 충돌 쌍이 감지되지 않고 테이블에 메시지가 표시됩니다.

6. Inspect Multiple Collision Pairs#

뷰포트에서 여러 쌍을 강조 표시하기 위해 여러 행을 선택할 수 있습니다. 여러 행을 선택하려면 클릭 시 Ctrl 또는 Shift를 누르세요. 선택을 빠르게 위아래로 이동하려면 Ctrl+Up 또는 Ctrl+Down을 누르세요.
