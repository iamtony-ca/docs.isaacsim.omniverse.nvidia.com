Asset Transformer Rules Reference#

이 페이지는 Asset Transformer에서 사용 가능한 모든 변환 규칙의 참조를 제공합니다.

일반적인 사용법은 Asset Transformer를 참고하세요. API 및 커스텀 규칙 개발은 Asset Transformer API를 참고하세요.

Rules Overview#

규칙은 기능에 따라 네 가지 패키지로 구성됩니다:

패키지

규칙

목적

Core Routing

SchemaRoutingRule, PropertyRoutingRule, PrimRoutingRule, RemoveSchemaRule

USD 의견을 전용 레이어로 라우팅

Performance

MaterialsRoutingRule, GeometriesRoutingRule

중복 제거 및 인스턴싱을 통한 에셋 최적화

Structure

FlattenRule, VariantRoutingRule, InterfaceConnectionRule

USD 컴포지션 구조 재편성

Isaac Sim

RobotSchemaRule, MakeListsNonExplicitRule, PhysicsJointPoseFixRule, MergeMeshRule, MjcToPhysxConversionRule, UrdfToMjcPhysxConversionRule

Isaac Sim 전용 변환 적용

Note

Asset Transformer UI에서 Rule Type 드롭다운은 각 규칙을 짧은 클래스 이름(예: SchemaRoutingRule)으로 나열하고 Package 열에 범위별로 규칙을 묶습니다. 해당 범위 토큰은 이 페이지의 카테고리에 다음과 같이 매핑됩니다: Core Routing은 core, Performance는 perf, Structure는 structure, Isaac Sim은 isaac_sim. 각 규칙에 대해 아래에 나열된 Fully Qualified Type은 프로파일 JSON에 저장되고 API에 전달되는 값입니다. UI는 선택한 범위와 클래스 이름으로 이를 구성합니다.

Available Rules#

카테고리 탭을 선택하여 사용 가능한 규칙을 확인합니다. 각 규칙을 확장하면 자세한 매개변수와 실행 로직을 볼 수 있습니다.

이 규칙들은 USD 의견(스키마, 속성, prim)을 소스 레이어에서 전용 대상 레이어로 라우팅합니다. 라우팅 과정은 대상 레이어에 재정의 의견을 생성하고 소스 레이어에서 원본 의견을 제거하여 컴포지션 의미론을 보존합니다.

SchemaRoutingRule

적용된 API 스키마와 관련 속성을 별도의 레이어로 라우팅합니다. 이를 통해 물리 스키마, 로봇 스키마 또는 기타 API 스키마를 선택적으로 로드할 수 있는 모듈식 구성이 가능합니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.core.schemas.SchemaRoutingRule

Parameters:

매개변수

타입

설명

schemas

list

라우팅할 API 스키마 패턴 목록. 와일드카드 지원 (예: Physics*는 PhysicsRigidBodyAPI, PhysicsMassAPI와 일치)

ignore_schemas

list

라우팅에서 제외할 스키마 패턴. schemas의 양수 매칭보다 우선합니다.

stage_name

str

출력 USD 파일명 (기본값: schemas.usda)

prim_names

list

처리할 prim 이름을 필터링하는 와일드카드 패턴 (기본값: ["*"] 전체 일치)

ignore_prim_names

list

처리에서 제외할 prim 이름 패턴

실행 로직:

  1. 스키마 탐색: 스테이지의 모든 prim을 순회하며 지정된 패턴과 일치하는 적용된 API 스키마를 수집합니다.

  2. 속성 네임스페이스 해석: 각 일치하는 스키마에 대해 속성 네임스페이스 접두사를 결정합니다 (예: PhysicsRigidBodyAPI는 physics: 네임스페이스 사용, PhysicsDriveAPI:angular는 drive:angular: 사용).

  3. 스키마 전송: USD의 TokenListOp을 사용하여 소스 레이어의 apiSchemas 메타데이터에서 스키마 토큰을 제거하고 대상 레이어에 prepend합니다.

  4. 속성 전송: Sdf.CopySpec을 사용하여 스키마 네임스페이스에 속하는 모든 속성을 소스 레이어에서 대상 레이어로 복사한 다음 소스에서 제거합니다.

  5. 레이어 관리: 소스와 일치하도록 대상 레이어의 기본 prim을 설정하고 두 레이어를 저장합니다.

PropertyRoutingRule

이름 패턴과 일치하는 속성을 별도의 레이어로 라우팅합니다. 이를 통해 특정 속성 네임스페이스(예: 물리 속성, 커스텀 속성)를 모듈식 레이어로 구성할 수 있습니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.core.properties.PropertyRoutingRule

Parameters:

매개변수

타입

설명

properties

list

속성 이름을 매칭하는 정규 표현식 패턴 (예: physics:.*는 모든 physics 네임스페이스 속성과 일치)

ignore_properties

list

라우팅에서 제외할 정규식 패턴

stage_name

str

출력 USD 파일명 (기본값: properties.usda)

scope

str

검색 범위를 제한하는 루트 prim 경로 (기본값: / 전체 스테이지 검색)

prim_names

list

처리할 prim 이름을 필터링하는 와일드카드 패턴

ignore_prim_names

list

제외할 prim 이름 패턴

실행 로직:

  1. 패턴 컴파일: 효율적인 매칭을 위해 제공된 정규식 패턴을 컴파일합니다.

  2. 속성 탐색: 범위 내 prim의 모든 속성과 관계를 순회하며 이름을 컴파일된 패턴과 대조합니다.

  3. 속성 복사: 각 일치하는 속성에 대해 Sdf.CopySpec을 사용하여 prim 스택에서 가장 강한 의견을 대상 레이어로 복사합니다.

  4. 소스 제거: 중복 의견을 방지하기 위해 prim 스택의 모든 소스 레이어(대상 레이어 제외)에서 속성 스펙을 제거합니다.

  5. 레이어 마무리: 대상 레이어를 내보내고 수정된 모든 소스 레이어를 저장합니다.

PrimRoutingRule

타입 패턴과 일치하는 전체 prim을 별도의 레이어로 라우팅합니다. 이를 통해 물리 prim, 렌더 prim 또는 기타 타입이 있는 prim을 모듈식 레이어로 구성할 수 있습니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.core.prims.PrimRoutingRule

Parameters:

매개변수

타입

설명

prim_types

list

라우팅할 prim 타입 패턴. 와일드카드 지원 (예: Physics*는 PhysicsJoint, PhysicsScene과 일치)

ignore_prim_types

list

라우팅에서 제외할 prim 타입 패턴

stage_name

str

출력 USD 파일명 (기본값: prims.usda)

scope

str

검색 범위를 제한하는 루트 prim 경로 (기본값: /)

prim_names

list

처리할 prim 이름을 필터링하는 와일드카드 패턴

ignore_prim_names

list

제외할 prim 이름 패턴

실행 로직:

  1. 타입 매칭: fnmatch를 사용하여 타입 이름이 지정된 패턴과 일치하는 범위 내 모든 prim을 수집합니다.

  2. 구성된 복사: 완전히 구성된 prim 정의(모든 속성, 메타데이터, 적용된 스키마 포함)를 대상 레이어로 복사합니다.

  3. 완전 제거: 모든 소스 레이어에서 prim 스펙을 제거합니다. 모든 속성 스펙을 명시적으로 삭제하고(SchemaRoutingRule과 같은 다른 규칙이 작성한 재정의 속성 처리), apiSchemas 메타데이터를 지우고, 부모 네임스페이스에서 prim 스펙을 삭제합니다.

  4. 레이어 관리: 대상 레이어를 내보내고 수정된 소스 레이어를 저장합니다.

RemoveSchemaRule

대상 레이어에서 특정 적용된 API 스키마(및 선택적으로 관련 속성)를 제거합니다. 다른 물리 백엔드용 에셋을 준비할 때 시뮬레이터 전용 스키마를 제거하는 데 유용합니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.core.remove_schema.RemoveSchemaRule

Parameters:

매개변수

타입

설명

stage_name

str

편집할 대상 USD 파일명 (예: mujoco.usda)

schema_patterns

list

제거할 API 스키마 이름과 일치하는 와일드카드 패턴 (예: PhysicsDriveAPI.*)

prim_path_patterns

list

영향 받는 prim 경로를 제한하는 정규식 패턴 (기본값: [".*"])

clear_properties

bool

일치하는 스키마 네임스페이스에 속하는 속성도 제거 (기본값: False)

실행 로직:

  1. 패턴 매칭: 대상 레이어의 모든 prim을 순회하며 적용된 API 스키마를 지정된 와일드카드 패턴과 대조합니다.

  2. 스키마 제거: TokenListOp 조작을 사용하여 각 prim의 apiSchemas 메타데이터에서 일치하는 스키마 토큰을 제거합니다.

  3. 속성 정리: clear_properties가 활성화된 경우 prim 스펙에서 일치하는 스키마 네임스페이스의 모든 속성을 제거합니다.

  4. 레이어 저장: 수정된 레이어를 저장합니다.

이 규칙들은 중복 제거 및 인스턴싱을 통해 더 나은 시뮬레이션 및 렌더링 성능을 위해 에셋을 최적화합니다.

MaterialsRoutingRule

재질 prim을 공유 레이어로 라우팅하고 전역 중복 제거를 수행하며, 원래 위치에 인스턴스 참조를 생성하고 텍스처/MDL 에셋을 지정된 폴더로 이동합니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.perf.materials.MaterialsRoutingRule

Parameters:

매개변수

타입

설명

scope

str

재질 검색 범위를 제한하는 루트 prim 경로 (기본값: /)

materials_layer

str

재질 레이어 출력 USD 파일명 (기본값: materials.usda)

textures_folder

str

대상에서 텍스처 에셋의 상대 폴더 이름 (기본값: Textures)

deduplicate

bool

콘텐츠 해시 기반 재질 중복 제거 활성화 (기본값: True)

download_textures

bool

원격 텍스처(예: Nucleus)를 로컬 폴더로 다운로드 (기본값: False)

실행 로직:

  1. 재질 탐색: 범위 내 모든 재질 prim(UsdShade.Material)을 찾아 각 재질을 정의하는 레이어를 추적합니다. PhysicsMaterialAPI가 적용된 재질은 건너뜁니다. 이러한 물리 전용 재질은 material:binding:physics 관계가 계속 해석될 수 있도록 원래 경로의 베이스 레이어에 남습니다.

  2. 에셋 수집: 재질이 참조하는 모든 텍스처 및 MDL 파일 경로를 로컬 및 원격(Nucleus) 에셋 모두 처리하여 해석합니다. MDL 파일을 파싱하여 포함된 텍스처 참조를 발견합니다.

  3. 콘텐츠 해싱: 일관된 중복 제거를 위해 해석된 에셋 경로를 사용하여 각 재질의 콘텐츠(타입, 속성, 연결, 관계)의 SHA-256 해시를 계산합니다.

  4. 에셋 전송: 전역 중복 제거로 모든 고유 에셋을 텍스처 폴더에 복사합니다. 파일명 충돌은 숫자 접미사를 추가하여 처리합니다. MDL 파일을 업데이트하여 전송된 텍스처를 가리키도록 합니다.

  5. 재질 레이어 생성: 재질 레이어에 /Materials 범위를 생성합니다. 각 고유 재질(해시 기준)에 대해 업데이트된 에셋 경로로 재질 정의를 복사합니다.

  6. 인스턴스 참조: 각 원래 재질 위치를 재질 레이어의 중복 제거된 재질에 대한 인스턴스 참조로 업데이트합니다.

  7. 바인딩 업데이트: 재질 바인딩이 있는 모든 prim에 MaterialBindingAPI가 적용되도록 합니다.

  8. 정리: 어떤 표면에도 바인딩되지 않은 재질의 인스턴스 참조를 제거합니다.

GeometriesRoutingRule

지오메트리 prim을 공유 레이어로 라우팅하고 중복 제거를 수행하며, 인스턴스별 재정의를 위한 별도의 인스턴스 레이어를 생성합니다. 완전히 평탄화된 스테이지(참조 및 인스턴스가 이미 해석됨)에서 작동합니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.perf.geometries.GeometriesRoutingRule

Parameters:

매개변수

타입

설명

scope

str

지오메트리 검색 범위를 제한하는 루트 prim 경로 (기본값: /)

geometries_layer

str

기본 지오메트리 정의 출력 USD 파일명 (기본값: geometries.usd)

instance_layer

str

인스턴스별 재정의 출력 USD 파일명 (기본값: instances.usda)

deduplicate

bool

동일한 지오메트리 정의 재사용 (기본값: True)

save_base_as_usda

bool

베이스 스테이지를 USD ASCII 형식으로 저장 (기본값: True)

verbose

bool

자세한 변환 분해 정보 로깅 (기본값: False)

실행 로직:

  1. 지오메트리 탐색: 범위 내 모든 지오메트리 prim(Mesh, Gprim 타입)을 식별합니다.

  2. 콘텐츠 해싱: 메시 데이터(포인트, 면 수, 인덱스), 변환, 내재적 속성(subdivisionScheme, orientation 등 타입별 속성)을 기반으로 지오메트리 해시를 계산합니다.

  3. 내재적 vs 인스턴스 속성: 내재적 지오메트리 속성(메시 데이터, UV, 법선, 탄젠트)과 인스턴스별 속성(시각적 재질 바인딩, CollisionAPI 등 적용된 스키마, 커스텀 속성)을 분리합니다. 물리 목적 재질 바인딩(material:binding:physics)은 VisualMaterials를 통해 재라우팅되지 않고 베이스 레이어의 원래 대상 경로를 가리키는 인스턴스 델타에 그대로 보존됩니다.

  4. 지오메트리 레이어 생성: 지오메트리 레이어의 /Geometries/{name}/{name} 아래에 지오메트리 정의를 생성합니다. 동일한 지오메트리는 같은 정의를 공유합니다.

  5. 인스턴스 레이어 생성: 인스턴스별 델타(재질 바인딩, 적용된 API 스키마, 변환 재정의, 커스텀 속성)를 캡처하는 인스턴스 항목을 생성합니다.

  6. 베이스 스테이지 업데이트: 원래 지오메트리 prim을 인스턴스 참조로 교체하여 지오메트리 정의를 참조하도록 베이스 스테이지를 업데이트합니다.

  7. 델타 병합: 인스턴스 레이어의 중복성을 줄이기 위해 동일한 델타를 가진 인스턴스를 그룹화합니다.

이 규칙들은 모듈식 에셋 구성을 위해 USD 컴포지션 구조를 재편성합니다.

FlattenRule

Fully Qualified Type: isaacsim.asset.transformer.rules.structure.flatten.FlattenRule

매개변수

selected_variants

변형 세트 이름과 변형 선택의 딕셔너리. 기본 prim의 변형 세트에만 적용. 예: {"Physics": "PhysX", "Gripper": "None"}

  2. 변형 선택 적용: selected_variants가 지정된 경우 기본 prim의 변형 세트에 변형 선택을 적용합니다. 대소문자 구분 없이 PhysX 요청 시 physx와 같은 변형을 찾습니다.

  6. 스테이지 전환: 관리자가 이후 규칙을 평탄화된 스테이지에서 작동하도록 전환할 출력 경로를 반환합니다.

VariantRoutingRule

변형 세트 콘텐츠를 별도의 레이어 파일로 라우팅합니다. 각 변형은 변형 세트 폴더별로 구성된 개별 USDA 파일로 추출됩니다. 변형 내의 컴포지션 아크는 소스 에셋을 복사하고 의존성을 재매핑하여 처리됩니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.structure.variants.VariantRoutingRule

Parameters:

매개변수

타입

설명

variant_sets

list

처리할 변형 세트 이름의 선택적 목록. 비어 있으면 기본 prim의 모든 변형 세트가 처리됩니다.

case_insensitive

bool

출력 파일명과 참조에 대해 변형 옵션 이름을 소문자로 변환 (기본값: True)

collect_dependencies

bool

외부 의존성(참조된 에셋, 텍스처)을 dependencies 폴더에 수집 (기본값: True)

excluded_variants

list

전체 처리에서 제외할 변형 이름. 제외된 변형은 빈 USDA 파일이 생성되지만 콘텐츠는 처리되지 않습니다. none, default와 같은 변형에 유용합니다.

실행 로직:

  1. 변형 세트 분석: 기본 prim의 변형 세트를 검사하고 변형 에셋(각 변형 내의 페이로드/참조)에서 변형 이름으로의 매핑을 구성합니다.

  2. 변형 파일 매핑: 원래 변형 에셋 경로에서 새 출력 파일 경로({VariantSetName}/{variant_name}.usda)로의 매핑을 생성합니다.

  3. 의존성 수집: UsdUtils.ComputeAllDependencies를 사용하여 각 변형 소스가 참조하는 모든 에셋(서브레이어, 참조, 페이로드, 텍스처)을 발견합니다. 의존성을 dependencies 폴더에 복사합니다.

  4. 재매핑을 통한 에셋 복사: 페이로드/참조가 있는 변형의 경우 소스 에셋을 변형 출력 파일로 복사합니다. UsdUtils.ModifyAssetPaths를 사용하여 모든 내부 경로를 새 변형 파일이나 수집된 의존성을 가리키도록 재매핑합니다.

  5. 델타 적용: 변형 스펙의 직접 재정의(속성, 관계, 자식 prim, 컴포지션 아크)를 복사된 콘텐츠 위에 가장 강한 의견으로 적용합니다.

  6. 변형 간 재매핑: 변형 간의 참조를 새 변형 파일을 가리키도록 업데이트합니다. 수집된 의존성의 경로를 일관성을 위해 재매핑합니다.

  7. 제외된 변형 처리: 제외된 변형에 대해 기본 prim만 있는 빈 USDA 파일을 생성합니다.

InterfaceConnectionRule

USD 에셋을 구성하는 컴포지션 아크로 최종 인터페이스 레이어를 생성합니다. 베이스 에셋을 참조/페이로드하는 최상위 에셋 파일을 생성하고 선택적으로 폴더 구조에서 변형 세트를 생성합니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.structure.interface.InterfaceConnectionRule

매개변수

dict

변형 세트 이름과 기본 변형 선택의 딕셔너리. 지정되지 않은 변형 세트는 기본적으로 none이 됩니다.

실행 로직:

  1. 인터페이스 레이어 생성: 원본 입력 에셋의 이름을 따서 패키지 루트에 인터페이스 레이어를 생성합니다.

  3. 기본 연결: 지정된 연결 타입을 사용하여 기본 레이어를 기본 prim에 연결합니다(prim 목록에 Reference 또는 Payload를 prepend하거나 Sublayer를 삽입).

  4. 폴더 변형 생성: 활성화된 경우 USD 파일이 포함된 하위 폴더의 payloads 폴더를 스캔합니다. 각 하위 폴더는 변형 세트가 되며, none 변형(페이로드 없음)과 각 USD 파일에 대한 변형(페이로드됨)이 생성됩니다.

  5. 커스텀 연결: 커스텀 연결 사양을 적용합니다. Sublayer 연결은 레이어의 서브레이어 경로에 추가합니다. Reference/Payload 연결은 기본 prim에 추가합니다. 인터페이스 레이어 또는 지정된 에셋 레이어를 수정할 수 있습니다.

  6. 불필요한 prim 복구: defaultPrim 외부의 루트 수준 prim(예: /Render, /PhysicsScene)에 대해 베이스 레이어를 스캔합니다. Sdf.CopySpec을 사용하여 각각을 같은 루트 수준의 인터페이스 레이어에 복사하고, 베이스 레이어에서 제거한 후 저장합니다. 이를 통해 구성된 스테이지에서 해당 prim에 계속 접근할 수 있으며 파이프라인을 멱등성 있게 만듭니다. 참조 기반 왕복에서 살아남지 못할 prim은 재변환에 걸쳐 지속되는 인터페이스 레이어로 승격됩니다.

  7. 변형 선택 기본값: 인터페이스 레이어의 기본 prim에 기본 변형 선택을 설정합니다.

이 규칙들은 로봇 에셋에 Isaac Sim 전용 변환을 적용합니다.

RobotSchemaRule

대상 prim에 Isaac Sim 로봇 스키마를 적용합니다. 로봇 스키마 유틸리티를 사용하여 아티큘레이션 구조를 감지하고 로봇 관계(링크, 조인트, 사이트)를 채웁니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.isaac_sim.robot_schema.RobotSchemaRule

Parameters:

매개변수

타입

설명

prim_path

Robot 스키마를 적용할 대상 prim 경로. 지정하지 않으면 기본적으로 스테이지 기본 prim을 사용합니다.

stage_name

str

로봇 스키마 의견의 출력 USD 파일명 (기본값: robot_schema.usda)

add_sites

sites_last

bool

False이면 사이트가 관계 순서에서 부모 링크 뒤에 추가됩니다. True이면 모든 사이트가 끝에 추가됩니다. (기본값: False)

sublayer

str

로봇 스키마 적용 전에 작업 스테이지에 포함할 선택적 서브레이어 경로. 조인트를 정의하는 물리 레이어를 포함할 때 유용합니다.

실행 로직:

  1. 대상 레이어 설정: 로봇 스키마 의견을 위한 대상 레이어를 생성하거나 엽니다. 소스 스테이지에 서브레이어로 추가합니다.

  2. 서브레이어 삽입: 서브레이어가 지정된 경우(예: 물리 레이어) 소스 스테이지에 가장 강한 서브레이어로 삽입합니다.

  3. 스키마 감지: 대상 prim에 RobotAPI가 이미 적용되어 있는지 확인합니다.

  4. 스키마 적용:

    • 새 스키마: 대상 prim에 RobotAPI를 적용한 다음 PopulateRobotSchemaFromArticulation을 호출하여 아티큘레이션 구조를 감지하고 isaac:links, isaac:joints, 선택적으로 isaac:sites 관계를 채웁니다.

    • 기존 스키마: RecalculateRobotSchema를 호출하여 기존 관계 순서를 보존하면서 스키마를 업데이트합니다. 유효하지 않은 항목을 제거하고 새로 발견된 링크/조인트/사이트를 추가합니다.

  5. 스키마 업데이트: 필요한 경우 더 이상 사용되지 않는 스키마 버전을 업데이트합니다.

  6. 레이어 격리: 편집 레이어를 저장하고 루트 레이어의 변경 사항을 버립니다(편집이 로봇 스키마 레이어에만 격리되므로).

MakeListsNonExplicitRule

prim 메타데이터 및 속성의 명시적 목록 연산을 비명시적 목록 연산(prepended 또는 appended)으로 변환합니다. 이는 USD 컴포지션에서 중요합니다. 명시적 목록은 더 약한 모든 의견을 재정의하는 반면, prepended/appended 목록은 더 약한 의견과 결합되기 때문입니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.isaac_sim.make_lists_non_explicit.MakeListsNonExplicitRule

Parameters:

매개변수

타입

설명

metadata_names

list

변환할 prim 메타데이터 이름. 와일드카드 지원 (예: api*는 apiSchemas와 일치).

property_names

list

변환할 prim 속성 이름. 와일드카드 지원 (예: material:*).

list_op_type

str

대상 목록 연산 타입: prepend(항목이 더 약한 의견 앞에 옴) 또는 append(항목이 뒤에 옴). 기본값: prepend.

    • prim 스펙에 새 목록 연산을 설정

  3. 관계 변환: 일치하는 관계에 대해:

    • targetPathList에서 명시적 타겟 경로를 추출

    • 관계 스펙을 재생성(가변성, 커스텀 플래그 등 다른 메타데이터 보존)

    • 명시적 목록 대신 Prepend() 또는 Append() 호출을 사용하여 타겟을 재작성

  4. 속성 연결 변환: 일치하는 속성 연결에 대해:

    • 명시적 연결 경로 추출

    • 연결 목록을 지우고 Prepend() 또는 Append()를 사용하여 재작성

  5. 레이어 관리: 수정된 모든 레이어를 저장합니다.

PhysicsJointPoseFixRule

GeometriesRoutingRule과 같은 업스트림 규칙이 바디 월드 변환을 변경한 후 물리 조인트 로컬 포즈를 수정합니다. 원본 입력 에셋에서 계산된 조인트 월드 포즈를 현재 작업 스테이지와 비교하고 월드 포즈가 벗어난 조인트에 대해 localPos0/1과 localRot0/1 속성을 업데이트합니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.isaac_sim.physics_joint_pose_fix.PhysicsJointPoseFixRule

Parameters:

매개변수

타입

설명

original_composition_path

str

원본 컴포지션 스테이지의 선택적 Usd.Stage 객체 또는 명시적 경로. 기본적으로 변환 관리자가 전달한 input_stage(수정되지 않은 소스 에셋)를 사용합니다.

tolerance_position

float

조인트 월드 포즈 비교 시 허용되는 최대 위치 차이(유클리드 거리) (기본값: 1e-6)

tolerance_orientation

float

조인트 월드 포즈 비교 시 1.0에서 쿼터니언 내적의 최소 허용 편차 (기본값: 1e-6)

실행 로직:

  1. 원본 스테이지: input_stage를 통해 원본 입력 에셋(변환 규칙 실행 전)을 엽니다.

  2. 조인트 탐색: 작업 스테이지에서 모든 UsdPhysics.Joint prim을 순회합니다.

  3. 월드 포즈 비교: 각 조인트에 대해 local_pose * body_world_transform을 사용하여 원본 스테이지와 작업 스테이지에서 body0와 body1 모두의 조인트 월드 포즈를 계산합니다.

  4. 편차 감지: 작업 스테이지의 조인트 월드 포즈가 설정된 허용 오차를 초과하여 원본과 다른 경우 영향 받은 바디 쪽에 수정 플래그를 표시합니다.

  5. 로컬 포즈 수정: joint_world_orig * inverse(body_world_entry)로 수정 로컬 포즈를 계산하고 결과 변환과 회전을 조인트의 localPos와 localRot 속성에 씁니다.

  6. 레이어 저장: 수정 사항이 적용된 경우 수정된 편집 레이어를 저장합니다.

MergeMeshRule

Scene Optimizer 병합 작업을 사용하여 공통 강체 부모를 공유하는 시각적 메시 prim을 병합합니다. 이는 지오메트리 prim의 수를 줄이고 렌더링 및 시뮬레이션 성능을 향상시킬 수 있습니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.isaac_sim.merge_mesh.MergeMeshRule

Parameters: 없음

실행 로직:

  1. 강체 탐색: RigidBodyAPI가 적용된 모든 prim을 순회합니다.

  2. 메시 그룹화: 각 강체에 대해 중첩된 강체 경계에서 중지하면서 default 또는 render 목적을 가진 자식 지오메트리 prim(Mesh, Cube, Sphere 등)을 수집합니다.

  3. Scene Optimizer 병합: 각 비어 있지 않은 메시 그룹에 대해 Scene Optimizer의 merge 명령을 호출하여 재질 속성을 보존하면서 메시를 단일 지오메트리 prim으로 결합합니다.

MjcToPhysxConversionRule

MJCF 액추에이터 및 조인트 속성을 PhysX 드라이브 및 조인트 스키마로 변환합니다. 이를 통해 MJCF 게인/바이어스 매개변수를 PhysX 강성/감쇠로 변환하여 다중 물리 엔진 지원이 가능합니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.isaac_sim.mjc_to_physx_conversion.MjcToPhysxConversionRule

Parameters: 없음

실행 로직:

  1. 액추에이터 변환: 각 MjcActuator prim에 대해 gainPrm, biasPrm, gainType, biasType, forceRange 속성을 읽습니다. 위치 제어(kp/kd) 및 속도 제어(kd) 패턴을 DriveAPI 강성과 감쇠로 변환합니다.

  2. 조인트 변환: 각 회전/직선 조인트에 대해 MJCF 전용 속성을 변환합니다(frictionloss → PhysxJointAPI.jointFriction, armature → PhysxJointAPI.armature, ref → DriveAPI.targetPosition).

미믹 조인트는 조인트 prim에 NewtonMimicAPI로 남겨지고 런타임에 직접 사용됩니다. 동등한 PhysxMimicJointAPI는 작성되지 않습니다.

UrdfToMjcPhysxConversionRule

URDF 조인트 속성을 MJCF 액추에이터와 PhysX 조인트 스키마 모두로 변환합니다. 이는 URDF로 가져온 에셋에 대한 다중 물리 변환 단계로 이중 백엔드 표현을 생성합니다.

Fully Qualified Type: isaacsim.asset.transformer.rules.isaac_sim.urdf_to_mjc_physx_conversion.UrdfToMjcPhysxConversionRule

Parameters: 없음

실행 로직:

  1. Physics 범위 생성: 액추에이터 prim을 위해 기본 prim 아래에 Physics 범위가 있는지 확인합니다.

  2. URDF에서 PhysX로 변환: 각 회전/직선 조인트에 대해 URDF 속성을 변환합니다(effort → DriveAPI.maxForce, velocity → PhysxJointAPI.maxJointVelocity, damping → DriveAPI.damping, friction → PhysxJointAPI.jointFriction, calibration → DriveAPI.targetPosition).

  3. MjcActuator 생성: 위치 또는 속도 제어 모드의 드라이브 강성과 감쇠에서 파생된 gainPrm/biasPrm 배열로 MjcActuator prim을 생성합니다.

  4. PhysX에서 MJC로 변환: PhysX 조인트 속성을 MJCF 속성으로 변환합니다(targetPosition → mjc:ref, jointFriction → mjc:frictionloss, armature → mjc:armature).

미믹 조인트는 조인트 prim에 NewtonMimicAPI로 남겨지고 런타임에 직접 사용됩니다. 동등한 PhysxMimicJointAPI는 작성되지 않습니다.

Idempotency Requirement#

모든 변환 규칙은 반드시 멱등성이 있어야 합니다: 동일한 에셋에 전체 프로파일을 두 번 실행해도(원본에 한 번, 첫 번째 실행 출력에 한 번) 절대 경로를 포함하는 doc 메타데이터 필드를 유일한 예외로 하여 동일한 USD 레이어를 생성해야 합니다. 익스텐션에는 이 요구 사항을 적용하는 멱등성 테스트(test_profile_idempotency.py)가 포함되어 있습니다.

새 규칙을 개발할 때 주의해야 할 비멱등성의 일반적인 원인:

  • 부동 소수점 편차 – 행렬 컴포지션/분해 왕복은 노이즈를 유발합니다. 값을 고정된 유효 자릿수로 양자화하거나, 가능한 경우 정규 xformOp 값을 직접 읽습니다.

  • 오래된 컴포지션 아크 – 변형 스펙에서 콘텐츠를 추출하면 대상 prim에 자기 참조 페이로드나 참조가 남을 수 있습니다. 소스에 없었던 아크를 제거합니다.

  • 일관되지 않은 defaultPrim – 재변환에 걸쳐 기본 prim을 안정적으로 해석하려면 self.source_stage.GetRootLayer().defaultPrim(루트 레이어만) 대신 self.source_stage.GetDefaultPrim()(구성된 스테이지)을 사용합니다.

  • 비결정론적 병합 결정 – 자식을 부모에 병합할지 결정할 때 이전 실행에 남은 스캐폴딩 prim(예: VisualMaterials 범위, 빈 over)을 고려합니다.

  • 불필요한 루트 수준 prim – 평탄화 후 defaultPrim 계층 구조 외부의 루트 수준 prim(예: 뷰포트/렌더 아티팩트)은 참조 기반 왕복에서 살아남지 못합니다. InterfaceConnectionRule은 이러한 prim을 베이스 레이어에서 인터페이스 레이어로 이동하여 자동으로 처리하지만, 루트 수준 prim을 도입하는 커스텀 규칙은 이 컴포지션 제약을 인지해야 합니다.
