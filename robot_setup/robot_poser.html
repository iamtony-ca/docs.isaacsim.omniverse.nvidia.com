Robot Poser#

Robot Poser는 Robot Schema를 가진 로봇의 명명된 포즈를 생성, 편집 및 적용합니다. 대화형 IK 목표 워크플로우와 영구적인 USD 저장을 결합하여 작성된 포즈가 로봇 에셋과 함께 저장되고 GUI와 코드 모두에서 재생할 수 있습니다.

기능은 두 개의 확장으로 분리되어 있습니다:

  • isaacsim.robot.poser – IK 해결, 명명된 포즈 CRUD, 조인트 상태 적용, JSON 임포트/익스포트를 위한 헤드리스 API.

  • isaacsim.robot.poser.ui – Robot Poser 창, Named Pose 속성 패널, IsaacNamedPose 프림용 커스텀 스테이지 아이콘으로 구성된 GUI.

Robot Poser Window#

Robot Poser 창은 Tools > Robotics > Robot Poser를 통해 접근할 수 있습니다.

Robot Selection#

창 상단의 드롭다운은 현재 스테이지에서 IsaacRobotAPI가 적용된 모든 프림을 나열합니다. 로봇을 선택하면 아래 테이블에 기존 명명된 포즈가 로드되고 시작/끝 링크 선택에 사용되는 사용 가능한 사이트 후보가 채워집니다.

Named Poses Table#

창의 주요 영역에는 로봇의 명명된 포즈 테이블이 표시됩니다. 각 행에는 다음이 표시됩니다:

Column

Description

Named Pose

포즈의 편집 가능한 이름. 이름을 변경하면 기본 IsaacNamedPose 프림이 업데이트됩니다.

Start Site

운동학적 체인의 시작 링크. 링크 또는 사이트일 수 있습니다. 일반적으로 로봇의 베이스입니다.

End Site

운동학적 체인의 끝 링크 또는 사이트. 링크 또는 사이트일 수 있습니다. 일반적으로 엔드 이펙터 프레임 또는 도구 마운트입니다.

참고

로봇의 고정 프레임이 아닌 사이트를 선택하여 운동학적 체인 과정에서 로봇의 원점에 영향을 미치는 경우 로봇이 베이스 원점을 이동할 수 있습니다. 나중에 이를 수정해야 하는 경우, 로봇 원점 링크의 포즈를 수동으로 재설정하고 명명된 포즈를 다시 적용할 수 있습니다.

참고

팁: 로봇의 기본 위치에 대한 명명된 포즈를 생성하면, 작성 중 로봇이 초기 위치에서 이동한 경우 이를 사용하여 로봇을 초기 위치로 재설정할 수 있습니다.

Row Actions#

각 행에는 액션 버튼이 있습니다:

  • Apply Pose () – 저장된 조인트 구성으로 로봇을 설정합니다. 시뮬레이션이 정지된 경우 조인트가 직접 텔레포트되고, 실행 중인 경우 값이 조인트 드라이브 타겟으로 작성됩니다.

  • Track Target () – 포즈에 대한 실시간 IK 추적을 활성화합니다. 활성화된 동안 뷰포트에서 IsaacNamedPose 프림을 이동(조작기 기즈모를 통해)하면 지속적으로 IK를 해결하고 저장된 조인트 값을 업데이트합니다. IK가 수렴하지 못하면 엔드 이펙터 링크 체인에 빨간색 윤곽선이 표시되어 실패를 나타냅니다.

Managing Poses#

  • Add – 로봇의 Named_Poses 스코프 아래에 새 IsaacNamedPose 프림을 생성하고, isaac:robot:namedPoses 관계에 등록하고, 테이블에 행을 추가합니다. 새 포즈는 선택된 시작/끝 사이트 쌍에 대한 로봇의 현재 조인트 상태를 캡처합니다.

  • Remove – 스테이지에서 선택된 포즈 프림을 삭제하고 로봇의 관계에서 제거합니다.

  • Search – 이름으로 테이블 행을 필터링합니다.

  • 드래그 앤 드롭 – 행을 재정렬하면 namedPoses 관계의 타겟도 재정렬됩니다.

Simulation Behavior#

Robot Poser는 시뮬레이션 타임라인이 재생 중인지 여부에 따라 다르게 동작합니다.

시뮬레이션 정지 – 조인트 값은 USD 스테이지의 조인트 속성을 직접 텔레포트하여 적용됩니다. 로봇은 즉시 목표 포즈로 이동하고 활성 편집 타겟 레이어에 작성됩니다. 이것은 작성 중 예상되는 모드입니다: 물리 간섭 없이 로봇을 정확하게 위치시키고 저장에 적합한 깔끔한 USD 의견을 생성합니다.

시뮬레이션 실행 – 조인트 값은 조인트 드라이브 타겟으로 작성됩니다. 그러면 물리 엔진이 후속 시뮬레이션 단계에서 역학, 조인트 한계, 접촉력을 고려하여 요청된 구성으로 로봇을 이동시킵니다. 결과 동작은 부드럽지만 힘이나 충돌로 인해 정확한 타겟에 도달하지 못할 수 있습니다. 목표 포즈는 활성 편집 타겟 레이어에 작성되지 않고 대신 세션 레이어에 작성되며, 시뮬레이션이 정지되면 지워집니다. 시뮬레이션이 정지된 후 포즈를 유지하려면 시뮬레이션 설정을 변경하여 정지 시 로봇 상태를 재설정하지 않도록 하세요.

IK 추적은 동일한 규칙을 따릅니다: 추적이 활성화되고 시뮬레이션이 정지된 경우, 타겟이 이동하는 매 프레임마다 조인트가 텔레포트됩니다. 시뮬레이션이 실행 중인 경우 드라이브 타겟이 대신 업데이트됩니다.

Asset Structure Recommendations#

명명된 포즈와 그에 작성되는 조인트 값은 현재 편집 타겟 레이어에 작성됩니다. Asset Structure 가이드라인을 따르면:

  • 명명된 포즈 프림(IsaacNamedPose)은 Robot Schema의 나머지 부분과 함께 로봇 스키마 레이어에 작성되어야 합니다. 로봇의 기능을 설명하고 에셋과 함께 이동합니다.

  • 로봇 베이스 포즈(스테이지에서 로봇의 초기 위치와 방향)는 베이스 레이어, 또는 에셋 베이스(데이터 손실 방지)와 물리 레이어 위에 적용된 전용 작성 레이어에 작성되어야 합니다. 이렇게 하면 장면 수준의 배치가 로봇 정의 자체와 분리되고 소스 또는 물리 레이어를 수정하는 것을 방지합니다.

명명된 포즈를 작성할 때 물리가 활성화되지 않은 경우, 운동학적 체인을 해결할 수 있도록 물리 레이어를 서브레이어로 임시로 추가한 다음 저장 전에 제거하세요 – Robot Schema 자체를 적용할 때 사용하는 것과 동일한 워크플로우입니다.

Named Pose Properties Panel#

Stage 패널에서 IsaacNamedPose 프림을 선택하면 속성 패널에서 명명된 포즈 속성을 편집할 수 있습니다. 이 패널은 Robot Poser 창을 열지 않고도 포즈를 직접 편집할 수 있습니다.

명명된 포즈를 선택할 때마다 해당 변환이 명명된 포즈 값에서 로봇의 순방향 운동학으로 자동 업데이트됩니다.

Site Selection#

패널 상단의 두 콤보 박스는 프림의 Start Link 및 End Link 관계를 제어합니다. 사이트를 변경하면 운동학적 체인이 재구성되고 아래 조인트 테이블이 다시 계산됩니다. 콤보 박스는 검색 필터링을 지원하여 복잡한 로봇에서 사이트를 빠르게 찾을 수 있습니다.

Action Buttons#

Button

Description

Set Robot to Pose ()

Track Target ()

이 포즈에 대한 독립 실행형 IK 추적을 시작합니다. 추적 루프는 Robot Poser 창과 독립적으로 실행됩니다 – 창을 열 필요가 없습니다. 추적 중에는 매 프레임마다 명명된 포즈 프림의 Xform(위치와 방향)이 IK 타겟으로 사용됩니다. 뷰포트 조작기를 통해 프림을 이동하면 로봇이 실시간으로 구동됩니다.

Example: Authoring Named Poses#

이 안내는 Robot Poser 창과 IK 추적을 사용하여 로봇 팔에 세 개의 명명된 포즈(홈 위치, 픽 준비 자세, 배치 타겟)를 생성합니다. IsaacRobotAPI가 적용된 로봇 프림이 하나 이상 포함된 스테이지로 시작합니다. 이 예제에서는 Robotiq 2F-85 그리퍼가 있는 UniversalRobots의 UR10e 로봇을 사용합니다. 1. 에셋 브라우저의 UniversalRobots 폴더에서 UR10e 로봇을 찾습니다. 2. UR10e 로봇을 스테이지로 드래그하여 추가합니다. 3. 속성 패널의 Gripper Variants에서 Robotiq 2F-85 그리퍼를 선택합니다.

1. Select the Robot#

Tools > Robotics > Robot Poser를 통해 Robot Poser 창을 엽니다. Active Robot 드롭다운에서 UR10e가 선택되어 있는지 확인하세요.

2. Add a “Home” Pose#

로봇이 기본 조인트 구성에 있는 상태에서 시작 링크를 `base_link`로, 끝 링크를 `grip_frame`으로 선택합니다. 드롭다운에 `base_link` 항목이 두 개 있다는 것을 알 수 있습니다. 이름 위에 마우스를 올려 그리퍼가 아닌 ur10e의 base_link인지 확인하세요. Robot Poser 창에서 Add를 클릭하세요. Named Poses 테이블에 새 행이 나타납니다. 이름 셀을 더블 클릭하여 Home으로 이름을 변경하세요. 운동학적 체인의 원하는 Start Site와 End Site를 선택하세요.

이 포즈는 로봇의 현재 조인트 상태를 캡처하고 잘 알려진 재설정 위치로 사용됩니다.

3. Add a “Pick Ready” Pose with IK Tracking#

Add를 다시 클릭하여 두 번째 포즈를 생성하고 이름을 PickReady로 변경하세요. 새 행의 Track Target 버튼()을 클릭하여 IK 추적을 활성화하세요.

추적이 활성화된 상태에서 뷰포트에서 PickReady 명명된 포즈 프림을 선택하고 이동/회전 기즈모를 사용하여 엔드 이펙터를 원하는 사전 파지 위치로 드래그하세요. 변환 속성 패널을 통해 포즈를 변경할 수도 있습니다. 로봇은 IK가 매 프레임마다 해결되면서 실시간으로 포즈를 업데이트합니다.

로봇이 목표 포즈에 도달하면 Track Target을 다시 클릭하여 추적을 중지하세요. 이제 조인트 값이 PickReady 프림에 저장됩니다.

4. Add a “Place” Pose#

프로세스를 반복합니다: Add를 클릭하고, 포즈 이름을 Place로 변경하고, Track Target을 활성화하고, 명명된 포즈 프림을 원하는 배치 위치로 드래그합니다. 만족하면 추적을 중지하세요.

5. Verify the Poses#

각 행의 Apply Pose 버튼()을 클릭하여 포즈를 순환하고 로봇이 예상된 구성에 도달하는지 확인하세요. 다른 순서를 원하면 드래그 앤 드롭으로 행을 재정렬하세요.

워크플로우 시연을 위한 아래 영상을 참조하세요.

Things to try#

  • 시뮬레이션 실행 중 추적을 활성화하고 뷰포트에서 명명된 포즈 프림을 이동하세요. 로봇이 명명된 포즈 프림을 실시간으로 따릅니다.

  • Animation curve Editor를 사용하여 조인트에 대한 시간 코딩된 조인트 타겟 값을 설정하여 명명된 포즈로 로봇 애니메이션을 만드세요.

  • 각 엔드 이펙터에 대한 명명된 포즈를 생성하여 다중 엔드 이펙터 로봇에서 작업하세요.

  • 로봇의 기본 위치에 대한 명명된 포즈를 생성해 보세요. 작성 중 로봇이 초기 위치에서 이동한 경우 이를 사용하여 로봇을 초기 위치로 재설정할 수 있습니다.

  • 두 개의 다른 엔드 이펙터 사이에 로봇의 베이스 링크를 새 위치로 이동시키는 명명된 포즈를 만들어 보세요. 베이스 링크 변환을 재설정하여 로봇을 원점으로 재설정하고, Zero-Pose 명명된 포즈를 다시 적용하세요.

Named Pose Schema Reference#

각 명명된 포즈는 USD 스테이지에 IsaacNamedPose 유형의 프림으로 저장됩니다. 스키마, 관계, 속성 및 쿼리 유틸리티는 Robot Schema 참조에 문서화되어 있습니다.

isaacsim.robot.poser 확장은 명명된 포즈를 프로그래밍 방식으로 작업하기 위한 CRUD 및 I/O 함수를 제공합니다. 전체 API 목록은 Robot Schema 문서의 Named Pose CRUD 섹션을 참조하세요.
