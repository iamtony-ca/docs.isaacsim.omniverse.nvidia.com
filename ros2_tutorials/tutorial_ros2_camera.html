  • 씬과 로봇에 추가 카메라 추가

  • OmniGraph에서 카메라 퍼블리셔 추가

  • 메뉴 단축키를 사용하여 카메라 퍼블리셔 추가

  • ROS 토픽을 통해 그라운드 트루스 합성 인식 데이터 전송

Getting Started#

필수 조건

  • ROS 2가 설치되고, ROS 2 확장이 활성화되고, 제공된 Isaac Sim ROS 2 워크스페이스가 빌드되고, 필요한 환경 변수가 설정되도록 ROS 2 Installation (Default)를 완료하세요.

  • ROS 토픽과 퍼블리셔와 구독자 동작에 대한 기본 이해.

  • Isaac Sim OmniGraph Tutorial 및 Tutorial 4: Add Camera and Sensors to a Robot 튜토리얼을 완료하세요.

  • 스테이지에 Turtlebot이 준비되도록 URDF Import: Turtlebot을 완료하세요.

참고

Isaac Sim 6.0에서 ROS2 Camera Helper 및 ROS2 Camera Info Helper의 frameSkipCount 입력은 더 이상 사용되지 않습니다. 게시 속도를 제어하려면 카메라 프림(반드시 OmniSensorAPI가 적용되어야 함)에 omni:sensor:tickRate를 설정하세요. 마이그레이션 가이드는 Multi-Tick Rendering을 참조하세요.

Camera Publisher#

Setting Up Cameras#

뷰포트에 기본으로 표시되는 카메라는 Perspective 카메라입니다. 뷰포트 디스플레이 내부 왼쪽 상단 모서리의 Camera 버튼으로 확인할 수 있습니다. Camera 버튼을 클릭하면 몇 가지 다른 사전 설정 카메라 위치가 있는 것을 볼 수 있습니다:

  • 위

  • 앞

  • 오른쪽 측면 뷰

이 튜토리얼에서는 /World/Camera_1과 /World/Camera_2에 두 개의 고정 카메라를 추가하여 두 가지 다른 관점에서 방을 봅니다. 이 튜토리얼 시리즈에서 나중에 사용되는 OmniGraph 노드 및 스크립트(예: ROS2 Setting Publish Rates)와 프림 경로가 일치하도록 정확히 이 이름으로 /World 바로 아래에 각 카메라를 생성합니다. 스테이지에 카메라를 추가하는 절차는 Tutorial 4: Add Camera and Sensors to a Robot에서 찾을 수 있습니다.

동시에 여러 카메라 뷰를 관찰하기 위해 추가 뷰포트를 엽니다. 추가 뷰포트를 열려면:

  1. 뷰포트를 열기 위해 Window > Viewports > Viewport 2

  2. 뷰포트의 왼쪽 상단 모서리에 있는 Cameras 버튼에서 원하는 카메라 뷰를 선택합니다.

Building the Graph for an RGB Publisher#

  1. Stage 패널에서 /World/Camera_1을 선택하면 새 Action Graph가 게시하는 카메라 프림에 인접하여 생성됩니다. 그래프를 직접 영향을 받는 센서에 부모로 설정하면 에셋 구성 가능성이 유지됩니다: Camera_1이 다른 씬에 참조될 때 퍼블리셔 그래프가 함께 이동합니다.

  2. Graph Editors 열기, Window > Graph Editors > Action Graph.

  3. Action Graph 창 중앙의 New Action Graph 아이콘을 클릭하거나, 기존 Action Graph에 카메라 퍼블리셔를 추가하려면 Edit Action Graph를 클릭합니다. 그래프 이름을 Camera_1_Graph로 지정합니다; 결과 그래프 경로는 /World/Camera_1/Camera_1_Graph입니다.

  4. 다음으로 Action Graph를 구성합니다:

    • 다음 이미지의 노드와 연결

    • 아래 표의 파라미터

    파라미터:

    노드

    입력 필드

    값

    Isaac Create Render Product

    cameraPrim

    /World/Camera_1

    enabled

    True

    ROS 2 Camera Helper

    type

    rgb

    topicName

    rgb

    frameId

    turtle

이 그래프를 틱하면 Camera_1에 할당된 새 렌더 프로덕트가 자동으로 생성됩니다.

Graph Explained#

  • On Playback Tick Node: 시뮬레이션이 "재생 중"일 때 틱을 생성합니다. 이 노드에서 틱을 받는 노드들은 매 시뮬레이션 스텝마다 계산 함수를 실행합니다.

  • ROS 2 Context Node: ROS 2는 미들웨어 통신에 DDS를 사용합니다. DDS는 Domain ID를 사용하여 물리적 네트워크를 공유하면서도 다른 논리적 네트워크가 독립적으로 운용될 수 있도록 합니다. 동일한 도메인의 ROS 2 노드는 서로를 자유롭게 검색하고 메시지를 전송할 수 있지만, 다른 도메인의 ROS 2 노드는 그럴 수 없습니다. ROS 2 컨텍스트 노드는 주어진 Domain ID로 컨텍스트를 생성합니다. 기본값은 0으로 설정됩니다. Use Domain ID Env Var가 체크되면 Isaac Sim의 현재 인스턴스를 실행한 환경에서 ROS_DOMAIN_ID를 가져옵니다.

  • Isaac Create Render Product: 렌더 프로덕트 프림을 생성하여 주어진 카메라 프림에서 렌더링된 데이터를 획득하고 렌더 프로덕트 프림의 경로를 출력합니다. enabled 필드를 체크하거나 해제하여 렌더링을 명령에 따라 활성화하거나 비활성화할 수 있습니다.

  • Isaac Run One Simulation Frame: 이 노드는 파이프라인이 시작 시 한 번만 실행되도록 합니다.

  • ROS 2 Camera Helper: 게시할 데이터 유형과 게시할 ROS 토픽을 지정합니다.

Camera Helper Node

Camera Helper Node는 복잡한 후처리 네트워크를 사용자로부터 추상화합니다.

Camera Helper Node가 연결된 상태에서 Play를 누른 후, Action Graph 창 왼쪽 상단 아이콘을 클릭하면 Action Graph 목록에 새로운 그래프가 나타나는 것을 볼 수 있습니다: /Render/PostProcessing/SDGPipeline.

이 그래프는 Camera Helper Node에 의해 자동으로 생성됩니다. 파이프라인은 렌더러에서 관련 데이터를 가져와 처리하고 해당 ROS 퍼블리셔로 전송합니다. 이 그래프는 실행 중인 세션에서만 생성됩니다. 에셋의 일부로 저장되지 않으며 Stage 트리에 나타나지 않습니다.

여러 카메라에 대해 여러 ROS 토픽을 게시하는 예제는 Isaac Sim Content Browser에서 Isaac Sim>Samples>ROS2>Scenario>turtlebot_tutorial.usd로 이동하여 찾을 수 있습니다.

Camera Info Helper Node#

Camera Info Helper 퍼블리셔 노드는 다음 방정식을 사용하여 K, P, R 카메라 고유 행렬을 계산합니다.

파라미터 계산:

\(fx = width * focalLength / horizontalAperture\)

\(fy = height * focalLength / verticalAperture\)

\(cx = width * 0.5\)

\(cy = height * 0.5\)

K 행렬 (고유 파라미터 행렬)

K 행렬은 3x3 행렬입니다.

\[K = { fx, 0, cx 0, fy, cy 0, 0, 1 }\]

P 행렬 (투영 행렬)

스테레오 카메라의 경우 첫 번째 카메라에 대한 두 번째 카메라의 x 및 y 방향 스테레오 오프셋은 Tx와 Ty로 표시됩니다. 두 개의 렌더 프로덕트가 노드에 연결되면 이 값들이 자동으로 계산됩니다.

단안 카메라의 경우 \(Tx = Ty = 0\).

P 행렬은 3x4 행 우선 행렬입니다.

\[P = { fx, 0, cx, Tx, 0, fy, cy, Ty, 0, 0, 1, 0 }\]

R 행렬 (정류 행렬)

R 행렬은 카메라 좌표계를 이상적인 스테레오 이미지 평면에 정렬하여 두 스테레오 이미지의 에피폴라 라인이 평행하게 되도록 적용되는 회전 행렬입니다. R 행렬은 스테레오 카메라에만 사용되며 3x3 행렬로 설정됩니다.

Graph Shortcut#

여러 카메라 센서 그래프를 구성하는 메뉴 단축키가 있습니다. Tools > Robotics > ROS 2 OmniGraphs > Camera로 이동합니다.

ROS 2 그래프가 표시되지 않으면 ROS 2 브릿지를 활성화해야 합니다. 그래프를 생성하는 데 필요한 파라미터를 묻는 팝업 창이 아래에 나타납니다. Graph Path, Camera Prim, frameId, 네임스페이스(있는 경우)를 제공하고 게시할 데이터에 대한 확인란을 체크해야 합니다.

기존 그래프에 그래프를 추가하려면 Add to an existing graph? 박스를 체크합니다. 이렇게 하면 기존 그래프에 노드가 추가되고, 틱 노드, 컨텍스트 노드, 시뮬레이션 시간 노드가 있으면 이를 재사용합니다.

Verifying ROS Connection#

전달되는 원시 정보를 관찰하려면 ros2 topic echo /<topic>을 사용하세요.

다시 rqt_image_view 방법을 사용하여 깊이를 시각화합니다: ros2 run rqt_image_view rqt_image_view /depth.

RViz2에서 이번에 게시된 이미지를 확인하려면:

  1. ROS 2 소싱된 터미널에서 rviz2 명령을 입력하여 RViz를 엽니다.

  2. Image 디스플레이 유형을 추가하고 토픽을 rgb로 설정합니다.

Troubleshooting#

깊이 이미지가 흑백 섹션만 표시되는 경우, 시야에 "무한" 깊이가 있어 명암 대비가 왜곡되었을 가능성이 높습니다. 이미지의 깊이 범위가 제한되도록 시야각을 조정하세요.

Additional Publishing Options#

요청 시 또는 지정된 속도로 주기적으로 이미지를 게시하려면 Python 스크립팅을 사용해야 합니다. 예제는 ROS 2 Camera로 이동하세요.
