Learning Objectives#

  • ROS2 컴포넌트 수동 스텝

이 페이지 상단의 Standalone Environment 배너에서 구성을 확인하거나 선택하여 설정에 맞는 실행 명령어를 확인하세요. 선택한 항목이 이 페이지 나머지 부분의 플랫폼별 명령어를 결정합니다.

Standalone Environment

Platform:

Getting Started#

사전 준비사항

  • 두 가지 워크플로 (Standalone 및 Extension)를 이해하기 위해 Workflows와 Hello World를 완료하세요.

  • Using Terminal or Enable ROS 2 Python Standalone Scripts의 단계를 완료하여 독립형 워크플로에서 ROS2 메시징을 활성화하는 데 필요한 환경 변수를 설정합니다.

Windows 독립형 ROS 2 예제는 Pixi Python 환경을 사용합니다. ROS 2 Installation (Other Platforms)의 Pixi 설정을 따른 다음, Pixi workspace에서 이 명령어들을 실행하세요.

Note

Windows 11에서는 머신 구성에 따라 RViz2가 제대로 열리지 않을 수 있습니다.

Manually Stepping ROS2 Components#

독립형 스크립팅은 일반적으로 시뮬레이션 스텝의 수동 제어에 이상적입니다. OnImpulseEvent OmniGraph 노드는 publisher와 subscriber의 빈도를 세심하게 제어할 수 있도록 ROS2 OmniGraph 노드에 연결할 수 있습니다.

ROS2 Domain ID를 1로 사용하여 ROS2 Publish Clock 노드가 있는 새 action graph를 정확하게 제어하도록 설정하는 방법의 예시:


import omni.graph.core as og

# Create a new graph with the path /ActionGraph
og.Controller.edit(
    {"graph_path": "/ActionGraph", "evaluator_name": "execution"},
    {
        og.Controller.Keys.CREATE_NODES: [
            ("ReadSimTime", "isaacsim.core.nodes.IsaacReadSimulationTime"),
            ("Context", "isaacsim.ros2.bridge.ROS2Context"),
            ("PublishClock", "isaacsim.ros2.bridge.ROS2PublishClock"),
            ("OnImpulseEvent", "omni.graph.action.OnImpulseEvent"),
        ],
        og.Controller.Keys.CONNECT: [
            # Connecting execution of OnImpulseEvent node to PublishClock so it will only publish when an impulse event is triggered
            ("OnImpulseEvent.outputs:execOut", "PublishClock.inputs:execIn"),
            # Connecting simulationTime data of ReadSimTime to the clock publisher node
            ("ReadSimTime.outputs:simulationTime", "PublishClock.inputs:timeStamp"),
            # Connecting the ROS2 Context to the clock publisher node so it will run under the specified ROS2 Domain ID
            ("Context.outputs:context", "PublishClock.inputs:context"),
        ],
        og.Controller.Keys.SET_VALUES: [
            # Assigning topic name to clock publisher
            ("PublishClock.inputs:topicName", "/clock"),
            # Assigning a Domain ID of 1 to Context node
            ("Context.inputs:domain_id", 1),
            # Disable useDomainIDEnvVar to ensure we use the above set Domain ID
            ("Context.inputs:useDomainIDEnvVar", False),
        ],
    },
)

모든 프레임에서 다음을 실행하여 impulse 이벤트를 설정하면 clock publisher가 한 번 틱됩니다:

og.Controller.set(og.Controller.attribute("/ActionGraph/OnImpulseEvent.state:enableImpulse"), True)

Note

독립형 스크립팅에서 렌더링 및 물리 시뮬레이션 스텝을 명시적으로 제어하기 때문에, 각 스텝 완료에 걸리는 시간은 계산 부하에 따라 달라지며 실시간과 일치하지 않을 수 있습니다. 이로 인해 동일한 애플리케이션을 독립형 스크립팅과 GUI를 사용하여 실행할 때 관찰되는 동작 속도에 불일치가 발생할 수 있습니다. 이런 경우 시뮬레이션 클록을 기준으로 사용하세요.

Examples#

일부 튜토리얼 예제가 독립형 Python 예제로 변환되었습니다. 실행 방법은 다음과 같습니다.

ROS 2 Clock#

이 샘플은 ROS 2 컴포넌트 노드가 있는 action graph를 생성하고 다른 속도로 틱하는 방법을 시연합니다.

다음을 실행하여 샘플을 실행할 수 있습니다:

./python.sh standalone_examples/api/isaacsim.ros2.bridge/clock.py

pixi run python standalone_examples/api/isaacsim.ros2.bridge/clock.py

다음 토픽들을 에코하여 발행되는 메시지를 확인합니다:

ros2 topic echo /sim_time
ros2 topic echo /manual_time

Isaac Sim UI를 사용하여 ROS 2 Clock publisher를 생성하고 설정하려면 ROS 2 Clock 튜토리얼을 참조하세요.

ROS 2 Camera#

다음 두 샘플은 ROS 2 RGB 이미지, depth 이미지, 카메라 정보 publisher를 설정하는 데 사용되는 ROS 2 Camera Helper와 Camera Info Helper OmniGraph 노드가 있는 action graph를 생성하는 방법을 시연합니다. 두 샘플 모두 다른 속도로 ROS 2 이미지 데이터를 발행하는 동일한 결과를 달성하지만 다른 방법을 사용합니다.

  • 매 프레임마다:

    • Camera Info가 발행됨

  • 5프레임마다:

    • RGB 이미지가 발행됨

  • 60프레임마다:

    • Depth 이미지가 발행됨

Periodic Image Publishing#

각 ROS 2 이미지 및 카메라 정보 publisher의 실행 속도 (N 프레임마다)는 SDGPipeline 그래프에서 각각의 Isaac Simulation Gate OmniGraph 노드를 수정하여 설정됩니다. 실행 속도를 설정하면 이미지 publisher가 N 렌더링 프레임마다 자동으로 틱됩니다.

다음을 실행하여 샘플을 실행할 수 있습니다:

./python.sh standalone_examples/api/isaacsim.ros2.bridge/camera_periodic.py

pixi run python standalone_examples/api/isaacsim.ros2.bridge/camera_periodic.py

샘플을 종료하려면 터미널에서 CTRL-C로 프로세스를 종료할 수 있습니다.

Manual Image Publishing#

ROS 2 이미지 및 카메라 정보 publisher는 각 publisher 노드와 해당 Isaac Simulation Gate OmniGraph 노드 사이에 Branch OmniGraph 노드를 삽입하여 수동으로 제어됩니다. Branch 노드는 커스텀 게이트처럼 작동하며 언제든지 활성화하거나 비활성화할 수 있습니다. Branch 노드가 활성화될 때마다 연결된 ROS 2 publisher 노드가 틱됩니다.

다음을 실행하여 샘플을 실행할 수 있습니다:

./python.sh standalone_examples/api/isaacsim.ros2.bridge/camera_manual.py

pixi run python standalone_examples/api/isaacsim.ros2.bridge/camera_manual.py

샘플을 종료하려면 터미널에서 CTRL-C로 프로세스를 종료할 수 있습니다.

Visualizing Results#

RViz2에서 두 샘플 중 하나의 결과를 시각화하려면, 새 ROS2가 소스된 터미널에서 Isaac Sim 패키지 디렉터리로 이동하여 다음 명령어를 실행합니다:

rviz2 -d <ros2_ws>/src/isaac_tutorials/rviz2/camera_manual.rviz

Note

RViz2 문제로 인해 depth 이미지 표시에 검은 프레임이 나타날 수 있습니다. Isaac Sim이 depth 이미지를 올바르게 발행하는지 확인하려면, ros2 run rqt_image_view rqt_image_view를 실행하고 토픽을 /depth로 설정합니다.

Carter Stereo#

이 샘플은 ROS 2 컴포넌트 노드가 포함된 action graph가 있는 기존 USD stage를 가져와 기본 설정을 수정하는 방법을 시연합니다. stereo 카메라 쌍이 자동으로 활성화되고 두 번째 뷰포트 창이 UI에 도킹됩니다.

  • 매 프레임마다:

    • ROS 2 클록이 발행됨

    • RTX Lidar에서 발생한 ROS 2 PointCloud2 메시지가 발행됨

    • Odometry가 발행됨

    • Twist subscriber가 스핀됨

    • TF 메시지가 발행됨

    • 왼쪽 및 오른쪽 카메라가 발행됨

  • 2프레임마다:

    • Twist 명령 메시지가 발행됨

다음을 실행하여 샘플을 실행할 수 있습니다:

./python.sh standalone_examples/api/isaacsim.ros2.bridge/carter_stereo.py

pixi run python standalone_examples/api/isaacsim.ros2.bridge/carter_stereo.py

샘플을 종료하려면 터미널에서 CTRL-C로 프로세스를 종료할 수 있습니다.

결과를 시각화하려면:

새 터미널에서 다음 명령어를 실행하여 RViz2를 로드합니다:

rviz2 -d <ros2_ws>/src/isaac_tutorials/rviz2/carter_stereo.rviz

Displays 내의 Right Camera - RGB와 Left Camera - RGB가 활성화되어 RGB 이미지를 시각화할 수 있는지 확인합니다.

Note

RViz2에 일부 이미지가 표시되지 않으면 이미지가 표시되도록 시뮬레이터에서 Stop과 Play를 누르세요.

Multiple Robot ROS 2 Navigation#

이 샘플은 기존 USD stage를 실행하는 방법을 보여줍니다.

출력을 시각화하려면 interactive version of the sample을 참조하세요:

  • 매 프레임마다:

    • ROS 2 클록 컴포넌트가 발행됨

    • RTX Lidar에서 발생한 ROS 2 PointCloud2 메시지가 발행됨

    • Odometry가 발행됨

    • Twist subscriber가 스핀됨

    • TF 메시지가 발행됨

샘플은 hospital 및 office 환경 모두에서 실행할 수 있습니다. 다음 중 하나를 실행하여 지정된 환경으로 샘플을 실행합니다:

./python.sh standalone_examples/api/isaacsim.ros2.bridge/carter_multiple_robot_navigation.py --environment hospital

pixi run python standalone_examples/api/isaacsim.ros2.bridge/carter_multiple_robot_navigation.py --environment hospital

./python.sh standalone_examples/api/isaacsim.ros2.bridge/carter_multiple_robot_navigation.py --environment office

pixi run python standalone_examples/api/isaacsim.ros2.bridge/carter_multiple_robot_navigation.py --environment office

샘플을 종료하려면 터미널에서 CTRL-C로 프로세스를 종료할 수 있습니다.

Note

문제가 발생하면 Troubleshooting을 참조하세요.

MoveIt2#

이 샘플은 여러 USD stage를 추가하는 방법을 보여줍니다. 또한 ROS 2 컴포넌트 노드가 있는 action graph를 수동으로 생성하고 자동으로 틱하는 방법을 시연합니다.

출력을 시각화하려면 interactive version of the sample을 참조하세요:

  • 매 프레임마다:

    • ROS 2 클록이 발행됨

    • Joint State 메시지가 발행됨

    • Joint State subscriber가 스핀됨

    • TF 메시지가 발행됨

다음을 실행하여 샘플을 실행할 수 있습니다:

./python.sh standalone_examples/api/isaacsim.ros2.bridge/moveit.py

pixi run python standalone_examples/api/isaacsim.ros2.bridge/moveit.py

터미널에서 CTRL-C로 종료할 수 있습니다.

Receiving ROS 2 Messages#

이것은 ROS2 빈 메시지를 받으면 씬의 큐브가 임의의 위치로 이동하는 기본 subscriber 예제입니다. 렌더링이 활성화된 상태로 실행되므로 씬과 큐브가 이동하는 것을 확인할 수 있습니다. 이 예제를 실행하려면:

./python.sh standalone_examples/api/isaacsim.ros2.bridge/subscriber.py

pixi run python standalone_examples/api/isaacsim.ros2.bridge/subscriber.py

샘플을 종료하려면 터미널에서 CTRL-C로 프로세스를 종료할 수 있습니다.

큐브가 있는 씬이 로드된 후, 다른 터미널에서 빈 메시지를 수동으로 발행할 수 있습니다. 1Hz 속도를 사용합니다.

ros2 topic pub -r 1 /move_cube std_msgs/msg/Empty
