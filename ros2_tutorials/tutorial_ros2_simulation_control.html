ROS2 Simulation Control#

Learning Objectives#

이 페이지에서는 다음 내용을 배웁니다:

  • Isaac Sim을 위한 ROS 2 Simulation Control extension 이해

  • ROS 2 서비스와 액션을 사용하여 Isaac Sim 시뮬레이션 제어

  • ROS 2 인터페이스를 사용하여 시뮬레이션 엔티티와 월드 조작

  • 프로그래밍 방식으로 시뮬레이션 스텝 진행

Getting Started#

사전 준비사항

  • ROS 2 Installation (Default)을 완료하세요.

  • 여러 시스템을 사용하는 경우, Isaac Sim을 실행하기 전에 ROS 2 Installation (Default)의 지침에 따라 FASTRTPS_DEFAULT_PROFILES_FILE 환경 변수를 설정하고, ROS 메시지를 주고받을 모든 터미널에서도 설정하며, ROS2 Extension이 활성화되어 있어야 합니다.

  • Isaac Sim 5.0 이상

  • ROS 2 (Humble 이상)

  • 다음을 사용하여 Simulation Interfaces 패키지를 설치합니다:

    sudo apt install ros-humble-simulation-interfaces

    sudo apt install ros-jazzy-simulation-interfaces

    Simulation Interfaces 패키지의 소스 코드는 여기를 참조하세요.

참고

이 extension은 simulation_interfaces 1.5.0 이상을 대상으로 합니다. 설치된 패키지에서 서비스나 SimulatorFeatures 상수가 없는 경우 해당 서비스는 등록을 건너뛰고 기능이 GetSimulatorFeatures 응답에서 생략됩니다 — 시작 시 사용 불가능한 유형을 명시하는 경고를 로깅합니다.

RoboStack Jazzy (pixi 기반 ROS 2 워크스페이스에서 사용)는 현재 SpawnEntities를 포함하지 않는 simulation_interfaces 1.2.0을 제공합니다. simulation_interfaces 1.5.0+은 SpawnEntities를 추가하며 곧 RoboStack Jazzy에 추가될 예정입니다. 그때까지 해당 서비스는 pixi/jazzy 워크스페이스에서 사용할 수 없습니다.

Overview#

ROS 2 Simulation Control extension은 ROS 2 Simulation Interfaces를 사용하여 Isaac Sim 기능을 제어합니다. 이 extension은 여러 서비스와 액션이 최소한의 성능 오버헤드로 동시에 실행될 수 있도록 확장 가능하게 설계되었습니다.

extension은 ROS 2 서비스와 액션을 통해 Isaac Sim에 대한 포괄적인 제어를 제공합니다:

  • 시뮬레이션 상태 제어: 시뮬레이션 재생, 일시 정지, 중지 및 스텝 진행

  • 엔티티 관리: 시뮬레이션 엔티티 (prim) 생성, 삭제, 조작

  • 월드 관리: 시뮬레이션 월드 (USD 파일) 로드, 언로드, 조회

  • 상태 조회: 엔티티, 시뮬레이션 상태, 사용 가능한 리소스에 대한 정보 가져오기

이를 통해 Simulation Interfaces 패키지를 사용하여 자동화된 테스트와 같은 워크플로를 활성화하는 ROS 2 명령줄 인터페이스를 통해 프로그래밍 방식으로 Isaac Sim을 제어할 수 있습니다. 이 페이지에는 Isaac Sim이 지원하는 Simulation Interfaces의 모든 서비스와 액션에 대한 예제 명령어가 나열되어 있습니다.

Enabling the Extension#

Enable Automatically#

  1. 다음 명령으로 터미널에서 Isaac Sim을 엽니다:

    ./isaac-sim.sh --/isaac/startup/ros_sim_control_extension=True

Enable Manually#

  1. Isaac Sim 열기

  2. Extension Manager에서 extension 활성화: isaacsim.ros2.sim_control

Available Services and Actions#

extension은 다음 ROS 2 서비스를 제공합니다:

  • /get_simulator_features: 이 Isaac Sim 구현에서 지원되는 기능 목록 나열

  • /set_simulation_state: 시뮬레이션을 특정 상태 (stopped/playing/paused/quitting)로 설정

  • /get_simulation_state: 현재 시뮬레이션 상태 가져오기

  • /get_entities: 시뮬레이션의 모든 엔티티 (prim) 목록 가져오기

  • /get_entity_info: 특정 엔티티에 대한 자세한 정보 가져오기 (현재 OBJECT 카테고리 유형 반환)

  • /get_entity_state: 특정 엔티티의 포즈, 트위스트, 가속도 가져오기

  • /get_entities_states: 필터링을 사용하여 여러 엔티티의 상태 (포즈, 트위스트, 가속도) 가져오기

  • /delete_entity: 시뮬레이션에서 특정 엔티티 (prim) 삭제

  • /spawn_entity: 지정된 위치에 시뮬레이션으로 새 엔티티 생성 (deprecated — 대신 /spawn_entities 사용)

  • /reset_simulation: 시뮬레이션 환경을 초기 상태로 재설정

  • /set_entity_state: 시뮬레이션의 특정 엔티티 상태 (포즈, 트위스트) 설정

  • /step_simulation: 시뮬레이션을 특정 프레임 수만큼 앞으로 진행

  • /load_world: 시뮬레이션으로 월드 또는 환경 파일 로드

  • /unload_world: 현재 월드를 언로드하고 빈 stage 생성

  • /get_current_world: 현재 로드된 월드에 대한 정보 가져오기

  • /get_available_worlds: 로드할 수 있는 사용 가능한 월드 파일 목록 가져오기

  • /spawn_entities: 단일 요청으로 여러 엔티티를 배치 생성 (Humble은 simulation_interfaces >= 1.4.0, Jazzy는 >= 1.5.0 필요)

  • /get_entity_bounds: 엔티티의 월드 공간 축 정렬 경계 박스 가져오기

  • /get_spawnables: 생성 가능한 USD 에셋 탐색

다음 ROS 2 액션도 제공합니다:

  • /simulate_steps: 진행 피드백과 함께 시뮬레이션을 스텝 진행하는 액션

Using the ROS 2 Simulation Control Services#

이 섹션에서는 사용 가능한 각 서비스를 자세히 사용하는 방법을 설명합니다.

GetSimulatorFeatures Service#

GetSimulatorFeatures 서비스는 simulation_interfaces에서 Isaac Sim이 지원하는 서비스 및 액션의 하위 집합을 나열합니다.

ros2 service call /get_simulator_features simulation_interfaces/srv/GetSimulatorFeatures

참고 사항:

  • 지원되는 기능 목록 반환 (예: SPAWNING, DELETING, ENTITY_STATE_GETTING)

  • spawn_formats 필드를 사용하여 USD 파일 형식 지원 보고

  • 구현에 대한 세부 정보와 함께 custom_info 제공

  • 시뮬레이터 기능의 전체 목록 참조

SetSimulationState Service#

SetSimulationState 서비스는 시뮬레이션의 전역 상태를 SimulationState.msg에 정의된 enum에 해당하는 (stopped/playing/paused/quitting)으로 업데이트합니다 (STATE_STOPPED, STATE_PLAYING, STATE_PAUSED, STATE_QUITTING).

  1. 시뮬레이션 상태를 playing으로 설정하려면:

    ros2 service call /set_simulation_state simulation_interfaces/srv/SetSimulationState "{state: {state: 1}}"  # 1=playing

  2. 시뮬레이션 상태를 paused로 설정하려면:

    ros2 service call /set_simulation_state simulation_interfaces/srv/SetSimulationState "{state: {state: 2}}"  # 2=paused

  3. 시뮬레이션 상태를 stopped로 설정하려면:

    ros2 service call /set_simulation_state simulation_interfaces/srv/SetSimulationState "{state: {state: 0}}"  # 0=stopped

  4. 시뮬레이터를 종료하려면:

    ros2 service call /set_simulation_state simulation_interfaces/srv/SetSimulationState "{state: {state: 3}}"  # 3=quit

참고 사항:

  • 상태 0 (Stopped)은 시뮬레이션을 일시 정지하고 재설정하는 것과 동일합니다

  • 상태 1 (Playing)은 시뮬레이션 타임라인을 시작합니다

  • 상태 2 (Paused)는 현재 시간에 시뮬레이션을 일시 정지합니다

  • 상태 3 (Quitting)은 Isaac Sim을 종료합니다

GetSimulationState Service#

GetSimulationState 서비스는 전체 시뮬레이션의 현재 상태를 SimulationState.msg에 정의된 enum에 해당하는 (stopped/playing/paused/quitting)으로 가져옵니다 (STATE_STOPPED, STATE_PLAYING, STATE_PAUSED, STATE_QUITTING).

ros2 service call /get_simulation_state simulation_interfaces/srv/GetSimulationState

참고 사항:

  • stopped는 0, playing은 1, paused는 2를 반환합니다

  • 스텝 진행 같은 작업을 수행하기 전에 시뮬레이션 상태를 조회하는 데 사용됩니다

GetEntities Service#

GetEntities 서비스는 선택적 필터링 (정규식 패턴 사용)과 함께 시뮬레이션에 있는 모든 엔티티 목록을 검색합니다.

  1. 시뮬레이션의 모든 엔티티 가져오기:

    ros2 service call /get_entities simulation_interfaces/srv/GetEntities "{filters: {filter: ''}}"

  2. 전체 경로 또는 부분 경로로 엔티티 가져오기. 이 경우 경로에 'camera'가 포함된 prim 필터링:

    ros2 service call /get_entities simulation_interfaces/srv/GetEntities "{filters: {filter: 'camera'}}"

  3. '/World'로 시작하는 경로를 가진 엔티티 가져오기:

    ros2 service call /get_entities simulation_interfaces/srv/GetEntities "{filters: {filter: '^/World'}}"

  4. 'mesh'로 끝나는 경로를 가진 엔티티 가져오기:

    ros2 service call /get_entities simulation_interfaces/srv/GetEntities "{filters: {filter: 'mesh$'}}"

참고 사항:

  • filter 파라미터는 엔티티 이름 (prim 경로)을 매칭하기 위한 POSIX 확장 정규식을 허용합니다

  • Isaac Sim은 전체 USD prim 경로를 엔티티 이름으로 사용합니다

GetEntityInfo Service#

GetEntityInfo 서비스는 특정 엔티티에 대한 유형 및 속성과 같은 자세한 정보를 제공합니다.

ros2 service call /get_entity_info simulation_interfaces/srv/GetEntityInfo "{entity: '/World/robot'}"

참고 사항:

  • 엔티티가 존재하면 EntityInfo와 함께 RESULT_OK 반환

  • 엔티티가 존재하지 않으면 RESULT_OPERATION_FAILED 반환

  • EntityInfo 포함 내용: * category: 현재 항상 OBJECT로 설정 (EntityCategory.OBJECT) * description: 빈 문자열 (향후 사용을 위해 예약됨) * tags: 빈 배열 (향후 사용을 위해 예약됨)

GetEntityState Service#

GetEntityState 서비스는 주어진 참조 프레임에 대한 특정 엔티티의 포즈, 트위스트, 가속도를 가져옵니다. 현재는 월드 프레임만 지원됩니다.

ros2 service call /get_entity_state simulation_interfaces/srv/GetEntityState "{entity: '/World/robot'}"

참고 사항:

  • RigidBodyAPI가 있는 엔티티는 포즈와 속도 모두 반환됩니다

  • RigidBodyAPI가 없는 엔티티는 속도가 0인 포즈만 반환됩니다

  • 가속도 값은 항상 0으로 보고됩니다 (현재 API에서 제공하지 않음)

  • 엔티티 상태를 성공적으로 검색하면 RESULT_OK 반환

  • 엔티티가 존재하지 않으면 RESULT_NOT_FOUND 반환

  • 엔티티 상태 검색 중 오류가 발생하면 RESULT_OPERATION_FAILED 반환

GetEntitiesStates Service#

GetEntitiesStates 서비스는 시뮬레이션에서 여러 엔티티의 월드 프레임 상태 (포즈, 트위스트, 가속도)를 가져옵니다.

  1. 시뮬레이션의 모든 엔티티 상태 가져오기:

    ros2 service call /get_entities_states simulation_interfaces/srv/GetEntitiesStates "{filters: {filter: ''}}"

  2. 경로에 'robot'이 포함된 엔티티 상태 가져오기:

    ros2 service call /get_entities_states simulation_interfaces/srv/GetEntitiesStates "{filters: {filter: 'robot'}}"

  3. '/World'로 시작하는 경로를 가진 엔티티 상태 가져오기:

    ros2 service call /get_entities_states simulation_interfaces/srv/GetEntitiesStates "{filters: {filter: '^/World'}}"

참고 사항:

  • GetEntities와 GetEntityState 서비스의 기능을 결합합니다

  • 먼저 정규식 패턴 매칭을 사용하여 엔티티를 필터링합니다

  • 각 필터링된 엔티티의 상태를 검색합니다

  • 엔티티 경로 및 해당 상태 목록을 반환합니다

  • RigidBodyAPI가 있는 엔티티는 포즈와 속도 모두 반환됩니다

  • RigidBodyAPI가 없는 엔티티는 속도가 0인 포즈만 반환됩니다

  • 가속도 값은 항상 0으로 보고됩니다 (현재 API에서 제공하지 않음)

  • 이 서비스를 사용하면 여러 엔티티의 상태가 필요할 때 여러 GetEntityState 호출보다 효율적입니다

  • 엔티티 상태를 성공적으로 검색하면 RESULT_OK 반환

  • 필터링 또는 상태 검색 중 오류가 발생하면 RESULT_OPERATION_FAILED 반환

DeleteEntity Service#

DeleteEntity 서비스는 시뮬레이션에서 지정된 엔티티를 삭제합니다.

ros2 service call /delete_entity simulation_interfaces/srv/DeleteEntity "{entity: '/World/robot'}"

참고 사항:

  • 엔티티가 성공적으로 삭제되면 RESULT_OK 반환

  • 엔티티가 보호되어 삭제할 수 없으면 RESULT_OPERATION_FAILED 반환

  • 삭제를 시도하기 전에 prim을 삭제할 수 있는지 확인하기 위해 prim_utils.is_prim_no_delete()를 사용합니다

GetSpawnables Service#

GetSpawnables 서비스는 생성 가능한 USD 에셋을 탐색합니다. 기본적으로 Isaac 에셋 루트 경로에서 /Isaac/Samples/ROS2/Robots를 검색합니다. sources 필드를 사용하여 추가 커스텀 검색 경로를 지정할 수 있습니다.

  1. 모든 기본 생성 가능 항목 나열:

    ros2 service call /get_spawnables simulation_interfaces/srv/GetSpawnables

  2. 추가 로컬 경로에서 생성 가능 항목 나열:

    ros2 service call /get_spawnables simulation_interfaces/srv/GetSpawnables "{sources: ['/home/user/custom_robots']}"

  3. 여러 소스에서 생성 가능 항목 나열:

    ros2 service call /get_spawnables simulation_interfaces/srv/GetSpawnables "{sources: ['/home/user/robots', '/opt/isaac_assets/robots']}"

참고 사항:

  • 기본 검색 경로는 /Isaac/Samples/ROS2/Robots (깊이 2까지 검색)

  • sources의 추가 경로는 재귀적으로 검색됩니다 (무제한 깊이)

  • 각 결과는 uri (전체 에셋 URI), description (확장자 없는 파일 이름), 빈 spawn_bounds를 가진 Spawnable 반환

  • 에셋 루트 경로를 사용할 수 없고 sources가 제공되지 않으면 RESULT_OPERATION_FAILED 반환

참고

Windows 사용자 (WSL 포함): sources에 경로를 제공할 때 네이티브 Windows 경로를 사용하세요 (예: C:/Users/foo/robots). /mnt/c/Users/foo/robots와 같은 WSL 스타일 경로는 Isaac Sim Windows 프로세스에서 접근할 수 없으며 결과를 반환하지 않습니다.

SpawnEntity Service#

경고

Deprecated. 대신 /spawn_entities 서비스를 사용하세요. SpawnEntities는 배치 작업과 엔티티별 오류 보고를 지원하여 단일 및 다중 엔티티 워크플로 모두에 적합합니다.

SpawnEntity usage (deprecated)

SpawnEntity 서비스는 지정된 위치에 새 엔티티를 시뮬레이션으로 생성합니다.

  1. 기본 위치로 기본 엔티티 생성:

    ros2 service call /spawn_entity simulation_interfaces/srv/SpawnEntity "{name: 'MyEntity', allow_renaming: false, uri: '/path/to/model.usd'}"

  2. 특정 위치와 방향으로 생성:

    ros2 service call /spawn_entity simulation_interfaces/srv/SpawnEntity "{name: 'PositionedEntity', allow_renaming: false, uri: '/path/to/model.usd', initial_pose: {pose: {position: {x: 1.0, y: 2.0, z: 3.0}, orientation: {w: 1.0, x: 0.0, y: 0.0, z: 0.0}}}}"

  3. 빈 Xform 생성 (URI 없음):

    ros2 service call /spawn_entity simulation_interfaces/srv/SpawnEntity "{name: 'EmptyXform', allow_renaming: false, uri: ''}"

  4. 자동 이름 바꾸기 활성화:

    ros2 service call /spawn_entity simulation_interfaces/srv/SpawnEntity "{name: 'AutoRenamedEntity', allow_renaming: true, uri: '/path/to/model.usd'}"

  5. namespace 지정:

    ros2 service call /spawn_entity simulation_interfaces/srv/SpawnEntity "{name: 'NamespacedEntity', allow_renaming: false, uri: '/path/to/model.usd', entity_namespace: 'robot1'}"

참고 사항:

  • URI가 제공되면 USD 파일을 주어진 prim 경로의 참조로 로드합니다

  • URI가 제공되지 않으면 주어진 prim 경로에 Xform을 생성합니다

  • 모든 생성된 prim은 추적을 위해 simulationInterfacesSpawned 속성으로 표시됩니다

  • 엔티티가 성공적으로 생성되면 RESULT_OK 반환

  • allow_renaming이 false이고 엔티티 이름이 이미 존재하면 NAME_NOT_UNIQUE (101) 반환

  • allow_renaming이 false이고 엔티티 이름이 비어 있으면 NAME_INVALID (102) 반환

  • USD 파일을 파싱하거나 로드하는 데 실패하면 RESOURCE_PARSE_ERROR (106) 반환

SpawnEntities Service#

SpawnEntities 서비스는 단일 요청으로 여러 엔티티를 생성합니다. spawn_requests의 각 항목은 SpawnEntity.msg 형식을 따르며 독립적으로 처리됩니다. 엔티티별 성공 또는 실패는 results 목록에 보고됩니다.

참고

이 서비스는 ROS 2 Humble에서는 simulation_interfaces >= 1.4.0, ROS 2 Jazzy에서는 >= 1.5.0이 필요합니다.

  1. 특정 위치에 두 엔티티 생성:

    ros2 service call /spawn_entities simulation_interfaces/srv/SpawnEntities "{
  spawn_requests: [
    {
      name: 'Robot1',
      allow_renaming: false,
      entity_resource: {uri: '/path/to/robot.usd'},
      initial_pose: {pose: {position: {x: 0.0, y: 0.0, z: 0.0}, orientation: {w: 1.0, x: 0.0, y: 0.0, z: 0.0}}}
    },
    {
      name: 'Robot2',
      allow_renaming: true,
      entity_resource: {uri: '/path/to/robot.usd'},
      initial_pose: {pose: {position: {x: 2.0, y: 0.0, z: 0.0}, orientation: {w: 1.0, x: 0.0, y: 0.0, z: 0.0}}}
    }
  ]
}"

참고 사항:

  • spawn_requests의 각 항목은 SpawnEntity에서 사용하는 플랫 uri 필드가 아닌 entity_resource.uri 필드를 사용합니다

  • 엔티티별 결과는 results 목록에 반환됩니다; 개별 성공 또는 실패를 위해 각 SpawnResult를 확인하세요

  • 개별 생성이 하나라도 실패하면 집계 result 필드가 ENTITIES_SPAWN_FAILED로 설정됩니다

  • 다른 모든 SpawnEntity 규칙이 각 항목에 적용됩니다 (자동 이름 바꾸기, simulationInterfacesSpawned 태깅 등)

GetEntityBounds Service#

GetEntityBounds 서비스는 엔티티의 월드 공간 축 정렬 경계 박스 (AABB)를 계산하고 반환합니다.

ros2 service call /get_entity_bounds simulation_interfaces/srv/GetEntityBounds "{entity: '/World/Cube'}"

참고 사항:

  • AABB의 최소 모서리와 최대 모서리라는 두 개의 points와 type=TYPE_BOX를 가진 Bounds 메시지 반환

  • 경계 박스는 기본 USD 타임 코드에서 계산되며 default 목적을 가진 prim만 포함합니다

  • 엔티티가 존재하지 않으면 RESULT_NOT_FOUND 반환

  • 경계 계산이 실패하면 RESULT_OPERATION_FAILED 반환

ResetSimulation Service#

ResetSimulation 서비스는 시뮬레이션 환경을 초기 상태로 재설정합니다.

ros2 service call /reset_simulation simulation_interfaces/srv/ResetSimulation

참고 사항:

  • 시뮬레이션 타임라인 중지

  • simulationInterfacesSpawned 속성을 가진 모든 prim을 찾아 제거합니다

  • 생성된 모든 엔티티가 제거되도록 여러 패스를 사용합니다

  • 시뮬레이션 타임라인 재시작

  • 성공적으로 재설정되면 RESULT_OK 반환

  • 시뮬레이션 재설정 중 오류가 발생하면 RESULT_OPERATION_FAILED 반환

SetEntityState Service#

SetEntityState 서비스는 시뮬레이션의 특정 엔티티 상태 (포즈, 트위스트)를 설정합니다. 현재는 월드 프레임의 변환만 허용됩니다.

  1. 위치와 방향만 설정:

    ros2 service call /set_entity_state simulation_interfaces/srv/SetEntityState "{
  entity: '/World/Cube',
  state: {
    header: {frame_id: 'world'},
    pose: {
      position: {x: 1.0, y: 2.0, z: 3.0},
      orientation: {w: 1.0, x: 0.0, y: 0.0, z: 0.0}
    },
    twist: {
      linear: {x: 0.0, y: 0.0, z: 0.0},
      angular: {x: 0.0, y: 0.0, z: 0.0}
    }
  }
}"

  2. 위치, 방향 및 속도 설정 (rigid body physics가 있는 엔티티의 경우):

    ros2 service call /set_entity_state simulation_interfaces/srv/SetEntityState "{
  entity: '/World/RigidBody',
  state: {
    header: {frame_id: 'world'},
    pose: {
      position: {x: 1.0, y: 2.0, z: 3.0},
      orientation: {w: 1.0, x: 0.0, y: 0.0, z: 0.0}
    },
    twist: {
      linear: {x: 0.5, y: 0.0, z: 0.0},
      angular: {x: 0.0, y: 0.0, z: 0.1}
    }
  }
}"

참고 사항:

  • 위치와 방향은 항상 모든 엔티티에 대해 업데이트됩니다

  • 속도는 RigidBodyAPI가 있는 엔티티에만 설정됩니다

  • 비강체의 경우 위치와 방향만 설정됩니다 (속도 설정은 무시됨)

  • 가속도 설정은 현재 지원되지 않으며 무시됩니다

  • 엔티티 상태를 성공적으로 설정하면 RESULT_OK 반환

  • 엔티티가 존재하지 않으면 RESULT_NOT_FOUND 반환

  • 엔티티 상태 설정 중 오류가 발생하면 RESULT_OPERATION_FAILED 반환

StepSimulation Service#

StepSimulation 서비스는 유한한 수의 스텝을 시뮬레이션하고 PAUSED 상태로 돌아옵니다.

  1. 1 프레임씩 시뮬레이션 스텝 진행 (참고: 내부적으로 2 스텝을 사용합니다):

    ros2 service call /step_simulation simulation_interfaces/srv/StepSimulation "{steps: 1}"

  2. 10 프레임씩 시뮬레이션 스텝 진행:

    ros2 service call /step_simulation simulation_interfaces/srv/StepSimulation "{steps: 10}"

  3. 100 프레임씩 시뮬레이션 스텝 진행:

    ros2 service call /step_simulation simulation_interfaces/srv/StepSimulation "{steps: 100}"

참고 사항:

  • 스텝 진행을 수행하기 전에 시뮬레이션이 paused 상태여야 합니다

  • 서비스 호출은 모든 스텝이 완료될 때까지 블로킹됩니다

  • 스텝 진행이 완료되면 시뮬레이션은 자동으로 paused 상태로 돌아옵니다

  • 스텝 진행이 성공적으로 완료되면 RESULT_OK 반환

  • 서비스가 호출될 때 시뮬레이션이 paused 상태가 아니면 RESULT_INCORRECT_STATE 반환

  • 스텝 진행 중 오류가 발생하면 RESULT_OPERATION_FAILED 반환

LoadWorld Service#

LoadWorld 서비스는 현재 씬을 지우고 시뮬레이션을 stopped 상태로 설정하면서 월드 또는 환경 파일을 시뮬레이션으로 로드합니다. 현재는 USD 형식 월드를 지원합니다.

  1. USD 파일에서 월드 로드:

    ros2 service call /load_world simulation_interfaces/srv/LoadWorld "{uri: '/path/to/world.usd'}"

  2. Isaac Sim 샘플 환경으로 샘플 월드 로드:

    ros2 service call /load_world simulation_interfaces/srv/LoadWorld "{uri: 'https://omniverse-content-production.s3-us-west-2.amazonaws.com/Assets/Isaac/6.0/Isaac/Environments/Simple_Room/simple_room.usd'}"

  3. ROS2 시나리오로 샘플 월드 로드:

    ros2 service call /load_world simulation_interfaces/srv/LoadWorld "{uri: 'https://omniverse-content-production.s3-us-west-2.amazonaws.com/Assets/Isaac/6.0/Isaac/Samples/ROS2/Scenario/carter_warehouse_apriltags_worker.usd'}"

참고 사항:

  • USD 파일(.usd, .usda, .usdc, .usdz)만 지원됩니다

  • 월드를 로드하기 전에 시뮬레이션이 stopped 또는 paused 상태여야 합니다 (playing 상태에서 불가)

  • 월드를 로드하면 현재 씬이 지워지고 USD 파일에서 새 stage가 생성됩니다

  • 주어진 경로를 직접 찾을 수 없으면 Isaac Sim이 자동으로 기본 에셋 루트 경로를 앞에 추가하려고 시도합니다

  • 월드가 성공적으로 로드되면 RESULT_OK 반환

  • 파일 형식이 지원되지 않으면 UNSUPPORTED_FORMAT 반환

  • USD 파일을 파싱하거나 로드할 수 없으면 RESOURCE_PARSE_ERROR 반환

  • 서비스가 호출될 때 시뮬레이션이 playing 상태이면 RESULT_OPERATION_FAILED 반환

UnloadWorld Service#

UnloadWorld 서비스는 현재 씬을 지우고 새 빈 stage를 생성하여 시뮬레이션에서 현재 월드를 언로드합니다. 이전에 생성된 모든 엔티티는 제거됩니다.

ros2 service call /unload_world simulation_interfaces/srv/UnloadWorld

참고 사항:

  • 월드를 언로드하기 전에 시뮬레이션이 stopped 또는 paused 상태여야 합니다 (playing 상태에서 불가)

  • 현재 월드를 언로드한 후 새 빈 stage를 생성합니다

  • 월드가 성공적으로 언로드되면 RESULT_OK 반환

  • 현재 월드가 로드되지 않은 경우 NO_WORLD_LOADED 반환

  • 서비스가 호출될 때 시뮬레이션이 playing 상태이면 RESULT_OPERATION_FAILED 반환

GetCurrentWorld Service#

GetCurrentWorld 서비스는 현재 로드된 월드에 대한 정보 (URI, 이름, 형식 포함)를 반환합니다.

ros2 service call /get_current_world simulation_interfaces/srv/GetCurrentWorld

참고 사항:

  • 파일에서 월드가 로드된 경우 URI와 이름을 포함한 월드 정보 반환

  • 메모리에서 생성된 월드 (새 stage)의 경우 빈 URI와 "untitled_world"를 이름으로 반환

  • 성공하면 월드 정보와 함께 RESULT_OK 반환

  • 현재 월드가 로드되지 않은 경우 NO_WORLD_LOADED 반환

GetAvailableWorlds Service#

GetAvailableWorlds 서비스는 시뮬레이션으로 로드할 수 있는 사용 가능한 월드 파일 목록을 반환합니다. TagsFilter 기반 필터링을 지원하면서 USD 월드 파일에 대한 기본 Isaac Sim 경로를 검색합니다.

  1. 모든 기본 사용 가능한 월드 가져오기:

    ros2 service call /get_available_worlds simulation_interfaces/srv/GetAvailableWorlds

  2. 태그 필터링으로 월드 가져오기 (파일 이름에서 특정 태그가 있는 기본 월드 검색):

    ros2 service call /get_available_worlds simulation_interfaces/srv/GetAvailableWorlds "{filter: {tags: ['warehouse', 'carter']}, continue_on_error: true}"

  3. 추가 커스텀 경로 검색:

    ros2 service call /get_available_worlds simulation_interfaces/srv/GetAvailableWorlds "{additional_sources: ['/custom/worlds/path'], continue_on_error: true}"

  4. 추가 로컬 소스와 함께 오프라인 전용 검색:

    ros2 service call /get_available_worlds simulation_interfaces/srv/GetAvailableWorlds "{additional_sources: ['/home/user/custom_worlds', '/opt/isaac_worlds'], offline_only: true, continue_on_error: true}"

참고 사항:

  • 기본 Isaac Sim 경로를 검색합니다: /Isaac/Environments 및 /Isaac/Samples/ROS2/Scenario

  • 태그 매칭을 위한 FILTER_MODE_ANY (기본값) 또는 FILTER_MODE_ALL과 함께 TagsFilter를 지원합니다

  • additional_sources에 지정된 추가 커스텀 경로를 검색할 수 있습니다

  • 로컬 파일시스템 경로만 검색하려면 offline_only: true를 설정합니다

  • 일부 경로가 실패해도 계속 검색하려면 continue_on_error: true를 설정합니다

  • 사용 가능한 월드 목록과 함께 RESULT_OK 반환

  • 기본 에셋 경로에 접근할 수 없고 추가 소스가 제공되지 않으면 DEFAULT_SOURCES_FAILED 반환

참고

Windows 사용자 (WSL 포함): additional_sources에 경로를 제공할 때 네이티브 Windows 경로를 사용하세요 (예: C:/Users/foo/worlds). /mnt/c/Users/foo/worlds와 같은 WSL 스타일 경로는 Isaac Sim Windows 프로세스에서 접근할 수 없으며 결과를 반환하지 않습니다.

Using the ROS 2 Simulation Control Actions#

SimulateSteps Action#

SimulateSteps 액션은 유한한 수의 스텝을 시뮬레이션하고 각 스텝 후 피드백과 함께 PAUSED 상태로 돌아옵니다.

  1. 기본 사용법 - 10 프레임씩 시뮬레이션 스텝 진행:

    ros2 action send_goal /simulate_steps simulation_interfaces/action/SimulateSteps "{steps: 10}"

  2. 피드백과 함께 - 20 프레임씩 시뮬레이션 스텝 진행 및 피드백 표시:

    ros2 action send_goal /simulate_steps simulation_interfaces/action/SimulateSteps "{steps: 20}" --feedback

참고 사항:

  • 스텝 진행을 수행하기 전에 시뮬레이션이 paused 상태여야 합니다

  • 스텝이 완료되면 시뮬레이션은 paused 상태로 돌아옵니다

  • 각 스텝 후 완료 및 남은 스텝을 나타내는 피드백을 받습니다

  • 실행 중 액션을 취소할 수 있습니다

Technical Details#

extension은 시뮬레이션 상태를 제어하기 위해 omni.timeline 인터페이스를 사용하며 표준 서비스를 통해 깔끔한 ROS 2 인터페이스를 제공합니다. 구현에는 다음이 포함됩니다:

  • 단일 노드를 통해 모든 ROS 2 서비스를 처리하는 싱글톤 ROS2ServiceManager

  • Isaac Sim의 타임라인과 인터페이스하는 SimulationControl 클래스

  • Action Graph 인터페이스와 독립적으로 ROS 2 스피닝을 위한 스레드 안전 구현

Extending#

더 많은 시뮬레이션 제어 서비스를 추가하려면 SimulationControl 클래스를 확장하고 ROS2ServiceManager에 추가 서비스를 등록하세요.
