Robot Schema#

Robot Schema는 OpenUSD를 확장하여 로봇 구조를 표준화되고 조합 가능한 방식으로 설명하는 적용 API 스키마 세트를 제공합니다. USD Common 정의와 운동학적 트리 정의를 위한 Physics Schema를 기반으로 하며, Isaac Sim에서 로봇의 표준 표현을 제공합니다.

스키마는 두 가지 확장으로 구현됩니다:

  • isaacsim.robot.schema – 로봇 구조를 탐색하고 유지하기 위한 스키마 정의, 애플리케이션 헬퍼 및 프로그래밍 유틸리티.

  • isaacsim.robot.schema.ui – 다양한 표시 모드로 로봇 운동학적 트리를 보고, 컴포넌트를 선택적으로 마스킹 및 우회하고, 링크를 월드에 고정하고, 뷰포트에서 조인트 연결을 시각화하는 대화형 Robot Inspector Window.

Schema Overview#

Robot Schema는 5개의 적용 API 스키마와 2개의 타입 스키마를 정의합니다:

스키마

목적

IsaacRobotAPI

로봇의 최상위 prim에 적용되는 루트 정의. 메타데이터와 링크 및 조인트의 정렬된 목록을 보유합니다.

IsaacLinkAPI

강체(또는 다른 시뮬레이션 바디)를 로봇 구성의 링크로 플래그 지정합니다.

IsaacJointAPI

물리 조인트를 로봇 구성의 일부로 플래그 지정하고 DOF 순서 정보를 담습니다.

IsaacSiteAPI

로봇의 관심 지점(도구 마운트, 센서 위치, 엔드 이펙터 프레임)을 표시합니다.

IsaacAttachmentPointAPI

표면 그리퍼에서 사용되는 부착 지점을 정의합니다.

IsaacNamedPose

IK 목표 변환과 함께 명명된 조인트 구성을 저장하는 타입 prim 스키마로, Robot Poser에서 사용됩니다.

IsaacSurfaceGripper

표면 그리퍼 메커니즘(그립 힘, 거리, 재시도 동작)을 위한 타입 prim 스키마.

Robot API#

IsaacRobotAPI는 로봇의 루트 prim에 적용되며, 로봇의 구성과 메타데이터에 대한 단일 소스 역할을 합니다.

관계(Relationships)

관계

설명

isaac:physics:robotLinks

베이스 링크로 시작하는 로봇을 구성하는 링크의 정렬된 목록. 부모 링크 뒤에 사이트가 삽입될 수 있습니다.

isaac:physics:robotJoints

링크를 연결하는 조인트의 정렬된 목록.

isaac:robot:namedPoses

로봇의 저장된 조인트 구성을 정의하는 IsaacNamedPose prim 목록.

속성(Attributes)

속성

타입

설명

isaac:description

String

로봇의 목적과 기능을 설명하는 자유 형식 텍스트.

isaac:namespace

String

컴포넌트 메시징에 사용되는 고유 네임스페이스 식별자.

isaac:robotType

Token

Manipulator, Humanoid, Mobile Base 등의 로봇 카테고리.

isaac:license

Token

로봇 에셋이 배포되는 라이선스.

isaac:source

String

원본 에셋 소스에 대한 URL 또는 참조.

isaac:version

String

로봇 에셋의 시맨틱 버전 번호.

isaac:changelog

String[]

에셋 개정 사항에 걸친 변경 설명의 정렬된 목록.

Note

링크 및 조인트 목록은 보고에 관련된 요소만 포함하면 됩니다. 전체 운동학적 트리에는 이 목록에 없는 추가 요소가 포함될 수 있습니다.

Link API#

IsaacLinkAPI는 로봇 구성에 참여하는 각 바디에 적용됩니다. prim이 로봇 상태 보고에 표시되어야 함을 나타내는 플래그 역할을 합니다.

속성

타입

설명

isaac:nameOverride

String

로봇 상태 보고 시 prim 이름 대신 사용되는 선택적 커스텀 이름.

링크는 강체로 제한되지 않습니다. API는 변형 가능한 바디 또는 다른 시뮬레이션 타입에 적용할 수 있지만, 비강체 링크에서 로봇 상태를 계산하려면 커스텀 처리가 필요합니다.

로봇에 속하는 모든 링크는 IsaacRobotAPI 링크 목록에 나타나는지 여부에 관계없이 IsaacLinkAPI가 적용되어 있어야 합니다.

Joint API#

IsaacJointAPI는 로봇 구성에 참여하는 물리 조인트에 적용됩니다. 조인트를 포함 대상으로 플래그 지정하고 DOF 순서 정보를 담습니다.

속성

타입

설명

isaac:nameOverride

String

로봇 상태 보고 시 prim 이름 대신 사용되는 선택적 커스텀 이름.

isaac:physics:DofOffsetOpOrder

Token[]

평탄화된 DOF 인덱스 순서를 정의하는 자유도 토큰(TransX, TransY, TransZ, RotX, RotY, RotZ)의 정렬된 목록. 단일 DOF 조인트(회전, 프리즈매틱) 및 제로 DOF 조인트(고정)는 이 속성이 필요하지 않습니다.

로봇에 속하는 모든 조인트는 IsaacRobotAPI 조인트 목록에 나타나는지 여부에 관계없이 IsaacJointAPI가 적용되어 있어야 합니다.

Note

이전 버전에서는 토큰 배열 대신 축별 DOF 오프셋 속성(isaac:physics:Tr_X:DoFOffset 등)이 사용되었습니다. 이는 더 이상 사용되지 않습니다. 기존 에셋 마이그레이션을 위해 UpdateDeprecatedJointDofOrder 또는 UpdateDeprecatedSchemas를 사용하십시오.

Site API#

IsaacSiteAPI는 로봇의 관심 지점(도구 부착 프레임, 센서 마운트 위치, 엔드 이펙터 참조 프레임 등)을 설명합니다.

속성

타입

설명

isaac:Description

String

"Tool Attachment Point"와 같은 사이트 설명.

isaac:forwardAxis

Token

사이트의 전방 방향으로 간주되는 축(X, Y, 또는 Z).

사이트는 robotLinks 관계에 포함됩니다. 부모 링크 바로 다음에 배치하거나 목록 끝에 그룹화할 수 있으며, 이는 채우기 시 sites_last 파라미터로 제어됩니다.

Note

IsaacSiteAPI는 더 이상 사용되지 않는 IsaacReferencePointAPI를 대체합니다. 구 스키마를 여전히 사용하는 로봇은 기능하지만 더 이상 사용 중단 경고를 표시합니다. 마이그레이션을 위해 UpdateDeprecatedSchemas를 사용하십시오.

Named Pose#

IsaacNamedPose는 Xform에서 상속받는 타입 prim 스키마로, 로봇의 운동학적 체인 세그먼트에 대한 재사용 가능한 조인트 구성을 저장합니다. 각 명명된 포즈는 시작 링크와 끝 링크/사이트 사이의 조인트, 해당 조인트 값, 그리고 prim의 Xform ops에 인코딩된 목표 엔드 이펙터 변환을 캡처합니다.

명명된 포즈는 로봇 루트 prim 하위의 Named_Poses 범위 아래에 수집되며 IsaacRobotAPI의 isaac:robot:namedPoses 관계를 통해 등록됩니다. Robot Poser UI를 통해 또는 isaacsim.robot.poser API를 통해 프로그래밍 방식으로 생성 및 관리됩니다.

관계(Relationships)

관계

설명

isaac:robot:pose:startLink

이 포즈가 다루는 운동학적 체인의 시작 링크.

isaac:robot:pose:endLink

운동학적 체인의 끝 링크 또는 사이트.

isaac:robot:pose:joints

시작 링크와 끝 링크 사이의 체인에서 조인트 prim의 정렬된 목록.

속성(Attributes)

Composing Robots#

로봇 구성은 각 서브 로봇의 루트 prim에 IsaacRobotAPI를 적용하여 빌드됩니다. 최종 어셈블리는 다음 중 하나의 방법으로 달성됩니다:

  • 서브 로봇의 루트 prim을 부모 로봇의 링크 및 조인트 목록에 추가하면, 부모가 서브 로봇의 전체 운동학적 트리를 재귀적으로 포함합니다.

  • 서브 로봇에서 특정 링크 및 조인트를 선택하여 부모 로봇의 목록에 직접 추가합니다.

Applying the Robot Schema#

Isaac Sim 에셋 라이브러리의 모든 로봇과 URDF Importer Extension 또는 MJCF Importer Extension을 통해 가져온 로봇에는 Robot Schema가 사전 적용되어 있습니다. 이전 버전에서 가져왔거나 외부 소스에서 가져온 로봇의 경우 스키마를 수동으로 적용해야 합니다.

Through the GUI#

  1. Stage 패널에서 로봇의 루트 prim을 선택합니다.

  2. Properties 패널에서 + Add 버튼을 클릭합니다.

  3. Isaac > Robot Schema > Robot API를 선택합니다.

이렇게 하면 루트 prim에 IsaacRobotAPI가 적용되고, 물리 아티큘레이션을 자동으로 탐색하여 발견된 모든 바디와 조인트에 IsaacLinkAPI 및 IsaacJointAPI를 적용합니다.

각 스키마의 속성은 해당 API 섹션(자주색으로 표시) 아래 Properties 패널에 나타납니다.

시간이 지남에 따라 로봇 구조가 변경되는 경우(예: 새로운 링크 또는 조인트가 추가된 경우), 새 prim에 개별 API를 수동으로 적용하거나 루트 prim에 Robot API를 재적용하여 자동 채우기를 다시 실행합니다.

Note

스키마를 적용할 때, 에셋이 Asset Structure 가이드라인을 따르는 경우, 인터페이스 레이어가 아닌 베이스 레이어 또는 전용 로봇 스키마 레이어에 적용하십시오. 자동 채우기에는 작성된 물리가 필요하므로, 스키마 적용 중에는 물리 레이어를 서브레이어로 일시적으로 추가한 다음 저장 전에 제거합니다.

Through Code#

다음 스니펫은 Robot Schema를 프로그래밍 방식으로 적용합니다. Asset Structure 가이드라인에 따라, 스키마는 별도의 레이어에 작성되어 다른 페이로드와 독립적으로 유지되고 스키마가 발전함에 따라 쉽게 업데이트할 수 있습니다.

import omni.usd
import pxr
import usd.schema.isaac.robot_schema as rs
from pxr import Sdf, Usd, UsdGeom

stage = omni.usd.get_context().get_stage()
robot_asset_path = "/".join(stage.GetRootLayer().identifier.split("/")[:-1])  # Get the asset path from the stage
robot_asset = ".".join(
    stage.GetRootLayer().identifier.split("/")[-1].split(".")[:-1]
)  # Get the asset name from the stage
schema_asset = f"configuration/{robot_asset}_robot_schema.usda"
edit_layer = Sdf.Layer.FindOrOpen(f"{robot_asset_path}/{schema_asset}")
if not edit_layer:
    edit_layer = Sdf.Layer.CreateNew(f"{robot_asset_path}/{schema_asset}")
# Add sublayer to the stage, but as a relative path, only if not already present
if schema_asset not in stage.GetRootLayer().subLayerPaths:
    stage.GetRootLayer().subLayerPaths.append(schema_asset)
# Make all edits in the edit layer
with pxr.Usd.EditContext(stage, edit_layer):

    default_prim = stage.GetDefaultPrim()

    # Apply the Robot API to the default prim, and auto-populate the Links and Joints lists
    rs.ApplyRobotAPI(default_prim)


edit_layer.Save()
stage.Save()

Editing in the Properties Panel#

IsaacRobotAPI가 있는 prim을 선택하면, Properties 패널에 로봇 메타데이터, 링크/조인트 관계 목록, 그리고 유지 관리 도구 모음을 단일 Figma 스타일 레이아웃으로 통합한 전용 Robot Schema 위젯이 표시됩니다. 이 위젯은 isaacsim.gui.property에서 제공하며, 이전에 이러한 속성을 표시하던 일반 관계/속성 편집기를 대체합니다.

위젯은 세 개의 접을 수 있는 섹션으로 나뉩니다.

Robot Joints and Robot Links#

다음 두 섹션은 isaac:physics:robotJoints 및 isaac:physics:robotLinks 관계를 스크롤 가능하고 드래그로 순서를 변경할 수 있는 목록으로 표시합니다.

각 행에는:

  • 숫자 인덱스(직접 자식에만 해당, 서브 로봇 행에는 없음).

  • 드래그 앤 드롭 순서 변경을 위한 잡기 핸들. 행을 드래그하면 삽입 지점에 수평 드롭 표시기가 표시됩니다.

  • 대상 prim의 표시 이름. 마우스를 올리면 전체 prim 경로가 포함된 툴팁이 표시됩니다. 더블 클릭하면 스테이지에서 해당 prim이 선택됩니다.

  • 관계에서 항목을 제거하는 뒤따르는 제거 버튼.

Adding entries#

Add Joint 및 Add Link 버튼은 호환 가능한 prim으로 미리 필터링된 스테이지 prim 선택기를 엽니다:

  • Add Joint – IsaacJointAPI 또는 IsaacRobotAPI가 있는 prim을 표시합니다.

  • Add Link – IsaacLinkAPI, IsaacSiteAPI, 또는 IsaacRobotAPI가 있는 prim을 표시합니다.

Sub-robot rows#

행이 IsaacRobotAPI를 가진 prim을 가리키는 경우, 레이블 옆에 공개 삼각형이 나타납니다. 행을 확장하면 해당 서브 로봇의 일치하는 관계(조인트 목록의 조인트, 링크 목록의 링크)의 읽기 전용 들여쓰기 미리보기가 표시됩니다. 미리보기는 깊이 중첩된 구성도 읽기 쉽도록 최대 4단계까지 재귀합니다.

Other Robot Schema Widgets#

다른 Robot Schema API 중 하나가 있는 prim을 선택하면 해당 스키마에 대한 집중 위젯이 표시됩니다:

  • Robot Link (IsaacLinkAPI) – 선택적 isaac:nameOverride 속성을 표시합니다.

  • Robot Joint (IsaacJointAPI) – isaac:nameOverride, isaac:physics:DofOffsetOpOrder, 및 isaac:actuator 플래그를 표시합니다.

  • Robot Site (IsaacSiteAPI) – 사이트 설명과 전방 축 토큰을 표시하며, 모든 Xformable prim에 사용 가능합니다.

각 위젯에는 스키마를 제거하고 작성된 속성을 지우는 헤더 제거 버튼이 있습니다. + Add 메뉴의 Isaac/Robot Schema/... 항목을 사용하여 이러한 스키마 중 하나를 새 prim에 적용합니다.

Parsing Robot Structure#

로봇 운동학적 트리는 Robot Schema 관계로 보강된 Physics Schema에서 파생됩니다. 파싱은 다음과 같이 진행됩니다:

  1. IsaacRobotAPI prim의 robotLinks 관계에서 링크를 수집합니다.

  2. robotJoints 관계에서 조인트를 수집합니다.

  3. 첫 번째 링크(베이스 링크)에서 시작하여, 조인트를 통해 연결된 링크까지 너비 우선 탐색을 수행하여 트리를 빌드합니다.

트리는 비순환이어야 합니다. 루프를 형성하는 조인트는 Exclude from Articulation 속성이 설정되어야 합니다; 그렇지 않으면 방문 순서에 따라 파싱 중에 임의로 루프가 끊어집니다.

Example#

  1. Content Browser에서 UR10e 로봇(Robots/UniversalRobots/ur10e/ur10e.usd)을 스테이지로 드래그합니다.

  2. Properties 패널의 Variant 선택 메뉴에서 Robotiq 2f-140 그리퍼 변형을 선택합니다.

  1. Window > Script Editor를 통해 Script Editor를 열고 실행합니다:

    import omni.usd

robot_tree = robot_schema.utils.GenerateRobotLinkTree(stage, prim)

robot_schema.utils.PrintRobotTree(robot_tree)

Schema Population#

함수

설명

PopulateRobotSchemaFromArticulation(stage, robot_prim, articulation_prim, *, detect_sites, sites_last)

DFS를 통해 물리 아티큘레이션 그래프를 탐색하고, 발견된 prim에 IsaacLinkAPI 및 IsaacJointAPI를 적용하고, 정렬된 robotLinks 및 robotJoints 관계를 작성합니다. 선택적으로 링크 하위의 리프 Xform을 감지하고 IsaacSiteAPI를 적용합니다.

RecalculateRobotSchema(stage, robot_prim, articulation_prim, *, detect_sites, sites_last)

PopulateRobotSchemaFromArticulation과 유사하지만 유효한 항목의 기존 순서를 유지합니다. 새 링크와 조인트가 추가되고 유효하지 않은 대상은 제거됩니다. 증분 업데이트에 사용합니다.

두 함수 모두 다음을 허용합니다:

  • detect_sites (bool): True이면, 각 링크 하위의 자식 없는 자식 Xform을 감지하고 IsaacSiteAPI를 자동으로 적용합니다.

  • sites_last (bool): False이면, 감지된 사이트는 부모 링크 바로 다음에 삽입됩니다. True이면, 모든 사이트가 링크 목록 끝에 추가됩니다.

로봇 서브트리를 탐색하고 IsaacReferencePointAPI를 IsaacSiteAPI로 교체합니다. 또한 조인트의 더 이상 사용되지 않는 축별 DOF 오프셋 속성을 마이그레이션합니다.

UpdateDeprecatedJointDofOrder(joint_prim)

단일 조인트의 더 이상 사용되지 않는 축별 DoFOffset 속성을 DofOffsetOpOrder 토큰 배열로 마이그레이션합니다. 편집 레이어에서 더 이상 사용되지 않는 속성을 제거합니다.

Kinematics#

isaacsim.robot.schema 확장은 순수 Python 순방향 운동학, 자코비안 계산 및 역운동학 스택을 포함합니다. 이 모듈들은 Robot Poser 내부적으로 사용되며 직접 사용도 가능합니다.

from usd.schema.isaac.robot_schema.ik_solver import IKSolver, IKSolverRegistry
from usd.schema.isaac.robot_schema.kinematic_chain import KinematicChain
from usd.schema.isaac.robot_schema.math import Joint, Transform

Math Primitives#

math 모듈(usd.schema.isaac.robot_schema.math)은 USD 스테이지나 시뮬레이션 의존성 없이 기초 데이터 구조와 순수 수학 유틸리티를 제공합니다.

데이터 구조

클래스

설명

Transform

강체 SE(3) 변환 (이동 t + 쿼터니언 q, [w, x, y, z] 순서). @를 통한 합성, inv()를 통한 역전 지원.

Joint

운동학적 체인의 단일 조인트. 스크류 축(w는 회전형, v는 프리즈매틱형), 홈 포즈, 조인트 제한(lower, upper), 선택적 끝 팁 오프셋, USD prim_path를 저장합니다. exp(q) 메서드는 주어진 조인트 값에 대한 상대 변환을 반환합니다.

쿼터니언 유틸리티

함수

설명

quat_mul(q1, q2)

두 쿼터니언의 해밀턴 곱.

quat_conj(q)

쿼터니언 켤레.

quat_rotate(q, v)

단위 쿼터니언으로 3D 벡터 회전.

axis_angle_to_quat(axis, angle)

축-각도 표현에서 단위 쿼터니언 빌드.

quat_to_matrix(q)

단위 쿼터니언을 3x3 회전 행렬로 변환.

선형 대수

함수

설명

skew(v)

v와의 외적을 위한 3x3 스큐 대칭 행렬.

adjoint(T)

강체 변환 T에 대한 6x6 수반 행렬.

Kinematic Chain#

kinematic_chain 모듈(usd.schema.isaac.robot_schema.kinematic_chain)은 로봇의 운동학적 트리를 캐시하고 FK 및 IK 계산을 위한 시작 prim과 끝 prim 사이의 정렬된 조인트 체인을 빌드하는 KinematicChain 클래스를 제공합니다.

메서드 / 속성

설명

KinematicChain(stage, robot_prim, start_prim, end_prim)

생성자. 운동학적 트리를 한 번 빌드하고 두 prim 사이의 조인트 체인을 추출합니다. start_prim과 end_prim은 선택적입니다; 생략하면 캐시된 트리를 IK 없이 텔레포트 작업에 사용할 수 있습니다.

compute_fk(q)

조인트 구성 q에 대한 엔드 이펙터 FK를 계산합니다. (Transform, per_joint_transforms)를 반환합니다.

compute_fk_and_jacobian(q)

통합 단일 패스 FK 및 공간 자코비안 계산. (Transform, 6xN Jacobian)을 반환합니다.

read_joint_states()

체인 조인트에 대한 현재 USD 조인트 상태를 읽습니다. prim 경로에서 값(라디안 또는 미터)의 딕셔너리를 반환합니다.

teleport(joint_dict)

운동학적 트리를 통해 FK 바디 변환을 전파하여 조인트 값을 적용합니다. 시뮬레이션이 중지된 경우 사용합니다.

teleport_anchored(joint_dict)

고정된 prim의 월드 위치를 변경하지 않고 조인트 값을 적용합니다. FK 전파 후 로봇을 강체 보정하여 역방향(자식에서 부모) 조인트를 처리합니다.

joints

체인에서 Joint 객체의 정렬된 목록.

tree_root

캐시된 운동학적 트리 루트 노드.

IK Solver Interface#

ik_solver 모듈(usd.schema.isaac.robot_schema.ik_solver)은 추상 솔버 인터페이스와 전역 레지스트리를 정의합니다.

클래스 / 함수

설명

IKSolver

추상 기본 클래스. 서브클래스는 목표 포즈를 달성하는 조인트 값을 반환하는 solve(chain, target, q0, **kwargs)를 구현합니다.

IKSolverRegistry.register(name, solver_cls, *, default)

주어진 이름으로 IK 솔버 클래스를 등록합니다. 기본 솔버로 만들려면 default=True로 설정합니다.

IKSolverRegistry.get(name)

주어진 이름으로 등록된 솔버의 새 인스턴스를 반환합니다. None은 기본 솔버를 반환합니다.

IKSolverRegistry.available()

등록된 모든 솔버 이름을 나열합니다.

pose_error(Td, T)

원하는 변환과 실제 변환 사이의 6-DOF 포즈 오차를 계산합니다. 6-벡터 [rot_x, rot_y, rot_z, pos_x, pos_y, pos_z]를 반환합니다.

커스텀 IK 솔버는 임포트 시 등록할 수 있으며 solver_name 파라미터에 이름을 전달하여 Robot Poser에서 사용할 수 있습니다.

Levenberg-Marquardt Solver#

lm_ik 모듈(usd.schema.isaac.robot_schema.lm_ik)은 "lm"으로 등록된 기본 IK 솔버를 제공합니다. 적응형 댐핑, 조인트 제한 클램핑, 조인트 중간 범위를 향한 영공간 바이어스, 조인트별 고정 마스크를 갖춘 Levenberg-Marquardt 최적화를 구현합니다.

파라미터

기본값

설명

lam

1e-3

초기 LM 댐핑 인수. 자동으로 적응: 진전 시 감소, 오버슈트 시 증가.

iters

30

최대 반복 횟수.

tol

1e-6

가중 비용에 대한 수렴 허용 오차.

w_rot

1.0

회전 오차 성분에 대한 가중치.

w_pos

1.0

위치 오차 성분에 대한 가중치.

max_step

0.5

반복당 최대 조인트 공간 스텝(급격한 점프 방지).

null_space_bias

0.05

조인트 중간 범위를 향한 영공간 바이어스 강도. 특이 구성에서 탈출하는 데 도움이 됩니다.

joint_fixed

None

체인 길이의 불리언 마스크. True는 해당 DOF를 잠급니다(자코비안 열이 0이 됨).

Named Pose CRUD#

isaacsim.robot.poser 확장은 IsaacNamedPose prim에 대한 상위 수준 CRUD 및 I/O 작업을 제공합니다. 이 함수들은 Kinematic Chain 및 IK 솔버를 기반으로 명명된 포즈를 생성, 검색, 적용 및 내보냅니다.

from isaacsim.robot.poser import (
    apply_pose_by_name,
    delete_named_pose,
    export_poses,
    get_named_pose,
    import_poses,
    list_named_poses,
    store_named_pose,
)

함수

설명

store_named_pose(stage, robot_prim, pose_name, pose_result)

IsaacNamedPose prim을 생성하고, 조인트 값, 관계 및 목표 Xform을 작성하고, 로봇의 namedPoses 관계에 등록합니다.

get_named_pose(stage, robot_prim, pose_name)

저장된 포즈를 PoseResult 데이터클래스로 검색합니다.

JSON 파일에서 명명된 포즈를 가져와 로봇에 저장합니다.

Note

pose_name은 prim 이름으로 사용되기 전에 pxr.Tf.MakeValidIdentifier()를 통해 정제됩니다. ASCII [A-Za-z0-9_] 범위 밖의 문자는 _로 대체되고, 입력이 숫자로 시작하는 경우 선행 _가 앞에 추가됩니다. 같은 식별자로 정제되는 서로 다른 입력(예: "home:v2"와 "home/v2")은 같은 저장된 포즈를 참조하며; store_named_pose는 기존 포즈를 덮어쓸 때 경고를 기록합니다.

명명된 포즈의 대화형 작성에 대해서는 Robot Poser 문서를 참조하십시오.

isaacsim.robot.schema 확장은 다음을 통해 접근 가능한 utils 모듈에서 포괄적인 유틸리티 함수 세트를 제공합니다:

from usd.schema.isaac.robot_schema import utils

Schema Population#

함수

설명

PopulateRobotSchemaFromArticulation(stage, robot_prim, articulation_prim, *, detect_sites, sites_last)

BFS를 통해 물리 아티큘레이션 그래프를 탐색하고, 발견된 prim에 IsaacLinkAPI 및 IsaacJointAPI를 적용하고, 정렬된 robotLinks 및 robotJoints 관계를 작성합니다. 선택적으로 링크 하위의 리프 Xform을 감지하고 IsaacSiteAPI를 적용합니다.

RecalculateRobotSchema(stage, robot_prim, articulation_prim, *, detect_sites, sites_last)

PopulateRobotSchemaFromArticulation과 유사하지만 유효한 항목의 기존 순서를 유지합니다. 새 링크와 조인트가 추가되고 유효하지 않은 대상은 제거됩니다. 증분 업데이트에 사용합니다.

두 함수 모두 다음을 허용합니다:

  • detect_sites (bool): True이면, 각 링크 하위의 자식 없는 자식 Xform을 감지하고 IsaacSiteAPI를 자동으로 적용합니다.

  • sites_last (bool): False이면, 감지된 사이트는 부모 링크 바로 다음에 삽입됩니다. True이면, 모든 사이트가 링크 목록 끝에 추가됩니다.

Asset Structure#

Asset Structure 가이드라인에 따라, Robot Schema를 별도의 레이어에 적용하고 로봇 에셋의 서브레이어로 로드합니다. 이렇게 하면 스키마가 물리 및 지오메트리 페이로드와 독립적으로 유지되어, 릴리스에 따라 스키마가 발전함에 따라 쉽게 업데이트할 수 있습니다.
