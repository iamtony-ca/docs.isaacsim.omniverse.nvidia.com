cuMotion World Interface Tutorial#

이 튜토리얼은 cuMotion 통합의 CumotionWorldInterface 클래스를 사용하여 모션 계획 및 제어를 위한 세계 상태, 장애물, 로봇 베이스 변환을 관리하는 방법을 보여줍니다.

이 튜토리얼을 마치면 다음을 이해할 수 있습니다:

  • CumotionWorldInterface 설정 방법

  • SceneQuery를 사용하여 장애물을 탐색하는 방법

  • 장애물 표현 방식과 안전 허용 오차를 설정하는 방법

  • 다양한 업데이트 방식으로 세계 상태를 동기화하는 방법

사전 요구 사항

  • Scene Interaction 튜토리얼을 검토하여 SceneQuery, WorldBinding, ObstacleStrategy, WorldInterface 인터페이스를 이해하세요.

  • Isaac Sim에서 오브젝트에 콜라이더 프리셋을 추가하는 방법을 숙지하세요. 스테이지 트리에서 오브젝트를 선택하고 Property 탭으로 이동한 후 + Add 버튼을 클릭하고 Physics > Colliders Preset (또는 동적 오브젝트의 경우 Physics > Rigid Body with Colliders Preset)을 선택하여 추가할 수 있습니다.

튜토리얼을 따라하려면 Isaac Sim 인스턴스를 실행하세요. 그런 다음 Window > Extensions를 열고 cuMotion Examples (isaacsim.robot_motion.cumotion.examples)를 검색하여 활성화하세요. 찾을 수 없는 경우 Extensions 검색바에서 @feature를 제거하고 다시 검색하세요. isaacsim.robot_motion.cumotion.examples Extension에는 CumotionWorldInterface를 사용하여 세계 상태를 관리하고, 장애물을 탐색하고, 세계 업데이트를 동기화하는 완전히 기능하는 예제가 있습니다.

Key Concepts#

CumotionWorldInterface는 Isaac Sim의 USD 씬과 cuMotion의 충돌 세계를 연결하는 브릿지입니다. Motion Generation API 클래스와 함께 작동하여 완전한 세계 상태 관리를 제공합니다:

  • WorldInterface 구현: CumotionWorldInterface는 Motion Generation API의 WorldInterface 인터페이스를 구현하여 WorldBinding 및 기타 Motion Generation API 컴포넌트와 함께 사용 가능합니다.

  • cuMotion 세계 관리: cuMotion의 내부 세계 표현에서 장애물을 관리하며, 충돌 쿼리를 위한 world_view를 제공합니다.

  • 디버그 시각화: 디버깅을 위한 충돌 지오메트리의 선택적 시각화

CumotionWorldInterface는 Motion Generation API 클래스와 함께 작동합니다: * SceneQuery는 USD 씬에서 오브젝트를 탐색합니다. * ObstacleStrategy는 장애물 지오메트리가 근사화되는 방식을 설정합니다. * WorldBinding은 세계 인터페이스를 사용하여 USD에서 cuMotion으로 세계 상태를 자동으로 동기화합니다.

Searching for Obstacles#

첫 번째 단계는 SceneQuery를 생성하고 USD 씬에서 오브젝트를 탐색하는 것입니다:

    # Create scene query to discover obstacles
    scene_query = SceneQuery()

    # Find all objects in a bounding box
    objects = scene_query.get_prims_in_aabb(
        search_box_origin=[0.0, 0.0, 0.0],
        search_box_minimum=[-100.0, -100.0, -100.0],
        search_box_maximum=[100.0, 100.0, 100.0],
        tracked_api=TrackableApi.PHYSICS_COLLISION,
    )

    print("Discovered objects:", objects)

SceneQuery는 지정된 축 정렬 경계 상자 내에서 지정된 충돌 API가 적용된 prim을 검색합니다.

Configuring Obstacle Representations#

충돌 검사를 위해 장애물 지오메트리를 근사화해야 합니다. ObstacleStrategy는 다양한 지오메트리 유형이 어떻게 표현되는지 관리합니다:

  • cuMotion의 경우, Cone이나 Cylinder 형상에 대한 네이티브 장애물 유형이 없으므로 두 가지 모두에 OBB 표현을 사용합니다. CumotionWorldInterface는 이를 네이티브 cumotion.CUBOID 유형으로 매핑합니다.

  • Mesh 오브젝트의 경우, 빠른 충돌 검사를 위해 OBB 표현을 사용하지만 더 정확한 충돌 검사를 위해 TRIANGULATED_MESH 표현을 사용할 수도 있습니다.

  • 안전 허용 오차는 계획 중 안전한 여유 거리를 확보하기 위해 장애물 주변에 추가 패딩을 추가합니다.

참고

ObstacleRepresentation은 StrEnum이므로, ObstacleConfiguration 오브젝트 생성 시 열거형 값 (예: ObstacleRepresentation.OBB) 또는 문자열 (예: "obb")을 사용할 수 있습니다. 장애물 표현 설정에 대한 자세한 내용은 Scene Interaction을 참조하세요.

    # Set up obstacle strategy
    obstacle_strategy = ObstacleStrategy()

    # Set default safety tolerance for all obstacles
    obstacle_strategy.set_default_safety_tolerance(0.06)

    # Configure specific geometry types
    obstacle_strategy.set_default_configuration(Mesh, ObstacleConfiguration("obb", 0.01))
    obstacle_strategy.set_default_configuration(Cone, ObstacleConfiguration("obb", 0.01))
    obstacle_strategy.set_default_configuration(Cylinder, ObstacleConfiguration("obb", 0.01))

Creating the World Interface and World Binding#

WorldBinding을 사용하여 CumotionWorldInterface를 USD 씬과 연결할 수 있습니다. 이 튜토리얼에서는 visualize_debug_prims=True 설정을 사용하여 다음을 수행합니다:

  • cumotion.World에 존재하는 장애물의 디버그 시각화 생성

  • 해당 오브젝트는 충돌 검사가 활성화된 경우 (금지 구역) 빨간색으로 표시됩니다.

  • 해당 오브젝트는 충돌 검사가 비활성화된 경우 (허용 구역) 초록색으로 표시됩니다.

    # Create world interface with optional debug visualizations
    world_interface = CumotionWorldInterface(visualize_debug_prims=True)

    # Create world binding
    world_binding = WorldBinding(
        world_interface=world_interface,
        obstacle_strategy=obstacle_strategy,
        tracked_prims=objects,
        tracked_collision_api=TrackableApi.PHYSICS_COLLISION,
    )

    # Initialize the world binding (populates obstacles into cuMotion)
    world_binding.initialize()

WorldBinding은 추적된 prim에서 충돌 지오메트리를 자동으로 추출하고, 장애물 전략에 따라 적절한 데이터를 CumotionWorldInterface에 전달하여 cuMotion 장애물 표현으로 변환합니다. CumotionWorldInterface는 WorldBinding 없이 (또는 부분적으로) 사용할 수도 있습니다. 예를 들어:

  • WorldBinding을 사용하여 씬을 초기 설정한 후, 변환이 USD 씬이 아닌 인식 알고리즘에서 올 때 CumotionWorldInterface.update_obstacle_transforms()를 수동으로 호출합니다.

  • 간단한 씬에는 Isaac Sim Core API 오브젝트 (Sphere, Cube)를 사용하여 CumotionWorldInterface를 직접 설정합니다.

Synchronizing the World Binding#

움직이는 장애물과 속성 변경을 추적하기 위해 매 프레임마다 세계 바인딩을 동기화해야 합니다. Scene Interaction 튜토리얼에서 다루는 세 가지 동기화 방법이 있습니다.

변환만 동기화 장애물의 변환 (위치 및 방향)만 업데이트합니다:

    # Update only transforms (fastest, for static obstacles with moving robot base)
    world_binding.synchronize_transforms()

속성만 동기화 장애물의 속성 (크기, 형상)만 업데이트합니다:

참고

cuMotion은 장애물의 속성 업데이트를 지원하지 않습니다. 따라서 형상 속성을 업데이트하는 WorldInterface의 모든 함수는 구현되지 않으며, 호출 시 NotImplementedError가 발생합니다. cuMotion에서 업데이트를 지원하는 유일한 속성은 충돌 활성화 상태입니다.

    # Update only properties (for obstacles that change size/shape or collision enabled state)
    # Note: cuMotion does not support updating the properties of obstacles.
    # The only property which supports updating in cuMotion is the collision enabled state.
    world_binding.synchronize_properties()

둘 다 동기화 변환과 속성 모두 업데이트합니다:

    # Update both transforms and properties (most complete, but not recommended for high-frequency updates)
    world_binding.synchronize()

일반적인 업데이트 루프에서는 매 프레임마다 WorldBinding.synchronize_transforms() 메서드를 호출하고, 속성이 변경된 것을 알 때만 또는 더 드물게 WorldBinding.synchronize_properties() 메서드를 호출합니다.

참고

위에서 설명한 동기화 방법은 추적 중인 충돌 오브젝트에만 사용됩니다. 로봇 베이스 위치는 자동으로 동기화되지 않으며, 로봇 베이스 포즈는 일반적으로 WorldInterface 인터페이스의 일부가 아닙니다. CumotionWorldInterface는 로봇 베이스 포즈를 업데이트하는 추가 함수를 제공하며, 이는 다음 섹션에서 다룹니다.

Updating Robot Base Transforms#

세계 인터페이스는 월드 좌표와 로봇 베이스 프레임 좌표 간 변환을 위해 로봇 베이스 변환을 알아야 합니다:

    # Update world interface with robot base transform
    world_binding.get_world_interface().update_world_to_robot_root_transforms(articulation.get_world_poses())

이 함수는 로봇 베이스가 이동할 때마다 호출해야 하며, cumotion.World의 모든 장애물 변환을 로봇 베이스 프레임에 상대적으로 업데이트합니다.

Exploring the Tutorial#

참고

이 튜토리얼을 대화형으로 실험하려면 isaacsim.robot_motion.cumotion.examples Extension의 isaacsim/robot_motion/cumotion/examples/world_interface/scenario.py에 있는 scenario.py 파일을 참조하세요.

이 튜토리얼은 CumotionWorldInterface를 실험하기 위한 대화형 환경을 제공합니다. 다음은 튜토리얼을 탐색하고 CumotionWorldInterface를 배우는 몇 가지 방법입니다:

기본 사용법: 씬에서 단일 장애물을 이동하고 세계 인터페이스가 이를 추적하는 방법을 확인하세요.

아래 예제 비디오에서 수행하는 작업:

  • 예제 실행

  • 큐브 장애물 이동 (Shift + 드래그)

  • 동기화가 synchronize_transforms로 설정되면 USD 오브젝트가 이동하고, 내부 cuMotion 장애물 변환이 내부적으로 업데이트되어 빨간색 OBB로 표시됩니다.

  • 동기화가 synchronize_properties로 설정되면 USD 오브젝트가 이동하지만, 내부 cuMotion 장애물 변환은 내부적으로 업데이트되지 않습니다.

  • 동기화가 synchronize로 설정되면 USD 오브젝트가 이동하고, 내부 cuMotion 장애물 변환이 내부적으로 업데이트됩니다.

장애물 추가

아래 예제 비디오에서 수행하는 작업:

  • 구체 생성

  • 구체에 콜라이더 Rigid Body with Colliders 프리셋 추가 (물리 충돌 및 동적 오브젝트로 설정)

  • 예제 재설정 (CumotionWorldInterface와 관리되는 cumotion.World 재생성)

  • 구체는 SceneQuery에 의해 자동으로 발견되고 WorldBinding과 CumotionWorldInterface를 통해 cumotion.World에 추가됩니다.

장애물 활성화 및 비활성화

아래 예제 비디오에서 수행하는 작업:

  • 캡슐 생성

  • 캡슐에 콜라이더 프리셋 추가 (정적 물리 충돌 오브젝트로 설정)

  • 예제 재설정 (CumotionWorldInterface와 관리되는 cumotion.World 재생성)

  • 캡슐은 SceneQuery에 의해 자동으로 발견되고 WorldBinding과 CumotionWorldInterface를 통해 cumotion.World에 추가됩니다.

  • 충돌을 활성화/비활성화하면 내부 cuMotion 장애물 변환이 활성화/비활성화 상태를 반영하도록 업데이트됩니다 (synchronize_transforms 동기화 방법을 사용하지 않는 경우).

장애물 표현 변경

더 배우려면 ObstacleStrategy의 일부 설정을 변경하고 CumotionWorldInterface와 cumotion.World가 어떻게 반응하는지 확인해 보세요. 예를 들어 Mesh 장애물을 추가하고 OBB 대신 TRIANGULATED_MESH로 표현하여 충돌 정확도 차이를 확인할 수 있습니다. 아래 두 비디오는 ObstacleStrategy의 기본 표현이 OBB와 TRIANGULATED_MESH로 설정되었을 때 메시가 추가되는 예시를 보여줍니다.

Summary#

이 튜토리얼에서 보여준 내용:

  1. World Interface 설정: CumotionWorldInterface 생성 (중앙화된 세계 상태 관리를 위해 WorldBinding 자주 사용)
