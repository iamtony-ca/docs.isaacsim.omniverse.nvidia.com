ROS 2 Installation (Other Platforms)#

NVIDIA Isaac Sim은 ROS 시스템 통합을 위한 ROS 2 브리지를 제공합니다. 시뮬레이터가 게시하고 수신하는 데이터 유형을 정의하는 데 동일한 공통 구성 요소 세트가 사용됩니다.

Isaac Sim 지원 ROS 배포판은 다음과 같습니다:

플랫폼

ROS 배포판

ROS 설치 참고사항

Ubuntu 24.04

Jazzy (권장)

ROS 2 Installation (Default) 참조

Ubuntu 22.04

Humble

기본 설치(Python 3.10) 사용. Isaac Sim에서 사용자 정의 ROS 인터페이스를 사용하려면 Python 3.12 빌드의 ROS 2 작업 공간을 사용하세요.

Ubuntu 22.04

Jazzy

소스에서 빌드(Python 3.10). Isaac Sim에서 사용자 정의 ROS 인터페이스를 사용하려면 Python 3.12 빌드의 ROS 2 작업 공간을 사용하세요.

Windows 11

Humble

WSL 설치(더 이상 사용되지 않음). 사용자 정의 ROS 인터페이스는 지원되지 않습니다.

Windows 11

Jazzy

Pixi: Pixi 기반 네이티브 ROS 설치. Isaac Sim은 Pixi ROS 설치를 직접 소스할 수 있습니다. WSL(더 이상 사용되지 않음): 사용자 정의 ROS 인터페이스는 지원되지 않습니다.

ROS 2 브리지의 경우 Isaac Sim은 ROS 2 Humble 및 ROS 2 Jazzy와 호환됩니다.

Attention

실험적: 네이티브 ROS 2 배포판 지원 – Isaac Sim은 이제 실험적으로 플랫폼(Ubuntu 22.04 또는 Ubuntu 24.04)에 네이티브로 설치된 모든 ROS 2 배포판 로드를 지원합니다. 이를 사용하려면 로컬에 설치된 ROS 2 배포판을 소스하고 동일한 터미널에서 Isaac Sim을 실행합니다. Humble과 Jazzy가 공식적으로 테스트되고 권장되는 배포판으로 남아 있지만, 기타 네이티브로 지원되는 배포판도 이 워크플로에서 작동할 수 있습니다.

Ubuntu 24.04의 ROS 2 Jazzy가 권장됩니다. 해당 설치 방식이라면 ROS 2 Installation (Default)를 참조하세요. 그렇지 않으면 구성을 확인하거나 선택하여 계속하세요:

Install ROS 2#

  1. 공식 웹사이트의 지침에 따라 ROS 2를 다운로드합니다:

    • ROS 2 Humble Ubuntu 22.04

  2. (선택 사항) ROS 2 브리지에서 일부 메시지 유형(바운딩 박스 게시에 사용되는 Detection2DArray 및 Detection3DArray)은 vision_msgs_package에 의존합니다. 아래 명령을 실행하여 시스템에 패키지를 설치합니다. 소스에서 ROS 2를 빌드한 경우 패키지를 클론하고 다시 빌드하기 전에 ROS 2 설치 작업 공간에 포함시킵니다. vision_msgs 게시자를 실행할 필요가 없다면 이 단계를 건너뛸 수 있습니다.

    sudo apt install ros-humble-vision-msgs

  3. (선택 사항) ROS 2 브리지에서 일부 메시지 유형(AckermannDriveStamped, Ackermann 조향 명령 게시 및 구독에 사용됨)은 ackermann_msgs_package에 의존합니다. 아래 명령을 실행하여 시스템에 패키지를 설치합니다. 소스에서 ROS 2를 빌드한 경우 패키지를 클론하고 다시 빌드하기 전에 ROS 2 설치 작업 공간에 포함시킵니다. ackermann_msgs 게시자나 구독자를 실행할 필요가 없다면 이 단계를 건너뛸 수 있습니다.

    sudo apt install ros-humble-ackermann-msgs

  4. ROS 환경이 터미널 또는 ~/.bashrc 파일에 소스되어 있는지 확인합니다. ROS 명령을 사용하기 전 매번 이 단계를 수행해야 합니다:

    source /opt/ros/humble/setup.bash

Note

Linux에서는 Isaac Sim을 실행하는 동일한 터미널에서 이 설치를 소스할 수 없습니다. Isaac Sim을 실행하기 전에 Isaac Sim 내부 ROS 라이브러리인 Python 3.12로 소스하세요.

  1. 공식 웹사이트의 지침에 따라 소스에서 ROS 2 Jazzy를 다운로드하고 빌드합니다:

    • ROS 2 Jazzy Ubuntu 22.04

  2. (선택 사항) ROS 2 브리지에서 일부 메시지 유형(Detection2DArray 및 Detection3DArray, 바운딩 박스 게시에 사용됨)은 vision_msgs_package에 의존합니다. 링크된 리포지토리를 클론하고 ROS 작업 공간에서 빌드합니다. vision_msgs 게시자를 실행할 필요가 없다면 이 단계를 건너뛸 수 있습니다.

  3. (선택 사항) 링크된 리포지토리를 클론하고 ROS 작업 공간에서 빌드합니다. 소스에서 ROS 2를 빌드한 경우 패키지를 클론하고 다시 빌드하기 전에 ROS 2 설치 작업 공간에 포함시킵니다. ackermann_msgs 게시자나 구독자를 실행할 필요가 없다면 이 단계를 건너뛸 수 있습니다. ROS 2 브리지에서 일부 메시지 유형(AckermannDriveStamped, Ackermann 조향 명령 게시 및 구독에 사용됨)은 ackermann_msgs_package에 의존합니다.

  4. ROS 환경이 터미널 또는 ~/.bashrc 파일에 소스되어 있는지 확인합니다. ROS 명령을 사용하기 전 매번 이 단계를 수행해야 합니다:

    . ~/ros2_jazzy/install/local_setup.bash

Note

Linux에서는 Isaac Sim을 실행하는 동일한 터미널에서 이 설치를 소스할 수 없습니다. Isaac Sim을 실행하기 전에 Isaac Sim 내부 ROS 라이브러리인 Python 3.12로 소스하세요.

Note

Pixi 기반 네이티브 ROS 2 설치는 현재 ROS 2 Jazzy만 지원합니다. Jazzy를 ROS 2 배포판으로 선택하거나, Humble을 사용하려면 WSL2 설치 방법으로 전환하세요.

WSL2를 사용하여 Windows에서 ROS 2를 실행하고, 내부 ROS 2 라이브러리를 사용하여 실행되는 Isaac Sim ROS 브리지와 통신합니다.

  1. Windows 머신에 WSL2를 설치합니다.

  2. 관리자 권한으로 Powershell을 열고 WSL 버전을 2로 변경합니다.

    wsl --set-default-version 2

  3. WSL 내에 Ubuntu 22.04 배포판을 설치합니다.

    wsl --install -d Ubuntu-22.04

  4. 설치가 완료되면 머신을 재시작하고 Windows에서 Ubuntu 22.04 앱을 엽니다. 설치하는 데 몇 분이 걸립니다.

    Note

    가상화 활성화 오류가 발생하면 Windows 가상화 활성화 지침을 따르세요.

  5. WSL2에 Ubuntu 22.04가 설치된 후 공식 웹사이트의 지침에 따라 ROS 2를 다운로드합니다:

    • ROS 2 Humble Ubuntu 22.04

  6. (선택 사항) ROS 2 브리지에서 일부 메시지 유형(Detection2DArray 및 Detection3DArray, 바운딩 박스 게시에 사용됨)은 vision_msgs_package에 의존합니다. 아래 명령을 실행하여 시스템에 패키지를 설치합니다. 소스에서 ROS 2를 빌드한 경우 패키지를 클론하고 다시 빌드하기 전에 ROS 2 설치 작업 공간에 포함시킵니다. vision_msgs 게시자를 실행할 필요가 없다면 이 단계를 건너뛸 수 있습니다.

    sudo apt install ros-humble-vision-msgs

  7. (선택 사항) ROS 2 브리지에서 일부 메시지 유형(AckermannDriveStamped, Ackermann 조향 명령 게시 및 구독에 사용됨)은 ackermann_msgs_package에 의존합니다. 아래 명령을 실행하여 시스템에 패키지를 설치합니다. 소스에서 ROS 2를 빌드한 경우 패키지를 클론하고 다시 빌드하기 전에 ROS 2 설치 작업 공간에 포함시킵니다. ackermann_msgs 게시자나 구독자를 실행할 필요가 없다면 이 단계를 건너뛸 수 있습니다.

    sudo apt install ros-humble-ackermann-msgs

Setup ROS 2 Workspaces#

ROS 2 튜토리얼과 예제를 실행하려면 작업할 터미널에서 ROS 2 설치 작업 공간을 소스해야 합니다.

  1. Isaac Sim ROS 작업 공간을 빌드하려면 ROS 2 설치를 완료했는지 확인하세요.

    Important

    Docker 컨테이너에서 ROS 실행에서 설명된 대로 ROS Docker 컨테이너를 사용하여 작업 공간을 빌드할 수도 있습니다. Docker 컨테이너를 설정한 후 이 단계로 돌아오세요.

  2. https://github.com/isaac-sim/IsaacSim-ros_workspaces에서 Isaac Sim ROS Workspace Repository를 클론합니다.

    Isaac Sim ROS 2 튜토리얼 시리즈를 진행하려면 몇 가지 ROS 패키지가 필요합니다. 필요한 패키지를 포함한 전체 ROS 2 작업 공간이 포함되어 있습니다.

  3. 소스에서 ROS 2를 빌드한 경우 추가 작업 공간을 빌드하기 전에 source /opt/ros/<ros_distro>/setup.bash 명령을 source <path_ros2_ws>/install/setup.bash로 교체합니다.

  4. ROS 2 작업 공간을 빌드하려면 추가 패키지를 설치해야 할 수 있습니다:

    # For rosdep install command
sudo apt install python3-rosdep build-essential
# For colcon build command
sudo apt install python3-colcon-common-extensions

  5. 네이티브 ROS 2가 소스되어 있는지 확인합니다:

    source /opt/ros/humble/setup.bash

  6. ROS 2 작업 공간의 루트에서 다음 명령을 실행하여 패키지 의존성을 해결합니다:

    cd humble_ws
git submodule update --init --recursive # If using docker, perform this step outside the container and relaunch the container
rosdep install -i --from-path src --rosdistro humble -y

  7. 작업 공간을 빌드합니다:

    colcon build

    루트 디렉터리 아래에 새 build, install, log 디렉터리가 생성됩니다.

  8. 이 작업 공간에서 빌드된 ROS 2 패키지를 사용하려면 새 터미널을 열고 다음 명령으로 작업 공간을 소스합니다:

    source /opt/ros/humble/setup.bash
cd humble_ws
source install/local_setup.bash

cd jazzy_ws
source install/local_setup.bash

    Note

    첫 번째 성공적인 빌드 후 Pixi는 pixi shell에 들어가거나 pixi run을 호출할 때마다 install\\setup.bat을 자동으로 소스합니다. 빌드된 패키지는 수동 소싱 없이 사용할 수 있습니다.

Pixi 작업 공간으로 Isaac Sim 실행

Note

다음을 각각 별도의 Command Prompt에서 순서대로 시작합니다:

터미널 1 — Zenoh 라우터 시작

ROS 2 노드가 시작하기 전에 Zenoh 라우터가 실행 중이어야 합니다:

cd C:\\IsaacSim-ros_workspaces\\jazzy_ws
pixi run zenoh

터미널 2 — Isaac Sim 시작

작업 공간은 ROS 2 브리지가 이미 활성화된 상태로 Isaac Sim을 실행하는 sim 작업을 노출합니다:

cd C:\\IsaacSim-ros_workspaces\\jazzy_ws
pixi run sim

Note

sim 작업은 pixi.toml의 [target.win.activation] 아래에 정의된 isaac_sim_package_path 환경 변수를 사용합니다(기본값: C:\\isaacsim). Isaac Sim이 다른 위치에 설치된 경우 해당 값을 편집하세요.

Note

Pixi 워크플로에서는 Pixi Python 환경으로 독립 실행형 스크립트를 실행하세요. 예를 들어 pixi run python <path\\to\\script.py> 또는 pixi shell 내에서 python <path\\to\\script.py>. Pixi가 PyPI에서 활성 환경에 Isaac Sim을 설치하므로 번들된 python.bat 또는 python.sh 런처를 사용하지 마세요.

Isaac Sim 내에서 ROS 2 브리지를 확인하려면:

  • Tools > Robotics > ROS 2 OmniGraphs > Clock > OK를 클릭합니다.

  • Play 버튼을 누릅니다.

터미널 3 — ROS 2 명령 실행

pixi run으로 ros2 명령을 실행하거나(pixi shell에 한 번 들어가서 직접 ros2를 호출):

cd C:\\IsaacSim-ros_workspaces\\jazzy_ws

/parameter_events
/rosout

의존성 추가

package.xml 파일에 새 의존성을 추가할 때 다음 pixi install에서 새 패키지가 포함되도록 pixi.toml을 재생성합니다:

pixi ros init --distro jazzy

conda 패키지를 직접 추가하려면:

pixi add <package-name>

PyPI 패키지를 추가하려면:

pixi add --pypi <package-name>

pixi ros init을 실행한 후 pixi.toml에 # NOT FOUND로 표시된 항목이 있는 경우 기본 채널에서 패키지를 해결할 수 없습니다. 옵션:

  • 사용자 정의 채널로 init 재실행: pixi ros init --distro jazzy --channel https://prefix.dev/my-channel.

  • 수동으로 채널 추가: pixi project channel add <channel-url>.

  • package.xml의 패키지 이름을 확인합니다.

  • PyPI에서 대신 설치: pixi add --pypi <package-name>.

  • 기본 ROS 채널에서 누락된 경우 ros-jazzy on RoboStack에 패키지를 기여합니다.

일반적인 문제

  • 빌드 실패 — pixi install로 모든 의존성이 깔끔하게 설치되었는지 확인한 다음 정리하고 다시 빌드합니다:

    pixi run clean
pixi run build

  • ``ros2`` 명령을 찾을 수 없음 — ROS 환경이 활성화되어 있어야 합니다. 명령에 pixi run을 접두사로 붙이거나(예: pixi run ros2 topic list) 먼저 pixi shell에 들어가세요.

  • 활성 ROS 환경 감지됨 — pixi ros init이 활성 ROS 설치에 대해 경고하면 머신의 다른 곳에서 ROS가 소스되어 있는 것입니다. 시작 스크립트에서 수동 source 호출이나 환경 재정의를 제거하고 터미널을 다시 시작하세요. Pixi는 ROS 환경을 자동으로 관리합니다 — 수동 소싱이 필요하지 않습니다.

Configuring Options and Enabling Internal ROS Libraries#

이미 ROS 2의 Python 3.12 빌드와 ROS 2 작업 공간의 Python 3.12 빌드를 소스하고 있으므로 Isaac Sim과 함께 제공되는 내부 ROS 2 라이브러리를 활성화할 필요가 없습니다.

다음 구성에 해당하는 경우 Isaac Sim과 함께 제공되는 내부 ROS 라이브러리로 Isaac Sim을 실행해야 합니다:

  • ROS Docker 컨테이너를 사용해야 하는 경우

  • 로컬에 ROS 2 작업 공간이 빌드되어 있지만 기본 또는 일반 ROS 인터페이스(예: std_msgs, geometry_msgs, nav_msgs)만 사용할 계획인 경우

Ubuntu 22.04에서 Isaac Sim 대화형 GUI는 다른 ROS 라이브러리가 소스되지 않은 경우 자동으로 내부 ROS 2 Humble 라이브러리를 로드합니다. ROS 2 브리지가 활성화된 상태로 Isaac Sim을 실행하려면 일반 실행 명령을 사용합니다.

./isaac-sim.sh

Note

ROS_DISTRO 환경 변수는 ROS가 소스되었는지 확인하는 데 사용됩니다.

독립 실행형 스크립트 실행

./python.sh를 사용하여 독립 실행형 Isaac Sim 스크립트를 실행하는 경우 내부 libs를 수동으로 활성화해야 합니다.

특정 내부 ROS 2 라이브러리를 직접 설정하려면 Isaac Sim을 실행하기 전에 새 터미널이나 명령 프롬프트에서 다음 환경 변수를 설정해야 합니다. Isaac Sim이 기본이 아닌 위치에 설치된 경우 isaac_sim_package_path 환경 변수를 Isaac Sim 설치 루트 폴더 경로로 교체합니다.

  • 독립 실행형 스크립트 실행:

export RMW_IMPLEMENTATION=rmw_fastrtps_cpp

# Setting this command multiple times will append the internal library path again potentially leading to conflicts
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:$isaac_sim_package_path/exts/isaacsim.ros2.core/humble/lib

$isaac_sim_package_path/python.sh <path/to/standalone/script>

# Only set this once per session to avoid path conflicts
$env:PATH = "$env:PATH;$env:isaac_sim_package_path\exts\isaacsim.ros2.core\jazzy\lib"

# Run Run Isaac Sim Standalone scripts
& "$env:isaac_sim_package_path\python.bat" <path/to/standalone/script>

Pixi 작업 공간에서는 ROS 2 Jazzy와 Zenoh 미들웨어가 사전 구성되어 있습니다. 내부 ROS 2 라이브러리 경로를 수동으로 설정할 필요가 없습니다.

Pixi Python 환경으로 독립 실행형 스크립트 실행:

pixi run python <path\\to\\standalone\\script.py>

Enabling the ROS 2 Bridge#

Fast DDS를 사용한 ROS 2 브리지 활성화 지침이 ROS 2 브리지를 활성화하는 권장 방법입니다.

다음도 활성화할 수 있습니다:

  • Cyclone DDS를 사용한 ROS 2 브리지 활성화.

  • Zenoh를 사용한 ROS 2 브리지 활성화(ROS 2 Jazzy).

Enabling the ROS 2 Bridge using Fast DDS#

동일한 머신에서 실행 중인 ROS 2 노드와 통신하기 위해 ROS 2 브리지를 사용하는 경우 FastDDS의 기본 구성을 사용합니다. 이렇게 하면 공유 메모리 전송을 사용하여 최상의 시뮬레이션 성능을 보장합니다.

같은 네트워크의 다른 머신에 있는 ROS 노드에 연결하기 위해 ROS 2 브리지를 사용하려는 경우 Isaac Sim을 실행하기 전에 ROS 2 메시지를 전달할 모든 터미널에서 Fast DDS 미들웨어를 설정하고 UDP 전송을 활성화해야 합니다:

  1. fastdds.xml이 있고 환경 변수가 설정되어 있는지 확인합니다:

    • ROS 2 작업 공간 설정을 따랐다면 <ros2_ws> 폴더 루트에 fastdds.xml 파일이 있습니다. ROS 2 함수를 사용하는 모든 터미널에 export FASTRTPS_DEFAULT_PROFILES_FILE=<path_to_ros2_ws>/fastdds.xml을 입력하여 환경 변수를 설정합니다.

    • ROS 2 작업 공간 설정을 따르지 않았다면 ~/.ros/ 아래에 fastdds.xml이라는 파일을 만들고, 다음 스니펫 링크를 파일에 붙여넣습니다:

<license>Copyright (c) 2022-2026, NVIDIA CORPORATION.  All rights reserved.

and proprietary rights in and to this software, related documentation
and any modifications thereto.  Any use, reproduction, disclosure or
distribution of this software and related documentation without an express
license agreement from NVIDIA CORPORATION is strictly prohibited.</license>


<profiles xmlns="http://www.eprosima.com/XMLSchemas/fastRTPS_Profiles" >
   <transport_descriptors>
         <transport_descriptor>
            <transport_id>UdpTransport</transport_id>
            <type>UDPv4</type>

  2. ROS 2 함수를 사용하는 터미널에서 export FASTRTPS_DEFAULT_PROFILES_FILE=~/.ros/fastdds.xml을 실행합니다.

  3. (선택 사항) Isaac Sim을 실행하기 전에 export ROS_DOMAIN_ID=(id_number)를 실행합니다. 나중에 환경 내에서 이 ROS_DOMAIN_ID를 사용할지, 아니면 특정 토픽에 다른 ID 번호를 명시적으로 사용할지 결정할 수 있습니다.

  4. Isaac Sim을 실행하기 전에 ROS 2 설치 또는 내부 ROS 2 라이브러리와 작업 공간을 소스합니다.

WSL2의 ROS 노드에 연결하기 위해 ROS 2 브리지를 사용하려면 ROS 2 메시지를 전달할 모든 터미널에서 Fast DDS 미들웨어를 설정하고 UDP 전송을 활성화해야 합니다:

  1. ROS 2 작업 공간 설정을 따르지 않았다면 C:\.ros\ 아래에 fastdds.xml이라는 파일을 만들고, 다음 스니펫 링크를 파일에 붙여넣습니다:

<profiles xmlns="http://www.eprosima.com/XMLSchemas/fastRTPS_Profiles" >
   <transport_descriptors>
         <transport_descriptor>
            <transport_id>UdpTransport</transport_id>
            <type>UDPv4</type>
         </transport_descriptor>

  2. ROS 2 함수를 사용하는 터미널에서 set FASTRTPS_DEFAULT_PROFILES_FILE=C:\.ros\fastdds.xml을 실행합니다.

  3. (선택 사항) Isaac Sim을 실행하기 전에 set ROS_DOMAIN_ID=(id_number)를 실행합니다. 나중에 환경 내에서 이 ROS_DOMAIN_ID를 사용할지, 아니면 특정 토픽에 다른 ID 번호를 명시적으로 사용할지 결정할 수 있습니다.

  4. Isaac Sim을 실행하기 전에 동일한 터미널에서 내부 ROS 2 라이브러리가 소스되어 있는지 확인합니다.

Note

Fast DDS는 Pixi 기반 설치에는 적용되지 않습니다. Pixi는 지원되는 Zenoh 미들웨어를 사용합니다. Zenoh를 사용한 ROS 2 브리지 활성화(ROS 2 Jazzy)를 참조하세요.

Enabling the ROS 2 Bridge using Cyclone DDS#

Isaac Sim은 ROS 2 Humble 및 Jazzy와 함께 Linux에서 Cyclone DDS 미들웨어를 지원합니다. RMW_IMPLEMENTATION이 설정되지 않은 경우 ROS 2 브리지는 Fast DDS를 사용합니다. Cyclone DDS로 전환하려면 Isaac Sim을 중지하고, 터미널에서 export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp를 실행한 다음 다시 시작합니다.

Note

Isaac Sim은 Linux에서만 Cyclone DDS 미들웨어를 지원합니다. Windows WSL2는 현재 지원되지 않습니다.

Note

Cyclone DDS는 Pixi 기반 설치에는 적용되지 않습니다. Pixi는 지원되는 Zenoh 미들웨어를 사용합니다. Zenoh를 사용한 ROS 2 브리지 활성화(ROS 2 Jazzy)를 참조하세요.

Enabling the ROS Bridge using Cyclone DDS (Linux Only)#

Note

Windows WSL2는 현재 지원되지 않습니다.

Note

Cyclone DDS는 Pixi 기반 설치에는 적용되지 않습니다. Pixi는 Zenoh 미들웨어를 사용합니다. Zenoh를 사용한 ROS 2 브리지 활성화(ROS 2 Jazzy)를 참조하세요.

  1. ROS 2 Humble 및 Jazzy 설치를 위한 Cyclone DDS를 설정하려면 ROS 2 Humble 설치 단계 또는 ROS 2 Jazzy 설치 단계를 따릅니다.

    Note

    Isaac Sim ROS 2 Humble 및 Jazzy 내부 라이브러리에는 Python 3.12로 컴파일된 Cyclone DDS가 포함되어 있습니다.

  2. Isaac Sim을 실행하기 전에 RMW_IMPLEMENTATION 환경 변수를 설정했는지 확인하세요. 앞으로 예제에서 환경 변수를 rmw_fastrtps_cpp로 설정하는 것을 보여주면 아래 명령으로 교체할 수 있습니다:

    export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp

Enabling the ROS 2 Bridge using Zenoh (ROS 2 Jazzy)#

Note

WSL2를 통한 Zenoh 미들웨어 지원은 현재 Windows에서 테스트되지 않았습니다.

Pixi 작업 공간은 기본 미들웨어로 Zenoh(rmw_zenoh_cpp)를 사용합니다. Zenoh는 작업 공간 설정의 일부로 사전 구성되어 있습니다. 실행 지침은 위의 Isaac Sim ROS Workspaces 섹션을 참조하세요.

Note

Ubuntu 22.04의 ROS 2 Humble에서 Zenoh 미들웨어 지원은 Python 3.12 컴파일 요구 사항으로 인해 테스트되지 않았습니다. ROS 2 Humble의 경우 Fast DDS 또는 Cyclone DDS를 사용하세요.

Note

Ubuntu 22.04의 ROS 2 Jazzy에서 Zenoh 미들웨어 지원은 현재 테스트되지 않았습니다.

Isaac Sim은 Linux 및 ROS 2 Jazzy에서 Zenoh 미들웨어를 지원합니다. Zenoh는 이기종 시스템 간의 효율적인 데이터 배포를 위해 설계된 오픈 소스 통신 프로토콜로, 전통적인 DDS 구현의 대안을 제공합니다.

Note

Isaac Sim에는 내부 Zenoh 라이브러리가 제공되지 않습니다. Isaac Sim을 실행하기 전에 Python 3.12로 Zenoh를 빌드하고 소스해야 합니다.

Zenoh에 대한 자세한 내용은 ROS 2 Zenoh 문서를 검토하세요.

Zenoh 설치 및 실행

  1. https://github.com/isaac-sim/IsaacSim-ros_workspaces에서 Isaac Sim ROS Workspace Repository를 이미 클론했는지 확인합니다. 그렇지 않은 경우 ROS 2 작업 공간 설정의 단계를 따르세요.

  2. rmw_zenoh 리포지토리를 jazzy_ws/src 디렉터리에 클론합니다:

    cd IsaacSim-ros_workspaces/jazzy_ws/src
git clone https://github.com/ros2/rmw_zenoh.git -b jazzy

  3. Dockerfile을 사용하여 Python 3.12로 ROS 2와 작업 공간(Zenoh 포함)을 빌드합니다:

    cd IsaacSim-ros_workspaces

./build_ros.sh -d jazzy -v 22.04

    Zenoh를 포함한 빌드된 작업 공간은 build_ws/jazzy/jazzy_ws 아래에 있습니다.

  4. 별도의 터미널에서 Zenoh 라우터를 시작합니다. 라우터는 ROS 2 노드가 서로 검색하기 전에 실행 중이어야 합니다.

    Docker 컨테이너에서 ROS 실행 섹션의 Docker 컨테이너를 사용합니다. 아직 ros_ws_docker 컨테이너를 시작하지 않았다면 먼저 해당 단계를 따르세요. 그런 다음 컨테이너 내에서 ros-jazzy-rmw-zenoh-cpp를 설치하고 라우터를 시작합니다:

    # Inside the ros_ws_docker container
apt-get update && apt-get install -y ros-jazzy-rmw-zenoh-cpp
source /opt/ros/jazzy/setup.bash
export RMW_IMPLEMENTATION=rmw_zenoh_cpp
ros2 run rmw_zenoh_cpp rmw_zenohd

    Note

    Zenoh 라우터 없이는 노드의 세션 구성에서 기본적으로 멀티캐스트 검색이 비활성화되어 있기 때문에 노드가 서로를 검색할 수 없습니다. 대신 노드는 Zenoh 라우터의 가십 기능을 통해 다른 피어에 대한 검색 정보를 받습니다.

  5. Isaac Sim을 실행하기 전에 Isaac Sim을 실행할 터미널에서 RMW_IMPLEMENTATION 환경 변수를 설정합니다. 앞으로 예제에서 환경 변수를 rmw_fastrtps_cpp로 설정하는 것을 보여주면 아래 명령으로 교체할 수 있습니다:

    export RMW_IMPLEMENTATION=rmw_zenoh_cpp

  6. jazzy_ws와 isaac_sim_ros_ws의 Python 3.12 빌드를 모두 소스한 다음 Isaac Sim 실행:

    cd IsaacSim-ros_workspaces
source build_ws/jazzy/jazzy_ws/install/local_setup.bash
source build_ws/jazzy/isaac_sim_ros_ws/install/local_setup.bash
cd ~/isaacsim
./isaac-sim.sh

  7. Isaac Sim과 통신해야 하는 ROS 2 노드를 실행하는 추가 터미널에서 동일한 Docker 컨테이너에 새 터미널을 엽니다:

    docker exec -it ros_ws_docker /bin/bash -c \
  "source /opt/ros/jazzy/setup.bash && \
   export RMW_IMPLEMENTATION=rmw_zenoh_cpp && \
   cd /jazzy_ws && source install/local_setup.bash && \
   bash"

Disabling the ROS Bridge in isaac-sim.sh#

Note

Windows에서는 ROS 브리지가 기본적으로 비활성화되어 있습니다.

ROS 브리지를 비활성화하려면 다음 단계를 따르세요:

  1. ~/isaacsim/apps/isaacsim.exp.full.kit에 있는 파일을 엽니다.

  2. isaac.startup.ros_bridge_extension = "isaacsim.ros2.bridge" 줄을 찾습니다.

  3. ROS 2 브리지를 비활성화하려면 isaac.startup.ros_bridge_extension = ""으로 변경합니다.

  4. 파일을 저장하고 닫습니다.

Running ROS in Docker Containers#

Note

Docker 워크플로는 Windows WSL2에서 지원되지 않습니다.

Note

Docker 워크플로는 Pixi 기반 설치에는 필요하지 않습니다. Pixi 작업 공간은 모든 의존성을 네이티브로 처리합니다.

  1. Isaac Sim ROS Workspace Repository를 이미 클론했는지 확인합니다.

  2. 클론된 리포지토리의 루트로 이동하여 다음 명령을 실행합니다. 리포지토리가 다른 위치에 클론된 경우 ~/IsaacSim-ros_workspaces의 경로를 올바른 경로로 업데이트하세요:

    cd ~/IsaacSim-ros_workspaces
git submodule update --init --recursive

  3. 적절한 ROS 2 Docker 컨테이너를 실행하고 Isaac Sim ROS Workspaces 리포지토리에서 적절한 작업 공간을 마운트합니다. 리포지토리가 다른 위치에 클론된 경우 -v ~/IsaacSim-ros_workspaces의 경로를 올바른 경로로 업데이트하세요.

    xhost +

docker run -it --net=host --env="DISPLAY" --env="ROS_DOMAIN_ID" -v ~/IsaacSim-ros_workspaces/humble_ws:/humble_ws --name ros_ws_docker osrf/ros:humble-desktop /bin/bash

    xhost +

docker run -it --net=host --env="DISPLAY" --env="ROS_DOMAIN_ID" -v ~/IsaacSim-ros_workspaces/jazzy_ws:/jazzy_ws --name ros_ws_docker osrf/ros:jazzy-desktop /bin/bash

    --net=host는 Isaac Sim과 ROS Docker 컨테이너 간의 통신을 허용하고, xhost +와 --env="DISPLAY"는 Docker 컨테이너에서 rviz와 같은 GUI 애플리케이션을 열 수 있게 하는 DISPLAY 환경 변수를 통과시킵니다. --name <container name>는 고정된 이름으로 컨테이너를 참조할 수 있게 합니다.

  4. Docker 컨테이너 내에서 ros 작업 공간으로 이동합니다.

    cd /${ROS_DISTRO}_ws

  5. Docker 컨테이너 내에서 Fast DDS를 사용한 ROS 2 브리지 활성화의 지침에 따라 FASTRTPS_DEFAULT_PROFILES_FILE 환경 변수를 설정합니다.

  6. 추가 의존성 설치, 작업 공간 빌드 및 빌드 후 작업 공간 소스하기:

    cd /${ROS_DISTRO}_ws
apt-get update
git submodule update --init --recursive # If using Docker, perform this step outside the container and relaunch the container
rosdep install --from-paths src --ignore-src --rosdistro=$ROS_DISTRO -y
source /opt/ros/$ROS_DISTRO/setup.sh
colcon build
source install/local_setup.bash

  7. 새 터미널을 열어야 하는 경우 기존 Docker를 엽니다:

    docker exec -it ros_ws_docker /bin/bash -c 'source /opt/ros/$ROS_DISTRO/setup.bash; exec bash'

  8. (선택 사항) 설치를 테스트하기 위해 OmniGraph 노드 Isaac Sim OmniGraph Tutorial를 사용하여 Isaac Sim 내에 기본 클락 게시자를 설정할 수 있습니다:

    1. 시뮬레이터에서 play를 누릅니다.

    2. 별도의 터미널을 열고, Docker를 열고, FASTRTPS_DEFAULT_PROFILES_FILE 환경 변수를 설정합니다.

    3. ROS 2를 소스합니다.

    4. ros2 topic echo /clock이 Isaac Sim에서 오는 타임스탬프를 출력하는지 확인합니다.

  9. (선택 사항) 설치된 모든 의존성과 빌드된 작업 공간이 포함된 컨테이너를 새 Docker 이미지로 저장하여 나중에 사용하려면:

    1. 호스트에서 새 터미널을 열고 컨테이너를 이미지로 커밋합니다:

      docker commit ros_ws_docker isaac_sim_ros_ws:latest

    2. 같은 이름으로 새 컨테이너를 시작하기 전에 이전 컨테이너를 제거합니다:

      docker rm ros_ws_docker

    3. 향후 세션에서 저장된 이미지를 재사용하려면 새 컨테이너를 시작합니다:

      xhost +

docker run -it --net=host --env="DISPLAY" --env="ROS_DOMAIN_ID" -v ~/IsaacSim-ros_workspaces/humble_ws:/humble_ws --name ros_ws_docker isaac_sim_ros_ws:latest /bin/bash

      xhost +

docker run -it --net=host --env="DISPLAY" --env="ROS_DOMAIN_ID" -v ~/IsaacSim-ros_workspaces/jazzy_ws:/jazzy_ws --name ros_ws_docker isaac_sim_ros_ws:latest /bin/bash

    4. 컨테이너 내에서 작업 공간을 소스하면 다시 빌드하지 않고 사용할 준비가 됩니다:

      source /opt/ros/$ROS_DISTRO/setup.bash
cd /${ROS_DISTRO}_ws
source install/local_setup.bash
