Chat IRO: Natural Language Interface for Isaac Sim Replicator Object#

비전-언어 및 씬 생성 워크플로에서는 사용자가 Isaacsim.replicator.object(IRO)용 YAML 구성 파일을 직접 작성해야 하는 경우가 많습니다. 특히 복잡한 레이아웃, 하모나이저, 물리 설정, 카메라 리그의 경우 오류가 발생하기 쉽고 시간이 오래 걸릴 수 있습니다.

Chat IRO는 일반 영어 설명을 실행 가능한 IRO YAML 구성으로 변환하여 Isaac Sim 내에서 직접 실행하는 자연어 인터페이스입니다. IRO 확장 프로그램 위에 구축되어 구성 작성, 검증, 실행을 자동화합니다.

Chat IRO에는 다음과 같은 기능이 있습니다:

  • 영어 설명을 IRO YAML 씬으로 변환합니다.

  • 수천 개의 프로덕션 YAML 예시가 있는 RAG(Retrieval-Augmented Generation) 시스템을 사용하여 정확도를 높이고 모범 사례를 재사용합니다.

  • 생성된 YAML의 구문 및 일반적인 구조적 문제를 실행 전에 검증합니다.

  • Isaac Sim 뷰포트에서 생성된 씬을 즉시 미리 봅니다.

  • 반복적인 워크플로를 위해 구성 파일을 저장하고 다시 로드합니다.

Workflow#

Chat IRO는 다음 워크플로를 사용하여 씬을 생성합니다:

  1. Create a scene with 10 random size and color cubes와 같은 자연어 요청을 Chat IRO 창에 입력합니다.

  2. 확장 프로그램은 선택적으로 기존 IRO YAML 파일의 RAG 인덱스를 쿼리하고 관련 예시를 LLM 컨텍스트에 삽입합니다.

  3. LLM은 isaacsim.replicator.object에 대한 후보 YAML 구성을 생성합니다.

  4. Chat IRO가 YAML을 검증하고, 일반적인 문제를 수정하고, IRO를 통해 실행하여 씬을 생성하거나 업데이트합니다.

  5. 결과로 나온 합성 씬이 뷰포트에 렌더링됩니다. 후속 프롬프트를 보내 구성을 반복적으로 개선할 수 있습니다.

Prerequisites#

Chat IRO를 사용하기 전에 다음 요구 사항이 충족되어 있는지 확인합니다:

  • isaacsim.replicator.object.ui 확장 프로그램이 활성화되어 있습니다.

  • 지원되는 운영 체제(Linux이 기본 플랫폼; Windows는 실험적).

  • CUDA를 지원하는 NVIDIA GPU(권장).

  • 최소 8GB RAM(대형 씬의 경우 16GB 이상 권장).

  • omni.ai.langchain.agent.chat_iro 확장 프로그램이 활성화되어 있습니다.

  • LLM 액세스를 위한 유효한 NVIDIA API 키.

Note

LLM 기능을 사용하려면 유효한 NVIDIA API 키와 충분한 크레딧이 필요합니다. 키를 얻고 크레딧을 관리하려면 NVIDIA API 포털을 방문하세요. 자세한 내용은 NVIDIA API 참조 페이지를 참조하세요.

Enable Chat IRO Extension#

  1. Omniverse Extension Manager 가이드를 따라 omni.ai.langchain.agent.chat_iro 확장 프로그램을 활성화합니다.

  2. Isaac Sim을 실행하고 Extension Manager가 아직 열려 있지 않다면 엽니다:

    • 메인 메뉴에서 Window > Extensions를 선택합니다.

    • Chat IRO를 검색합니다.

    • 확장 프로그램을 활성화하고, 이후 실행 시 자동으로 로드되도록 하려면 AUTOLOAD를 선택합니다.

  3. 환경 변수로 NVIDIA API 키를 설정하여 구성합니다.

    Linux/macOS

    # Set API key for the current shell session
export NVIDIA_API_KEY="nvapi-YOUR-KEY-HERE"

# Make the setting persistent (for bash)
echo 'export NVIDIA_API_KEY="nvapi-YOUR-KEY-HERE"' >> ~/.bashrc
source ~/.bashrc

    Windows (Command Prompt)

    REM Set API key for the current Command Prompt session
set NVIDIA_API_KEY=nvapi-YOUR-KEY-HERE

REM To make the setting persistent, add the variable in
REM System Properties > Environment Variables.

Note

LLM 인증이 실패하면 NVIDIA_API_KEY가 설정되어 있고 남은 크레딧이 있는지 확인합니다.

Accessing the Chat IRO Panel#

확장 프로그램이 활성화되면:

  1. 메인 Chat IRO 창을 엽니다:

    • 메뉴 바에서 Window > Chat IRO를 선택합니다.

    • 일반적으로 뷰포트 오른쪽에 도킹 가능한 Chat IRO 패널이 열립니다.

  2. Model 드롭다운 메뉴에서 모델을 선택합니다. 검증된 작동 모델은 다음과 같습니다:

    • meta/llama-4-maverick-17b-128e-instruct(권장, 256K 컨텍스트, 기본값)

    • qwen/qwen3-next-80b-a3b-instruct(128K+ 컨텍스트)

    • openai/gpt-oss-120b(128K 컨텍스트)

    • openai/gpt-oss-20b(128K 컨텍스트)

  3. 모델을 선택한 후 Chat IRO 패널의 상태 표시줄을 확인합니다. 오류가 없으면 모델이 준비되어 있고 확장 프로그램이 인증된 것입니다.

Using Chat IRO#

Chat IRO는 다음과 같은 방법으로 사용할 수 있습니다:

  • UI 패널 사용

  • 새 IRO 씬 생성

  • 기존 IRO YAML 파일 편집

Using the UI Panel#

Chat IRO 패널을 사용하여 씬을 생성하고 미리 보려면:

  1. Chat IRO 입력 상자에 다음과 같은 프롬프트를 입력합니다:

    Create a scene with 7 cubes and 6 spheres. All objects are randomly positioned, random color, and sized.

  2. Enter를 눌러 프롬프트를 전송합니다.

  3. Chat IRO가 RAG 인덱스에서 관련 YAML 패턴을 검색하고, IRO 구성을 생성하고, 검증하고, Isaac Sim에서 실행합니다.

  4. 생성된 씬이 요청한 동작(객체 수, 색상, 위치, 조명, 카메라 배치)과 일치하는지 뷰포트를 검사합니다.

  5. 후속 프롬프트를 사용하여 기존 구성을 수정하여 씬을 개선합니다. 예를 들어:

    Make all cubes blue and add rigidbody physics

    확장 프로그램이 YAML 구성을 그 자리에서 업데이트하고, 다시 적용하고, 뷰포트를 새로고침합니다.

  6. 기본적으로 구성 파일은 다음과 유사한 디렉터리에 자동으로 저장됩니다:

    ~/Documents/ChatIRO_Results/config_files/my_scene.yaml

    Chat IRO에 파일을 다른 위치에 저장하도록 요청하여 커스텀 경로를 지정할 수도 있습니다.

Generating New IRO Scenes#

Chat IRO는 간결하고 명확하게 지정된 프롬프트에서 완전한 IRO 씬을 생성하도록 최적화되어 있습니다. 좋은 프롬프트에는 다음이 포함됩니다:

  • Create 20 purple cubes arranged in a circular formation with radius 900 at Y = 50.

  • Pack 8 cubes and 6 spheres scaled 1.2x into a bin sized (300, 400, 500) at (5, 0, 0).

예를 들어 다음 프롬프트:

Create a scene with 7 cubes and 6 spheres. All objects are randomly positioned,
random color, and sized.

는 일반적으로 다음과 유사한 IRO 구성을 생성합니다:

isaacsim.replicator.object:
  version: 0.10.0
  parent_config: standard
  seed: 42
  num_frames: 10
  output_path: /Documents/ChatIRO_Results
  screen_height: 2160
  screen_width: 3840
  focal_length: 14.228393962367306
  horizontal_aperture: 20.955

  camera_parameters:
    screen_width: $[/screen_width]
    screen_height: $[/screen_height]
    focal_length: $[/focal_length]
    horizontal_aperture: $[/horizontal_aperture]
    near_clip: 0.001
    far_clip: 100000

  cube:
    count: 7
    type: geometry
    subtype: cube
    tracked: true
    color:
      distribution_type: range
      start:
      - 0
      - 0
      - 0
      end:
      - 1
      - 1
      - 1
    transform_operators:
    - rotateX: 0
    - rotateY: 0
    - rotateZ: 0
    - translate:
        distribution_type: range
        start:
        - -500
        - 50
        - -500
        end:
        - 500
        - 50
        - 500
    - scale:
        distribution_type: range
        start:
        - 0.5
        - 0.5
        - 0.5
        end:
        - 1.5
        - 1.5
        - 1.5

  sphere:

Editing Existing IRO YAML Files#

Chat IRO는 수동으로 또는 다른 도구로 생성한 YAML 구성 파일을 로드하고 수정할 수도 있습니다.

일반적인 워크플로:

  1. Chat IRO에 파일을 로드하도록 요청합니다:

    load /home/user/Documents/ChatIRO_Results/config_files/my_scene.yaml

  2. 뷰포트에서 생성된 씬을 검사합니다.

  3. 자연어를 사용하여 편집을 적용합니다. 예를 들어:

    Add 5 more cubes with random colors.

Increase dome light intensity to 3000.

Add a rotating camera that orbits the scene 360 degrees.

  4. 업데이트된 구성을 저장합니다:

save /absolute/path/to/my_scene_v2.yaml

내부적으로 Chat IRO는 새로 생성된 구성에 사용되는 것과 동일한 검증 및 실행 파이프라인을 재사용합니다.

Managing Output Files and Directories#

기본적으로 Chat IRO는 생성된 구성 파일과 시뮬레이션 출력을 홈 폴더 아래의 구조화된 디렉터리에 저장합니다.

Default Output Location#

모든 Chat IRO 출력은 다음에 구성됩니다:

~/Documents/ChatIRO_Results/
├── config_files/              # YAML configuration files
├── simulation_results/        # IRO simulation outputs
└── .cache/                    # Temporary files (hidden)

  • config_files/ 디렉터리에는 씬을 정의하는 YAML 파일이 포함됩니다.

  • simulation_results/ 디렉터리에는 YAML 구성을 실행할 때 생성된 렌더링된 이미지, 센서 데이터 및 기타 출력이 포함됩니다.

  • .cache/ 디렉터리에는 임시 처리 파일이 저장됩니다.

Note

~/Documents/ChatIRO_Results/가 존재하지 않으면 Chat IRO가 처음 사용 시 자동으로 생성합니다.

Changing the Output Directory#

환경 변수를 사용하여 기본 출력 디렉터리를 변경할 수 있습니다:

# Linux/macOS
export CHAT_IRO_OUTPUT_DIR="~/MyProjects/IRO_Results"

# To make it persistent, add to your shell startup file, for example:
echo 'export CHAT_IRO_OUTPUT_DIR="~/MyProjects/IRO_Results"' >> ~/.bashrc
source ~/.bashrc

REM Windows (Command Prompt)
set CHAT_IRO_OUTPUT_DIR=C:\Users\YourName\IRO_Results

REM Add to System Environment Variables for persistence

Note

고급 사용자는 Chat IRO 확장 프로그램 설정 또는 확장 프로그램과 함께 제공되는 Python API를 통해 기본 출력 디렉터리를 구성할 수도 있습니다.

Natural‑Language File Commands#

Chat IRO는 구성을 로드, 저장, 실행하기 위한 간단한 텍스트 명령을 이해합니다:

파일 로드

load /path/to/my_scene.yaml

파일 저장

save /absolute/path/to/my_warehouse_scene.yaml

save this as /absolute/path/to/production_config.yaml

특정 파라미터로 시뮬레이션

simulate with seed 123

Note

안정적인 동작을 위해 저장 시 항상 절대 경로를 지정하세요. 예: save /absolute/path/to/my_scene.yaml. 파일 이름만 사용하는 것(예: save my_scene.yaml)은 저장 위치가 환경과 구성에 따라 다를 수 있으므로 권장하지 않습니다.

Chat IRO RAG Configuration#

Chat IRO에는 기존 IRO 씬과 모범 사례 구성에 대한 심층적인 지식을 제공하는 RAG(Retrieval-Augmented Generation) 시스템이 포함되어 있습니다.

RAG 시스템의 동작은 extension.toml에서 커스터마이즈할 수 있습니다:

[settings.exts."omni.ai.langchain.agent.chat_iro"]
enable_rag = true                  # Enable/disable RAG (default: true)
rag_top_k = 15                     # Number of documents to retrieve
rag_max_tokens = 8000              # Maximum tokens for RAG context
enable_multi_query_rag = true      # Enable multi‑query decomposition
max_sub_queries = 3                # Maximum number of sub‑queries

# Optional cross‑encoder reranking
enable_rag_reranking = false
reranker_model = "BAAI/bge-reranker-large"

활성화하면 RAG를 통해 Chat IRO가 다음을 수행할 수 있습니다:

  • 복잡한 프롬프트를 여러 집중된 하위 쿼리로 분해합니다.

  • 기하학, 하모나이저, 카메라에 대한 관련 YAML 스니펫을 검색합니다.

  • 결과를 병합하고 재순위화하여 더 높은 품질의 구성을 제공합니다.

Note

크로스 인코더 재순위화를 활성화하면 일반적으로 요청당 추가 지연(약 100-200ms)의 비용으로 검색 정확도가 10-30% 향상됩니다. 간단한 프롬프트나 낮은 지연 환경에서는 enable_rag_reranking = false를 유지하세요.

Best Practices#

Chat IRO는 자연어를 해석하는 LLM에 의존합니다. 명확하고 구체적인 프롬프트가 더 신뢰할 수 있는 IRO 구성으로 이어집니다.

권장 프롬프트 지침:

  • 막연한 용어 대신 구체적인 숫자를 지정합니다.

    좋은 예: Create 20 cubes in a circular formation with radius 900 at Y = 50.

    피해야 할 예: Create some objects in a circle.

  • 크기, 위치, 물리 요구 사항을 명시적으로 설명합니다.

  • 씬을 반복적으로 구축하고 각 단계를 뷰포트에서 검증합니다.

  • 작업 중인 구성을 자주 저장하고 개선하면서 버전을 관리합니다.

생성된 YAML이 실행되지 않거나 씬이 비어 있는 경우:

  • 수정된 구조로 재생성하도록 Chat IRO에 요청합니다. 예를 들어:

    Regenerate the configuration using valid YAML syntax and complete
all missing parameters.

  • 전체 씬을 다시 작성하는 대신 특정 오류(간격, 회전, 수, 물리 플래그)에 집중하여 수정 프롬프트를 사용합니다.

Troubleshooting#

일반적인 문제 및 해결 방법:

LLM 인증 실패

  • 증상: 누락되거나 잘못된 API 키에 관한 오류 메시지; YAML이 생성되지 않음.

  • 조치: 환경과 extension.toml에서 NVIDIA_API_KEY를 확인하고, 계정에 남은 크레딧이 있는지 확인하고, Isaac Sim을 재시작합니다.

씬이 렌더링되지 않음

  • 증상: Chat IRO가 응답하지만 뷰포트가 비어 있습니다.

  • 조치:

    • Chat IRO 창에서 생성된 YAML을 검사합니다.

    • Isaac Sim 콘솔이나 로그에서 오류 메시지를 찾습니다.

    • Create 5 cubes와 같은 간단한 프롬프트를 시도하여 기본 동작을 확인합니다.

YAML 구문 오류

  • 증상: Failed to parse YAML과 같은 메시지.

  • 조치:

    • Chat IRO에 YAML 구문을 수정하도록 요청합니다.

    • 프롬프트를 단순화하고 단일 자체 포함 구성을 요청하는지 확인합니다.

느린 응답

  • 증상: 프롬프트를 보내고 응답을 받을 때까지 눈에 띄는 지연이 있습니다.

  • 조치:

    • extension.toml에서 rag_top_k를 줄이고 재순위화를 비활성화합니다.

    • 매우 복잡한 씬을 여러 개의 작은 프롬프트로 분할합니다.

Session Management#

매우 긴 세션에서 LLM이 원래 제약 조건에서 벗어나거나 일관성 없는 구성을 생성할 수 있습니다.

대화를 초기화하려면:

  • Chat IRO 창 왼쪽 상단의 \(+\) 버튼을 클릭하여 새 세션을 시작합니다.

  • 동작이 계속 일관성 없으면 선택적으로 Isaac Sim을 재시작합니다.

  • 다음과 같은 명확한 지시로 새 세션을 시작합니다:

    You are a YAML configuration generator for Isaac Sim Replicator Object.
Generate only valid YAML with proper structure. Create a scene with
10 cubes in a grid layout.
