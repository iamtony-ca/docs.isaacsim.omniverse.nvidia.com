Actor Simulation and Synthetic Data Generation#

다양한 환경에서 인간 캐릭터와 로봇과 같은 애니메이션 액터 또는 에이전트를 감지하고 추적하는 것은 소매, 제조, 물류 등 다양한 산업에서 상당한 가치를 제공합니다. 레이아웃 최적화, 안전 향상 및 효율성 증대에 도움이 됩니다. 그러나 탐지 모델 훈련을 위한 실세계 데이터 수집은 비용이 많이 들고 확장이 어려운 경우가 많습니다.

합성 데이터 생성은 유연하고 확장 가능한 솔루션을 제공합니다. Omni.Metropolis.Pipeline(OMP), Isaacsim.Replicator.Agent(IRA), Isaacsim.Anim.Robot.Core(IAR) 익스텐션은 함께 3D 환경에서 인간 캐릭터와 로봇을 설정하고 합성 데이터를 생성하는 방법을 제공합니다. 이 프레임워크는 설정 파일을 통해 액터 동작, 환경 및 센서에 대한 제어도 제공합니다. 컴퓨터 비전 모델 훈련과 소프트웨어 인더루프 시스템 테스트를 위한 GPU 가속 솔루션을 제공하는 것을 목표로 합니다.

이 프레임워크는 다음과 같은 기능으로 시뮬레이션 커스터마이징을 간소화합니다:

  • Codeless Interaction: 설정은 YAML 파일로 표현됩니다. 합성 데이터를 얻기 위해 코드가 필요하지 않습니다.

  • Simplified Setup: Isaac Sim에 포함되어 있으며 인터랙티브 및 헤드리스 워크플로를 위한 GUI와 스크립팅 인터페이스를 모두 제공합니다.

  • High-Fidelity Data: Omniverse의 SimReady 에셋, 물리 및 렌더링을 활용하여 AI 훈련에 필수적인 현실적인 이미지와 정확한 어노테이션을 생성합니다.

  • Seamless Integration: Kit 익스텐션의 일부로서 omni.anim.behavior, omni.anim.navigation, omni.replicator.core와 기본으로 호환됩니다.

이 익스텐션을 활성화하기 전에 Isaac Sim에 대해 알아보려면 What Is Isaac Sim?을 읽고 Isaac Sim을 설치하려면 Installation을 따르십시오.


Enable Extensions#

  1. Omniverse Extension Manager 가이드를 따라 Omni.Metropolis.Pipeline, Isaacsim.Anim.Robot.Core, Isaacsim.Replicator.Agent.Core & UI를 활성화합니다.

    • 익스텐션은 시작 시 Isaac Sim Assets에서 샘플 에셋을 가져옵니다. 에셋 로딩에 문제가 발생하면 Isaac Sim Assets를 참고하십시오.

    • UI 로딩이 멈추는 것처럼 보이면 --/persistent/isaac/asset_root/timeout=1.0 플래그로 Isaac Sim을 시작해 보십시오.

  2. UI 패널은 Tools > Action and Event Data Generation > Actor SDG로 접근할 수 있으며 화면 오른쪽에 열립니다.

참고

  • 시작 시 익스텐션이 자동으로 로드되게 하려면 익스텐션 매니저에서 autoload 체크박스를 선택하십시오.

  • 익스텐션 의존성으로 인해 Isaac Sim 앱을 다시 시작해야 할 수 있습니다.

팁

예기치 않은 오류가 발생하면 이전 사용자 설정을 초기화하기 위해 --reset-user 플래그로 Isaac Sim을 실행해 보십시오.

./isaac-sim.sh --reset-user

Getting Started in the UI#

처음 사용자라면 UI로 시작하십시오. Isaac Sim 헤드리스 모드에서 Python 스크립트로 실행하는 방법은 Running from script 섹션을 참고하십시오.

  1. Enable Extensions 섹션을 따라 UI 패널을 엽니다.

  2. 기본적으로 최소 설정이 로드됩니다. 폴더 브라우저 아이콘을 사용하여 별도의 설정 파일을 로드할 수도 있습니다.

    • 모든 샘플 설정 파일은 [Isaac Sim App Path]/extscache/isaacsim.replicator.agent.core-[current-version]/data/sample_configs/에 있습니다. 각 파일에 대한 설명은 Sample Configs를 참고하십시오.

    • 최소 설정 파일에는 액터나 카메라가 없습니다. 더 포괄적인 예제는 위 폴더의 full_pipeline.yaml을 사용하십시오. 이 예제는 로딩 시간이 더 걸릴 수 있습니다.


  1. [선택 사항] 설정 파일을 필요에 맞게 수정합니다.

    • New 아이콘을 사용하여 새 설정 파일을 생성합니다.

    • Reload 아이콘을 사용하여 UI의 변경 사항을 초기화하고 원래 설정 파일을 다시 로드합니다.

    • Save 또는 Save As 아이콘을 사용하여 UI의 변경 사항을 설정 파일에 저장합니다.

    • Verbose save 체크박스를 사용하여 설정 파일 저장 시 얼마나 많은 세부 정보를 쓸지 제어합니다. 꺼져 있으면 파일이 간결하게 유지되며 수정된 값만 씁니다. 켜져 있으면 비어 있지 않은 모든 필드가 포함되어 사용 가능한 모든 옵션의 완전한 참조를 원하거나 완전히 명시적인 설정을 공유해야 할 때 유용합니다.

  2. UI 상단에서 Set Up Simulation 버튼을 클릭합니다. 설정에 따라 시뮬레이션 에셋(씬, 카메라, 액터)을 로드하기 시작합니다.

    • 씬은 에셋을 올바르게 생성하고 제어하기 위해 NavMesh가 필요합니다. 예제 설정의 씬은 NavMesh가 미리 설정되어 있습니다. 외부 씬을 사용하는 경우 NavMesh 설정은 Navigation Mesh를 참고하십시오.

    • Window > Navigation > NavMesh로 이동하여 NavMesh 설정에서 Auto-Bake를 끌 수도 있습니다. 끄면 성능이 향상될 수 있습니다.

    참고

    Set Up Simulation을 클릭하면 항상 현재 설정에서 씬을 완전히 다시 로드합니다. 여기에는 기본 환경 USD를 다시 열고 설정에 따라 모든 액터(캐릭터 및 로봇), 센서, 프롭 레이어를 처음부터 다시 생성하는 것이 포함됩니다. 이 작업은 이전 설정 후 스테이지에 수동으로 편집한 내용을 버립니다. 설정을 반복하려면 먼저 UI 또는 설정 파일에서 변경 사항을 적용한 다음 Set Up Simulation을 클릭하십시오.

  3. UI 상단에서 Start Data Generation 버튼을 클릭합니다. 시뮬레이션과 데이터 생성은 Actor SDG Setup 패널의 Simulation Duration 필드에 지정된 시간(초) 동안 실행됩니다.

  4. 데이터 생성이 완료되면 출력 데이터는 Replicator 패널에 지정된 Output Directory에서 사용할 수 있습니다.

    • 기본값은 Windows의 경우 %USERPROFILE%\IRA_output이고 Linux의 경우 ~/IRA_output입니다.

    • 폴더 레이아웃, 예상 파일명 및 성공적인 실행을 확인하는 체크리스트는 Expected Output을 참고하십시오.

Running from Script#

대규모 데이터 생성의 경우 스크립트에서 실행하는 것이 더 효율적입니다. IRA는 오프라인 데이터 생성을 실행하기 위한 자동 스크립트(actor_sdg.py)를 제공합니다.

스크립트에서 실행하려면 Isaac Sim이 설치된 위치에서 터미널을 열고 다음 명령을 실행합니다:

  • Linux의 경우:

    ./python.sh tools/actor_sdg/actor_sdg.py -c [config file path]

  • Windows의 경우:

    .\python.bat tools\actor_sdg\actor_sdg.py -c [config file path]

참고

  • [config file path]는 IRA 설정 파일의 경로입니다.

  • 스크립트를 실행하려면 Isaac Sim에 번들된 python.sh 또는 python.bat를 사용해야 합니다.

  • 예제 설정 파일도 /tools/actor_sdg 폴더에 제공됩니다. 샘플 Linux 실행의 경우 다음을 실행합니다: ./python.sh tools/actor_sdg/actor_sdg.py -c tools/actor_sdg/sample_config.yaml

Expected Output#

데이터 생성이 완료된 후 IRA는 설정의 replicator 블록에서 output_dir로 지정된 디렉터리에 시뮬레이션 출력을 씁니다. output_dir이 설정되지 않으면 기본 위치는 다음과 같습니다:

  • Linux: ~/IRA_output

  • Windows: %USERPROFILE%\IRA_output

정확한 레이아웃은 활성 라이터에 따라 다릅니다(전체 매개변수 목록은 Writer Configuration 참고). 기본 IRABasicWriter와 번들된 full_pipeline.yaml 샘플 설정으로는 다음과 유사한 구조가 예상됩니다:

API Usage#

이 익스텐션은 자신의 스크립트에서 시뮬레이션을 설정하고 데이터를 생성하는 데 사용할 수 있는 Python API도 노출합니다. isaacsim.replicator.agent.core가 활성화되어 있는지 확인하고 다음 예제와 같이 API를 사용하십시오.

참고

아래 스니펫은 익스텐션에 번들된 최소 설정(data/sample_configs/minimal.yaml)을 사용합니다.

import os

from isaacsim import SimulationApp

# Start the application
simulation_app = SimulationApp({"headless": True})

# Get the utility to enable extensions
from isaacsim.core.utils.extensions import enable_extension

# Enable the IRA extension
enable_extension("isaacsim.replicator.agent.core")
simulation_app.update()


def _get_config_path() -> str:
    """Return path to the minimal IRA config bundled with the extension."""
    import omni.kit.app

    core_ext_path = (
        omni.kit.app.get_app().get_extension_manager().get_extension_path_by_module("isaacsim.replicator.agent.core")
    )
    default_config_file_path = os.path.join(core_ext_path, "data", "sample_configs", "minimal.yaml")
    if not os.path.isfile(default_config_file_path):
        raise FileNotFoundError(f"IRA config not found: {default_config_file_path}")
    return default_config_file_path


async def run_ira_data_generation(setup_simulation: bool = False, run_data_generation: bool = False):
    from isaacsim.replicator.agent.core import api as IRA

    # IRA: load config. Specify the config file path.
    config_path = _get_config_path()
    result = IRA.load_config_file(config_path)
    if not result:
        raise RuntimeError(f"Failed to load IRA config: {config_path}")

    # IRA: get config, you can modify the config here
    config = IRA.get_config_file()
    IRA.set_config(config)

    # IRA: setup simulation (only when setup_simulation is True)
    if setup_simulation:
        await IRA.setup_simulation()

        # Allow a few frames for scene to settle
        import omni.kit.app

        app = omni.kit.app.get_app()
        for _ in range(10):
            await app.next_update_async()

        # IRA: generate data (only when run_data_generation is True)
        if run_data_generation:
            await IRA.start_data_generation_async(will_wait_until_complete=True)


from omni.kit.async_engine import run_coroutine

task = run_coroutine(run_ira_data_generation(setup_simulation=False, run_data_generation=False))
while not task.done():
    simulation_app.update()

Migrating from IRA 0.x.x#

IRA 0.x.x(Isaac Sim 5.1 이전 버전에 포함)에서 업그레이드하는 경우, IRA 1.x.x(Isaac Sim 6.0+)는 완전한 아키텍처 재설계임을 알아두십시오. 다음을 포함한 모든 핵심 기능은 유지됩니다:

  • 환경 로딩

  • 캐릭터 및 로봇 스폰

  • 센서 배치

  • 합성 데이터 생성

그러나 설정 스키마, 동작 시스템, Python API가 모두 변경되었습니다. 기존 0.x 설정 파일과 스크립트는 수정 없이는 작동하지 않습니다.

재설계의 주요 이유:

  • 더 간단한 워크플로 – 외부 명령 파일을 생성, 저장, 로드하는 다단계 프로세스가 사라졌습니다. 동작이 이제 YAML 설정에 인라인으로 정의되어 UI에서 두 번의 클릭으로 설정이 줄어듭니다.

  • 더 큰 유연성 – 명명된 그룹을 사용하면 단일 설정에서 독립적인 설정으로 여러 캐릭터, 로봇, 센서 집합을 정의할 수 있습니다. 여러 데이터 라이터가 라이터별 타이밍 및 센서 선택으로 동시에 실행될 수 있습니다.

  • 더 강한 유효성 검사 – Pydantic v2 모델이 로드 시 설정을 검증하고 명확한 오류 메시지를 표시하여 시뮬레이션이 시작되기 전에 실수를 포착합니다.

  • USD 네이티브 아키텍처 – 액터 설정이 USD 스키마와 프림으로 지속되어 스테이지에서 직접 검사하고 편집할 수 있습니다.

이전/이후 예제와 단계별 체크리스트로 모든 주요 변경 사항을 다루는 전체 마이그레이션 가이드는 Replicator Agent (IRA)를 참고하십시오.

UI 또는 코드로 설정 파일을 편집하는 방법은 Configuration Editor API를 참고하십시오:

Actor Behaviors#

액터 동작은 OMP, IRA, IAR이 함께 달성합니다.

액터는 재생 시 "routine-trigger" 동작 루프를 수행합니다. 이 패턴은 액터에 할당된 동작과 트리거로 설정 가능합니다.


The Routine Trigger Loop#

액터 트리거가 활성화되지 않으면 액터는 actor global seed를 사용하여 확률에 따라 가중치가 부여된 루틴에서 수행할 동작을 반복적으로 선택함으로써 루틴 루프를 수행합니다.

트리거가 활성화되면 액터가 루틴을 일시 중지하고 각 활성 트리거 아래의 동작을 수행합니다. 높은 우선순위의 트리거가 발동되면 실행 중인 트리거를 일시 중지하고 대기열에 넣습니다(낮은 우선순위의 트리거는 건너뜀). 트리거는 모든 동작이 완료되면 완료로 표시됩니다. 그런 다음 대기열의 첫 번째 트리거가 재개됩니다. 모든 활성 트리거가 완료되면 액터가 루틴으로 돌아갑니다.


Configure Behaviors#

설정 파일에서 액터가 씬에 로드된 후 설정은 USD API 스키마와 USD 프림에 임베드됩니다. 각 액터는 MetroAgentAPI 스키마와 해당 파생 타입으로 표현됩니다. 인간 캐릭터의 경우 SkelRoot 프림에 연결된 IRACharacterAPI이고, 애니메이션 로봇의 경우 로봇 페이로드의 루트 프림에 연결된 AnimRobotAPI입니다. 각 동작과 트리거는 액터 USD API가 참조할 수 있는 개별 USD 프림이 되며, 각 액터 트리거 프림도 동작 목록에 대한 참조를 가질 수 있습니다.

액터 USD API 스키마는 액터의 기본 정보를 정의합니다:

  • 이름

  • 그룹

  • 시드

  • 루틴 참조 슬롯과 트리거 참조 슬롯

재생 시 이름, 그룹, 시드가 결합되어 actor global seed로 해시됩니다. 이 시드는 액터 자체의 무작위 루틴 선택과 속도 범위에서 속도 선택과 같은 각 동작 내의 선택을 포함한 액터의 모든 "무작위성"에 사용됩니다. 이는 또한 다른 설정과 환경이 변경되지 않으면 동일한 actor global seed가 동일한 결과를 보여준다는 것을 의미합니다.

각 타입의 액터 동작은 USD 프림 타입으로 표현됩니다. 동작의 설정을 정의합니다:

  • 가중치

  • 반복

  • 동작

인간 캐릭터의 경우 동작 프림 타입은 CharacterXXXBehavior 명명 패턴을 따릅니다. 애니메이션 로봇의 경우 RobotXXXBehavior입니다.

각 액터 트리거도 USD 프림입니다. 트리거 우선순위를 정의하고 이 트리거가 활성화될 때 순차적으로 실행될 동작 목록에 대한 참조를 가집니다. 인간 캐릭터와 애니메이션 로봇은 OMP에서 MetroXXXTrigger 명명 패턴으로 정의된 동일한 트리거 타입을 공유합니다.

또한 액터는 애니메이션 구현으로 omni.behavior.behavior(인간 캐릭터)와 isaacsim.anim.robot.core(애니메이션 로봇)를 활용합니다. 자세한 내용은 다음 문서를 참고하십시오:

  • Animated Robot Controller

Behavior Tree (Experimental)#

경고

행동 트리 캐릭터 지원은 실험적이며 향후 릴리스에서 변경될 수 있습니다.

위에서 설명한 routine-trigger 동작 시스템 외에도 IRA 1.3.0은 행동 트리를 통한 캐릭터 동작 구동 지원을 도입합니다. 행동 트리는 omni.behavior.tree.core와 omni.anim.behavior.tree 익스텐션으로 저작됩니다.

Behavior Tree Overview

행동 트리는 의사 결정을 위한 계층적 모델입니다. 함께 동작하는 다양한 노드 타입으로 구성됩니다:

  • Action 노드는 캐릭터가 구체적인 행동(예: MoveTo, Wait)을 수행하는 리프 수준 노드입니다.

  • Composite 노드는 로직과 실행 흐름을 제어합니다. 예를 들어 Sequence 노드는 자식을 순서대로 실행하고, Selector 노드는 하나가 성공할 때까지 자식을 시도합니다.

  • Modifiers와 Decorators는 다른 노드를 래핑하여 동작을 변경합니다. 예를 들어 Repeat(서브트리 반복) 또는 RandomNavMeshPoint(무작위 목적지 제공)입니다.

이러한 노드 타입을 조합하면 코드 없이 단일 트리 정의 내에서 단순한 배회 루프부터 다단계 조건부 시퀀스까지 임의로 복잡한 동작을 구성할 수 있습니다. 가중치에 따라 무작위로 동작을 선택하는 IRA 루틴 시스템과 비교하여 행동 트리는 순서, 분기 및 루프에 대한 완전한 결정론적 제어를 제공합니다.

Relationship to Routines and Triggers

참고

행동 트리 캐릭터 그룹에는 트리거가 현재 지원되지 않습니다. 반응형 또는 조건부 로직은 행동 트리 자체의 노드로 저작해야 합니다.

행동 트리 모드는 routine-trigger 시스템의 대안입니다. 설정의 각 캐릭터 그룹은 둘 중 하나를 사용합니다: 그룹은 routines와 triggers(IRA 스타일)를 정의하거나 behavior_tree를 참조합니다. 그러나 단일 설정 파일이 여러 그룹을 포함할 수 있으므로 IRA 스타일과 행동 트리 그룹이 나란히 공존할 수 있습니다.

Workflow

  1. omni.behavior.tree.ui를 사용하여 행동 트리를 저작하고 JSON 파일로 저장합니다. 트리는 액션, 컴포짓, 모디파이어 노드를 위해 omni.behavior.tree.core와 omni.anim.behavior.tree 노드 라이브러리를 참조합니다. 행동 트리 저작 방법은 Behavior Tree's User Guide를 참고하십시오.

  2. IRA 설정 YAML에서 JSON 파일을 가리키는 behavior_tree 필드를 포함하는 캐릭터 그룹을 생성합니다. 선택적으로 트리 파일을 수정하지 않고 서로 다른 캐릭터 그룹에 노드 매개변수를 할당하기 위해 overrides 필드를 제공합니다.

    경고

    overrides 필드는 현재 UI를 통해 설정할 수 없습니다. YAML 설정 파일 또는 API에서 직접 편집해야 합니다.

  3. 평소와 같이 시뮬레이션을 실행합니다. 행동 트리 캐릭터는 IRA 캐릭터와 동일한 스폰, NavMesh, 데이터 생성 파이프라인을 공유합니다.

행동 트리 캐릭터 그룹이 포함된 일부 샘플 설정 파일은 [Isaac Sim Assets Path]/Samples/BehaviorTree 폴더와 isaacsim.replicator.agent.core 익스텐션의 data/sample_configs/behavior_tree/ 폴더에 번들로 제공됩니다(Sample Configs 참고). 설정 세부 사항, 매개변수 참조, YAML 예제는 설정 파일 가이드의 Behavior Tree Character Group (Experimental)을 참고하십시오.

경고

IRA가 설정한 행동 트리 캐릭터에는 여전히 IRACharacterAPI 스키마가 적용되어 있지만, 이는 데이터 생성 식별(이름, 그룹, 시맨틱 레이블 및 관련 필드)에만 사용됩니다. 캐릭터의 동작은 omni.behavior.tree.core(OBT)를 통해 행동 트리가 완전히 제어합니다. seed와 같은 IRA 수준 설정은 행동 트리 캐릭터에 영향을 미치지 않습니다.

Terminology#

Isaacsim.Replicator.Agent.Core

시뮬레이션 상태를 관리하는 코어 익스텐션입니다. 시뮬레이션을 설정하고 합성 데이터를 캡처하기 위한 핵심 API와 모듈을 포함합니다. 해당 모듈은 독립적으로 호출할 수 있습니다.

IRA를 위한 UI 익스텐션입니다. 이 익스텐션이 로드되면 코어 익스텐션이 자동으로 로드됩니다. 이 익스텐션에는 익스텐션과의 쉬운 상호작용을 위한 UI 컴포넌트가 포함됩니다.

Configuration File

시뮬레이션의 핵심 구성 요소(무작위화 시드, 시뮬레이션 지속 시간, 액터 수, 출력 형식 포함)를 정의하는 설정 데이터가 포함된 .yaml 파일입니다. 익스텐션을 사용하려면 먼저 설정 파일을 로드하거나 UI를 사용하여 YAML 파일을 생성해야 합니다.

Actor

액터는 각 컨트롤러(omni.behavior.tree와 isaacsim.anim.robot)에 의해 제어되며 시뮬레이션에서 행동을 수행합니다. 익스텐션은 인간 캐릭터와 로봇(Nova Carter, iw.hub)을 액터로 지원합니다. "액터"와 "에이전트"라는 용어는 이 문서에서 혼용됩니다.

Seed

무작위화 시드. 동일한 시드로 익스텐션은 카메라 및 에이전트 위치와 에이전트 동작에 대해 동일한 무작위화 결과를 생성할 수 있습니다. 동일한 시드와 동일한 작업 순서로 동일한 데이터가 생성됨이 보장됩니다.

Replicator (Omni.Replicator.Core)

이 익스텐션이 기반으로 하는 데이터 캡처 익스텐션입니다. Replicator 익스텐션에 대한 자세한 내용은 Replicator Official Documentation에서 확인할 수 있습니다.

Behavior Tree

의사 결정 및 행동을 구성하기 위한 계층적 데이터 모델입니다. 행동 트리는 JSON 파일로 정의되며 IRA 설정에서 참조됩니다. 캐릭터 동작을 제어하기 위한 routine-trigger 시스템의 대안을 제공합니다. 자세한 내용은 Behavior Tree documentation를 참고하십시오.
