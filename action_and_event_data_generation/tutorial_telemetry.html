Telemetry and Performance Tracking#

Action and Event Data Generation 익스텐션에는 성능 지표 및 사용 패턴을 추적하는 내장 텔레메트리 기능이 포함되어 있습니다. 텔레메트리 시스템은 익스텐션 전반에 걸쳐 다양한 지표를 캡처하여 시스템 동작, 성능 특성 및 사용 패턴에 대한 유용한 인사이트를 제공합니다.

Overview#

Action and Event Data Generation 생태계의 텔레메트리는 개발자와 사용자가 다음을 할 수 있게 도와줍니다:

  • 성능 모니터링: 실행 시간, 리소스 사용량 및 시스템 성능 추적

  • 사용 패턴 이해: 기능이 어떻게 사용되는지에 대한 인사이트 확보

  • 문제 식별: 병목 현상 및 성능 문제 조기 감지

  • 사용자 경험 향상: 데이터 기반 인사이트를 사용하여 워크플로 최적화

텔레메트리 시스템은 여러 익스텐션에 걸쳐 구현되어 있으며 지표 수집 및 보고에 표준화된 접근 방식을 제공합니다.

로컬 텔레메트리 로그는 ~/.nvidia-omniverse/logs/ 디렉터리에서 찾을 수 있습니다.

Telemetry Architecture#

텔레메트리 시스템은 NVIDIA Omniverse의 구조화된 로깅 프레임워크를 기반으로 구축되며 다음으로 구성됩니다:

  • Schema Definition: 텔레메트리 이벤트와 해당 속성을 정의하는 구조화된 스키마

  • Event Generation: 스키마 정의에서 자동으로 생성된 Python 바인딩

  • Data Collection: 텔레메트리 이벤트를 내보내는 계측된 코드

  • Storage and Analysis: 로컬에 기록되고 분석 플랫폼으로 전송되는 이벤트

자세한 내용은 Omniverse Telemetry Walkthrough를 참고하십시오.

Telemetry Modes#

텔레메트리 시스템은 다음 운영 모드를 지원합니다:

  • Production Mode: --/telemetry/mode=prod - 프로덕션 배포의 기본 모드.

  • Test Mode: --/telemetry/mode=test - QA, 검증 및 테스트를 위한 내부 모드.

  • Dev Mode: --/telemetry/mode=dev - 개발을 위한 내부 모드.

각 모드는 데이터 수집 및 전송 정책이 다릅니다. 전송 또는 구조화된 로깅을 비활성화하려면 아래의 Configuring Telemetry를 참고하십시오.

헤드리스 모드에서 실행 중이면 텔레메트리가 기본적으로 비활성화됩니다. 텔레메트리를 활성화하려면 애플리케이션 설정을 사용하여 --/telemetry/mode=dev를 전달합니다:

import os
from isaacsim import SimulationApp

base_exp_path = os.path.join(
    os.environ["EXP_PATH"],
    "isaacsim.exp.action_and_event_data_generation.base.kit"
)
app_config = {
    "headless": True,
    "width": 1920,
    "height": 1080,
    "extra_args": ["--/telemetry/mode=dev"], # Enables telemetry in dev mode
}
sim_app = SimulationApp(launch_config=app_config, experience=base_exp_path)

모드에 관계없이 데이터는 ~/.nvidia-omniverse/logs/ 디렉터리에 사용자 홈 디렉터리로 로컬에 저장됩니다.

Configuring Telemetry#

텔레메트리 전송(데이터 수집)을 비활성화하려면 커맨드라인 또는 애플리케이션 설정에서 --/telemetry/enableAnonymousData=false(또는 0)를 설정합니다. 텔레메트리 이벤트는 여전히 로컬에 기록됩니다. 또는 앱의 .kit 파일에서 [settings.telemetry] 아래에 enableAnonymousData = false를 설정합니다(Data Collection & Usage 참고).

모든 텔레메트리(전송 및 로컬 로깅)를 비활성화하려면 커맨드라인 또는 애플리케이션 설정에서 --/structuredLog/enable=false(또는 0)를 설정합니다.

텔레메트리는 개별 extension.toml 파일을 통해 익스텐션 수준에서 활성화하거나 비활성화할 수도 있습니다.

다음 익스텐션에는 특정 텔레메트리 설정이 포함되어 있습니다:

  • isaacsim.replicator.agent — 설정 위치: EXTS_PATH/isaacsim.replicator.agent.core/config/extension.toml

  • omni.metropolis.utils — 설정 위치: EXTS_PATH/omni.metropolis.utils/config/extension.toml

참고

isaacsim.replicator.agent는 두 부분(isaacsim.replicator.agent.core와 isaacsim.replicator.agent.ui)으로 구성됩니다. core 익스텐션의 설정을 업데이트하면 두 부분 모두에 영향을 미칩니다.

EXTS_PATH는 플랫폼에 따라 다릅니다. 예를 들어 Windows에서는 C:\isaacsim\extscache\isaacsim.replicator.agent.core-1.x.y\config\extension.toml처럼 보이고, Linux에서는 ~/isaacsim/extscache/isaacsim.replicator.agent.core-1.x.y/config/extension.toml처럼 보입니다(버전 1.x.y는 다를 수 있음).

[settings]
exts."isaacsim.replicator.agent".telemetry_enabled = true

런타임에 텔레메트리 설정을 수정하려면:

import carb.settings

settings = carb.settings.get_settings()

# Enable telemetry for specific extensions
settings.set("/exts/isaacsim.replicator.agent/telemetry_enabled", True)
settings.set("/exts/omni.metropolis.utils/telemetry_enabled", True)

# Disable telemetry for specific extensions
settings.set("/exts/isaacsim.replicator.agent/telemetry_enabled", False)
settings.set("/exts/omni.metropolis.utils/telemetry_enabled", False)

Available Telemetry Events#

다음 텔레메트리 이벤트는 Action and Event Data Generation 익스텐션 전반에서 사용 가능합니다:

omni.metropolis.utils

  • file_read - 파일 읽기 작업 추적

  • file_write - 파일 쓰기 작업 추적

isaacsim.replicator.agent.core

  • data_generation - 데이터 생성 작업 추적

  • load_asset_to_scene - 에셋 로딩 이벤트 추적

  • stage_setup_event - 스테이지 설정 작업 추적

  • writer_initialized_event - 라이터 초기화 이벤트 추적
