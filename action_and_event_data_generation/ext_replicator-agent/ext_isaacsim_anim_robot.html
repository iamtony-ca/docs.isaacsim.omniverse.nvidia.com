Animated Robot Controller#

isaacsim.anim.robot.core 확장 프로그램은 캡처된 시뮬레이션 모션 데이터의 재생을 통해 현실적인 로봇 애니메이션을 구현합니다. 물리 기반 시뮬레이션과 애니메이션을 연결하여 실시간 물리 계산의 계산 부담 없이 정밀한 로봇 움직임을 재현할 수 있습니다. 이 확장 프로그램은 물리 지원 로봇 모델을 운동학적 정확성과 시각적 충실도를 보존하면서 애니메이션 표현으로 변환합니다.

isaacsim.anim.robot.core는 omni.metropolis.pipeline 에이전트 프레임워크와 통합됩니다. 각 로봇은 AnimRobot으로, 애니메이션 재생을 구동하는 유한 상태 기계를 갖춘 Metro Agent입니다. 로봇은 운동학, 상태, 전환, 애니메이션 데이터를 정의하는 YAML 파일을 통해 구성됩니다.

Supported Robots#

이 확장 프로그램은 다음 로봇에 대한 샘플 구성을 포함합니다:

  • Nova Carter – 차동 구동 모바일 로봇

  • iw.hub – 리프트 기능이 있는 차동 구동 창고 로봇

  • Forklift – 차동 구동 지게차

새 YAML 구성 파일을 생성하여 커스텀 로봇을 추가할 수 있습니다 (Customization 참조).

Architecture#

이 확장 프로그램은 여러 주요 모듈로 구성됩니다:

  • AnimRobot – omni.metropolis.pipeline과 통합되는 에이전트 클래스. USD 스키마 속성을 파싱하고, 구성을 로드하고, 상태 기계를 설정하고, 로봇의 수명 주기를 관리합니다.

  • StateMachine – 애니메이션 상태와 전환을 관리하는 유한 상태 기계. 각 상태는 로봇의 관절에서 재생되는 관련 애니메이션 데이터를 가질 수 있습니다.

  • Actions – 상태 기계를 업데이트하고 각 프레임에 모션을 적용하여 로봇을 구동하는 고수준 명령(MoveTo, Idle, Turn, Sequence)입니다.

  • Drive – 탐색 경로를 프레임별 위치 및 방향 업데이트로 변환하는 드라이브 베이스 구현(OmniDirectionalDrive, DifferentialDrive)입니다.

  • PathPlanner – get_path_points(start, end, agent_radius=0.5)를 노출하고 웨이포인트의 list[Gf.Vec3d](또는 경로가 없을 때 None)를 반환하는 경로 계획 백엔드. NavMeshPathPlanner는 start와 end를 가장 가까운 유효한 navmesh 위치로 스냅하고, 최단 경로를 쿼리하고, 거의 직선에 가까운 웨이포인트를 제거한 후 반환합니다. 기본 PathPlanner는 장애물 확인 없이 start에서 end까지의 두 점 직선을 반환합니다.

  • Behaviors – 액션을 자율 에이전트 루틴으로 구성하는 런타임 동작(Wander, Patrol, Halt)과 트리거(Event, Time, Collision).

Customization#

YAML 에이전트 구성 파일을 수정하거나 새로 생성하여 로봇 동작과 애니메이션을 커스터마이즈할 수 있습니다. 샘플 구성은 Isaac Sim 에셋 서버의 /Isaac/Samples/AnimRobot/sample_configs/{robot type}.yaml에 제공됩니다. get_IAR_sample_config_path()(Public API 참조)는 런타임에 이 경로를 해결하고 에셋 서버에 연결할 수 없을 때는 {isaacsim.anim.robot.core extension path}/data/sample_configs/의 로컬 복사본으로 폴백합니다.

Robot Attributes#

각 로봇에 대해 다음 속성을 구성할 수 있습니다(BaseAgentConfig 스키마 기반):

  • agent_name (str): 에이전트의 표시 이름(기본값: "BaseAgent"). 공백은 자동으로 밑줄로 대체됩니다.

  • linear_velocity (float): 초당 미터 단위의 전진 이동 속도(기본값: 1.0, >= 0이어야 함).

  • angular_velocity (float): 초당 도 단위의 회전 속도(기본값: 45.0, >= 0이어야 함).

  • forward_vec (list[float]): 초기 전진 방향 벡터, 정확히 3개의 요소(기본값: [1.0, 0.0, 0.0]).

  • joints (list[str]): 애니메이션화할 수 있는 관절 prim 상대 경로 목록.

  • drive_base (str): 로봇의 구동 시스템 유형. 지원 값: differential, omni_directional(기본값: omni_directional).

  • path_planner (str): 경로 계획기 유형. 지원 값: navmesh, base(기본값: navmesh).

  • states (list[str]): FSM 상태 이름 목록(기본값: ["idle", "turn_left", "turn_right", "forward"]). 드라이브 구현은 FSM을 이러한 특정 상태 이름으로 푸시합니다. 커스텀 states 목록은 여전히 idle, forward, turn_left, turn_right를 포함해야 하며, 그렇지 않으면 드라이브 베이스가 모션 중에 “유효한 상태가 아닙니다” 오류를 기록합니다.

  • transitions (dict[str, list[str]]): 상태 간 유효한 전환을 정의하는 상태 전환 그래프. 소스 및 대상 상태는 states 목록에 대해 검증됩니다. 기본값은 {idle: [turn_left, turn_right, idle, forward], turn_left: [idle, forward], turn_right: [idle, forward], forward: [turn_left, turn_right, idle]}.

  • animation_paths (dict[str, str]): 상태 이름을 애니메이션 USD가 포함된 폴더 경로에 매핑. 경로는 확장 프로그램의 설치 디렉터리를 참조하는 ${ext_path} 변수를 사용할 수 있습니다.

  • asset_path (str | null): 에이전트 USD에 대한 상대 또는 절대 경로/URL. 상대 경로인 경우 먼저 로컬 파일을 시도한 다음 Isaac Sim 에셋 루트를 시도합니다. 생략하면 prim의 기존 참조가 사용됩니다.

  • radius (float | null): navmesh 쿼리 반경에 대한 내부 런타임 재정의. YAML 로더는 현재 이 필드를 채우지 않으므로(이는 RuntimeAgentConfig에서 init=False로 선언되었으며 에이전트의 경계 상자에서 파생되지 않음) 로봇별 YAML에서 설정해도 현재는 효과가 없습니다. IRA에서 생성된 로봇의 계획 반경을 제어하려면 대신 IRA 로봇 그룹에서 agent_radius를 설정하세요.

구성 예시 (iw_hub.yaml):

agent_name: iw_hub
linear_velocity: 0.5
angular_velocity: 30.0
forward_vec: [1.0, 0.0, 0.0]
joints:
  - /chassis/lift
  - /chassis/left_wheel
  - /chassis/right_wheel
  - /chassis/left_swivel/left_caster
  - /chassis/right_swivel/right_caster
  - /chassis/left_swivel
  - /chassis/right_swivel
drive_base: differential
path_planner: navmesh
animation_paths:
  turn_left: ${ext_path}/data/sample_animations/iw_hub/turn_left
  turn_right: ${ext_path}/data/sample_animations/iw_hub/turn_right
  forward: ${ext_path}/data/sample_animations/iw_hub/forward
  lift_up: ${ext_path}/data/sample_animations/iw_hub/lift_up
  lift_down: ${ext_path}/data/sample_animations/iw_hub/lift_down
states: [idle, turn_left, turn_right, forward, lift_up, lift_down]
transitions:
  idle: [turn_left, turn_right, idle, forward, lift_up, lift_down]
  turn_left: [forward, idle]
  turn_right: [forward, idle]
  forward: [turn_left, turn_right, idle]
  lift_up: [idle]
  lift_down: [idle]
asset_path: Isaac/Samples/AnimRobot/iw_hub.usd
radius: 0.8

Actions#

이 확장 프로그램은 로봇을 프로그래밍 방식으로 제어하기 위한 다음 액션 유형을 제공합니다:

  • MoveTo – 구성된 경로 계획기와 드라이브 베이스를 사용하여 에이전트를 대상 위치로 이동합니다. 대상은 [x, y, z] 좌표 또는 USD prim 참조(Sdf.Path, prim 경로 문자열, 또는 Usd.Prim)일 수 있습니다. prim 대상은 0.25초마다 다시 해결되며 prim이 0.25분 이상 이동하면 경로를 재계획하므로 에이전트가 이동하는 대상을 추적합니다. 에이전트는 경로를 계획한 다음 회전 및 전진 상태를 전환하며 따라갑니다.

  • Idle – 지정된 기간(초) 동안 에이전트를 유휴 상태로 유지합니다.

  • Turn – 에이전트를 제자리에서 주어진 3D 방향 벡터를 향해 회전시킵니다(yaw만; Z 구성 요소는 무시됨).

  • PlayAnimation – 에이전트에서 명명된 FSM 상태를 재생합니다. duration이 0.0(기본값)이면 상태가 마지막 타임 샘플까지 한 번 재생되고 액션이 완료됩니다. duration > 0이면 해당 초 동안 상태가 유지됩니다.

  • Sequence – 비어 있지 않은 액션 목록을 순서대로 하나씩 실행합니다.

함수형 API를 사용하여 액션을 생성할 수 있습니다:

from isaacsim.anim.robot.core import (
    idle,
    move_to,
    play_animation,
    resolve_anim_robot,
    sequence,
    turn,
)

# Resolve the live runtime agent from its USD prim.
agent = resolve_anim_robot(prim)
cfg, sm = agent.runtime_config, agent.state_machine

# Move to a world-space position
action = move_to(cfg, sm, [10.0, 5.0, 0.0])

# Or move to (and track) a USD prim target
action = move_to(cfg, sm, "/World/pickup_target")

# Idle for 3 seconds
action = idle(cfg, sm, 3.0)

# Turn to face a direction (yaw-only)
action = turn(cfg, sm, [0.0, 1.0, 0.0])

# Play a named FSM state (duration=0 plays once to the clip's end)
action = play_animation(cfg, sm, "lift_up")

# Compose a sequence and inject it into the running agent
action = sequence(
    [
        move_to(cfg, sm, [10.0, 5.0, 0.0]),
        idle(cfg, sm, 2.0),
        turn(cfg, sm, [0.0, -1.0, 0.0]),
    ],
    execute_now=True,
)

각 액션은 execute() 메서드를 통해 실행 중인 에이전트에 주입할 수도 있습니다(sequence(..., execute_now=True) 사용). 주입은 액션을 에이전트의 루틴에 작성하므로 에이전트가 RoutineTrigger 제어 모드일 때만 적용됩니다. 행동 트리로 제어되는 에이전트는 BT 노드를 틱하여 액션을 받습니다.

Behaviors#

Behaviors는 액션을 구성하는 고수준 자율 루틴을 정의합니다. USD 스키마를 통해 구성되며 omni.metropolis.pipeline 트리거 시스템 내에서 실행됩니다.

  • Wander – 로봇이 구성 가능한 거리 범위 내의 임의의 navmesh 도달 가능 지점으로 이동한 다음 임의의 기간 동안 유휴 상태가 됩니다. 구성 가능한 속성에는 탐색 영역, 유휴 시간 범위, 이동 거리 범위가 포함됩니다.

  • Patrol – 로봇이 순서대로 웨이포인트 시퀀스(좌표 [x, y, z] 또는 USD prim 대상)를 방문합니다. 도달할 수 없는 웨이포인트는 경고와 함께 건너뜁니다.

  • Halt – 로봇이 구성 가능한 시간 범위 내에서 임의의 기간 동안 유휴 상태가 됩니다.

이러한 동작은 다음에 의해 트리거될 수 있습니다:

  • 이벤트 트리거 – 외부 이벤트에 의해 활성화됨

  • 시간 트리거 – 시간 일정에 따라 활성화됨

  • 충돌 트리거 – 에이전트의 구성된 충돌 볼륨에 다른 prim이 진입할 때 활성화됨

Drive Types#

이 확장 프로그램은 로봇이 물리적으로 이동하고 회전하는 방식을 제어하는 다양한 드라이브 베이스 구현을 지원합니다:

  • OmniDirectionalDrive – 에이전트가 먼저 회전하지 않고 어느 방향으로든 이동할 수 있습니다. 일정한 선형 속도로 경로를 따라 탐색하고 대상 방향을 향해 독립적으로 회전할 수 있습니다.

  • DifferentialDrive – 에이전트가 전진하기 전에 대상 방향을 향해 제자리에서 회전해야 합니다. 회전 후 주행 패턴을 따릅니다: 먼저 다음 웨이포인트를 향해 회전한 다음 직선으로 전진합니다.

드라이브 유형은 YAML 구성의 drive_base 필드를 기반으로 선택됩니다.

Customizing Animations#

커스텀 애니메이션을 생성하려면:

  1. Isaac Sim에서 로봇을 시뮬레이션합니다.

  2. omni.kit.stagerecorder.core를 사용하여 모션 데이터를 캡처합니다.

  3. 새 애니메이션 폴더 경로로 YAML 구성의 animation_paths 필드를 업데이트합니다.

애니메이션 파일은 상태별 폴더로 구성되어야 합니다. 각 관절의 애니메이션 데이터는 관절 이름으로 명명된 별도의 USD 파일로 저장됩니다. 예를 들어 iw.hub의 turn-left 애니메이션은 다음 위치에 있습니다: {isaacsim.anim.robot.core extension path}/data/sample_animations/iw_hub/turn_left/

Public API#

사용자 코드(행동 트리, 커스텀 런타임 상태, 독립 실행형 스크립트)는 isaacsim.anim.robot.core에서만 임포트해야 합니다. 다른 서브모듈은 내부용이며 예고 없이 변경될 수 있습니다.

액션 팩토리 – update(dt), is_done(), cancel() 메서드가 있는 불투명 액션 핸들을 반환합니다:

  • idle(runtime_config, state_machine, duration)

  • move_to(runtime_config, state_machine, target) – target은 Float3 유사 좌표 또는 USD prim 참조(Sdf.Path, prim 경로 문자열, 또는 Usd.Prim)일 수 있습니다.

  • turn(runtime_config, state_machine, direction) – yaw만 적용.

  • play_animation(runtime_config, state_machine, state_name, duration=0.0) – duration=0.0은 상태를 마지막 타임 샘플까지 한 번 재생합니다. duration > 0은 해당 초 동안 유지합니다.

  • sequence(actions, execute_now=False) – actions는 비어 있지 않아야 합니다. 별도의 execute() 호출 없이 시퀀스를 실행 중인 에이전트에 주입하려면 execute_now=True를 전달합니다.

에이전트 조회

  • resolve_anim_robot(prim) – Usd.Prim이 주어지면 해당 prim에 등록된 활성 AnimRobot(.runtime_config와 .state_machine 노출)을 반환하거나, prim이 유효하지 않거나, 에이전트가 아직 등록되지 않았거나(타임라인 시작 전에 일반적), 런타임이 해제된 경우 None을 반환합니다.

유틸리티

  • get_IAR_sample_config_path() – 번들된 샘플 구성 폴더의 해결된 경로. 도달 가능하면 Isaac Sim 에셋 서버 경로(/Isaac/Samples/AnimRobot/sample_configs/)를 반환하고, 그렇지 않으면 로컬 확장 프로그램 복사본을 폴백으로 반환합니다.
