Omni Metropolis Pipeline#

omni.metropolis.pipeline 익스텐션은 Action and Event Data Generation 익스텐션이 사용하는 공유 설정, 트리거 및 에이전트 레이어를 제공합니다.

YAML 설정 파일을 로드하고 익스텐션별 섹션을 파싱하며 비동기 설정을 실행하는 싱글톤 ConfigurationManager를 제공하여 단일 설정 파일에서 애플리케이션을 설정할 수 있게 합니다. 또한 TriggersManager와 내장 트리거 타입을 제공하여 이벤트(Physical Space Event Generation 등)를 특정 시간, carb 이벤트 또는 물리 충돌 시 시작할 수 있게 합니다. 또한 AgentManager와 기본 에이전트 인터페이스를 제공하여 에이전트(Actor Simulation and Synthetic Data Generation 등)를 생성하고 관리할 수 있게 합니다.

Overview#

  • Configuration – YAML 기반 애플리케이션 설정

    • ConfigurationManager – 설정 파일을 로드하고 등록된 익스텐션에 섹션을 전달하는 싱글톤. 익스텐션은 설정 파일이 로드되고 시뮬레이션이 설정될 때 사용되는 섹션 파서와 비동기 설정 함수를 등록합니다.

  • Triggers – 발동 시 콜백을 실행하는 이벤트 객체

    • TriggersManager – 딕셔너리 설명에서 트리거 인스턴스를 생성하고 관리하는 싱글톤. 스크립트에서 트리거를 생성하여 인시던트(화재, 넘어짐, 쏟아짐) 및 기타 이벤트 API에 전달하는 데 사용합니다.

    • Trigger types – 시간 기반, carb 이벤트 기반, 충돌 기반 트리거가 시작 시 익스텐션에 의해 등록됩니다. 각 트리거는 add_callback을 사용하여 콜백을 추가할 수 있으며, 트리거가 발동되면 모든 콜백이 호출됩니다.

  • Agents – 루틴을 수행하고 트리거에 응답할 수 있는 엔티티의 런타임 표현

    • AgentManager – 딕셔너리 설명에서 에이전트 인스턴스를 생성하고 관리하는 싱글톤. 스크립트에서 에이전트를 생성하여 에이전트 API에 전달하는 데 사용합니다.

    • Base agent interface – 기본 에이전트 인터페이스는 에이전트의 동작과 트리거를 정의하는 USD Prim입니다. 에이전트를 생성하고 관리하는 데 사용됩니다.

Configuration#

ConfigurationManager는 YAML 설정 파일을 로드하고 파서와 설정 함수를 등록한 익스텐션에 섹션을 전달합니다. 설정 파일은 현재 isaacsim.replicator.agent 익스텐션을 지원합니다. 형식 정보는 Configuration File Guide를 참고하십시오.

Using ConfigurationManager in Script#

ConfigurationManager.get_instance()를 사용하거나 omni.metropolis.pipeline.configuration의 모듈 수준 함수를 사용하여 싱글톤에 접근합니다. 설정 파일을 로드하기 전에 익스텐션의 섹션을 등록하고, load_config_file(path)와 await setup_simulation()을 호출합니다.

Registration

  • register_config_section(extension_name, section_header, section_name, section_parser, section_setup) – 설정 섹션을 등록합니다. section_header는 YAML 최상위 키(예: "omni.metropolis.pipeline" 또는 "isaacsim.replicator.agent.core")입니다. section_name은 오케스트레이터 헤더 아래의 키(예: "agent")입니다. section_parser는 해당 섹션의 원시 딕셔너리를 받아 파싱된 데이터를 반환하는 호출 가능 객체입니다. section_setup은 파싱된 페이로드를 받아 setup_simulation()이 호출될 때 실행되는 비동기 호출 가능 객체입니다.

  • is_section_registered(extension_name) – 해당 익스텐션이 섹션을 등록했는지 여부를 반환합니다.

  • unregister_config_section(extension_name) – 등록을 제거합니다.

Loading and access

  • load_config_file(file_path) – YAML 파일을 로드하고 파싱합니다. 성공 시 True를 반환합니다. False를 반환하면 get_load_error_message()를 사용합니다.

  • get_config(extension_name) – 해당 익스텐션의 파싱된 설정을 반환하거나, 존재하지 않거나 로드되지 않은 경우 None을 반환합니다.

  • get_config_file_path() – 현재 로드된 설정 파일의 경로를 반환하거나, 없으면 None을 반환합니다.

Setup

  • setup_simulation() – 비동기. 각 등록된 익스텐션의 설정 함수를 파싱된 설정과 함께 실행합니다. 모두 성공하면 True를 반환합니다. 실패 시 get_setup_error_message()를 사용합니다.

)

def parse_my_section(raw: dict):
    # Parser receives {section_header: section_data}. Return any structure your setup needs.
    return next(iter(raw.values()), {})

async def setup_my_section(parsed):
    # Run async setup using parsed config (for example, create prims, load assets).
    pass

# Register before loading. Use section_header and section_name that match your YAML.
register_config_section(

Triggers#

Using TriggersManager in Script#

매니저 싱글톤을 가져와 딕셔너리에서 트리거를 생성합니다. 시간 트리거를 생성하고 콜백을 추가하며 트리거를 IRI 이벤트 매니저(화재, 넘어짐, 쏟아짐)에 전달하는 전체 스크립트 예제는 Physical Space Event Generation과 해당 페이지의 Event Configuration in IRI Script 및 Triggers 섹션을 참고하십시오.

필수 조건

  • IRI 이벤트와 함께 트리거를 사용할 때 isaacsim.replicator.incident.core를 활성화하십시오.

  • omni.metropolis.pipeline 익스텐션은 Action and Event Data Generation 애플리케이션을 사용할 때 자동으로 로드됩니다.

Trigger Types#

익스텐션은 이 트리거 타입들을 등록합니다. 트리거 딕셔너리의 type 필드와 해당 매개변수를 사용하십시오. IRI 이벤트와 함께 각 트리거 타입의 YAML 및 스크립트 예제는 Physical Space Event Generation 튜토리얼의 Triggers를 참고하십시오.

time

시뮬레이션 타임라인이 지정된 시간(초)에 도달할 때 발동됩니다. 딕셔너리 형태: {"trigger": {"type": "time", "time": <seconds>}}.

carb_event

명명된 carb 이벤트가 디스패치될 때 발동됩니다. 선택적 페이로드는 발동 후 트리거에서 사용할 수 있습니다. 딕셔너리 형태: {"trigger": {"type": "carb_event", "event_name": "<event_name>"}}.

collision

콜라이더 프림의 물리 트리거 enter/exit 시 발동됩니다. 콜라이더에는 CollisionAPI, TriggerAPI, RigidbodyAPI가 있어야 합니다. metro:collider:name 속성을 사용하여 다른 콜라이더 이름으로 선택적 필터링이 가능합니다. 매개변수: collider_prim_path, trigger_enter, trigger_exit, other_collider_names.

Trigger API Summary#

  • TriggersManager.get_instance() – 싱글톤 TriggersManager를 반환합니다.

  • create_trigger_by_dict(dict_data) – {"trigger": {"type": "...", ...}}에서 트리거를 빌드합니다. 등록된 타입이 없으면 None을 반환합니다.

  • TriggerBase.add_callback(callback_fn) – 트리거 인스턴스를 인수로 받는 호출 가능 객체를 추가합니다. 트리거가 발동되면 호출됩니다.

  • TriggerBase.destroy() – 타임라인/이벤트에서 구독을 해제하고 콜백을 지웁니다. 트리거가 더 이상 필요하지 않을 때 호출합니다.

Example Usage#

import carb
from omni.metropolis.pipeline.triggers import TriggersManager

def callback(trigger):
    carb.log_info("Trigger fired!")

# Register a callback on a time trigger that fires at 1 second
trigger_manager = TriggersManager.get_instance()
trigger = trigger_manager.create_trigger_by_dict({"trigger": {"type": "time", "time": 1.0}})
trigger.add_callback(callback)

Agents#

Using AgentManager in Script#

Action and Event Data Generation 애플리케이션에서 에이전트를 생성하고 설정하는 방법(YAML, UI)은 Actor Simulation and Synthetic Data Generation을 참고하십시오.

Example Usage#

다음 패턴은 커스텀 에이전트 클래스를 등록하고 에이전트의 API로 프림을 생성합니다. 타임라인이 재생되면 AgentsManager가 프림을 발견하여 에이전트를 인스턴스화하고, 타임라인이 중지되면 런타임 인스턴스가 제거됩니다.

from typing import ClassVar
from pxr import Usd, UsdPhysics
import carb
import omni.usd
import omni.timeline
from omni.metropolis.pipeline.agent import Agent, AgentsManager

class MyAgent(Agent):
    AGENT_API: ClassVar[Usd.APISchemaBase] = UsdPhysics.RigidBodyAPI

    def get_world_position(self):
        return carb.Float3(0, 0, 0)

    def get_world_rotation(self):
        return carb.Float4(0, 0, 0, 1)

    def get_speed(self):
        return 0.0

    def get_facing_direction(self):
        return carb.Float3(1, 0, 0)

    def get_current_task_name(self):
        return None

    def on_update(self, delta_time: float):
        pass  # Custom behavior each frame

# Create a prim with the agent API so the manager can discover it
stage = omni.usd.get_context().get_stage()
stage.DefinePrim("/World", "Xform")
prim = stage.DefinePrim("/World/MyAgent", "Xform")
UsdPhysics.RigidBodyAPI.Apply(prim)

# Register the agent class and play to collect runtime instances
manager = AgentsManager.get_instance()
manager.register_agent_class(MyAgent)

timeline = omni.timeline.get_timeline_interface()
timeline.play()
# After play, manager.get_runtime_agent_instances() will contain MyAgent instances
# for each prim that has RigidBodyAPI. Call timeline.stop() to clear them.
