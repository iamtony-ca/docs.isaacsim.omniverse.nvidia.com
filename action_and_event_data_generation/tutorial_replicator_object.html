Object Simulation and Synthetic Data Generation#

isaacsim.replicator.object (IRO)는 코드 수정 없이 사용할 수 있는 합성 데이터 생성 도구로, 소매 객체 탐지부터 로봇공학까지 다양한 작업에 활용할 수 있습니다. 이 확장은 UI 또는 isaac-sim 컨테이너에서 실행할 수 있습니다.

가변 장면을 설명하는 YAML 설명 파일 또는 그러한 파일들이 중첩된 계층을 입력으로 받아, RGB, 2D/3D 경계 상자, 분할 마스크 등 그래픽 콘텐츠와 함께 설명 파일을 출력합니다.

Motivation#

합성 데이터로 딥러닝 모델을 학습시키는 수요가 높지만, 합성 데이터를 생성하는 3D 소프트웨어는 UI 패널에 익숙해지는 것을 포함해 학습에 오랜 시간이 걸리는 경우가 많습니다. IRO는 도메인 랜덤화가 적용된 장면을 쉽게 구성할 수 있는 방법을 제공하는 것을 목표로 합니다. 예를 들어, 이 제품의 일반적인 사용자는 Maya나 3ds Max와 같은 3D 모델링 소프트웨어 경험이 없는 데이터 과학자입니다.

도메인 랜덤화 시나리오에서 데이터 과학자는 3D 장면의 실제 세부 내용보다는 장면이 랜덤화되는 규칙과 이러한 랜덤화 규칙들 간의 관계에 더 집중하는 경우가 많습니다. IRO는 매크로를 사용하여 랜덤화된 3D 장면을 추상적이고 직관적이며 간결하게 설명할 수 있는 도구 세트를 제공합니다.

Chat IRO: Natural Language Interface for IRO#

Chat IRO는 일반 영어로 장면을 설명하면 IRO 설명 파일(YAML)을 자동으로 생성하는 새로운 확장입니다. 구성을 스테이지에 적용하고, 즉각적인 뷰포트 미리보기를 표시하며, 시뮬레이션을 실행할 수 있고, YAML 파일 저장 및 로드를 지원하여 수동 YAML 편집 없이 빠르고 반복적인 장면 제작이 가능합니다.

  • Chat IRO: Natural Language Interface for Isaac Sim Replicator Object

End-to-end Pipeline#

엔드투엔드 파이프라인은 IRO 사용에 필요한 큰 단계들의 그룹으로 구성됩니다.

그래픽 리소스 획득

랜덤화된 장면을 구성하려면 IRO에서 가져온 3D 모델이 USD 형식이어야 합니다. Wavefront OBJ와 같은 일반적인 3D 형식은 asset converter를 사용하여 USD로 변환할 수 있습니다.

설명 파일 작성

설명 파일의 사양은 이 다중 페이지 문서에 설명되어 있습니다. best practices의 동영상 가이드부터 시작하는 것을 권장합니다.

합성 데이터 생성

IRO를 실행하려면 아래 지침을 따르세요.

CV 모델 학습, 배포 및 실제 응용

IRO로 생성한 합성 이미지로 객체 탐지 모델을 학습하는 단계를 보여주는 예제 노트북이 TAO 6.0에 있습니다.

Run from the UI#

  1. Omniverse Extension Manager guide를 따라 isaacsim.replicator.object.core 및 isaacsim.replicator.object.ui 확장을 활성화하세요.

  2. 확장이 성공적으로 활성화되면 오른쪽 상단에 Object SDG 패널이 표시되고, Tools > Action and Event Data Generation에 Object SDG 및 Distribution Visualizer 옵션이 표시됩니다.

    그렇지 않으면 확장을 비활성화했다가 다시 활성화하세요. Object SDG 패널은 기본적으로 켜져 있으며, Tools > Action and Event Data Generation > Object SDG로 끄거나 다시 켤 수 있습니다.

  3. 위에 표시된 것처럼 열린 확장 패널의 오른쪽에 있는 폴더 아이콘 또는 Visual Studio Code 아이콘을 클릭하세요. 확장의 루트 폴더가 열립니다.

  4. PATH_TO_CORE_EXTENSION/isaacsim/replicator/object/core/configs 아래에 YAML 형식의 설명 파일이 많이 있습니다. demo_kaleidoscope.yaml부터 시작하는 것을 권장합니다. 빈 공간 감지의 경우 demo_empty_space.yaml을 사용하고 연결된 카탈로그 페이지를 참조하세요.

  5. global.yaml로 이동하여 시뮬레이션 출력을 저장할 수 있는 로컬 폴더로 output_path를 업데이트하세요.

참고

Simulate 버튼 아래의 드롭다운에서 설명 파일을 선택하세요. 확장이 로드되면 configs 폴더의 모든 .yaml 파일의 이름이 이 목록에 포함됩니다.

설명 파일의 플레이스홀더

일부 예제 설명 파일에는 플레이스홀더가 있습니다. 경로를 유효한 경로로 교체해야 합니다.

예를 들어:

  • global.yaml 및 minimum.yaml에서 PATH_TO_OUTPUT을 유효한 경로로 교체하세요.

  • demo_bottle.yaml에서 PATH_TO_LABEL_IMAGES를 JPEG 이미지가 포함된 폴더로 교체하세요.

  • tutorial_harmonizer_permutate.yaml, demo_macro.yaml, tutorial_macro.yaml, tutorial_scene_graph.yaml 및 tutorial_scene_graph_randomized.yaml을 실행하려면:

    • global.yaml의 PATH_TO_ORO를 확장의 루트 폴더에 있는 data/oro_tutorial_models/oro.usd의 절대 경로로 교체하세요.

  • doc_observatory.yaml에서:

    • PATH_TO_OBSERVATORY_SCOPE를 data/oro_tutorial_models/observatory_scope.usd의 절대 경로로 교체하세요

    • PATH_TO_OBSERVATORY_BASE를 data/oro_tutorial_models/observatory_base.usd의 절대 경로로 교체하세요

    • PATH_TO_OBSERVATORY_SHAFT를 data/oro_tutorial_models/observatory_shaft.usd의 절대 경로로 교체하세요.

  • demo_bin_pack.yaml, demo_bins_of_bins_rack_2_layers.yaml, demo_bins_of_bins_rack.yaml, demo_bins_of_bins.yaml, demo_table.yaml 및 demo_transform_operator.yaml이 작동하려면:

    • global.yaml의 PATH_TO_BOXES를 상자(또는 기타 USD 파일)의 USD 파일이 포함된 폴더로 교체하세요.

  • demo_shader_attributes.yaml에서,

    • PATH_TO_USD를 USD 파일 경로로 교체하세요.

  • demo_frustum.yaml에서:

    • PATH_TO_MAIN_OBJECTS를 주요 객체로 사용할 USD 파일이 포함된 폴더로 교체하세요.

    • PATH_TO_DISTRACTORS를 방해 요소로 사용할 USD 파일이 포함된 폴더로 교체하세요.

    • PATH_TO_BACKGROUND_IMAGES를 배경 이미지로 사용할 JPEG 이미지가 포함된 폴더로 교체하세요.

서로 다른 USD 파일의 크기가 다르기 때문에 올바르게 표시되지 않으면 스케일을 조정할 수 있습니다.

  1. 드롭다운 박스에서 demo_kaleidoscope를 선택하면 Description File 텍스트 박스에 demo_kaleidoscope가 표시됩니다. 전체 절대 경로 PATH_TO_CORE_EXTENSION/isaacsim/replicator/object/core/configs/demo_kaleidoscope.yaml를 사용하여 설명 파일을 로드할 수도 있습니다.

  2. Simulate를 클릭하여 시뮬레이션을 시작하세요. 진행 표시줄에 시뮬레이션 진행 상황이 표시됩니다.

위 내용과 이후 내용에서 PATH_TO_CORE_EXTENSION은 환경에 따라 다르며, Isaac on Windows의 경우 C:\isaacsim\extscache\isaacsim.replicator.object.core-0.x.y\isaacsim\replicator\object\core\configs\demo_kaleidoscope.yaml과 같고, Isaac on Linux의 경우 ~/isaacsim/extscache/isaacsim.replicator.object.core-0.x.y/isaacsim/replicator/object/core/configs/demo_kaleidoscope.yaml과 같습니다.

확장 사용 방법에 대한 가이드는 여기에서 확인할 수 있습니다.

Run from Docker#

Isaac Sim Docker 컨테이너를 설치하려면 Container Deployment를 방문하세요.

Isaac Sim Docker 컨테이너를 실행하려면:

docker run --gpus device=0 --entrypoint /bin/bash -v LOCAL_PATH:/tmp --network host -it ISAAC_SIM_DOCKER_CONTAINER_URL

따라서 global.yaml을 업데이트하여 output_path를 /tmp 아래의 임의 폴더로 설정하세요.

예를 들어, demo_kaleidoscope로 시뮬레이션을 실행하려면:

bash isaac-sim.sh --no-window --enable isaacsim.replicator.object.core --allow-root --/log/file=/tmp/isaacsim.replicator.object.log --/log/level=warn --/windowless=True --/config/file=PATH_TO_CORE_EXTENSION/isaacsim/replicator/object/core/configs/demo_kaleidoscope.yaml

/tmp/isaacsim.replicator.object.log에는 실행 메시지와 확장 메시지가 포함됩니다. METROPERF로 파일을 필터링하여 확장의 메시지를 검색할 수 있습니다.

참고

Docker 컨테이너 내에서 첫 번째 실행 시 아무것도 생성되지 않으면 다시 실행하세요.

Concepts#

Description File

설명 파일은 isaacsim.replicator.object라는 주요 키를 가진 YAML 파일입니다.

설명 파일은 키-값 쌍으로 구성됩니다. 각 키-값 쌍은 Mutable, Harmonizer, 또는 Setting입니다.

설명 파일은 지정된 대로 프레임을 생성합니다. 매 프레임마다 장면이 랜덤화되고, 그래픽 콘텐츠가 캡처되어 디스크에 출력됩니다. Settings는 장면 구성 방법과 데이터 출력 방법을 설명합니다. 예를 들어, 출력할 프레임 수, 2D 경계 상자 출력 여부, 물리 시뮬레이션의 중력 및 마찰을 설정할 수 있습니다.

설명 파일은 mutables라고 하는 객체로 장면을 채웁니다.

Mutable은 매 프레임마다 랜덤화됩니다. 때로는 랜덤화 방식을 제한하고 싶을 수 있습니다. 예를 들어, 다른 mutable이 어떻게 랜덤화되는지 알고 그에 맞게 랜덤화하려면 harmonizer를 정의하세요.

Example Minimal Description File Definition
isaacsim.replicator.object:
   version: 0.x.y
   num_frames: 3
   output_path: OUTPUT_PATH
   screen_height: 1080
   screen_width: 1920
   seed: 0
Simulation Workflow

시뮬레이션이 실행될 때마다 처음에 초기화 단계가 발생하고, 매 프레임마다 프레임별 시뮬레이션 단계가 발생합니다.

초기화 단계에서 설명 파일은 설명 파서에 의해 파싱됩니다. 실제 값을 얻기 위해 해석이 필요한 모든 mutable attribute에 대해 심볼이 생성됩니다. 이 심볼들은 초기화된 후 USD 장면과 한 번 상호작용할 때, 그리고 모든 프레임별 시뮬레이션에서 실제 값으로 해석됩니다.

심볼이 해석될 때마다 그에 의존하는 심볼도 재귀적으로 해석됩니다. 미해석된 harmonized mutable attribute가 발견되면 파서는 AWAITING_HARMONIZATION 상태로 진입하고, harmonizer가 조화를 이루어(pitch 속성에서 정보를 수집하고 랜덤화), 조화된 mutable attribute로 출력을 전파합니다. 모든 harmonized mutable attribute가 해석되면 파서는 AWAITING_HARMONIZATION 상태에서 벗어납니다.

이후 해석된 값들은 USD 장면을 업데이트하는 데 사용됩니다. 중력이 켜져 있으면 물리가 해석되어 객체들이 겹칠 때 서로 멀어지거나 표면에 떨어집니다(자세한 내용은 물리 시뮬레이션 설명 참조). 그리고 그래픽 콘텐츠가 캡처됩니다. 최종적으로 이 프레임의 장면 상태가 기록되어 저장되므로, 나중에 복원하거나 검사할 수 있습니다.

자세한 내용은 harmonization example에서 확인할 수 있습니다.

Compose a Description File#

테이블 위에 랜덤화된 객체가 떨어지는 장면을 생성하는 설명 파일을 작성하려면:

다음 에셋이 있다고 가정합니다:

  • PATH_TO_HDRI의 돔 라이트를 위한 HDRI 텍스처

  • PATH_TO_TABLE의 테이블로 사용할 USD 모델

  • PATH_TO_OBJECTS의 테이블에 배치할 객체의 USD 모델이 포함된 폴더

설명 파일을 작성하기 전에 그래픽 에셋의 분포를 계획하세요. 에셋을 뷰포트로 드래그하여 파악하고 이미지를 참조하세요. 여기서 돔 라이트를 생성하고 텍스처를 PATH_TO_HDRI로 설정합니다. 그런 다음 PATH_TO_TABLE에서 테이블을, PATH_TO_OBJECTS에서 객체 중 하나를 가져옵니다.

테이블에 배치될 객체의 위치를 적절한 범위로 조정하세요.

관찰에 따르면 (-13, 100, -70)에서 (13, 100, 70)까지가 상자 위치의 합리적인 범위입니다. 다음과 같이 설명 파일을 작성하세요:

isaacsim.replicator.object:
  # the minimum
  version: 0.x.y
  num_frames: 3
  seed: 0
  output_path: PATH_TO_OUTPUT
  screen_height: 2160
  screen_width: 3840

  # physics parameters
  gravity: 10000
  friction: 0.3
  simulation_time: 10
  linear_damping: 4

  # light
  bright_light:
    type: light
    subtype: dome
    intensity: 1000
    transform_operators:
    - rotateX: 270
    texture_path: PATH_TO_HDRI

  # camera; transforms page has more details on how to construct a list of transform operators
  focal_length: 14.228393962367306
  horizontal_aperture: 20.955
  camera_parameters:
    screen_width: $[/screen_width]
    screen_height: $[/screen_height]
    focal_length: $[/focal_length]
    horizontal_aperture: $[/horizontal_aperture]
    near_clip: 0.001
    far_clip: 100000
  default_camera:
    type: camera
    camera_parameters: $[/camera_parameters]
    transform_operators:
    - translate:
      - 0
      - 50
      - 0
    - rotateY:
        distribution_type: range
        start: -180
        end: 180
    - rotateX: -30
    - translate_local:
      - 0
      - 0
      - 400

  # boxes
  box:
    count: 10
    type: geometry
    subtype: mesh
    physics: rigidbody # reacts to gravity, collisions, etc.
    tracked: true # if true, the bounding boxes, segmentation, etc. will be recorded in the output
    usd_path:
      distribution_type: folder
      value: PATH_TO_OBJECTS
      suffix: usd
    transform_operators:
    - translate: # as planned
        distribution_type: range
        start:
        - -13
        - 100
        - -70
        end:
        - 13
        - 100
        - 70
    - rotateXYZ:
        distribution_type: range
        start:
        - -180
        - -180
        - -180
        end:
        - 180
        - 180
        - 180
    - scale:
      - 0.2
      - 0.2
      - 0.2

  table:
    type: geometry
    subtype: mesh
    physics: collision # rigidbodies will collide with it, but it doesn't move
    usd_path: PATH_TO_TABLE
    transform_operators:
    - rotateX: -90

Catalog#

연결된 카탈로그 파일의 규칙:

표의 Type은 예상 데이터 유형을 나타냅니다. 유형이 예상되는 경우, 해당 특정 유형의 나중 평가를 위해 매크로 문자열을 사용할 수 있습니다. 예를 들어, 값에 int가 예상되는 경우 int를 직접 제공하거나 $[index]와 같은 것을 사용할 수 있습니다. 자세한 내용은 Macro를 참조하세요.

mutable 내에서 이러한 옵션 외에도 이 유형으로 평가할 Mutable Attribute를 지정할 수 있습니다.

numeric은 리터럴 또는 평가된 float 또는 int를 의미합니다.
