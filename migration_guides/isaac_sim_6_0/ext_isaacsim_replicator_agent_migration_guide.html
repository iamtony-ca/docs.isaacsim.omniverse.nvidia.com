Replicator Agent (IRA)#

이 가이드는 IRA 0.x.x(Isaac Sim 5.1 이하에 포함)에서 IRA 1.x.x (Isaac Sim 6.0 이상)으로 업그레이드하는 사용자를 위한 주요 변경 사항 및 마이그레이션 단계를 다룹니다.

IRA 1.0.0은 완전한 아키텍처 개편입니다. 구성 스키마, Python API, 종속성 그래프 및 내부 모듈 레이아웃이 모두 변경되었습니다. 기존 0.x 구성 및 코드는 수정 없이는 작동하지 않습니다.

What Changed at a Glance#

IRA 1.x는 0.x의 모든 핵심 기능을 유지합니다:

  • 환경 로딩

  • 캐릭터 및 로봇 스포닝

  • 센서 배치

  • 합성 데이터 생성

각 영역의 구성 및 실행 방법을 재설계하면서. 아래 표는 지원하는 버전과 변경 사항을 포함하여 모든 주요 기능 영역을 요약합니다:

0.x

1.x

0.x에서는 명령 생성과 저장이 별도의 수동 단계였습니다. 1.x에서는 모든 액터 동작이 routines와 triggers를 사용하여 YAML 구성에 미리 정의되므로 설정에서 생성까지 두 번의 클릭으로 가능합니다.

Default Config File Location#

버전

위치

0.x

[ext-path]/config/default_config.yaml (시작 시 자동 로드)

1.x

[ext-path]/data/sample_configs/ (번들 샘플은 샘플 구성 참조)

1.x에서는 기본적으로 minimal.yaml이 로드됩니다. 캐릭터, 카메라, 라이터가 있는 전체 예제는 full_pipeline.yaml을 사용하십시오.

Headless or Script-Based Data Generation#

스크립트 진입점이 변경되었습니다:

# OLD (0.x)
./python.sh tools/actor_sdg/sdg_scheduler.py -c [config file path]

# NEW (1.x)
./python.sh tools/actor_sdg/actor_sdg.py -c [config file path]

sdg_scheduler.py 스크립트가 actor_sdg.py로 교체되었습니다. 이전 스크립트의 --save_usd 플래그는 더 이상 스크립트 인수로 문서화되지 않습니다.

Simulation Duration#

0.x에서 simulation_length는 프레임 단위로 지정되었습니다(30 FPS 가정). 사용자는 수동으로 변환해야 했습니다: 300 프레임 = 10 초.

1.x에서 simulation_duration은 직접 초 단위로 지정됩니다 (예: 60.0). 시뮬레이션은 여전히 내부적으로 고정 30 FPS로 실행됩니다.

Configuration File Changes#

YAML 구성 스키마가 처음부터 재설계되었습니다.

Version Field#

version 필드는 이제 확장 버전을 추적하며 반드시 1.x.x 주요 버전을 가져야 합니다.

# 0.x
isaacsim.replicator.agent:
  version: 0.7.0

# 1.x
isaacsim.replicator.agent:
  version: 1.6.0

0.x 필드

1.x 대체

섹션이 scene에서 environment로 이름 변경됨.

완전히 재구성됨 (아래 참조).

최상위 마이그레이션 예제:

# OLD (0.x)
isaacsim.replicator.agent:
  version: 0.7.0
  global:
    seed: 123456789
    simulation_length: 1800

# NEW (1.x)
isaacsim.replicator.agent:
  version: 1.6.0
  seed: 123456789           # optional, auto-generated if omitted
  simulation_duration: 60.0  # 1800 frames / 30 FPS = 60 seconds
  environment:
    base_stage_asset_path: "Isaac/Environments/Simple_Warehouse/full_warehouse.usd"  # now required

Environment (was scene)#

scene 섹션이 environment로 이름 변경되고 확장되었습니다.

# OLD (0.x)
# scene.asset_path was defined in the config, OR fell back to settings

# NEW (1.x)
environment:
  base_stage_asset_path: "Isaac/Environments/Simple_Warehouse/full_warehouse.usd"
  prop_asset_paths: []  # optional: additional USD assets loaded as sublayers

주요 차이점:

  • base_stage_asset_path는 필수입니다 (확장 설정에 대한 폴백 없음).

  • Isaac Sim 에셋 루트를 기준으로 해석되는 상대 경로, 전체 URL 및 절대 파일 시스템 경로를 지원합니다.

  • 스테이지에 추가 에셋을 레이어하기 위한 새 prop_asset_paths 필드.

Character Section#

character 섹션은 이제 명명된 그룹 시스템을 사용하여 인라인 동작 정의(루틴)를 사용합니다. 이는 외부 명령 파일이 있는 이전 플랫 구조를 대체합니다.

# OLD (0.x)
character:
  asset_path: ""                       # single path for all characters
  command_file: default_command.txt     # external text file for commands
  num: 8
  filters: []
  spawn_area: []
  navigation_area: []

# NEW (1.x)
character:
  root_prim_path: "/World/Characters"   # optional, defaults to /World/Characters
  motion_library_path: ""                # optional; 1.4.0+ moved this to character root (was per-group)
  groups:
    warehouse_workers:                   # named group
      num: 10
      asset_path: "Isaac/People/Characters/"
      spawn_areas: []                    # renamed from spawn_area
      semantic_labels: [["class", "character"]]
      routines:                          # replaces command_file
        - wander:
            weight: 1.0
            repeat: 1
            walk:
              speed_range: [0.8, 1.5]
              distance_range: [5.0, 10.0]
              navigation_areas: []       # renamed from navigation_area
            idle:
              - animation: idle
                weight: 1.0
                time_range: [2.0, 5.0]
      triggers: []                       # new: event-driven behavior interrupts
      colliders: null                    # new: optional collision shapes

주요 차이점:

  • 명명된 그룹: 캐릭터는 이제 독립적인 설정을 가진 character.groups 아래의 명명된 그룹으로 구성됩니다.

  • 명령 파일 없음: command_file 필드와 외부 .txt 명령 파일이 사라졌습니다. 동작은 routines를 사용하여 인라인으로 정의됩니다.

  • 루틴이 명령을 대체: 각 루틴 항목은 인라인 매개변수를 가진 동작 유형(wander, patrol, stop, custom_behavior)입니다.

  • filters 필드가 제거됨.

  • spawn_area가 spawn_areas로 이름 변경됨.

  • navigation_area는 이제 walk.navigation_areas 아래에서 동작별로 설정됩니다.

  • 캐릭터 그룹의 새 semantic_labels, triggers, colliders 필드. motion_library_path도 새로운 필드이지만 IRA 1.4.0부터 그룹 내부가 아닌 character: 루트 수준에 위치합니다. 그룹 내부에 배치하면 유효성 검사에 실패합니다.

Robot Section#

캐릭터와 유사하게, 로봇도 이제 인라인 루틴이 있는 명명된 그룹을 사용합니다.

# OLD (0.x)
robot:
  command_file: default_robot_command.txt
  nova_carter_num: 0
  iw_hub_num: 0
  write_data: false
  spawn_area: []
  navigation_area: []

# NEW (1.x)
robot:
  root_prim_path: "/World/Robots"
  groups:
    carters:
      num: 2
      config_file_path: "nova_carter.yaml"   # IAR robot config file
      spawn_areas: []
      agent_radius: null                      # optional NavMesh agent radius
      write_data: false
      camera_prim_paths: []                   # optional: specific cameras for data
      semantic_labels: [["class", "robot"]]
      routines:
        - wander:
            move:
              distance_range: [10.0, 15.0]
              navigation_areas: []
            idle:
              time_range: [2.0, 5.0]
      triggers: []

주요 차이점:

  • nova_carter_num / iw_hub_num이 제거되었습니다. 적절한 IAR 구성을 가리키는 num과 config_file_path가 있는 명명된 그룹을 사용하십시오.

  • command_file이 인라인 routines로 교체됨 (캐릭터와 동일).

  • 새 config_file_path 필드가 로봇의 Isaac Anim Robot(IAR) 구성을 참조합니다.

  • 데이터 생성에 사용할 온보드 카메라를 지정하는 새 camera_prim_paths.

  • spawn_area가 spawn_areas로 이름 변경됨.

Sensor Section#

센서 섹션이 명명된 그룹과 플러그인 가능한 배치 전략을 지원하도록 재설계되었습니다.

# OLD (0.x)
sensor:
  camera_num: 4
  # OR
  camera_list: ["/World/Camera1", "/World/Camera2"]

# NEW (1.x)
sensor:
  root_prim_path: "/World/Cameras"
  groups:
    ceiling_cameras:
      num: 6
      aim_at_targets:           # placement strategy
        height_range: [7.0, 10.0]
        look_down_angle_range: [30.0, 45.0]
        focal_length_range: [10.0, 15.0]
        distance_range: [5.0, 10.0]
    coverage_cameras:
      num: -1                   # auto-calculate for maximum coverage
      maximum_coverage:
        target_coverage_ratio: 0.8
        height_range: [2.0, 5.0]

주요 차이점:

  • camera_num / camera_list가 sensor.groups 아래의 명명된 그룹으로 교체됨.

  • 각 그룹은 배치 전략을 지정합니다: aim_at_targets 또는 maximum_coverage.

  • 확장 설정에 있던 카메라 무작위화 파라미터(높이, 각도, 초점 거리, 거리)가 이제 센서 그룹별 구성 YAML에 있습니다.

  • maximum_coverage와 함께 num: -1을 사용하면 자동 카메라 수 계산이 트리거됩니다.

Replicator Section#

replicator 섹션은 이제 라이터별 타이밍 제어를 통해 여러 동시 라이터를 지원합니다.

# OLD (0.x)
replicator:
  writer: IRABasicWriter          # single writer, selected by name
  parameters:                     # flat parameter dict
    object_info_bounding_box_2d_tight: false
    object_info_bounding_box_2d_loose: false
    semantic_filter_predicate: "class:character|robot;id:*"
    rgb: true
    camera_params: true

# NEW (1.x)
replicator:
  hide_debug_visualization: true  # new: hides NavMesh/skeleton debug viz
  writers:
    IRABasicWriter:               # writer name is the dict key
      rgb: true
      camera_params: true
      object_info_bounding_box_2d_tight: true
      object_info_bounding_box_2d_loose: true
      semantic_filter_predicate: "class:character|robot;id:*"
      start_frame: 30             # new: per-writer frame control
      # OR use time-based:
      # start_time: 1.0
      # end_time: 50.0
      # sensor_prim_list: []      # new: optional per-writer sensor selection

주요 차이점:

  • writer + parameters가 writers 딕셔너리로 교체됨. 키는 라이터 이름이고, 값은 직접 매개변수를 포함합니다(parameters 하위에 중첩 없음).

  • 여러 라이터: 단일 구성에서 여러 라이터를 정의할 수 있습니다(예: IRABasicWriter + CosmosIRAWriter).

  • 타이밍 제어를 위한 라이터별 start_frame과 end_frame 또는 start_time과 end_time.

  • 특정 카메라 선택을 위한 라이터별 sensor_prim_list.

  • 기본 start_frame은 30입니다 (초기 안정화 프레임 건너뜀).

  • 새 라이터: CosmosIRAWriter, SceneGraphWriter, CustomWriter. 기본 Replicator BasicWriter는 writer_name: "BasicWriter"를 사용하여 CustomWriter를 통해 사용할 수 있습니다.

  • IRABasicWriter 기본값 변경: object_info_bounding_box_2d_tight, object_info_bounding_box_2d_loose, object_info_bounding_box_3d가 이제 기본적으로 true로 설정됩니다.

Behavior System Architecture Changes#

액터 동작 시스템이 근본적으로 재설계되었습니다.

Old System (0.x): Command Files + omni.anim.people#

0.x에서 액터 동작은 외부 명령 파일(.txt)로 제어되었습니다:

  • 캐릭터는 텍스트 기반 명령 파일을 줄별로 읽는 omni.anim.people에 의해 구동되었습니다.

  • 로봇은 유사한 명령 파일을 가진 isaacsim.anim.robot에 의해 구동되었습니다.

  • UI의 임의 명령 생성 버튼이 확장 설정에 저장된 무작위화 설정을 기반으로 이러한 명령 파일을 생성했습니다.

  • 명령 전환 맵(.json)이 명령 간의 확률론적 전환을 제어했습니다.

New System (1.x): Routine-Trigger Loop + omni.anim.behavior#

1.x에서 액터 동작은 omni.anim.behavior.core로 지원되는 루틴-트리거 루프를 사용합니다:

  1. 루틴: YAML 구성에 정의된 가중치 기반 동작 목록. 액터는 액터의 개별 시드를 사용하여 확률 가중치에 따라 루틴 풀에서 동작을 반복적으로 선택합니다.

  2. 트리거: 이벤트 기반 또는 시간 기반 인터럽트. 활성화되면 액터는 루틴을 일시 중지하고 트리거의 동작 시퀀스를 실행합니다. 우선순위가 높은 트리거는 실행 중인 트리거를 선점할 수 있습니다.

  3. USD 스키마: 로드 후 각 액터의 구성이 USD API 스키마에 포함됩니다(캐릭터의 SkelRoot의 IRACharacterAPI, 로봇 루트 프림의 AnimRobotAPI). 동작과 트리거는 액터가 참조하는 개별 USD 프림이 됩니다.

Available Behavior Types#

캐릭터 동작:

  • wander – 구성 가능한 속도, 거리, 유휴 애니메이션으로 무작위 걷기 및 유휴 사이클

  • patrol – 3D 점 또는 목표 프림으로 정의된 경로 따르기

  • stop – 구성 가능한 시간 동안 정지 상태 유지

로봇 동작:

  • wander – 구성 가능한 거리와 대기 시간으로 무작위 이동 및 유휴

  • patrol – 3D 점 또는 목표 프림으로 정의된 경로 따르기

  • halt – 구성 가능한 지속 시간 동안 정지

트리거 유형 (캐릭터와 로봇이 공유):

  • event_trigger – 명명된 이벤트에서 발생

  • time_trigger – 지정된 시간(초) 후에 발생

  • collision_trigger – 액터의 명명된 충돌체가 다른 명명된 충돌체와 겹치기 시작하거나 끝날 때 발생

Behavior Tree Groups (alternative to routines/triggers)#

IRA 1.3.0(캐릭터)과 1.5.0(로봇)부터, 그룹은 그룹 수준에서 behavior_tree JSON 에셋과 선택적 overrides 블록을 선언하여 루틴-트리거 시스템에서 벗어날 수 있습니다. 모든 동작 로직은 참조된 트리 내부에서 저작됩니다. routines, triggers, colliders는 적용되지 않습니다. 전체 스키마와 예제는 구성 파일 가이드의 "동작 트리 캐릭터 그룹 (실험적)" 및 "동작 트리 로봇 그룹 (실험적)" 섹션을 검토하십시오.

Step-by-Step Migration Checklist#

Config File Migration#

  1. version을 1.x.x로 변경 (예: 1.5.0).

  2. global.seed를 루트 수준 seed로 이동.

  3. global.simulation_length(프레임)를 루트 수준의 simulation_duration(초)으로 변환.

    • 공식: simulation_duration = simulation_length / 30.0

  4. scene.asset_path를 environment.base_stage_asset_path로 이름 변경 (이 필드는 이제 필수).

  5. character를 num, asset_path 및 인라인 routines가 있는 character.groups.<group_name>으로 재구성.

  6. 외부 명령 파일(.txt)을 인라인 routines 항목으로 변환. 이전 명령을 새 동작 유형으로 매핑:

    • GoTo 명령 -> distance_range가 있는 wander 동작

    • Idle 명령 -> idle 목록이 있는 wander 동작

    • 순찰 유사 시퀀스 -> path_points 또는 target_prims가 있는 patrol 동작

    • 전체 동작 유형 문서는 구성 파일 가이드를 검토하십시오.

  7. robot을 num, config_file_path 및 인라인 routines가 있는 robot.groups.<group_name>으로 재구성.

    • nova_carter_num: N을 config_file_path: "nova_carter.yaml"을 사용하는 그룹 항목으로 교체.

    • iw_hub_num: N도 적절한 IAR 구성으로 유사하게 교체.

  8. sensor.camera_num과 camera_list를 배치 전략(aim_at_targets 또는 maximum_coverage)을 사용하는 sensor.groups.<group_name>으로 교체.

  9. 확장 설정의 카메라 무작위화 설정(높이, 각도, 초점 거리, 거리 범위)을 센서 그룹 구성으로 이동.

  10. replicator.writer + replicator.parameters를 replicator.writers.<WriterName>: { ... }로 교체.

  11. response, event, incident 섹션을 제거하고 개별 캐릭터/로봇 그룹의 triggers로 교체.

Quick Config Conversion Example#

이전 (0.x):

isaacsim.replicator.agent:
  version: 0.7.0
  global:
    seed: 123456789
    simulation_length: 1800
  sensor:
    camera_num: 4
  character:
    asset_path: ""
    command_file: default_command.txt
    num: 8
  robot:
    command_file: default_robot_command.txt
    nova_carter_num: 2
    iw_hub_num: 0
    write_data: false
  replicator:
    writer: IRABasicWriter
    parameters:
      semantic_filter_predicate: "class:character|robot;id:*"
      rgb: true
      camera_params: true
